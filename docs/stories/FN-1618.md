## Key Decisions
- **Single shared service over per-provider services.** Extending `fleetneuron-integrations-service` reuses the established webhook + signature-verification pattern from FN-758 and avoids paying for N additional Render web services. Recommended split-out only if a single provider's volume eventually dominates (unlikely given small-carrier scale).
- **Envelope encryption with Render env-var master key + per-row DEK.** Avoids an external KMS billing relationship while still giving rotation + per-row isolation. Same rotation pattern as the SendGrid Inbound Parse webhook secret. If we later move to AWS KMS, only the wrap step changes; envelope shape stays.
- **`hos_records` (daily roll-up) stays untouched; new `hos_log_events` table for raw duty-status events.** All Tier-1 ELD APIs return events; existing manual-entry flows continue to write `hos_records` directly. A worker re-derives daily totals from events.
- **Render Cron Jobs over an internal scheduler (BullMQ/Bree).** Operators already know the pattern from FMCSA imports; isolated container means a poller crash can't take down webhook receivers; independent scaling per cadence (1 min for ELD positions, 15 min for factoring).
- **Webhook security: provider HMAC is the actual boundary; `?secret=` query string is a first-line filter only.** Same shape as the SendGrid Inbound Parse webhook (FN-758) — gateway forwards to integrations-service, which dispatches to the per-provider adapter for signature verification, then idempotently records in `integration_webhook_events`.
- **At-least-once webhook delivery + DB-level dedupe** via `UNIQUE(provider, external_event_id)`. Industry standard; cheaper than exactly-once with no observable benefit.
