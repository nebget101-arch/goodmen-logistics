## Implementation Summary

### FN-1594 (subtask)

Built the Angular wizard, service, routing, duplicate-review modal, and result screen for spreadsheet load import. Wizard is split into 6 step components per the FN-1594 component tree (rather than the single-component pattern used by `fuel-import-wizard`). All state lives in the parent `LoadsImportWizardComponent`; the step components are presentational and emit navigation events.

**State machine**: `upload → ai_analysis → mapping → validate → commit → result`. The progress rail lets the user click back to any step they've already passed through; forward jumps are blocked.

**Lazy loading**: `LoadsImportModule` is registered in `app-routing.module.ts` at `loads/import` *before* the existing lazy `loads` route, so the more specific path resolves to the new module instead of falling through to `LoadsDashboardModule`. This keeps the heavy dashboard bundle (Leaflet, intelligence panel, etc.) out of the import wizard.

**AI mapping**: Step 2 renders Claude attribution and a confidence-badged mapping table. The user can `Accept & Adjust` (carries the AI suggestions into the mapping step) or `Map Manually` (clears the suggestions). Mapping overrides survive a back-and-forward navigation: the mapping-step component holds a local copy of the column map and emits the latest values to the parent on every navigation event (back AND next).

**Duplicate modal**: Auto-opens when `commitResult.duplicates.length > 0`. Lists each duplicate with row #, attempted load_number, broker, pickup→delivery, rate, and a deep-link to the existing FN load using the dashboard's existing `?loadId=` query-param convention. Includes a CSV export button. The result step exposes a "Review duplicates" CTA that re-opens the modal if the user dismissed it.

**FN-317 guard**: `app-ai-select` `[options]` is bound to a stable property (`headerOptions`) populated once on `/preview` response — never via getter. This avoids the infinite-CD loop documented in FN-317.

## Key Decisions

1. **Step components, not a single component.** The fuel-import-wizard reference is one component with conditional templates. FN-1594 explicitly enumerated separate step components, so the loads variant follows that. The trade-off is more files but a clearer surface for unit/E2E tests.

2. **Parent-owned state, presentational steps.** Steps take `@Input` data and emit `@Output` navigation events. The parent (`LoadsImportWizardComponent`) holds all session state. This makes back-and-forward navigation safe — going back doesn't lose the column map because the parent never resets it.

3. **Top-level lazy route at `loads/import`, not a child route inside LoadsDashboardModule.** Putting the wizard inside the dashboard module would force users to download the heavy dashboard bundle (Leaflet + intelligence panel + smart filters) just to open the import page. A separate lazy module keeps the import bundle small (~223 kB).

4. **`ViewEncapsulation.None` on the wizard with `.li-` prefixed classes.** All wizard styling lives in `loads-import-wizard.component.scss`. Step components have no styles. Class names are prefixed `.li-` to avoid global pollution.

5. **Deep-link uses `?loadId=` query param.** The loads-dashboard already reads `loadId` from the URL and opens the detail modal. The duplicate-modal uses the same convention.

6. **Permissions: `loads.create` OR `loads.edit` on the route; `loads.create` on the entry button.** The wizard creates loads; the route `canActivate` covers both perms in case staff with edit-only roles need to retry imports. The dashboard button is gated on `loads.create`.

7. **Backend contract is documented in `loads-import.model.ts`.** Backend (FN-1584) hasn't shipped yet. The wizard scaffolding compiles and lazy-loads correctly (`ng build --configuration development` exits 0 with the lazy bundle generated at ~223 kB). Full E2E will require the backend live (per FN-1594 description: "wizard scaffolding may start in parallel; commit step needs backend live for E2E"). The QA subtask FN-1595 is blocked on FN-1590 for the same reason.
