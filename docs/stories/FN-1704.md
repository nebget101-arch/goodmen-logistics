# FN-1704: Story C — Real-time forced-logout via WebSocket

## Requirement
Kick a displaced session instantly instead of waiting for its next request to 401. The gateway currently rooms sockets only by `tenant:<id>`; add per-user/per-session targeting and a `session:revoked` event.

## Acceptance Criteria
- [ ] Gateway Socket.IO connection joins `user:<userId>` (and `session:<sid>`) rooms in addition to `tenant:<tenantId>`.
- [ ] Internal `/internal/ws/emit` bridge + `websocket.service.js` helper can target a user/session room.
- [ ] On takeover/remote-logout, emit `session:revoked` `{ reason }` to ONLY the displaced session/user room — no token/secret, never tenant-wide.
- [ ] Clean no-op if the displaced session has no live socket (Story B 401 fallback covers it).
