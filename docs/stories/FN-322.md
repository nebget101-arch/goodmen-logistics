### FN-327 (subtask)
**Migration file**: `20260327200000_fn327_investigation_tokens_and_response_fields.js`

**Changes:**
1. **New table `employer_investigation_tokens`**: id, past_employer_id (FK), driver_id (FK), token_hash (varchar 64, indexed), expires_at, created_by (FK → users), status (default 'active'), created_at
2. **Extended `employer_investigation_responses`**: 16 new columns — employed_as, employment_from/to, drove_cmv, vehicle_types (jsonb), reason_for_leaving, accidents (jsonb), drug_alcohol_history (jsonb), no_safety_history, other_remarks, completed_by_name/title, signature_data (jsonb), pdf_storage_key, pdf_file_name
3. **Extended `driver_past_employers`**: share_token_id, inquiry_email_sent_to, inquiry_created_by

All new columns nullable. Follows existing pattern with hasTable/hasColumn guards for idempotency.
