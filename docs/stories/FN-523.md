# FN-523: Auto-populate consent form capture fields from submitted employment application

## Requirement
After a driver submits their employment application, their personal data (Full Name, DL Number, DOB, State Issued, Last 4 SSN) should be stored in session and used to pre-populate the capture fields on each consent form in the onboarding packet.

## Acceptance Criteria
- [x] After employment application submit, key fields are stored in session: fullName, dateOfBirth, ssnLast4, driversLicenseNumber, stateOfIssue
- [x] Each consent form capture field is pre-populated from session data on render
- [x] Pre-populated values remain editable by the driver
- [x] Session data is cleared when the onboarding session ends (sessionStorage auto-clears on tab close)
- [x] If no session data exists (driver accesses consent forms directly), fields are blank

## Dependencies
- None

## Agent
frontend

## Implementation Summary
1. **Employment application** (`employment-application.component.ts`): Added `storeSessionDataForConsentForms()` method called after successful submit. Stores `fullName` (first + middle + last), `dateOfBirth`, `ssnLast4` (last 4 of raw SSN), `driversLicenseNumber`, and `stateOfIssue` (from first license entry) to `sessionStorage` under key `fn_onboarding_applicant`.

2. **Consent form** (`consent-form.component.ts`): Added `prefillFromSession()` method called in `ngOnInit`. Reads `fn_onboarding_applicant` from `sessionStorage` and pre-populates capture fields. Values remain editable via existing `[(ngModel)]` bindings.

Both methods silently catch errors if `sessionStorage` is unavailable.

## Files Changed
- `frontend/src/app/onboarding/employment-application/employment-application.component.ts`
- `frontend/src/app/components/onboarding-packet/consent-form/consent-form.component.ts`

## Key Decisions
- Used `sessionStorage` (not `localStorage`) so data auto-clears when the browser tab closes â€” no manual cleanup needed
- SSN stored as last 4 digits only (not full SSN) to minimize sensitive data in client storage
- Prefill is unconditional (all fields populated regardless of `captureFields` config) since the HTML template already conditionally renders only the needed fields
