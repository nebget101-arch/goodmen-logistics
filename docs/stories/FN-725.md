## Key Decisions
1. **`localStorage` for Last Used** — No backend endpoint for recent drivers; localStorage is persistent and zero-latency.
2. **`TruckOption` local interface** — Extends `EquipmentOption` without touching the service interface, future-proofing if the API returns `equipment_owner` on some trucks.
3. **Driver auto-fill** — Mirrors the pattern in the existing `selectInlineDriver()` in `loads-dashboard.component.ts`.
4. **Step validity = driver selected** — Truck and trailer are optional (some loads don't have equipment assigned at creation time).
5. **Click-outside via `@HostListener` on `document:click`** — Uses `ElementRef.nativeElement.contains()` to close all three dropdowns cleanly.
