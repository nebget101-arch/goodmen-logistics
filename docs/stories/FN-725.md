**Files created:**

- `load-wizard.component.html` — Progress bar, `<ng-content>` body slot, footer navigation buttons, unsaved-changes dialog
- `load-wizard.component.scss` — Full-screen overlay panel, step progress bar with active/completed/locked states
- `load-wizard.module.ts` — Feature module; declares and exports `LoadWizardComponent`

**Integration points:**
- `app.module.ts` — imports `LoadWizardModule`
- `loads-dashboard.component.ts` — added `showLoadWizard`, `wizardActiveStep`, `wizardStepValid`, `wizardDirty` state; `openLoadWizard()`, `closeWizard()`, `onWizardSave()`, `onWizardSaveAndNew()` methods
- `loads-dashboard.component.html` — "Manual Load Entry" button now calls `openLoadWizard()`; `<app-load-wizard>` mounted at bottom of template with placeholder step content

**AC delivered:**
- [x] Progress bar shows 4 steps (Basics / Stops / Driver & Equipment / Attachments); active step highlighted
- [x] Jump-to-step works only if previous steps valid (`canJumpTo()` guard)
- [x] Keyboard shortcuts functional: Enter=Next, Esc=Cancel, Cmd+S=Save, Cmd+Shift+S=Save&New
- [x] Unsaved-changes dialog on close (when `isDirty=true`)

**Design decisions:**
1. **Content projection via `<ng-content>`** — the wizard shell renders whatever the parent projects. Step components (`*ngIf="wizardActiveStep === n"`) live in the parent template; step subtasks (FN-733–736) fill these slots.
2. **Two-way binding on `activeStep`** — `[(activeStep)]="wizardActiveStep"` keeps step state in the dashboard component so the host can respond to navigation events.
3. **`ChangeDetectorRef.markForCheck()`** — used throughout because keyboard event handlers modify state outside Angular's default CD cycle.
4. **Forward navigation guard** — `canJumpTo(n)` checks `stepValid.slice(0, n).every(Boolean)`. The parent marks each step valid after required-field validation (implemented in step sub-tasks).

## Key Decisions
1. **Shell-first approach**: FN-732 delivers the skeleton; step content follows in FN-733–736. Placeholder `<div>` slots are shown until real step components are built.
2. **Feature module over standalone**: Matches existing project pattern (`LocationsAdminModule`) for consistent module structure.
3. **Keyboard listener on `document:keydown`**: Ensures shortcuts work even when focus is on step content inside the wizard body.
