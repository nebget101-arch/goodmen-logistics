# FN-680 — Schema Enhancements for Locations, Bins & Warehouse-to-Shop Supply

Extend the `locations` table with contact/timezone/operating-hours fields, enforce a `location_type` CHECK constraint, and add supporting schema for bins and warehouse-to-shop supply flows.

## Implementation Notes

### FN-686 (subtask)

**Branch:** `database/FN-686/locations-fields-and-check`

**File:** `backend/packages/goodmen-database/migrations/20260416100000_enhance_locations_table.js`

**What it does:**
1. Adds 8 new columns to `locations` (idempotent):
   - `city` VARCHAR(100)
   - `state` VARCHAR(50)
   - `zip` VARCHAR(20)
   - `phone` VARCHAR(30)
   - `email` VARCHAR(255)
   - `contact_name` VARCHAR(255)
   - `timezone` VARCHAR(100) DEFAULT `'America/New_York'`
   - `operating_hours` JSONB
2. Backfills `location_type`: uppercases all existing values and maps free-text variants (e.g. "Repair Shop" → "SHOP") to the nearest valid enum value. Rows with unmappable values are set to NULL and logged via `console.warn`.
3. Adds named CHECK constraint `locations_location_type_check` that enforces `location_type IN ('SHOP','YARD','DROP_YARD','WAREHOUSE','OFFICE','TERMINAL')` or NULL. Applied **after** backfill to avoid constraint violations.

**Down migration:** drops the constraint then the 8 columns in reverse order.

**Idempotency:** every step checks `hasColumn` before altering; constraint uses `DROP CONSTRAINT IF EXISTS` before `ADD CONSTRAINT`.
