## Implementation Notes

### FN-686 (subtask)

**Branch:** `database/FN-686/locations-fields-and-check`

**File:** `backend/packages/goodmen-database/migrations/20260416100000_enhance_locations_table.js`

**What it does:**
1. Adds 8 new columns to `locations` (idempotent):
   - `city` VARCHAR(100)
   - `state` VARCHAR(50)
   - `zip` VARCHAR(20)
   - `phone` VARCHAR(30)
   - `email` VARCHAR(255)
   - `contact_name` VARCHAR(255)
   - `timezone` VARCHAR(100) DEFAULT `'America/New_York'`
   - `operating_hours` JSONB
2. Backfills `location_type`: uppercases all existing values and maps free-text variants (e.g. "Repair Shop" → "SHOP") to the nearest valid enum value. Rows with unmappable values are set to NULL and logged via `console.warn`.
3. Adds named CHECK constraint `locations_location_type_check` that enforces `location_type IN ('SHOP','YARD','DROP_YARD','WAREHOUSE','OFFICE','TERMINAL')` or NULL. Applied **after** backfill to avoid constraint violations.

**Down migration:** drops the constraint then the 8 columns in reverse order.

**Idempotency:** every step checks `hasColumn` before altering; constraint uses `DROP CONSTRAINT IF EXISTS` before `ADD CONSTRAINT`.

### FN-687 (subtask)

**Branch:** `database/FN-687/location-bins-table`

**File:** `backend/packages/goodmen-database/migrations/20260416180000_create_location_bins.js`

**What it does:**
- Creates `location_bins` table with UUID PK, `tenant_id`, `location_id` FK (CASCADE), `bin_code`, `bin_name`, `bin_type` (SHELF|RACK|FLOOR|CABINET|FREEZER|OUTDOOR), `zone`, `aisle`, `shelf`, `position`, `capacity_notes`, `active`, and timestamps.
- Indexes: `uq_location_bins_code` UNIQUE(location_id, bin_code), `idx_location_bins_tenant`, `idx_location_bins_location` (location_id, active).

**Down migration:** drops indexes then table.

---

### FN-688 (subtask)

**Branch:** `database/FN-688/inventory-bin-id-backfill`

**File:** `backend/packages/goodmen-database/migrations/20260416300000_link_inventory_to_bins.js`

**What it does:**
- Adds `bin_id UUID NULL FK -> location_bins.id ON DELETE SET NULL` to `inventory` (keeps `bin_location` text for backward compat).
- Adds `bin_id_override UUID NULL FK -> location_bins.id ON DELETE SET NULL` to `receiving_ticket_lines`.
- Backfills: for each distinct (location_id, bin_location) in `inventory`, creates a `location_bins` row and sets `inventory.bin_id`. Same for `receiving_ticket_lines.bin_location_override`.

**Down migration:** drops FKs and columns (bin_location text unchanged).

---

### FN-689 (subtask)

**Branch:** `database/FN-689/warehouse-supply-rules`

**File:** `backend/packages/goodmen-database/migrations/20260416400000_create_warehouse_shop_supply_rules.js`

**What it does:**
- Creates `warehouse_shop_supply_rules` table with UUID PK, `tenant_id`, `warehouse_location_id` FK → locations (CASCADE), `shop_location_id` FK → locations (CASCADE), `notes`, `active`, and timestamps.
- Constraints: `uq_wssr_warehouse_shop` UNIQUE(warehouse_location_id, shop_location_id), `chk_wssr_no_self_supply` CHECK(warehouse_location_id <> shop_location_id).
- Indexes: `idx_wssr_tenant`, `idx_wssr_warehouse`, `idx_wssr_shop`.
- Application layer is responsible for enforcing that warehouse_location_id.type = 'WAREHOUSE' and shop_location_id.type = 'SHOP'.

**Down migration:** drops indexes, UNIQUE index, then table.
