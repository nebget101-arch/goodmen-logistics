# FN-411: [Frontend] Update fuel reporting and overview to support Product Type and Category dimensions

## Summary
Added product type and category dimension support across the fuel module: overview dashboard, transaction list, and expense reports.

## Implementation Summary

### Backend Changes
- `routes/fuel.js` — Overview endpoint now returns `byProductType` aggregation (gallons, amount, count per product type for 30-day window)
- `routes/reports.js` — Fuel expense summary CTE now groups by product_type, producing separate rows for "Fuel", "Fuel (Def)", "Fuel (Reefer)" instead of a single "Fuel" row

### Frontend Changes
- `fuel.model.ts` — Added `byProductType` to `FuelOverview` interface
- `fuel-overview.component.html` — New "Spend by Product Type" panel showing color-coded cards (blue=Diesel, green=DEF, purple=Reefer) with amount, gallons, and transaction count
- `fuel-overview.component.css` — Styles for product type breakdown grid and color-coded labels
- `fuel-transactions.component.html` — Added Category column + filter dropdown, split row "S" badge indicator on date cell
- `fuel-transactions.component.ts` — Added `categoryFilter` property, passed to API
- `fuel-transactions.component.css` — Split badge styling

### Key Decisions
- Product type breakdown uses card layout (not chart) to avoid adding chart library dependency
- Split row indicator is a small purple "S" badge next to the date — minimal footprint, tooltip explains
- Reports grouping: diesel rows remain as "Fuel" (no suffix) for backwards compatibility; only DEF and reefer get suffixed labels
- No double-counting risk: reports SUM is unchanged, just adds GROUP BY product_type
