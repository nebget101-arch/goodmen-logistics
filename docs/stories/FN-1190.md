## Implementation Summary (FN-1222)

### New Files
- `backend/microservices/ai-service/src/voice/voice.controller.js` — handles `POST /api/voice/incoming` (Twilio webhook → TwiML), `POST /api/voice/consent` (digit gather → consent decision), and `WS /api/voice/stream` (Media Streams ASR feed)
- `backend/microservices/ai-service/src/voice/asr.service.js` — `AsrSession` EventEmitter that consumes Twilio Media Streams binary audio and emits `start`, `transcript` (interim/final), and `stop` events
- `backend/microservices/ai-service/src/voice/consent.service.js` — `recordConsent` / `getConsent` backed by the `voice_consent` table (FN-1223)
- `backend/microservices/ai-service/src/voice/did-mapping.service.js` — `lookupTenant` backed by the `did_mapping` table (FN-1223)
- `backend/microservices/ai-service/src/routes/voice.routes.js` — Express router mounted at `/api/voice`
- `backend/microservices/ai-service/src/handlers/__tests__/voice.controller.spec.js` — 7 unit tests, all passing

### Modified Files
- `backend/microservices/ai-service/server.js` — added `http.Server` + `WebSocketServer` for Media Streams upgrade; mounted `/api/voice` router
- `backend/microservices/ai-service/package.json` — added `ws ^8.18.0`

### Key Decisions
- TwiML generated as raw XML strings (no Twilio SDK needed for basic responses)
- `ws` package chosen for WebSocket server (minimal dependency)
- `AsrSession` is a stub that buffers audio and emits placeholders; Deepgram/Whisper integration is a follow-on to avoid blocking voice bridge launch on external ASR vendor contract
- Module references (not destructuring) used in voice.controller.js to keep the seam testable without a full DI framework

## Implementation Summary (FN-1223)

### New Files
- `backend/packages/goodmen-database/migrations/20260611120000_create_voice_consent.js` — `voice_consent` table: `caller_id`, `tenant_id`, `granted`, `granted_at`, `ip`, `recording_id`, `created_at`; composite index on `(tenant_id, caller_id, created_at DESC)` and sparse index on `recording_id WHERE NOT NULL`
- `backend/packages/goodmen-database/migrations/20260611120100_create_did_mapping.js` — `did_mapping` table: `did` (UNIQUE), `tenant_id`, timestamps; index on `tenant_id`

### Key Decisions
- No FK constraints to `tenants` — voice module intentionally decoupled from auth-users schema to allow separate deployment
- `recording_id` nullable — only set after caller grants consent and recording begins
- `did` unique constraint enforced at DB level (one DID per tenant globally)

## Deployment Handoff

### Services to restart/redeploy
- `fleetneuron-db-migrations` (Render job — run migrations first)
- `fleetneuron-ai-service` (after migrations and env vars from FN-1224)

### DB migrations to run
- `20260611120000_create_voice_consent.js`
- `20260611120100_create_did_mapping.js`

### Environment/config changes (FN-1224 to add to Render)
- `VOICE_BASE_URL` — public HTTPS URL of ai-service (for TwiML action URLs)
- `VOICE_WS_URL` — public WSS URL for Media Streams (may equal VOICE_BASE_URL with wss:// scheme)
- `TWILIO_AUTH_TOKEN` — for webhook signature validation (middleware hook in voice.routes.js)
- `TWILIO_ACCOUNT_SID` — for reference

### Deploy order
1. Apply DB migrations (voice_consent + did_mapping)
2. Set env vars (FN-1224)
3. Deploy `fleetneuron-ai-service`
4. Configure Twilio webhook → `POST https://<VOICE_BASE_URL>/api/voice/incoming`

### Post-deploy verification
- Call the provisioned Twilio DID → hear consent message → press 1 → hear prompt
- Call from unmapped DID → hear rejection message + hangup
- `SELECT COUNT(*) FROM voice_consent; SELECT COUNT(*) FROM did_mapping;` — both return 0, no error
