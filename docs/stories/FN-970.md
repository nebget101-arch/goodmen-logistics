# FN-970 — Provision qa-smoke@fleetneuron.test user on dev with full-permission role

**Type:** Story
**Epic:** [FN-969 — Enterprise Playwright UI testing framework — Phase 1 smoke coverage](https://nebget101.atlassian.net/browse/FN-969)
**Status:** In Progress
**Labels:** `agent:database`, `framework:playwright`, `scope:frontend-smoke`

## Requirement

Playwright `globalSetup` needs a stable, full-permission test account on dev to perform programmatic API login and persist `storageState`. Without this, Story 4 (auth fixture) cannot execute.

## Scope

- Seed account `qa-smoke@fleetneuron.test` in the auth-users DB on dev.
- Assign role/permissions equivalent to "full-access" so the smoke suite can navigate every NAV_TOP_LINKS / NAV_SECTIONS / user-menu route without permission redirects.
- Document `QA_SMOKE_USER` / `QA_SMOKE_PASSWORD` in `tests/e2e-playwright/.env.example` and provision them as GitHub Actions repository secrets. Credentials are NEVER stored in Render service envs.
- Confirm login works via curl against gateway `/api/auth/login`.

## Acceptance Criteria

- [x] Account exists on dev DB (seeded by migration `20260427120000_seed_qa_smoke_user.js` once `QA_SMOKE_USER` / `QA_SMOKE_PASSWORD` are set on the migration runner)
- [ ] `curl POST /api/auth/login` with the credentials returns 200 + JWT (validated under FN-987)
- [ ] Token decodes to a role with full permissions (validated under FN-987)
- [x] Credentials documented as env vars (NEVER committed) — see `tests/e2e-playwright/.env.example`
- [x] User flagged as test/QA so it can be excluded from billing/usage reports — `users.is_test = true`

### FN-985 (subtask)

**Branch:** `database/FN-985/seed-qa-smoke-user`

**What changed**
- New Knex migration: `backend/packages/goodmen-database/migrations/20260427120000_seed_qa_smoke_user.js`.
  - Adds `users.is_test BOOLEAN NOT NULL DEFAULT false` (idempotent via `hasColumn`) and `idx_users_is_test` index so billing/usage reports can exclude QA accounts.
  - Reads `QA_SMOKE_USER` / `QA_SMOKE_PASSWORD` from the migration runner's env. Hashes the password with `bcrypt.hash(password, 10)` — same cost factor used by `routes/auth.js`, `auth-users-service/routes/users.js`, and `scripts/reset-user-password.js`.
  - Upserts the user by `LOWER(email)` (or username) so re-running the migration rotates the password rather than failing on conflict.
  - Sets legacy `users.role = 'admin'` (broadest value in the existing enum check) and inserts a `user_roles` row mapping the user to the RBAC `super_admin` role (which `01_rbac_seed.js` assigns every permission). This satisfies both legacy role gates and the modern RBAC service.
  - Marks the seeded user `is_test = true` and `is_active = true`.
- If `QA_SMOKE_USER` / `QA_SMOKE_PASSWORD` are not set in the env (e.g. on staging/prod runners), the migration logs a warning and exits without inserting anything. The `is_test` column is still ensured.
- `down()` deletes the QA smoke user (and its `user_roles` rows) but leaves `is_test` in place.

**Why username = email**
The legacy login endpoint (`backend/packages/goodmen-shared/routes/auth.js`) calls `getUserByUsername(value)`, which matches on `username = $1 OR LOWER(email) = LOWER($1)` (see `backend/packages/goodmen-shared/internal/user.js`). Setting `users.username = users.email = 'qa-smoke@fleetneuron.test'` lets Playwright login with whichever field name the test fixture passes. The endpoint reads from `req.body.username`, so the smoke fixture must send `{"username": "qa-smoke@fleetneuron.test", "password": "…"}`.

**What did NOT change (intentional)**
- No changes to `routes/auth.js` or RBAC seeds — full permissions come from the existing `super_admin` role definition.
- No `tenant_id` is set; the login flow falls back to `tenantContextService.getDefaultContextForUser` when `users.tenant_id` is null.
- No new env vars on Render services. `QA_SMOKE_USER` / `QA_SMOKE_PASSWORD` live only on the migration runner and the GitHub Actions secrets per FN-986.

- **Environment/config changes:**
  - **`fleetneuron-db-migrations` (Render):** add `QA_SMOKE_USER=qa-smoke@fleetneuron.test` and `QA_SMOKE_PASSWORD=<chosen value>` *only on the dev environment*. Do not propagate to staging/prod.
  - **GitHub Actions repository secrets** (FleetNeuronAPP repo): add `QA_SMOKE_USER` and `QA_SMOKE_PASSWORD` matching the values above. See `tests/e2e-playwright/.env.example`.
  - **Other Render services:** none — these credentials must NOT be set on any application service.
- **Cache invalidation:** none.
- **Feature flags:** none.
- **Deploy order:**
  1. Land FN-985 PR → dev migration job picks up the new migration.
  2. DevOps: set `QA_SMOKE_USER` / `QA_SMOKE_PASSWORD` on the dev migration runner and redeploy it to apply the seed.
  3. Repo admin adds matching GitHub Actions secrets (FN-986 runbook).
  4. FN-987 verifies login + token decode.
- **Post-deploy verification:**
  ```sh
  curl -X POST -H 'Content-Type: application/json' \
    -d '{"username":"qa-smoke@fleetneuron.test","password":"<value>"}' \
    https://fleetneuron-logistics-gateway.onrender.com/api/auth/login
  ```
  Expect `200` with a JWT. Decode (e.g. `jwt.io` or `jq -R 'split(".")[1] | @base64d | fromjson'`) and confirm the `role` claim is `admin`. The user's RBAC `user_roles` mapping to `super_admin` grants every permission. This validation is owned by FN-987.
