# FN-970 — Provision qa-smoke@fleetneuron.test user on dev with full-permission role

**Type:** Story
**Epic:** [FN-969 — Enterprise Playwright UI testing framework — Phase 1 smoke coverage](https://nebget101.atlassian.net/browse/FN-969)
**Status:** Backlog
**Labels:** `agent:database`, `framework:playwright`, `scope:frontend-smoke`

## Requirement

Playwright `globalSetup` needs a stable, full-permission test account on dev to perform programmatic API login and persist `storageState`. Without this, Story 4 (auth fixture) cannot execute.

## Scope

- Seed account `qa-smoke@fleetneuron.test` in the auth-users DB on dev.
- Assign role/permissions equivalent to "full-access" so the smoke suite can navigate every NAV_TOP_LINKS / NAV_SECTIONS / user-menu route without permission redirects.
- Document `QA_SMOKE_USER` / `QA_SMOKE_PASSWORD` in `tests/e2e-playwright/.env.example` and provision them as GitHub Actions repository secrets. Credentials are NEVER stored in Render service envs.
- Confirm login works via curl against gateway `/api/auth/login`.

## Acceptance Criteria

- [ ] Account exists on dev DB
- [ ] `curl POST /api/auth/login` with the credentials returns 200 + JWT
- [ ] Token decodes to a role with full permissions
- [ ] Credentials documented as env vars (NEVER committed)
- [ ] User flagged as test/QA so it can be excluded from billing/usage reports

### FN-986 (subtask)

- Created this parent story doc.

**What did NOT change (intentional)**

**Manual repo-admin step required after FN-985 lands**
1. GitHub repo → Settings → Secrets and variables → Actions → New repository secret.
2. Add `QA_SMOKE_USER` = `qa-smoke@fleetneuron.test`.
3. Add `QA_SMOKE_PASSWORD` = the password value chosen by FN-985 (whatever is seeded into the dev auth-users DB).
4. Confirm both secrets appear in the repository secrets list.
