### FN-1259 (subtask)

**Branch:** `frontend/FN-1259/driver-portal-shell`

**Files changed (8):**
- `frontend/src/app/components/driver-portal/driver-portal.routes.ts` — lazy-loaded route config
- `frontend/src/app/components/driver-portal/shell/shell.component.ts/html/scss` — portal shell with responsive dark-theme nav
- `frontend/src/app/components/driver-portal/incident-list/incident-list.component.ts/html/scss` — incident list with status badges, time-since-opened, filter bar
- `frontend/src/app/components/driver-portal/incident-list/incident-list.component.spec.ts` — spec covering render, filter, timeSince, error, empty, and array-wrapped API shapes

**Key decisions:**
- Standalone components throughout (no new NgModule)
- `ChangeDetectionStrategy.OnPush` on both components
- Status badge palette (NEW/TRIAGED/DISPATCHED/RESOLVED/CANCELLED) all meet WCAG 4.5:1 contrast on the dark card background
- All interactive touch targets set to `min-height: 44px` (WCAG 2.5.5/2.5.8)
- Filter is purely client-side over already-loaded incident list; no extra HTTP calls
- `timeSince()` returns minutes < 60, hours < 24, then days
- Route registered at `driver-portal/*` via `DRIVER_PORTAL_ROUTES` — caller adds the lazy `loadChildren` entry to `app-routing.module.ts`
