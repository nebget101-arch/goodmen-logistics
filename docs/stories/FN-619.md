# FN-619: Custom Business Spans for Critical Flows (Phase 2)

## Requirement
Add custom OpenTelemetry spans to critical business flows so traces show business context. Settlement, fuel import, FMCSA lookup, AI calls, and authentication all get named spans with domain-specific attributes.

## Acceptance Criteria
- [ ] `withSpan()` utility in `goodmen-shared/utils/tracing-spans.js`
- [ ] Settlement spans: `settlement.create_draft` + 5 child spans with attributes
- [ ] Fuel import spans: `fuel.import.stage`, `fuel.import.commit` + matching child spans
- [ ] FMCSA span: `fmcsa.lookup` with cache_hit attribute
- [ ] AI spans: `ai.chat` with token usage attributes
- [ ] Auth spans: `auth.login`, `auth.token_verify`
