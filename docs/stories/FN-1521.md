## Implementation Summary

### FN-1536 (subtask) â€” AI

Researched and produced the 50-state + DC sales-tax rules reference dataset at `docs/reference/state-sales-tax-rules-2026.csv`. The CSV is the seed source for the `state_tax_rules` table built by FN-1537 and consumed by the tax engine in FN-1538.

**Approach.** State-by-state across DoR publications, statutes, and regulations. Aggregators (Avalara, Tax Foundation) used only as triage; primary state sources resolved every classification. Five spot-check states verified hand-to-DoR (TX, CA, FL, NY, PA).

**Key finding.** The labor-taxability axis is the dominant variable. Jurisdictions cluster:
- **No-tax (5)**: AK, DE, MT, NH, OR â€” all flags `false`, rate `0`
- **Labor exempt when separately stated (~26)**: AL, AZ, CA, CO, GA, ID, IL, IN, MA, MD, ME, MI, MN, MO, NE, NV, ND, OH, OK, RI, SC, TX, VA, VT
- **Labor fully taxable (~16 + DC)**: AR, CT, DC, FL, HI, IA, KS, KY, LA, MS, NC, NJ, NM, NY, PA, SD, TN, WA, WI, WV, WY
- **Edge cases**: MA inconsequential rule, VA diagnostic-labor exemption (2023), KY 2018 statute change, FL/NJ "labor taxable when parts furnished"

**Sanity-check confirmation** (FN-1536 acceptance criteria):
- TX â†’ labor `false`, parts `true`, rate `0.0625` (Tex. Tax Code Â§151.0048; Comptroller Pub. 94-113)
- AK / DE / MT / NH / OR â†’ all flags `false`, rate `0`

**Methodology + caveats**: see `docs/reference/state-sales-tax-rules-2026.md` for full methodology, the `fees_taxable` discussion, spot-check evidence table, and limitations (local rates, sector exemptions, warranty work).

**CSV consumer contract** (for FN-1537 seed script):
- 51 rows + header. UTF-8. RFC 4180 quoting (notes contain commas â†’ quoted).
- Booleans lowercase (`true`/`false`) â€” Postgres-friendly without coercion.
- `default_sales_tax_rate` is decimal (`0.0625`, not `6.25`).
- `effective_from` = `2026-01-01` for all rows.
- Recommended Postgres column types listed in the markdown.

### FN-1538 (subtask) â€” Backend

State-aware tax engine in `recomputeWorkOrderTotals`. Replaces the per-line
`taxable` flag (kept as legacy fallback) with `state_tax_rules` per-component
flags driven off `locations.state`.

**Rate selection priority** (matches the FN-1538 algorithm spec):
1. `tax_rate_override === true` â†’ user-supplied `tax_rate_percent / 100`.
2. Active rule found in `state_tax_rules` â†’ `rule.default_sales_tax_rate`.
3. Location with state but no rule â†’ legacy `0.085` + warn log.
4. Location with no state â†’ legacy `0.085` + warn log.
5. No `location_id` â†’ rate `0` (skip tax) + warn log.

**Per-component taxability**: rule flags win. Without a rule, the legacy
per-line `taxable` flag is used so historical numbers don't shift silently
for tenants that never had state rules. Manual rate override still respects
the rule's per-component flags (a TX override of 10% taxes only parts at
10%, not labor or fees).

**Discount apportionment**: discounts reduce the taxable subtotal in
proportion to its share of the overall subtotal. A 10% off coupon reduces
the taxable base by 10%, not the full discount amount.

**Schema additions** (`20260510120000_add_workorder_tax_breakdown.js`):
- `tax_breakdown` JSONB nullable â€” `{ rule_state, rate, override, fallback_reason, labor_taxable, parts_taxable, fees_taxable, labor_subtotal, parts_subtotal, fees_subtotal, taxable_subtotal, taxable_after_discount, discount_amount, tax_amount }`.
- `tax_rate_override` BOOLEAN default false â€” disambiguates "user explicitly set 0%" from "use the rule" when `tax_rate_percent = 0`.

**API surface**:
- `getWorkOrderById` already returns the full `work_orders` row, so `tax_breakdown` is exposed automatically (Knex/pg returns JSONB as a parsed object).
- `createWorkOrder` / `updateWorkOrder` / `updateCharges` accept `taxRateOverride: boolean` from the payload.
- `computeWorkOrderTotals` and `loadStateTaxRule` are exported for unit testing.

**Tests** (`backend/packages/goodmen-shared/test/work-orders-tax.test.js`,
10 cases, all passing under `node:test`):
- TX (parts only @ 6.25%), CA (all @ 7.25%), MT (none), FL (parts @ 6%).
- Manual override with rule flags applied.
- No-rule legacy 8.5% fallback against per-line flags.
- No-location skip (rate 0).
- Location with no state â†’ legacy 8.5%.
- Discount apportioned proportionally.
- Explicit 0% override on CA (zero tax even though rule rate is 7.25%).
