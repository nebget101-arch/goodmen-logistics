# FN-1116: Reports — Report-context chat side-panel

## Requirement
Right-side drawer launched from the report header lets users ask follow-up questions about the report being viewed. Caches the report dataset block server-side for cost-efficient multi-turn use.

## Acceptance Criteria
- [ ] `POST /api/ai/reports/chat` exists in `ai-service` and is gateway-routed
- [ ] Dataset block uses Anthropic prompt caching; `cache_read_input_tokens > 0` on 2nd message
- [ ] History truncation enforced server-side (default last 10, env-configurable)
- [ ] Data truncation when row count exceeds threshold; response includes `_truncated: true`
- [ ] Requires `REPORTS_SHOP` (escalated from VIEW); 403 otherwise
- [ ] FE drawer slides in from right; closes on Esc and overlay click
- [ ] Conversation cleared when user navigates to different report
- [ ] Drawer keyboard-trappable while open (a11y)
- [ ] Endpoint documented in Swagger
- [ ] Screenshots in `docs/stories/evidence/FN-1116/`

**Files added/changed:**

- `frontend/src/app/reports/pages/report-view/report-view.component.{ts,html}` — adds **Ask AI** trigger, hosts the drawer, clears the conversation when the user navigates between reports

**Behavior implemented (acceptance):**
- Right-side drawer slides in (180ms ease-out); closes on Esc and overlay click; close restores focus to the previously-focused element.
- Conversation lives in the drawer's component state. The parent `report-view` resets `[open]` to false when `config.key` changes; the drawer also wipes its own buffer in `ngOnChanges` when `reportKey` changes.
- Focus trap: while open, document-level keydown handler intercepts `Tab` / `Shift+Tab` and cycles between the first and last focusable elements inside the drawer panel.
- Sends `{ reportKey, filters, data, history, message, summary }` to `POST /api/ai/reports/chat` (gateway-routed → ai-service per FN-1137).
- Defensive client-side data truncation: outgoing `data` is capped at the first **100 rows**. Server is the source of truth for full truncation enforcement.
- Theme: matches existing dark-theme reports palette (slate / blue accent); no new colors introduced.

**Unit spec coverage** (`report-chat-drawer.component.spec.ts`):
- open/close (overlay click, Esc)
- send appends user msg, then assistant reply on success; isSending toggles
- multi-turn: prior history is included, current message is excluded from history
- error path: error text shown, user message preserved, in-flight guard
- data truncation to 100 rows
- conversation reset on `reportKey` change
- focus trap (Tab at last focusable cycles back, preventDefault called)

**Verification:** `tsc --noEmit` clean across the frontend project. Full `ng build` / `ng test` not run locally — environment Node is v14 and the Angular CLI requires ≥ v18.13. CI on the integration branch is the next type/lint/test gate. Browser screenshot deferred to FN-1138 (QA) per the parent story.

## API Contract
**Request** `POST /api/ai/reports/chat`
```json
{ "reportKey": "direct-load-profit", "filters": {...}, "data": [...], "history": [{"role":"user","content":"…"}], "message": "which dispatcher had the lowest margin?" }
```
**Response**
```json
{ "reply": "Dispatcher X at 8.2% margin…", "generatedAt": "...", "usage": { "cache_read_input_tokens": 5200, "_truncated": false } }
```
