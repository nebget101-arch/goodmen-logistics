### FN-1137 (subtask)
**AI chat endpoint with cached dataset block.** Branch: `ai/FN-1137/chat-endpoint` â†’ integrated into `integration/FN-1116`.

**Files added/changed:**
- `backend/microservices/ai-service/src/handlers/reports-chat-handler.js` (new) â€” multi-turn chat handler with prompt caching, history/data truncation, RBAC.
- `backend/microservices/ai-service/src/handlers/__tests__/reports-chat-handler.test.js` (new) â€” 30 unit tests (caching shape, truncation, RBAC, error paths).

- `backend/microservices/ai-service/src/services/auth-context.js` â€” adds `reports.shop` to `admin`/`company_admin` and adds `shop_manager`/`parts_manager` entries (mirrors `06_shop_clerk_seed.js`).

**Behavior implemented (acceptance):**
- `POST /api/ai/reports/chat` returns `{ reply, generatedAt, usage, meta }`.
- Dataset block uses Anthropic prompt caching: the system array carries one cached `text` block (`cache_control: { type: 'ephemeral' }`) containing `{reportKey, filters, summary, rowCount, originalRowCount, truncated, data}`. Static instructions live in a separate uncached system block. Verified by unit test "2nd-turn surfaces cache_read_input_tokens (warm cache)" returning `cache_read_input_tokens: 5200`.
- History truncation: server keeps the last `AI_REPORTS_CHAT_MAX_HISTORY` messages (default 10). Malformed entries (missing role/content, non-user/assistant role) are dropped.
- Data truncation: when `data.length > AI_REPORTS_CHAT_MAX_DATA_ROWS` (default 200), the handler slices to the limit and the response carries `usage._truncated: true`.
- RBAC: enforced via `hasReportsShop(req.user)` â€” `super_admin` always allowed; otherwise the caller must hold `reports.shop`. Holding only `reports.view` returns 403 (escalation enforced); the FN-1134 anomalies endpoint, which uses `reports.view`, is unaffected.
- Error handling: 400 on invalid `reportKey` / missing `message` / malformed body; 403 on RBAC denial; 502 on Anthropic upstream failure; 503 if Anthropic is not configured.
- Swagger: full annotation added in `ai-router.js` (request schema, response, error codes).

**Unit spec coverage** (`reports-chat-handler.test.js`):
- `hasReportsShop` super_admin / explicit perm / view-only-rejected / undefined user
- request validation: bad reportKey, missing message, non-array data, non-object filters, non-array history, message length cap
- `truncateHistory` keeps last N, drops malformed, empty/null
- `truncateData` passes through under limit, slices and flags over limit, non-array

- handler data truncation: `usage._truncated` true/false
- handler RBAC: 403 view-only, 403 missing user, 200 super_admin
- handler 400 paths: invalid reportKey, missing message
