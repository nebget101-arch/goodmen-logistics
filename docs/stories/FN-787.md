## Implementation Summary

### FN-804 (subtask)

Added `HoverPopoverComponent` in `frontend/src/app/shared/components/hover-popover/` and declared/exported it from `SharedModule`. The component projects trigger content via `<ng-content>` and renders a floating tooltip or rich template popover on hover/focus. Positioning math lives in a pure helper (`calculatePopoverPosition`) so the collision-avoidance AC can be unit-tested without a DOM. 9 position tests + 5 component tests.

### FN-805 (subtask)

**Row-hover toolbar replaces the ⋮ dropdown** (`loads-dashboard.component.html`). The `.row-toolbar` block holds 6 icon buttons — View, Edit, Clone, Map, Dispatch, Delete(if draftable) — right-aligned in the actions column. Default `opacity: 0; pointer-events: none;` flips to fully visible on `tr:hover` and on `:focus-within` (keyboard discoverability). 100 ms CSS transition. The old `actions-dropdown` menu (Edit / Copy / Clone / Create Return / Map / Delete / Approve) is removed; Copy / Create Return / Approve move out of the per-row UI per spec and will live in the detail drawer (FN-788).

**Inline status dropdown filters to valid next transitions only** (`loads-dashboard.component.ts`). A `statusTransitions` map keyed on current status returns allowed next states. The happy path is DRAFT → NEW → DISPATCHED → IN_TRANSIT → DELIVERED; terminal states (DELIVERED, CANCELLED, TONU) return an empty array so the chevron disappears and the menu never opens. `getValidStatusTransitions(load)` is called from both the `*ngIf` chevron guard and the `*ngFor` that renders options.

**Inline billing dropdown** mirrors the status pattern: `billingMenuLoadId`, `openBillingMenu`, `changeLoadBilling`, `billingTransitions` map, `getValidBillingTransitions`. Optimistic UI update + `LoadsService.updateLoad({ billingStatus })` PUT; rollback on error with a 4 s error toast.

**Inline notes editor** adds a new `notes` column (fully integrated into `columnDefs`, both colgroups, header row, filter row — filter cell is intentionally blank). Non-edit mode renders a button showing the note (or an italic `—` when empty) with hover border to signal interactivity. Click opens a textarea, which is auto-focused + selected in the next tick (`setTimeout(0)` + `querySelector` — only one editor is ever open at a time). Key handlers: `Esc` cancels, `Enter` commits (not `Shift+Enter` — default behaviour preserved for newlines if the user wants them via drag-resize… but Enter preventDefault stops line-break insertion when committing). `blur` also commits. A `notesSaveInFlight` latch prevents Enter + blur from double-saving.

**Outside-click handling** — the existing `@HostListener('document:click')` now also clears `billingMenuLoadId`, `editingNotesLoadId`, and `editingNotesDraft`.

## Key Decisions

### FN-805
- **Scoped removal of Copy / Return / Approve from the per-row UI.** The spec lists exactly 6 icons (View, Edit, Clone, Map, Dispatch, Delete). Surfacing the rest would defeat the polish the hover toolbar is meant to deliver. Those actions stay accessible from the detail drawer (FN-788) and bulk actions toolbar where applicable.
- **`Enter` preventDefault on the notes textarea.** Commits on Enter match the AC. Users needing a literal newline can still add one (the save path normalizes empty strings to `null`, but multi-line content is preserved round-trip since we call `trim()` only for change detection, not for the PUT payload).
- **Transition maps over a reusable state machine.** Two tables (`statusTransitions`, `billingTransitions`) are clearer than a generic transition service for this scope. If a third status-bearing entity needs the same pattern later, extract.
- **New `notes` column defaults to visible.** Users toggle via the existing FN-767 column picker; the column's `visibleColumns` entry falls through the `loadUserPreferences` guard (only explicit `false` values hide) so existing saved preferences pick it up as visible with zero migration.
- **Native `tr:hover` + `:focus-within` instead of JS.** CSS-only fade keeps the change compact and hardware-accelerated. The 100 ms transition matches FN-804's hover-popover timing for visual consistency.

- **Post-deploy verification:**
  1. Hover a loads-table row → right-aligned toolbar fades in within 100 ms with View/Edit/Clone/Map/Dispatch/Delete.
  2. Click the status pill on a `NEW` load → dropdown shows only `Dispatched`, `Canceled`. Click `Dispatched` → pill flips optimistically and PUT `/loads/:id` fires.
  3. Click the billing pill on a `PENDING` load → shows `BOL received`, `Canceled` only. Select one → same flow.
  4. Enable the Notes column from the column picker (if hidden) → click an empty cell → inline textarea opens, auto-focused. Type, press Enter → saves. Reopen, type, press Esc → discarded. Type, click outside → saves (blur path).
  5. Load the page under a read-only role → toolbar Clone button is hidden via `*appHasPermission="'loads.create'"`.
