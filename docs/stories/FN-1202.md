## Implementation Summary

### FN-1253 (backend subtask) — GPS heartbeat endpoint + matching service

**Files created:**
- `backend/packages/goodmen-shared/services/vendor-position.service.js` — Throttled heartbeat
  ingestion into `vendor_positions` table (1/30 s per vendor, checked via DB `recorded_at`).
- `backend/packages/goodmen-shared/services/vendor-matching.service.js` — Haversine-distance
  matching against vendors with fresh GPS positions (≤5 min). Filters on `active` status,
  `capacity > 0`, tenant scoping (private + marketplace), and required skills (JSONB array
  subset check). Ranks by composite score (70 % distance, 30 % capacity). 60-second in-process
  cache per `incident_id`; invalidated by `invalidateCache`.
- `backend/packages/goodmen-shared/routes/vendor-heartbeat.js` — Three endpoints:
  `POST /:id/heartbeat`, `POST /match`, `POST /match/invalidate`. Mounted under
  `/api/logistics/vendors` in logistics-service alongside the existing `roadsideVendorsRouter`.
- `backend/packages/goodmen-shared/services/vendor-matching.service.test.js` — Jest unit tests
  covering radius filter, skills filter, ranking correctness, cache hit, and cache invalidation.

**Updated:**
- `backend/microservices/logistics-service/server.js` — Added `vendorHeartbeatRouter` mount at
  `/api/logistics/vendors` (after existing `roadsideVendorsRouter`).

**Key decisions:**
- No PostGIS: production Postgres lacks the extension (confirmed in FN-1250 migration notes).
  Distance calculated in JS via Haversine; `vendor_positions` uses `lat`/`lng` DECIMAL columns
  (FN-1254 database subtask will create the table following this same pattern).
- In-process match cache: a single `Map` keyed by `incident_id` with TTL eviction on read.
  Sufficient for single-instance Render deployments; can be promoted to Redis if horizontal
  scaling is needed.
- Throttle via DB timestamp rather than in-memory state so it survives service restarts and works
  correctly across multiple instances (though current infra is single-instance).
- Gateway routing: `/api/logistics/vendors` proxy already registered by FN-1249; no gateway
  change required for the heartbeat or match sub-paths.
