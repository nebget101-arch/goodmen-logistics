# FN-1395: Redesign sidebar company switcher — rich identity, search, keyboard nav

## Requirement
Replace the current `<app-ai-select>`-based company switcher in the sidebar (and mobile topbar) with a purpose-built popover that shows clear company identity (avatar + full name + MC chip), supports search and keyboard navigation, and matches the AI dark theme. The current control truncates "Goodmen Logistics · MC 13…" to a single line and offers no search — unusable for users with many operating entities and visually unfinished compared to the rest of the shell.

## Acceptance Criteria
See full AC list in the Jira issue (FN-1395). Summary:
- [ ] Trigger card: avatar (initials), full name (2-line clamp), MC chip on its own line
- [ ] Read-only trigger when user has only one accessible entity
- [ ] Popover with search + listbox of company rows (avatar + name + MC chip + checkmark for selection)
- [ ] Search filters by name AND MC number, case-insensitive
- [ ] Keyboard nav: ArrowUp/Down/Home/End/Enter/Escape; focus returns to trigger on close
- [ ] ARIA: `aria-haspopup="listbox"`, `aria-expanded`, `role="listbox"`/`role="option"`/`aria-selected`
- [ ] Admin "All Entities" appears at top with section divider; non-admin does not see it
- [ ] Loading skeleton replaces "Loading…" text; no layout shift
- [ ] Mobile: compact pill in topbar; popover renders as bottom sheet `≤768px`; search input does NOT auto-focus on mobile
- [ ] Theme: existing CSS variables only; no new color literals (except 6-color initials palette)
- [ ] Existing `onOperatingEntityChange()` behavior preserved exactly

### FN-1396 (subtask)
Built standalone `CompanySwitcherComponent` (`frontend/src/app/shared/company-switcher/`) — Angular `OnPush` component with `[entities]`, `[selectedEntityId]`, `[showAllEntities]`, `[loading]`, `[compact]` inputs and `(entitySelected)` output. Trigger card renders avatar + name (2-line clamp) + MC chip; popover hosts a search input and `role="listbox"` of rows with avatars, MC chips, and a check on the selected entry. Single-entity case renders a non-interactive read-only trigger; loading case renders an animated skeleton; popover stays closed while loading. Search filters by name AND MC number, case-insensitive. Keyboard nav implemented for ArrowUp/Down/Home/End/Enter/Escape; focus returns to the trigger on close. ARIA roles applied per AC. Mobile (`compact` AND viewport `≤768px`) renders the popover as a fixed bottom sheet (max-height 70vh) and intentionally does not auto-focus the search input. Initials are derived from the first two whitespace-split tokens; avatar color comes from a deterministic FNV-style hash of `entity.id` mapped onto a fixed 6-color dark-theme palette.

Wired into `app.component.html` for both desktop sidebar and mobile topbar. `app.component.ts` now exposes plain fields `companySwitcherEntities` and `companySwitcherShowAll` (no getter-based bindings — FN-317 RCA), rebuilt from the same `context$()` subscription that previously fed the dual `operatingEntityOptions[/Mobile]` arrays. Those arrays and the `AiSelectOption` import were removed since they're no longer used in this component. Obsolete CSS selectors (`.company-switcher`, `.company-switcher-label`, `.company-switcher-select` + variants, `.company-switcher-loading`, and the mobile media-query rule that targeted `.company-switcher-select`) deleted from `frontend/src/styles.css`; the `.mobile-company-switcher` wrapper class is retained as a flex layout container.

## Key Decisions
- Scoped styles use existing CSS variables (`--border-color`, `--text-dark`, `--text-light`); the only new color literals are the 6-entry initials palette permitted by AC.
- "All Entities" row is rendered first when `showAllEntities` is true and gets a section divider above it (visible after it, before the entity rows). Filtering removes the "All Entities" entry only when its label doesn't match the query, so admins can search across both contexts.
- Initials hash is a small FNV-style polynomial hash of `entity.id` (falls back to name) — deterministic and non-cryptographic; collisions are acceptable since the palette is intentionally small.
- Used `(document:click)` host listener for outside-click detection rather than CDK overlay, to keep the bundle change minimal and avoid changing the existing sidebar layout context.
