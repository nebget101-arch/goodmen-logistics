# FN-482 — Redesign Fuel Cards: Hierarchical Account > Cards > Driver Assignment

## Overview
Separates fuel_card_accounts (company-level accounts like Comdata, EFS, TCS) from
individual fuel cards. Introduces a new `fuel_cards` table so the hierarchy becomes:
  Account -> Cards -> Driver Assignments

## Subtasks

### FN-483 — Add fuel_cards table and backend endpoints (Backend)
**Branch:** `backend/FN-483/fuel-card-hierarchy`
**Status:** In Progress

#### Migration
- Created `fuel_cards` table with UUID PK, tenant_id, fuel_card_account_id (FK), card_number_masked, card_number_last4, status, notes, timestamps
- Indexes: tenant_id, fuel_card_account_id, unique(tenant_id, card_number_masked)
- Added `fuel_card_id` (nullable UUID FK) to `fuel_card_driver_assignments`

#### Endpoints added/updated
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/fuel/accounts/:accountId/cards` | List cards under an account |
| POST | `/api/fuel/accounts/:accountId/cards` | Create a card under an account |
| PATCH | `/api/fuel/accounts/cards/:cardId` | Update card fields |
| GET | `/api/fuel/cards` (existing) | Now includes `card_count` per account |
