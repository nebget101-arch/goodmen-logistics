## Implementation Summary

### FN-1152 (subtask) â€” backend trends aggregator
- New gateway-local route `GET /api/insights/trends?range=7d` that JWT-authenticates the caller, derives `tenant_id` from the token, and returns four series (`loadVolume`, `maintenance`, `onTimePct`, `fuelCost`).
- Each series is `{ actual: [{ date, value }], predicted: [{ date, value }] }`. `actual` is the past 7 UTC days ending today (oldest â†’ newest); `predicted` is the next 7 days, computed via simple linear regression over the actual values.
- Sparse-data graceful nulls: missing day â†’ `value: null`. Insufficient history (<2 actual points) â†’ predicted is all-null. Per-series query failure is isolated; the failed series gets all-null arrays and is reported in `upstreamErrors[]` while the other three series still serve.
- Cache: in-memory map keyed by `tenantId:range`, 10-minute TTL. `?refresh=true` bypasses and refills the cache.
- Aggregator and cache are constructor-injected (`buildTrendAggregator({ knex, cache, now })`, `buildTrendCache({ ttlMs, now })`) so tests can inject a fake knex and a deterministic clock.

## Key Decisions

### FN-1152 (subtask)
- **Direct DB access at the gateway** rather than HTTP fan-out to downstream services. Reasons: (a) the AC says "derives trends from existing tables", (b) there is no existing 7-day time-series endpoint on logistics/vehicles/integrations services, and (c) the briefing/ask predecessors already established a `goodmen-shared/config/knex` import path from the gateway via `routes/stripe.js`. The aggregator is constructor-injected so it remains test-friendly.
- **Linear regression for the `predicted` array** rather than a heavier ML approach. Suitable for a 7-day sparkline preview, predictable, and degrades cleanly to nulls under sparse data. Easy to swap behind the same interface later (e.g., when FN-1132 adds maintenance forecasting).
- **`upstreamErrors[]`** mirrors the briefing-aggregator contract so the frontend (FN-1154) can render a "stale source" badge per series without coupling to specific failure modes.
- **`?refresh=true`** override on the route follows the existing `/api/ai/briefing?refresh=true` pattern so the frontend has a uniform cache-bust knob across Control Center widgets.
