# FN-419: AI-Powered Toll Data Normalization for CSV Imports

## Overview
Enable the tolls module to leverage the AI service for automatic CSV column
mapping and data normalization during toll transaction imports.

## Sub-tasks

### FN-435: BE -- POST /import/ai-normalize route and mapping profile storage
**Status:** In Progress
**Files changed:**
- `backend/packages/goodmen-shared/routes/tolls.js`

**Route:** `POST /api/tolls/import/ai-normalize`

**Request body:**
```json
{ "batchId": "uuid", "headers": ["col1", "col2"], "sampleRows": [[...], [...]] }
```

**Behaviour:**
1. Validates `batchId`, `headers[]`, and `sampleRows[]`.
2. Proxies `headers` + `sampleRows` to the AI service at
   `POST ${AI_SERVICE_URL}/api/ai/tolls/csv-normalize` with a 30-second timeout.
3. If the AI returns `overallConfidence >= 0.8` and a `columnMapping`, the route
   auto-saves (upserts) a record in `toll_import_mapping_profiles` scoped to the
   tenant and batch.
4. Returns the AI result (column mapping, confidence scores, normalized sample,
   warnings) plus a `mappingProfileSaved` flag.

**Error responses:**
| Status | Condition |
|--------|-----------|
| 400 | Missing or invalid body fields |
| 401 | No tenant context |
| 502 | AI service unreachable or upstream 5xx |
| 504 | AI service timeout (30s) |
