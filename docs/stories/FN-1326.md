### FN-1339 (subtask)

#### Implementation Summary
Added the four-tier severity ramp as CSS custom properties on the global `:root` in `frontend/src/styles.css` (the project keeps its global tokens as CSS variables — adding a SCSS partial would require wiring SCSS into the build pipeline, which isn't current convention). Tokens follow the existing AI dark-theme palette (slate / cyan / amber / red-400) so the badges and rows visually belong with `.card`, `.topbar`, `load-detail-drawer`, etc.

Built two standalone components under `frontend/src/app/shared/components/`:

- `app-severity-badge` — small uppercase pill with optional pip; reads `--sev-{tier}-{fg|bg|border|pip}` so theme overrides are zero-cost. Exposes static `SeverityBadgeComponent.rank(severity)` for parent ranking logic so FN-1329 (Action Queue) doesn't reinvent the order map.
- `app-grouped-alert-row` — full row used by FN-1329. Inputs cover the API-contract fields (severity, category, message, count, primaryAction). Outputs: `(primaryActionClick)`, `(dismiss)`, two-way `[(expanded)]`. The expanded body is exposed via `<ng-content>` so each consumer renders its own target list (vehicles, drivers, etc.).

Both components are registered in `SharedModule` (imports + exports) so non-standalone consumers can pick them up by simply importing `SharedModule`. They're already standalone, so the `severity-preview` page imports them directly.

Sample preview lives at `/dev/severity-preview` and is gated by `environment.production` in `app-routing.module.ts` — the route entry resolves to `[]` in the prod build, so it never ships to customers. The route uses `loadComponent` so it adds zero bundle weight unless visited.

#### Files Changed (FN-1339)
- `frontend/src/styles.css` — added four severity ramps (12 new CSS custom properties).
- `frontend/src/app/shared/components/severity-badge/{ts,html,scss,spec.ts,index.ts}` — new component.
- `frontend/src/app/shared/components/grouped-alert-row/{ts,html,scss,spec.ts,index.ts}` — new component.
- `frontend/src/app/dev/severity-preview/{ts,html,scss}` — sample preview page.
- `frontend/src/app/shared/shared.module.ts` — declare + export the two new standalone components.
- `frontend/src/app/app-routing.module.ts` — non-prod-only `/dev/severity-preview` route.

#### Key Decisions
- **CSS variables, not a SCSS partial.** Spec mentioned `_severity-tokens.scss`, but the project's global theming lives in `styles.css` as `:root` variables (`--ai-text-strong`, `--ai-accent-info`, etc.). Adding a SCSS partial would require importing SCSS into the global pipeline, which is not configured. CSS custom properties achieve the same goal without a build-config change.
- **`SeverityBadgeComponent.rank()` lives on the badge component**, not in a separate util. FN-1329 needs to sort grouped alerts by severity; placing the rank map on the badge avoids a circular import or a third file just to hold the constant.
- **Foreground color choices**: red-400 (#f87171), amber-400 (#fbbf24), cyan-300 (#67e8f9), slate-300 (#cbd5e1). All measured ≥ 4.5:1 contrast against `--background` (#020617) — passes WCAG AA. The pip uses a slightly darker hue (red-500/amber-500/cyan-500/slate-400) for a subtle rim effect.
