### FN-431 (subtask)

**Implementation Summary:**
Added 4 new endpoints to `backend/packages/goodmen-shared/routes/tolls.js`:
1. `POST /api/tolls/import/upload` — accepts CSV via multer, creates `toll_import_batch`, parses headers + sample rows
2. `POST /api/tolls/import/commit` — accepts batch_id + mapped rows, deduplicates via sha256 hash, auto-matches toll_devices to resolve truck_id/driver_id, inserts toll_transactions, creates toll_transaction_exceptions for unmatched rows
3. `GET /api/tolls/import/mapping-profiles` — lists tenant's saved mapping profiles
4. `POST /api/tolls/import/mapping-profiles` — saves new mapping profile with column_map JSON

**Key Decisions:**
- Reuses `parseFileBuffer` from `fuel-parser.js` for CSV/XLSX parsing (shared utility)
- Dedupe hash: sha256 of `tenant_id|transaction_date|amount|plaza_name|device_number|entry|exit`
- Auto-matching: looks up `toll_devices` by device_number or plate_number to resolve truck_id/driver_id
- Unmatched rows create `match_failed` exceptions in `toll_transaction_exceptions`
- Commit is transactional — all-or-nothing per batch

**Files Changed:**
- `backend/packages/goodmen-shared/routes/tolls.js` — added multer, crypto, 4 import endpoints
