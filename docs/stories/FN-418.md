### FN-431 (subtask) ŌĆö Backend endpoints

#### Endpoints added

| Method | Path | Description |
|--------|------|-------------|
| POST | `/import/upload` | Upload CSV/XLSX, parse headers + sample rows, create batch record |
| POST | `/import/commit` | Commit mapped rows with dedup, auto-match devices, create exceptions |
| GET | `/import/mapping-profiles` | List tenant mapping profiles |
| POST | `/import/mapping-profiles` | Create a new mapping profile |

#### Implementation notes
- CSV/XLSX parsing uses `xlsx` package (same pattern as `fuel-parser.js`)
- File upload via `multer` (memory storage, 10 MB limit)
- Dedupe hash: MD5 of `tenant_id|provider|transaction_date|amount|plaza_name`
- Auto-match looks up `device_number` and `plate_number` in `toll_devices`
- Unmatched rows create `toll_transaction_exceptions` with type `match_failed`
- Audit trail via `toll_import_batch_rows` for every row
- Batch counters updated on completion: `total_rows`, `success_rows`, `failed_rows`, `duplicate_rows`
