### FN-1674 (subtask â€” database)
Two Knex migrations create the share-link schema that backs the rest of Story E:

- **`load_share_links`** â€” one row per generated tracking link.
  - `id` UUID PK; `load_id` UUID NOT NULL â†’ `loads.id` (`ON DELETE CASCADE`); `created_by` UUID NOT NULL â†’ `users.id` (`ON DELETE RESTRICT`).
  - `token_hash` TEXT NOT NULL with a **UNIQUE** constraint (`uq_load_share_links_token_hash`) â€” the raw 32-byte base64url token is shown to the broker once and never stored; only its SHA-256 hash lives here, so public-page lookups are an indexed O(1) hash match.
  - Lifecycle/audit columns: `created_at` (default now), `expires_at` (nullable â€” API sets default 7 days post-delivery), `revoked_at` (nullable), `view_count` INTEGER NOT NULL default 0, `last_viewed_at` (nullable).
  - `reveal_options` JSONB NOT NULL default `'{}'` â€” driver/vehicle/breadcrumbs/route toggles consumed by the public page (Story F).
  - Secondary index `idx_load_share_links_load_id` for list/manage-by-load queries.
- **`load_share_link_views`** â€” one row per public-page view (audit / "viewed N times").
  - `id` UUID PK; `share_link_id` UUID NOT NULL â†’ `load_share_links.id` (`ON DELETE CASCADE`); `viewed_at` (default now); `ip_hash` and `user_agent_hash` TEXT nullable (hashed only â€” no PII in the clear).
  - Composite index `idx_load_share_link_views_link_viewed_at` on `(share_link_id, viewed_at DESC)` for newest-first view listing.

**Verification:** ran `knex migrate:latest` against an isolated scratch DB (all 205 migrations apply clean), confirmed both table structures with `\d` (columns, FKs, unique constraint, indexes match spec), and confirmed `migrate:down` drops both tables cleanly and re-apply succeeds.

### FN-1675 (subtask â€” backend)
The share-link API + token primitives, on top of the FN-1674 schema. Token-only auth (no PIN/email gate â€” intake decision).

- **`services/share-link-service.js`** (new) â€” security-sensitive primitives, kept out of the route so Story F can reuse them:
  - `generateToken()` â†’ 32 random bytes (`crypto.randomBytes`) as base64url; `hashToken()` â†’ SHA-256 hex (what we store/look up by). The raw token is returned to the broker **once** on create and never persisted.
  - `hashValue()` â†’ SHA-256 of viewer IP / user-agent (NULL for empty input), used by the audit helper.
  - `normalizeRevealOptions()` â†’ coerces the four optional toggles (`driverName`, `vehicleNumber`, `breadcrumbs`, `routeLine`) to booleans, defaulting **OFF**. Location/ETA/timeline are always-on and intentionally not stored as toggles.
  - `defaultExpiry()` / `resolveExpiry()` â†’ default expiry = **delivery date + 7 days** (intake decision), falling back to `now + 7d` when the load has no delivery date; an explicit `expiresAt` from the UI selector overrides.
  - `recordShareLinkView()` â†’ inserts one `load_share_link_views` row and bumps `view_count` / `last_viewed_at`. Exported for the Story F (FN-1658) public read endpoint â€” the only consumer that records views.
- **`routes/load-share-links.js`** (new) â€” three endpoints, restricted to `admin` / `dispatch`, all tenant-scoped (and operating-entity-scoped) via `req.context`:
  - `POST /api/loads/:id/share-links` â€” validates the load belongs to the caller tenant, mints a token, stores its hash + `expires_at` + `reveal_options`, returns `{ ...link, token, url }` (raw token + `/track/<token>` URL) once with `201`.
  - `GET /api/loads/:id/share-links` â€” lists a load's links with `view_count` / `last_viewed_at` and a derived `status` (`active` / `expired` / `revoked`). Never leaks `token_hash`.
  - `DELETE /api/share-links/:id` â€” sets `revoked_at` (idempotent); join to `loads` enforces tenant ownership. Returns `404` for cross-tenant or malformed-UUID ids.
- **`logistics-service/server.js`** â€” mounted the router at `/api` (after `/api/loads`) so it owns both `/loads/:id/share-links` and the bare `/share-links/:id` path; `loadsRouter` falls through on the unmatched sub-path.

**Verification:** `node --test routes/load-share-links.test.js` â€” 10/10 pass (auth 401, role 403, create + once-only token + default-expiry + reveal defaults, explicit reveal toggles, invalid `expiresAt` 400, cross-tenant 404, malformed-id 404, list without hash leak, revoke + idempotency, cross-tenant revoke 404). `node --check` clean on all three source files.

### FN-1676 (subtask â€” frontend)

- **`ShareLinkModalComponent`** (`load-detail-drawer/share-link-modal/`) â€” a hand-rolled dark-theme overlay (no shared modal component exists) with three views:

  - **created** â€” the one-time public URL with a copy button (`navigator.clipboard` + textarea fallback) and a "copy it now, it can't be shown again" notice.

- **FN-317 compliance** â€” the `app-ai-select` `[options]` binds a stable `expiryOptions` array (never a getter), avoiding the infinite change-detection loop.

## Key Decisions
- Token-only auth (no PIN/email gate) â€” intake decision. Default expiry 7 days post-delivery.
- **(FN-1676) URL shown once:** the UI can't re-display an existing link's URL because the raw token is never stored â€” by design, an existing active link shows view-count/expiry/revoke only; getting a fresh URL means revoke + create.
- **(FN-1676) Reveal defaults:** driver name + vehicle # OFF (intake privacy decision); breadcrumb trail OFF (granular GPS history); route line ON (planned polyline, not sensitive). Live location, ETA, and status timeline are always on and not toggleable.
- **(FN-1676) Expiry contract:** create body sends `{ expiryDays, revealOptions }` where `expiryDays` is days-after-delivery (`null` = no expiry); the backend computes the absolute `expires_at`.
- `token_hash` stored SHA-256 hashed with a UNIQUE constraint (provides the O(1) lookup index); raw token never persisted.
- Viewer `ip_hash` / `user_agent_hash` stored hashed only â€” enough to aggregate viewers without retaining PII.
- `load_id` and `share_link_id` FKs use `ON DELETE CASCADE` (links and their view audit die with the parent); `created_by` uses `RESTRICT` (preserve attribution).
