# FN-537: Auto-populate truck and trailer in load edit modal when driver is selected

## Requirement
On the **Edit Load** modal, when a driver is selected (or already assigned), the Truck and Trailer fields should automatically populate with the truck and trailer currently assigned to that driver in the Drivers dispatch view.

Currently, even when a driver is already assigned to a load, the Truck and Trailer fields in the edit modal show blank ‚Äî the user must re-type them manually.

## Acceptance Criteria
- [x] When a driver is selected in the Edit Load modal, the Truck field auto-populates with that driver's currently assigned truck unit number
- [x] When a driver is selected, the Trailer field auto-populates with the driver's currently assigned trailer unit number
- [x] If a driver has no truck/trailer assigned, fields remain blank (no error)
- [x] On load of the edit modal (driver already set), truck and trailer pre-fill from driver's dispatch assignment if load fields are blank
- [x] User can still override the auto-populated truck/trailer by manually editing the field

### FN-538 ‚Äî Frontend Implementation

**`loads.component.ts`**:
- Replaced plain `driverName` text input in edit modal with `app-ai-select` bound to `editLoad.driverId` with `(ngModelChange)="onEditDriverChange($event)"`
- Added `onEditDriverChange(driverId)`: finds driver in cached `this.drivers` list ‚Üí patches `editLoad.truck = driver.truckUnitNumber` and `editLoad.trailer = driver.trailerUnitNumber`; also updates `editLoad.driverName`
- Added `prefillTruckTrailerFromDriver(driverId, overwrite)`: shared helper used by both `onEditDriverChange()` (overwrite=true) and `openEditModal()` (overwrite=false ‚Äî only fills blank fields). Uses the cached drivers list first; falls back to `apiService.getDriver()` if driver not found in cache
- Updated `openEditModal()` to call `prefillTruckTrailerFromDriver(load.driverId, false)` when driver is set but truck/trailer are blank

**`loads.component.html`**:
- Replaced `Driver Name` plain text input with `app-ai-select` for `editLoad.driverId`
- Added `Truck (unit #)` and `Trailer (unit #)` editable text fields with "Auto-filled from driver" placeholder

### FN-539 ‚Äî Backend Audit

**Audit result**: No backend changes needed.

`GET /api/drivers/:id` uses `SELECT d.*` which already includes `truck_id` and `trailer_id`. `transformRow()` converts them to `truckId`/`trailerId`. Explicitly declared in response object with comment for clarity.
