# FN-537: Auto-populate truck and trailer in load edit modal when driver is selected

## Requirement
On the **Edit Load** modal, when a driver is selected (or already assigned), the Truck and Trailer dropdowns should automatically populate with the truck and trailer currently assigned to that driver in the Drivers dispatch view.

Currently, even when a driver is already assigned to a load, the Truck and Trailer dropdowns in the edit modal show "Select truck" / "Select trailer" — blank.

## Acceptance Criteria
- [ ] When a driver is selected in the Edit Load modal, the Truck dropdown auto-populates with that driver's currently assigned truck (from `drivers.truck_id`)
- [ ] When a driver is selected, the Trailer dropdown auto-populates with the driver's currently assigned trailer
- [ ] If a driver has no truck/trailer assigned, dropdowns remain blank (no error)
- [ ] On load of the edit modal (driver already set), truck and trailer pre-fill from existing load data or the driver's current dispatch assignment as fallback
- [ ] User can still override the auto-populated truck/trailer by manually selecting a different one

### FN-539 — Backend Audit

**Audit result**: No backend changes were required.

`GET /api/drivers/:id` uses `SELECT d.*` which includes `truck_id` and `trailer_id` from the `drivers` table. `transformRow()` (case-converter util) converts them to camelCase → `truckId` / `trailerId` in the JSON response.

**Change made**: Explicitly surfaced `truckId` and `trailerId` in the response object with a comment so the contract is obvious to FN-538 (frontend) and future callers:

```js
truckId: driver.truckId || null,
trailerId: driver.trailerId || null,
```
