### FN-393 (subtask) — Fix date picker z-index

**Implementation Summary:**
Added `position: relative` and `z-index: 10` to the `.header-filters` row in the loads dashboard table. This creates a stacking context for the filter row that sits above the `<tbody>`, allowing the date picker panel (which already has `z-index: 200` in its own component CSS) to render on top of the table body.

**Files Changed:**
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.scss`

**Key Decisions:**
- Fixed at the filter row level rather than modifying the date picker component, since the root cause is the table body painting over the thead due to table rendering order.
