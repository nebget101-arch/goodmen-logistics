# FN-1402: Loads list — Table view visual redesign (sticky header, gridlines, theme tokens, status-cell overflow)

## Requirement
Polish the **Table** tab of the Loads list so it visually matches the AI dark theme and is comfortable to scan, **without removing or changing any functionality**. FN-1352 already delivered the Cards and Kanban views (which look good) and the view toggle. This story is scoped strictly to the Table view tab — Cards and Kanban must remain untouched.

## Acceptance Criteria
- [ ] Table tab header row stays visible while the body scrolls; header columns remain pixel-aligned with body columns through resize, sort, density change.
- [ ] Zebra striping or row separators visible in all three density modes; consistent with AI dark theme.
- [ ] Subtle vertical column separators present using theme tokens.
- [ ] Status / Billing cell content (pill + ETA + indicator) does not clip at default column widths in compact, comfortable, and spacious density.
- [ ] Rate, Pickup Date, Completed Date columns use tabular numerals and right-align cleanly across rows.
- [ ] No new hex / `rgba(…)` color literals introduced in `loads-dashboard.component.scss` for Table-view styles; raw colors in realtime in-transit indicators replaced with AI theme CSS vars.
- [ ] Density toggle adjusts horizontal cell padding as well as vertical.
- [ ] Hover, selected (bulk-checked), and keyboard-focused row states are visually distinct in dark theme.
- [ ] Cards view and Kanban view are pixel-identical to the FN-1352 baseline (visual regression check via existing `docs/stories/evidence/FN-1352/*` screenshots).
- [ ] Column resize (FN-1060) still works on every non-pinned column header; reset action still clears persisted widths.
- [ ] Column visibility menu, density menu, sort, search, smart filter chips, header-row filters, bulk select, inline status/billing/notes editing, hover previews, row hover toolbar, virtual scroll all behave identically to pre-redesign.
- [ ] No backend, API, or data-shape changes in this PR.
- [ ] Verified in browser at `/loads` on dev across compact/comfortable/spacious density and across all three view tabs (Table re-styled; Cards/Kanban unchanged). Screenshots committed under `docs/stories/evidence/FN-1402/`.

## Implementation Summary

### FN-1403 (subtask)
Polish pass on the Loads dashboard **Table** view tab. Cards and Kanban templates and SCSS regions are untouched.

- **Sticky header**: `.table--header` (the dual-table header that sits outside the `cdk-virtual-scroll-viewport`) now uses `position: sticky; top: 0; z-index: 5;` with an opaque `--ai-surface-deep` background and a 1px shadow seam to `--border-color`. Body table continues to scroll inside the CDK viewport; header sticks to the page viewport so it stays visible when the user scrolls the surrounding shell. The dual-table architecture is preserved so FN-1060 column-resize handles keep the header `<colgroup>` and body `<colgroup>` in lockstep via the existing `columnWidths` map.
- **Zebra striping + vertical separators**: `.table--body tbody tr:nth-child(even) > td` paints the new `--ai-surface-zebra` token; `td + td` and `th + th` get a 1px `--border-color` left border so columns visually separate without a stronger gridline.
- **Status / Billing cell overflow fix**: increased default px widths in `loads-dashboard.component.ts` for `status` (110 → 140), `billing` (110 → 130), and `rate` (90 → 100). User-resized widths persisted via `patchLoadsDashboard({ columnWidths })` are unaffected (existing prefs win at restore time). Pill + ETA chip + chevron now fit on a single inline row at compact / comfortable / spacious without clipping.
- **Tabular numerals + right-aligned Rate**: `.cell-num` (Pickup Date, Completed Date, ETA chip) and `.cell-rate` (Rate) classes added to the Table-view body markup. Header `th.th-rate` right-aligns to match the body. ETA chip already used `tabular-nums` via the new SCSS rule.
- **Theme-token sweep on realtime in-transit indicators**: `.in-transit-dot` and `.in-transit-eta` (status-cell pulse + ETA chip) now read from `--ai-in-transit-pulse`, `--ai-in-transit-pulse-soft`, `--ai-in-transit-eta-fg`, `--ai-in-transit-eta-bg`, `--ai-in-transit-eta-border`. `row-delivered`, `row-cancelled`, `row-selected` row tints route through `--ai-row-delivered-bg`, `--ai-row-cancelled-bg`, `--ai-row-selected-bg`.
- **Density horizontal padding**: introduced `--loads-cell-px` CSS variable on `.loads-dashboard`. Base `th` and `td` rules bind `padding-left` / `padding-right` to it. Density modes set both `--loads-row-height` (existing) and `--loads-cell-px` (new): compact = 8px, comfortable = 12px, spacious = 16px. Vertical padding keeps the existing per-mode block.
- **Hover / selected / focus states**: row `:hover` and `:focus-within` paint `--ai-row-hover-bg` with an inset 3px `--ai-row-hover-stripe` left edge. `tr.row-selected > td` background uses `--ai-row-selected-bg` and is excluded from the hover override (`:not(.row-selected)`) so selection visually wins over transient hover.

## Key Decisions

- **Kept the dual-table architecture** instead of collapsing to a single `<table>` with `position: sticky thead`. The single-table approach would have required moving CDK virtual scroll onto the `<tbody>` (CDK supports it but the `itemSize` math gets brittle once `<thead>` is in the same scroll context) AND would risk regressing FN-1060 column resize, which currently relies on two `<colgroup>`s sharing one `columnWidths` map. The dual-table sticky-header approach is a one-line CSS change with no markup churn.
- **Increased `status` / `billing` default widths instead of restructuring the cell layout**. The ticket offered three options for the overflow fix; bumping the default width is the lowest-risk change that preserves the inline-edit CDK Overlay flow, the FN-818 confidence chip, and the FN-813 in-transit dot + ETA chip without touching the Cards-view layout (which the ticket explicitly forbids changing).
- **Introduced `--ai-surface-zebra` and `--ai-row-*` tokens in `styles.css`** rather than inlining `rgba()` values in the component SCSS. The ticket allows new tokens with justification — these are semantic surfaces (zebra row, hovered row, selected row, delivered row, cancelled row) that didn't exist in the AI palette and would otherwise reappear as raw `rgba()` literals every time a row state needs styling. Aliases keep the SCSS scannable for future contributors and pass the `grep -nE '#[0-9a-fA-F]{3,8}|rgba?\('` audit in the Table-view region.
- **Hover excludes `.row-selected`** via `:not(.row-selected)` so selected rows keep their tint when the user hovers them (selection is persistent state; hover is transient — selection should win the cascade).
