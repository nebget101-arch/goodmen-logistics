## Implementation Summary

### FN-625 (Create shared OTel tracing initializer)
- Created `backend/packages/goodmen-shared/config/tracing.js` — shared initializer all 9 services will import
- Accepts `serviceName` param, initializes NodeSDK with OTLP HTTP exporters for traces and metrics
- Auto-instrumentations enabled: http, express, pg, knex, dns (fs/net disabled as noisy)
- Graceful no-op if `OTEL_EXPORTER_OTLP_ENDPOINT` not set (safe for local dev without SigNoz)
- Graceful shutdown on SIGTERM/SIGINT flushes pending spans
- Added 7 `@opentelemetry/*` dependencies to goodmen-shared package.json

## Files Changed
- `backend/packages/goodmen-shared/config/tracing.js` (new — 107 lines)
- `backend/packages/goodmen-shared/package.json` (added 7 OTel deps)

## Key Decisions
- **HTTP exporter** (not gRPC) — simpler, works through Render's HTTP proxy without gRPC config
- **Graceful no-op** pattern — if endpoint not set, just logs and returns (no crash, no overhead)
- **Enhanced DB reporting** enabled on pg instrumentation — includes SQL query text in spans for debugging
- **fs/net instrumentations disabled** — too noisy, adds volume without business value
- **60s metric export interval** — balances freshness vs overhead for Render's Starter plan
