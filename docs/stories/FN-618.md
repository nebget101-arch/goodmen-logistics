# FN-618: OpenTelemetry Auto-Instrumentation for All Backend Services (Phase 1)

## Requirement
Add OpenTelemetry auto-instrumentation to all 9 backend services (gateway + 8 microservices) with **self-hosted SigNoz on Render** (ClickHouse + OTel Collector + Dashboard). This is the foundation — auto-instrumentation captures HTTP requests, PostgreSQL queries, Express middleware, and external HTTP calls without modifying existing code. Cost: ~$45-55/mo.

## Acceptance Criteria
- [ ] Self-hosted SigNoz deployed on Render (ClickHouse + OTel Collector + Dashboard)
- [ ] Data retention: traces 15 days, metrics 30 days
- [x] Shared tracing initializer in `goodmen-shared/config/tracing.js`
- [x] All 9 services have `tracing.js` + `require('./tracing')` as first line
- [ ] No `SIGNOZ_ACCESS_TOKEN` needed — self-hosted uses Render networking
- [ ] Gateway forwards W3C trace context headers (`traceparent`, `tracestate`)
- [ ] Docker-compose updated with OTel env vars
- [ ] All 9 services appear in self-hosted SigNoz Dashboard
- [ ] Distributed traces show gateway -> service -> DB chain

- Created `tracing.js` in each of the 9 service directories
- Each tracing.js uses `path.join()` to resolve the shared initializer (follows existing codebase pattern)

## Key Decisions
- **Direct path.join() instead of package name** — matches existing codebase pattern where services resolve shared modules via `path.join(__dirname, ...)` rather than `require('@goodmen/shared/...')`
- **No package.json changes** — OTel deps live in goodmen-shared which is already linked; gateway uses direct path resolution
- **tracing.js before dotenv** — OTel SDK must initialize before any modules load so auto-instrumentation can monkey-patch http, pg, etc.
