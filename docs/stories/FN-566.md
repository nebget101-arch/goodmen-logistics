# FN-566: Equipment Owner % resets to 0 on page reload — not persisted or returned by backend

## Bug Report
**Observed:** After entering an Equipment Owner % (e.g. 44%) and saving, navigating away and returning shows EO% = 0.

---

## Root Cause

`GET /api/drivers/:id` in `goodmen-shared/routes/drivers.js` queries:
- `drivers` table (base fields including `pay_basis`, `pay_rate`, `pay_percentage`)
- `driver_payee_assignments` (payee details)
- `expense_responsibility_profiles` (expense config)

But **never** `driver_compensation_profiles`, which is the only table that holds `equipment_owner_percentage`. So on reload the frontend receives `equipmentOwnerPercentage: undefined`, which maps to `null` and renders as 0.

The DB migration (`20260401_add_equipment_owner_percentage.js`), POST/PUT save logic, and frontend UI/mapping were all already correct. Only the GET load path was broken.

## Implementation Summary

Added a `driver_compensation_profiles` query in `GET /api/drivers/:id`:

```sql
SELECT percentage_rate, equipment_owner_percentage, profile_type, pay_model
FROM driver_compensation_profiles
WHERE driver_id = $1
  AND is_active = true
ORDER BY created_at DESC
LIMIT 1
```

Added `equipmentOwnerPercentage` to the response object from the profile result. The frontend `buildDriverFromSource()` at line 1360 already maps `source.equipmentOwnerPercentage ?? source.equipment_owner_percentage` — no frontend change needed.
