## Implementation Summary
_FN-1646 (frontend):_ Full rewrite of the Roadside Service AI board to the Claude Design spec.

- **Layout**: replaced the flat `.header`/`.hero-pill`/`.layout` 3-column grid with an `.ai-header` (kicker → H1 + `emergency_home` icon → subtitle + `New Call` primary) and a `320px` left rail + detail-pane grid.
- **Stepper-driven detail pane**: mounted `<app-status-stepper>` driving New → Triaged → Dispatched → Resolved. The four `setStatus` sibling buttons are gone; the stepper (and one primary CTA per pane) drives advancement. `currentStepKey` is derived from `selectedCall.status`; `activeStepKey` tracks the expanded panel and is moved by stepper clicks and summary-chip edits.
- **Summary chips**: completed steps collapse to `<app-summary-chip>` rows with an edit affordance (`editStep`).
- **Side-rail tabs**: Timeline / Location / Attachments / Public Link consolidated into `<app-side-rail-tabs>` with `[data-rail-pane]` panes.
- **Fields**: all 10 form `<select>` + 2 new left-rail filters → `<app-ai-select>` with stable `readonly` option arrays (no getter-based `[options]` — avoids the FN-317 CD loop). Inputs/textareas → `.form-input` / `.form-textarea` inside `<label class="form-field">`. Numeric ETA / confidence / amount → `.num-field` + `.unit` chip (min / % / $). `<app-ai-sparkle>` marks AI-written values (symptoms, recommended action, confidence, AI Q/A log).
- **Caller Profile**: read-mode `.read-field` rows with an inline pencil `.btn-icon` that flips the panel to edit mode (no longer duplicates the Create Call form).
- **Copy + toasts**: "Start a new roadside call", "No open calls. Take a breath.", "Timeline starts when you advance this call."; `ToastService.success(...)` on triage/dispatch/resolve/link/upload (replacing the old silent `successMessage`).
- **Loading**: skeleton call-card shells built from `<app-ai-skeleton>` replace the "Loading..." text row.
- All endpoints/behavior preserved (`RoadsideService` calls unchanged).

## Key Decisions
- **Panel-per-step model**: `activeStepKey` selects exactly one expanded `.ai-panel` (Caller Profile / Triage / Dispatch / Resolve). Stepper clicks and chip "edit" change which step is expanded; the stage's save action is the single `.btn-primary`.
- **Component-scoped theme classes**: `.ai-panel`, `.panel-head`, `.form-field`, `.lbl`, `.pill`, `.btn-icon`, `.ai-header` are defined in the component CSS (the established per-page pattern — they are not global). `.btn-primary`/`.btn-secondary`/`.btn-sm`/`.form-input`/`.num-field`/`.read-field` come from global `styles.css`; the stepper/summary-chip/side-rail-tabs/toast come from `SharedModule`.
- **No new colors**: every hex traces to the canonical palette in `docs/AI_THEME_VISUAL_REFERENCE.md` / `theme.css`.
- **Caller edit persistence**: there is no backend endpoint to PATCH caller fields on an existing call, so the inline pencil flips the fields to editable inputs bound to the loaded object (ephemeral). Persisting caller-profile edits would need a new backend endpoint — **out of scope** for this UI-only subtask (see Open Items).
