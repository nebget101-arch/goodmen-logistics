## Implementation Summary

### FN-1469 (subtask) â€” database
Added an idempotent migration that supplies the columns `work-orders.service.js` and `work-orders-hub.js` already read/write but the schema did not have. PostgreSQL only reports the first missing column per failed statement, which is why `requested_by_user_id` masked the rest. The migration uses `hasTable` + `hasColumn` guards so it is safe to apply on any environment, including ones where parts of the schema were already added by `20260310101000_add_multi_mc_scope_columns.js` (which independently added `tenant_id` and `operating_entity_id`).

Verified against the dev DB: `up â†’ up â†’ down â†’ down â†’ up` was fully idempotent. `down()` intentionally drops only the columns this migration owns (`cost_type`, `requested_by_user_id`) and the three indexes it creates â€” `tenant_id` / `operating_entity_id` belong to the prior migration and are preserved on rollback so we don't over-revert.

## Key Decisions
- **`cost_type` default `'BILLABLE'`** â€” matches the ticket and the consumer in `work-orders-hub.js` lowercases before comparing to `'internal'`, so any non-internal value (including `BILLABLE`) is treated as billable. No retroactive backfill needed.
- **FK semantics** â€”
  - `tenant_id â†’ tenants(id) ON DELETE RESTRICT` (matches the `20260310101000` pattern)
  - `operating_entity_id â†’ operating_entities(id) ON DELETE RESTRICT` (same pattern)
  - `requested_by_user_id â†’ users(id) ON DELETE SET NULL` (preserve WO history if a requesting user is deleted)
- **Asymmetric down()** â€” `up()` is hasColumn-guarded so it only adds what's missing; `down()` mirrors that by only dropping columns this migration introduced. Rolling back never touches columns owned by `20260310101000`.
- **Indexes** â€” `idx_work_orders_tenant`, `idx_invoices_tenant`, `idx_invoices_op_entity` created via `CREATE INDEX IF NOT EXISTS` (kept distinct from the `*_id`-suffixed indexes the `20260310101000` migration creates, so neither migration's `down()` clobbers the other's indexes).
