# FN-296: Fix date field labels, placeholders, centering, and PM fields alignment in vehicle form

## Requirement
Multiple date input fields in the vehicle form (trucks and trailers) have CSS issues causing labels to not display. The `app-ai-date-picker` component hides `mdc-notched-outline` (which contains `mat-label`), so labels passed via the `label` input are invisible. Additionally, placeholder text needs centering and PM fields need horizontal alignment.

## Implementation Summary

### FN-299 (subtask)
**Root cause**: The `app-ai-date-picker` component's SCSS hides `mdc-notched-outline` with `display: none !important`, which also hides the `<mat-label>` rendered inside it. Labels passed via the component's `label` input were therefore invisible.

**Fix approach**: Added explicit `<label>` elements in the vehicle form HTML above each `app-ai-date-picker` instance, and removed the `label` attribute from the components (using `for` attribute to link to the input ID). This follows the same pattern as other form fields in the vehicle form.

**Changes**:
1. Added visible `<label>` elements for all 4 date fields (inspection_expiry, registration_expiry, insurance_expiry, next_pm_due)
2. Removed `label` prop from `app-ai-date-picker` so the component uses its compact/external-label mode
3. Added CSS for placeholder text centering via `::placeholder { text-align: center }`
4. Added `pm-row` CSS class with explicit `grid-template-columns: 1fr 1fr` to force PM date and mileage fields side-by-side
5. Added responsive override for `pm-row` to stack on mobile

## Files Changed
- `frontend/src/app/components/vehicles/vehicle-form/vehicle-form.component.html` — Added external labels, removed label prop from date pickers, added pm-row class
- `frontend/src/app/components/vehicles/vehicle-form/vehicle-form.component.css` — Placeholder centering, PM row grid, responsive override

## Key Decisions
- Used external `<label>` elements instead of modifying the shared `ai-date-picker` component to avoid breaking other usages
- When no `label` prop is passed, the date picker component applies `ai-date-picker-field--external-label` class for compact height
