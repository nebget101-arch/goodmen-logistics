# FN-531: [Bug] SSN field on employment application resets on each keypress — [value] binding fights user input

## Problem
FN-520 attempted to fix SSN masking but did not fully resolve the root cause. The field used `[value]="getSsnDisplay()"` — a one-way Angular property binding. On every change detection cycle Angular overwrites the DOM input's `.value` with the masked string (e.g. `***-**-6789`), which happens *after* each keypress. This causes the cursor to jump and makes it appear that only 1 character was accepted per keypress — the user cannot fill the SSN field.

## Root Cause
`[value]="getSsnDisplay()"` + `(input)="onSsnInput($event)"` is a conflict:
- `(input)` fires, `onSsnInput` runs, updates `ssnRawValue`
- Angular change detection runs, sets `input.value = getSsnDisplay()` (masked)
- The browser's native input position is reset, losing the cursor position

## Fix
- **HTML**: Removed `[value]="getSsnDisplay()"` from the SSN `<input>`. Only `(input)="onSsnInput($event)"` remains — Angular never touches the DOM value.
- **TS**: At the end of `onSsnInput()`, imperatively set `input.value = this.getSsnDisplay()` and `input.setSelectionRange(display.length, display.length)` (cursor to end). This gives full control over the display without Angular CD interference.
