### Architecture

**Unified parts table** — replaces 3 separate UIs:
- Shows `workOrderParts[]` (API-backed inventory lines with full status data)
- Falls back to `workOrder.parts[]` inline form rows when no API lines exist (new/unsaved WO)
- Columns: SKU, Description, Requested, Reserved, Issued, Unit Price, Line Total, Bin, Status, Actions

**Add Part dialog** — single `+ Add Part` button opens a modal with 3 tab modes:
1. **Search Catalog** — autocomplete across `partsCatalog`, stock warning + Backorder option
2. **Scan Barcode** — direct text/scan input; calls `lookupBarcode()` then `reserveWorkOrderPart()` immediately (no staging)
3. **Manual Entry** — free-form name/SKU/qty/price; matches catalog by SKU if `workOrderId` set, otherwise pushes to `workOrder.parts` array

**Phone bridge** — SSE `scan` events now call `submitBarcode()` directly, eliminating the old
`scannedParts[]` staging table entirely.

**Bulk actions** — "Issue All Reserved" and "Return All Issued" buttons appear conditionally
above the table via `hasReservedLines` / `hasIssuedLines` getters.

### Key API note
`removeWorkOrderPart()` does not exist in `api.service.ts`. The Remove button for API-backed
lines shows an alert informing the user; form-backed rows (`workOrder.parts`) can be freely removed
via `removeFormPart()`. A backend story should be created to add `DELETE /work-orders/:id/parts/:lineId`.

## Key Decisions

1. **No new sub-components** — kept everything in the existing work-tab files for simplicity; the Jira spec suggested sub-folders but the tab file is the natural host given the existing OnPush pattern.
2. **Remove button behavior** — for API-backed parts, Remove shows an informational alert (API endpoint missing); for form-backed parts it splices `workOrder.parts`. This is documented as a known limitation.
3. **Manual entry falls back to form array** — when `workOrderId` is absent (new WO) or no catalog match is found, Manual Entry pushes to `workOrder.parts` so data isn't lost.
4. **ChangeDetectorRef added** — original file used OnPush without CDR; all async callbacks now call `cdr.markForCheck()`.
