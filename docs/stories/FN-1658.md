## Public read API contract (`GET /api/track/:token`)
**Owned by frontend (FN-1678); implemented by backend (FN-1679).** The TypeScript
source of truth is `frontend/src/app/public-track/public-track.models.ts`. The
endpoint MUST be exempt from the gateway auth guard and MUST apply the broker's
`reveal_options` server-side (omit/null sensitive fields when their toggle is off).

**Responses**
- `200` → `{ "success": true, "data": PublicTrackPayload }`
- `404` → unknown / malformed token (no info leak; generic body)
- `410` → expired (`expires_at` past) or revoked (`revoked_at` set)

**`PublicTrackPayload`** (camelCase — matches `reveal_options` keys from FN-1675):
```jsonc
{
  "loadNumber": "L-1042",
  "status": "pickup | in_transit | delivered",
  "statusLabel": "In transit",
  "eta": "2026-06-04T18:00:00.000Z | null",
  "lastUpdatedAt": "2026-06-03T12:00:00.000Z",   // drives "Last updated N min ago"
  "reveal": { "driverName": false, "vehicleNumber": false, "breadcrumbs": false, "routeLine": true },
  "currentPosition": { "lat": 32.78, "lon": -96.80 } | null,
  "origin":      { "label": "Dallas, TX",  "lat": 32.78, "lon": -96.80 },
  "destination": { "label": "Memphis, TN", "lat": 35.15, "lon": -90.05 },
  "milestones": [
    { "key": "pickup",     "label": "Picked up",  "state": "complete", "timestamp": "…", "location": "Dallas, TX" },
    { "key": "in_transit", "label": "In transit", "state": "current",  "timestamp": "…" },
    { "key": "delivered",  "label": "Delivered",  "state": "upcoming", "timestamp": null }
  ],
  // reveal-gated (present only when the matching reveal flag is true):
  "driverName": "Jordan P." | null,
  "vehicleNumber": "T-204" | null,
  "routeLine": [[lat, lon], …] | null,
  "breadcrumbs": [{ "lat": …, "lon": …, "at": "ISO" }, …] | null
}
```
- `milestones[].state` ∈ `complete | current | upcoming`.
- On each read the endpoint should record a view via
  `share-link-service.js#recordShareLinkView(shareLinkId, { ip, userAgent })`
  (best-effort; don't fail the read if auditing throws).

## Implementation Summary

### FN-1678 (subtask — frontend)
Standalone, unauthenticated `/track/:token` page. No app shell, no guard. Lazy
`loadComponent` route so the map library ships in its own chunk (keeps the
public initial payload small). Polls `GET /api/track/:token` every 60s via
`rxjs/timer` (no WebSocket); a separate 30s clock ticks the "Last updated N min
ago" label between polls. `reveal_options` are honored both server-side (the
endpoint filters) and defensively in the template/map (`*ngIf="p.reveal.x"`).

- **Map:** Leaflet, **dynamically imported** (`await import('leaflet')`) and run
  `outsideAngular`. Markers use `divIcon` + Material Symbols (no default-icon
  asset paths); `title`/`alt`/tooltip + `keyboard: true` for a11y. Route line
  (cyan) and breadcrumbs (faint circles) are reveal-gated.
- **States:** loading, ready, and three error states — `not_found` (404),
  `gone` (410), `error` (network/5xx, with a retry button). A transient `error`
  while data is already on screen keeps the last good payload instead of
  blanking the page.
- **a11y:** `aria-live` status announcement, `role="img"` + dynamic
  `aria-label` on the map, `aria-current="step"` on the active milestone,
  SR-only completed/current annotations, focus-visible outlines, AA-contrast
  dark palette, `prefers-reduced-motion` honored.
- **Branding:** light "Powered by FleetNeuron AI" footer; no app chrome.

### FN-1679 (subtask — backend)
Unauthenticated `GET /api/track/:token` read endpoint (`goodmen-shared/routes/public-track.js`).
Hashes the raw token with `share-link-service.js#hashToken` (SHA-256) and resolves
the share link by `token_hash` (UNIQUE) joined to the load — never looking up the
raw value. Returns the `PublicTrackPayload` envelope above, assembled from:
- **status / milestones** — coarse `pickup | in_transit | delivered` derived from
  `loads.status`; milestone `state` (`complete | current | upcoming`) and timestamps
  from `load_stops` (PICKUP/DELIVERY `stop_date`) + `loads.completed_date`.
- **currentPosition** — latest `vehicle_position_pings` row (FN-1660) for the load's
  `truck_id`, as `{ lat, lon }`.
- **origin / destination** — `load_stops` city/state labels + coords via `zip_codes`.
- **reveal-gated** — `driverName` (drivers join), `vehicleNumber` (`vehicles.unit_number`),
  `routeLine` (`[lat, lon]` stop coords), `breadcrumbs` (capped 100 pings / 24h window).

**Security:** unknown/malformed token → `404`, revoked/expired → `410`, both with
generic leak-free bodies (no load id / tenant / address). A shape-gate regex 404s a
malformed token before any DB hit. `Cache-Control: no-store` so shared caches can't
cross-serve viewers. Each successful read records a hashed `load_share_link_views`
row via `recordShareLinkView({ ip, userAgent })` (best-effort; X-Forwarded-For aware).

**Gateway exemption:** the gateway has no global auth guard — auth is enforced
per-route inside each service — so "exempt from the auth guard" = (1) a gateway
proxy entry `app.use('/api/track', buildProxy(LOGISTICS_SERVICE_URL, …))` and
(2) mounting the router in logistics **ahead of** every `app.use('/api', authMiddleware…)`
catch-all, with **no** auth/tenant middleware.

## Key Decisions
- Same app domain `/track/:token` (no subdomain) — intake decision.
- **ETA is `null`** — there is no ETA source on `dev` yet (telematics ingestion
  FN-1660 lands position pings but no ETA). The contract already allows `eta: null`,
  so the field is reserved for a later ETA pipeline without a shape change.
- Breadcrumbs capped to **100 pings / last 24h** to protect the ≤200KB payload budget.
- No new migration — consumes Story E tables (`load_share_links`,
  `load_share_link_views`) and FN-1660's `vehicle_position_pings`.
- Leaflet **dynamically imported** to keep it out of the page's initial bundle
  and protect the ≤ 200KB gzipped budget (verified by FN-1680).
- Public payload contract is **camelCase** to match the `reveal_options` keys
  already minted by Story E (`share-link.service.ts`), avoiding a snake/camel
  translation layer on the public page.
- The shared `AuthInterceptor` only attaches `Authorization` when a token is in
  `localStorage`, so a logged-out shipper hits the endpoint cleanly; a logged-in
  broker's token is simply ignored by the (unauthenticated) public route.
