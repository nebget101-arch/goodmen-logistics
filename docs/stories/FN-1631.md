# FN-1631 — render-dev.yaml: AI_SERVICE_URL missing on drivers-compliance-service-dev

**Type:** Bug
**Parent epic:** FN-1429 (AI Tools Phase 1)
**Related:** FN-1630 (prod equivalent — original PR #761 was scope-narrow to `render.yaml` only)
**Agent:** devops
**Branch:** `agent/devops/FN-1631/render-dev-ai-service-url`

## Problem

PR #761 (FN-1630) wired `AI_SERVICE_URL` into the prod `render.yaml` for
`fleetneuron-drivers-compliance-service` but missed the parallel addition in
`render-dev.yaml`. On dev, the env var is currently present **only** because the
user added it manually via the Render dashboard. The next Blueprint sync against
`render-dev.yaml` would silently drop the manual value, reintroducing the
`cdl_extract_ai_unavailable` failure on every dev CDL upload.

## Implementation Summary

Mirrored the prod fix in `render-dev.yaml`: added a single `AI_SERVICE_URL` env
var to the `fleetneuron-drivers-compliance-service-dev` envVars block, pointing
at the **dev** AI service URL (`https://fleetneuron-ai-service-dev.onrender.com`),
placed at the same position used in `render.yaml` (right after the R2 block,
just before `NODE_ENV: production`).

## Files Changed

- `render-dev.yaml` — added `AI_SERVICE_URL=https://fleetneuron-ai-service-dev.onrender.com`
  to `fleetneuron-drivers-compliance-service-dev` envVars (2 lines added at
  line 228, between `R2_SIGNED_URL_EXPIRES_SECONDS` and `NODE_ENV`).

## Key Decisions

- **Dev URL, not prod.** Acceptance criterion is explicit: this block is the
  dev compliance service and must point to the dev AI service. Using the prod
  URL would cross-wire dev → prod AI calls.
- **Insertion location matches prod.** Placed between R2 block and NODE_ENV to
  keep the two Blueprint files structurally parallel. Future audits or sweeps
  diffing render.yaml ↔ render-dev.yaml will see them line up cleanly.
- **No backend changes.** The "fail fast when AI_SERVICE_URL is unset" hardening
  in `cdl-extraction-service.js` is explicitly out of scope per the ticket and
  should be filed as a separate AI/backend ticket.

## Verification

- YAML parse OK (`ruby -ryaml -e "YAML.load_file('render-dev.yaml')"` exits clean).
- `grep -n AI_SERVICE_URL render-dev.yaml` shows the new entry at line 228 inside
  the `fleetneuron-drivers-compliance-service-dev` block (block starts line 182).
- Value reads `https://fleetneuron-ai-service-dev.onrender.com` (dev — confirmed
  not the prod URL).
