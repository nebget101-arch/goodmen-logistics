# FN-551: Equipment Owner & Effective Dates not persisted on Pay Routing save

## Bug
When saving Pay Routing, the Equipment Owner field and Effective Start/End Dates are lost on page reload.

**Root cause (frontend):** `savePayeeAssignment()` sends effective dates in the POST, but on load the GET response dates were never written back into `newDriver.effectiveStart` / `newDriver.effectiveEnd`.

**Root cause (backend):** Previous open-ended assignments were not end-dated when creating a new one, causing overlapping assignments.

## Acceptance Criteria
- [x] Effective Start Date persists after page reload
- [x] Effective End Date persists after page reload
- [x] Equipment Owner selection persists after page reload
- [x] `driver_payee_assignments` record written with `additional_payee_id`, `effective_start_date`, `effective_end_date`
- [x] GET endpoint for payee assignment returns effective dates so UI can pre-fill on load
- [x] POST `/api/settlements/drivers/:driverId/payee-assignment/resolve` accepts and stores effective dates
- [x] Previous open-ended assignments are end-dated when a new assignment is created

## Implementation Summary
