# FN-551: Equipment Owner & Effective Dates not persisted on Pay Routing save

## Bug
When saving Pay Routing, the Equipment Owner field and Effective Start/End Dates are lost on page reload.

**Root cause:** `savePayeeAssignment()` in `dispatch-drivers.component.ts` sends `additional_payee_id` (and effective dates) in the POST payload — but on load, the GET `/api/settlements/drivers/:driverId/payee-assignment` response is received yet `assignment.effective_start_date` / `assignment.effective_end_date` are never written back into `newDriver.effectiveStart` / `newDriver.effectiveEnd`. So the date pickers always appear blank after a reload even if the data was stored.

- [x] Effective End Date persists after page reload

## Implementation Summary

### FN-558 (frontend)
**Root cause confirmed:** `savePayeeAssignment()` already sends `effective_start_date` and `effective_end_date` (lines 1684–1690). The bug was in the load callback (`getPayeeAssignment` subscribe block, line ~820) — effective dates were read from the API response but never assigned to `newDriver.effectiveStart` / `newDriver.effectiveEnd`.

**Fix:** added two assignments in `getPayeeAssignment` success callback, immediately after the `additional_payee_id` block:
```typescript
if (assignment?.effective_start_date) {
  this.newDriver.effectiveStart = assignment.effective_start_date;
}
if (assignment?.effective_end_date) {
  this.newDriver.effectiveEnd = assignment.effective_end_date;
}
```

The HTML already has `app-ai-date-picker` inputs bound to `newDriver.effectiveStart` / `newDriver.effectiveEnd` — no HTML changes needed.
