# FN-521: Address autocomplete on employment application using SmartyStreets API

## Summary
Add address autocomplete to the employment application's Current Address and Employer Address fields using the SmartyStreets US Address Autocomplete API. A backend proxy keeps API keys server-side.

## Subtasks

| Key | Summary | Agent | Status |
|-----|---------|-------|--------|
| FN-525 | Backend: SmartyStreets address autocomplete proxy endpoint | backend | Done |
| FN-526 | Frontend: Address autocomplete UI for current/employer address fields | frontend | Selected for Development |
| FN-527 | QA: Validate address autocomplete on employment application | qa | Backlog |

## Implementation Notes

### FN-525 -- Backend proxy (completed)
- **Route**: `backend/packages/goodmen-shared/routes/address.js`
- **Endpoint**: `GET /api/address/autocomplete?q=<partial-address>`
- **Mounted in**: `drivers-compliance-service` (no auth middleware -- public route)
- **Gateway**: `/api/address` proxied to `DRIVERS_COMPLIANCE_SERVICE_URL`
- **Env vars required**: `SMARTY_AUTH_ID`, `SMARTY_AUTH_TOKEN`
- Returns `{ success, data: [{ street, city, state, zip }] }` (max 10 results)
- 503 if env vars missing, 502 if SmartyStreets fails
- Query must be >= 3 chars, capped at 100 chars

## Deployment Notes
- **Service**: `drivers-compliance-service` + `gateway`
- **Env vars to add on Render**: `SMARTY_AUTH_ID`, `SMARTY_AUTH_TOKEN` on `drivers-compliance-service`
