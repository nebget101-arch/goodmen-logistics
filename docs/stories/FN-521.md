# FN-521: Address autocomplete on employment application using SmartyStreets API

## Requirement
Add address autocomplete to the employment application's Current Address and Employer Address fields using the SmartyStreets US Address Autocomplete API. As the user types a street address, a dropdown of suggestions appears. Selecting a suggestion populates street, city, state, and zip automatically. The SmartyStreets API key is proxied through the backend — never exposed to the browser.

## Acceptance Criteria
- [ ] Current Address street field shows autocomplete suggestions from SmartyStreets as user types
- [ ] Employer Address street field shows autocomplete suggestions from SmartyStreets as user types
- [ ] Selecting a suggestion auto-fills city, state, and zip fields
- [ ] Backend proxy endpoint handles SmartyStreets API key — key is never exposed to the browser
- [ ] Suggestions are debounced (300ms) to avoid excessive API calls
- [ ] Field still allows manual entry if autocomplete is dismissed or unavailable

## Implementation Summary

### FN-525 — Backend proxy
- Route: `backend/packages/goodmen-shared/routes/address.js`
- Endpoint: `GET /api/address/autocomplete?q=<partial-address>`
- Mounted in `drivers-compliance-service` (public route, no auth)
- Gateway: `/api/address` proxied to `DRIVERS_COMPLIANCE_SERVICE_URL`
- Env vars required: `SMARTY_AUTH_ID`, `SMARTY_AUTH_TOKEN`
- Returns `{ success, data: [{ street, city, state, zip }] }` (max 10 results)
- 503 if env vars missing, 502 if SmartyStreets fails

### FN-526 — Frontend UI
- Added autocomplete to Current Address and Employer Address street inputs
- Debounced input (300ms), calls `GET /api/address/autocomplete?q=...`
- Dropdown shows suggestions; selecting one patches street, city, state, zip in the FormGroup
- Dismiss on Escape or outside click; silent degradation if API unavailable
