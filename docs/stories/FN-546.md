# FN-546: Applicant Certification: DOB, SSN Last 4, DL#, State of Issue not populating

## Problem
FN-535 added the Applicant Certification fields but three defects left them blank or wrong on page load:

1. **No initial sync** — `setupCertificationSync()` wired `valueChanges` subscriptions only. These fire on future input, not values already in the form. Normal flow: user fills Applicant Info first, then scrolls down to Certification — the subscriptions never fired for those values.
2. **SSN Last 4 subscription missing** — there was no `valueChanges` subscriber to watch the SSN field and update `certification.ssnLast4`. The field was only updated imperatively inside `onSsnInput()` but not on init.
3. **DOB shows today's date** — `certification.dateOfBirth` was being overwritten to today's date by an earlier init path with no subsequent fix from `applicant.dateOfBirth`.

## Fix

### `syncCertificationNow()` — new private helper
Does a one-time snapshot of all current form values into the certification section. Called:
- At the end of `setupCertificationSync()` (initial page load)
- In `prefillLicenseFromDriver()` success callback (after license data arrives from API)

### `setupCertificationSync()` — SSN subscription added
```typescript
this.form.get('applicant.ssn')?.valueChanges.subscribe(() => {
  const last4 = this.ssnRawValue.slice(-4);
  cert.get('ssnLast4')?.setValue(last4 || '', { emitEvent: false });
});
```

### `prefillLicenseFromDriver()` — sync after API prefill
```typescript
this.syncCertificationNow(); // re-sync after DL# / state are patched
```
