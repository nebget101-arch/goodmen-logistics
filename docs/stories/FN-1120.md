## Implementation Summary
**FN-1191** — re-tokenized every Reports stylesheet (`*.css` and `*.scss`) so no hex literal remains in `frontend/src/app/reports/`. Added a new AI-dark-theme palette to global `:root` (`--ai-text-*`, `--ai-surface-*`, `--ai-accent-*`) so reports + future modules share one token vocabulary. Added ARIA scaffolding: report-table now renders `<caption>` + `scope="col"` + per-column `id` and per-cell `headers=`, with `aria-sort` on the active sort column; report-chart canvas carries `role="img"`, `aria-label`, and `aria-describedby` pointing at a screen-reader-only semantic `<table>` listing each label/value pair; report-filters inputs each have a visible `<label for>`, an `aria-describedby` hint that explains what the filter does, and the date picker accepts a new `ariaDescribedBy` input that wires into the underlying matInput.

### FN-1191 (subtask)

**Token strategy.** Two layers: (1) reuse existing global tokens (`--primary-color`, `--text-dark`, `--border-color`, `--accent-color`, `--success-color`, `--warning-color`, `--danger-color`); (2) add a new AI-dark palette to `frontend/src/styles.css` (`--ai-text-strong/-body/-soft/-muted/-subtle/-faint`, `--ai-surface-deep/-raised/-elevated`, `--ai-accent-info/-info-strong/-info-stronger/-cyan/-indigo/-success/-danger/-danger-soft/-warning/-warning-soft`, `--ai-text-on-accent`). Reports CSS uses `var(--…)` exclusively — no hex.

**Chart color sourcing.** `ReportChartComponent` reads `--primary-color` via `getComputedStyle(document.documentElement)` so the canvas fill tracks the theme token at runtime; the `'#1a237e'` argument inside `themeColor(...)` is a defensive fallback when no document is available (SSR / tests) and is the same value `--primary-color` resolves to.

**Chart semantic fallback.** Each `app-report-chart` instance now emits a `.sr-only` `<table>` keyed off a per-instance random id; the `<canvas>` references it via `aria-describedby`, so screen-reader users get the same series the canvas paints. The table is generated only when `labels` and `data` are present.

**Table headers/IDs.** `ReportTableComponent` derives a per-instance `idPrefix` (random suffix) and a `columnHeaderId(key)` helper. Each `<th>` carries `scope="col"`, an `id` of the form `rpt-tbl-XXXX-col-<key>`, and `aria-sort` reflecting the active sort. Each `<td>` carries `headers="<that id>"`. The optional title is exposed as a visually-hidden `<caption>` referenced by `aria-labelledby` on the `<table>`. Pagination wraps in `role="navigation"` with disabled prev/next at the boundaries.

**Filter descriptions.** Each filter row now has a `<small class="filter-hint" id="…Hint">` rendered under the input, and the input/select carries `aria-describedby="…Hint"`. The date picker (`AiDatePickerComponent`) gained an `ariaDescribedBy` input that flows through to the matInput. The Location `<select>` now has a matching `id`/`for` pair.

## Key Decisions
- **Add tokens globally rather than per-component.** Defining `--ai-…` tokens in `:root` (one place) instead of duplicating them inside each component's `:host` keeps the slate-200/300/400/500/600 palette consistent across the dashboard and lets future modules drop into the same vocabulary. Blast radius is contained — we only *add* tokens; no existing token's value changes.
- **Chart accent via getComputedStyle, not Chart.js plugin.** The reports chart is a one-off `<canvas>`, not a Chart.js plugin pipeline; reading `--primary-color` from `:root` once per render is the smallest change that makes the chart honor a theme token.
- **Per-instance id prefix on table + chart.** When two `app-report-table` (or `app-report-chart`) instances coexist, each needs unique header/caption ids so `headers="…"` and `aria-describedby="…"` resolve correctly. A single `Math.random` suffix per component instance is enough — this is a presentation concern, not a correctness one, so we don't pull in a DI'd id-generator.
