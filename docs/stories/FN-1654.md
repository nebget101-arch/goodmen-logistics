## Key Decisions
- **Geometry storage = GeoJSON in `jsonb` (NOT PostGIS).** The PostGIS extension is **not** enabled on prod — the only extensions present across the entire migration history are `uuid-ossp` and `pgcrypto` (no `CREATE EXTENSION postgis`, no `geometry()`/`geography()` columns, no GIST indexes anywhere in `goodmen-database`). Introducing PostGIS would add a deploy-coupled extension prerequisite to every environment. Instead the `geometry` column is `jsonb` holding GeoJSON.
  - **`geometry` jsonb shape:**
    - circle → `{ "type": "Circle", "center": [lng, lat], "radius_m": <number> }` (GeoJSON has no native circle, so this is a documented custom shape)
    - polygon → `{ "type": "Polygon", "coordinates": [[[lng, lat], ...]] }` (standard GeoJSON; `[lng, lat]` order per spec)
  - **⚠️ Story C (FN-1655) impact:** point-in-polygon and point-in-circle tests MUST be done **app-side** (e.g. ray-casting for polygons, haversine/great-circle distance ≤ `radius_m` for circles). There is no DB-side `ST_Contains`/`ST_DWithin` available.
- **FK targets:** `geofences.address_id` → `locations.id` (`ON DELETE SET NULL`); there is no `addresses` table — `locations` is the addressable-place table (UUID PK). The FK is wired conditionally (only if `locations` exists) so the migration is portable. `geofence_triggers.geofence_id` → `geofences.id` (`CASCADE`); `geofence_triggers.vehicle_id` → `vehicles.id` (`CASCADE`, NULL = applies to all vehicles).
- **Added columns beyond the literal subtask list:** `is_active BOOLEAN DEFAULT true` (required to satisfy the parent AC `active` CRUD filter) and `updated_at` (conventional). Composite index `(tenant_id, is_active)` supports tenant-scoped active listing; `(created_by)` supports the `owned-by` filter. `near-point` filtering is app-side (no PostGIS).
- **Integrity CHECKs:** `kind ∈ (circle,polygon)`; `event_kind ∈ (enter,exit,dwell)`; `action ∈ (notify,update_load_status,webhook)`; dwell events require `dwell_minutes > 0`; webhook actions require `target_url`.
