# FN-671: Unified API Documentation Gateway & Shared Swagger Config

## Requirement
Create a unified API documentation experience so developers see all endpoints in one place. Implement shared Swagger configuration and a gateway-level aggregation endpoint.

## Acceptance Criteria
- [x] Shared Swagger config created at `backend/packages/goodmen-shared/config/swagger.js`
- [x] All 8 services import the shared definition via `buildSwaggerOptions()`
- [x] Gateway `/api-docs` aggregates all service specs into one unified view
- [x] `npm run swagger:export` generates `docs/api/openapi.json`
- [x] Gateway `/api-docs` is publicly accessible (no auth required)
- [x] Each service exposes `/api-docs-json` for raw spec consumption
- [ ] All endpoints visible in the unified gateway docs (verify after deploy)

devops

## Implementation Summary

### Shared Swagger Config
Created `backend/packages/goodmen-shared/config/swagger.js` with:
- `sharedDefinition` — common OpenAPI 3.0 definition (servers, security schemes, contact info)
- `buildSwaggerOptions({ title, description, apis })` — factory function each service calls to build their swagger-jsdoc options

### Service Updates (8 services)
Replaced inline swagger config in each service's `server.js` with:
```js
const { buildSwaggerOptions } = require('@goodmen/shared/config/swagger');
const swaggerOptions = buildSwaggerOptions({ title, description, apis });
```
Added `/api-docs-json` endpoint to each service for raw JSON spec consumption by the gateway.

### Gateway Aggregation
- Fetches `/api-docs-json` from all 8 downstream services at startup
- Merges paths, schemas, and tags into a single unified OpenAPI spec
- Serves via Swagger UI at `/api-docs` (no auth required)
- Caches merged spec for 5 minutes
- Handles service unavailability gracefully (skips services that don't respond)

### Swagger Export Script
- `backend/gateway/scripts/swagger-export.js` — fetches aggregated spec from gateway and writes to `docs/api/openapi.json`
- Run via `npm run swagger:export` in the gateway directory

## Key Decisions
- Used runtime aggregation (fetch + merge) instead of `swagger-combine` to avoid extra dependencies and support dynamic service availability
- Each service exposes `/api-docs-json` rather than using swagger-combine's URL-based config — simpler and more resilient
- Gateway caches the merged spec for 5 minutes to avoid excessive upstream fetches
- Shared config uses a factory function pattern so services retain their own title/description while sharing the security scheme and server definitions

- **Deploy order**: Deploy all 8 microservices FIRST (they need `/api-docs-json`), then gateway
