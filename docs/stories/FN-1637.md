FN-1637 needs partial re-grooming against the componentized architecture.

## Implementation Summary
FN-1640 modernized the **in-zone** portions of `dashboard.component.*` to the
FN-1636 AI primitives, scoped per the user's "Option 1 (trimmed)" decision
because most of the ticket's target UI has been refactored into sibling
components (`app-control-center`, `app-kpi-strip`, `app-quick-actions`) that the
story forbids editing.

Delivered (all within `dashboard.component.{html,ts,css}`):
- **Header** → canonical `.ai-header` / `.ai-kicker` / `title-icon` / `.subtitle`
  pattern (matches Audit/Roadside), with a company-context line and a single
  `.btn-primary.ai-glow-btn` "Generate Audit Report" CTA linking to `/audit`.
  The bespoke `.dashboard-header-pill` "Live compliance signal" chip was removed.
- **Loads progress** (4 cards) and **Billing status** (5 cards) clusters →
  `.ai-panel` containers rendering `<app-kpi-card>` from `loadsCards`/`billingCards`
  view-models. Status mapping preserves legacy emphasis (Delivered/Paid → `good`,
  Funded → `warning`, else `info`).
- **Loading** → per-panel `<app-ai-skeleton>` grids (4 + 5 placeholders) with
  `aria-busy="true"` on the panels; the bespoke `.orbital-loader` markup + CSS
  + `@keyframes spin` were removed.
- **Degraded banner** → `.ai-panel-flat[data-severity="warning"]` with a
  `.btn-secondary` Retry that calls `loadDashboard()`.
- CSS trimmed to page-shell, header, KPI-cluster grids, degraded layout, and
  responsive breakpoints; all `.panel*`, `.metric-card*`, `.orbital-loader*`,
  `.degraded-banner*`, `.dashboard-header*` rules deleted.

`<app-ask-bar>` and `<app-control-center>` remain mounted unchanged.

## Key Decisions
- **In-zone subset only.** The ticket's premise (monolithic dashboard) is stale;
  fleet-health/alerts/quick-actions are in sibling components. We modernized only
  what's genuinely in `dashboard.component.*` and deferred the rest rather than
  editing forbidden files or shipping a half-met AC silently.
- **`<app-kpi-card>` has no `queryParams` input.** The legacy loads/billing cards
  deep-linked with `[queryParams]` (e.g. `{status:'dispatched'}`); the FN-1636
  primitive only supports `routerLink`. Cards now link to `/loads` without the
  per-status filter. **Follow-up:** add an optional `queryParams` input to
  `<app-kpi-card>` to restore deep-link filtering (see Open Items).
- **No trend chip.** Backend doesn't emit `previousPeriodValue`; per the ticket
  we omit the delta rather than fabricate it.
- **No new hex.** Header reuses canonical tokens already present in
  `audit.component.css`; all other hex pre-existed in `dashboard.component.css`.
