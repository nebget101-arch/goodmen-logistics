## Implementation Summary

### FN-1449 (subtask)

Added an "Extract with AI" panel to the top of the tolls-import wizard's upload step. Service writers choose a toll invoice image, click **Extract with AI**, see a spinner with "Extracting line items…" copy, and on success the existing `InvoicePreviewDialogComponent` opens populated with the extracted rows. The CSV/Excel upload path is preserved below an "or upload a CSV" divider.

A new `AiService` (`frontend/src/app/services/ai.service.ts`) owns the call to `POST /api/ai/tolls/invoice-vision`. It base64-encodes the file via `FileReader.readAsDataURL`, posts `{ imageBase64, mediaType }`, and maps the response (`entry_location`/`exit_location`) to the dialog's existing `ExtractedTollTransaction` shape (`entry_point`/`exit_point`). When the API returns an overall `confidence < 0.7`, every row in the result is flagged `low_confidence: true`; the dialog template renders a new `Verify` badge for those rows and tints the row background.

The dialog template/CSS now also surface `low_confidence` via a `data-low-confidence="true"` attribute and a `.low-confidence` class for QA hooks.

## Key Decisions

- **AI route name.** The handler is mounted at `POST /api/ai/tolls/invoice-vision` (not `/api/ai/toll-invoice/vision` as the ticket draft stated). Per the implementation note, the frontend was wired to the actual route — no gateway change introduced.
- **PDF support deferred.** The vision handler explicitly only accepts `image/jpeg|png|webp|gif` (`SUPPORTED_MEDIA_TYPES` in `toll-invoice-vision-handler.js`). The `<input accept>` is restricted to those types client-side so users see a clear "JPG, PNG, WebP, GIF" hint instead of a 400 from the server. PDF support requires a backend handler change (Anthropic vision currently does not natively read PDFs without an extraction step) and is not part of this subtask.
- **Confidence threshold = 0.7.** The handler returns one overall `confidence` score, not per-row. Rather than expose a UX-disrupting "AI uncertain" banner, the frontend marks **every** row low_confidence when the overall score is below 0.7 — making each row's `Verify` badge a visible nudge to scan editable cells before accepting.
- **Manual-entry path preserved.** The CSV/Excel wizard (Upload → AI Analysis → Map → Preview → Commit) is unchanged. The vision panel sits above the existing drop-zone behind an "or upload a CSV" divider, so no flow was removed.
- **No new gateway mapping.** `app.use('/api/ai', ...)` already proxies the AI service, so the request flows through gracefully.
