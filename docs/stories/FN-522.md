# FN-522: Auto-populate License History on employment application from driver's most recent license record

## Overview
Pre-populate the License History section of the employment application form with data from the driver's most recent license record, reducing manual data entry for onboarding staff.

## Subtasks

| Key | Title | Status |
|-----|-------|--------|
| FN-528 | Backend: Add endpoint to return most recent license for a driver | Pending |
| FN-529 | Frontend: Pre-populate License History fields from most recent license on load | Done |

## Implementation Notes (FN-529)
- On component init, calls `ApiService.getDriver(driverId)` to fetch the driver record
- Maps `cdlNumber` -> `licenseNumber` and `cdlState` -> `state` in the first licenses FormArray entry
- Only patches fields that the user hasn't already typed into (no override)
- Silently skips pre-population if the API returns null or errors
- Uses the existing `getDriver` endpoint since no dedicated `license-latest` endpoint exists yet
