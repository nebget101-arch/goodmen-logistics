# FN-522: Auto-populate License History on employment application from driver's most recent license record

## Requirement
When a driver opens the public employment application, the **License History** section should be pre-populated with data from the driver's most recent license record in Personal Data (drivers table / DQF). Fields to auto-fill: SSN last 4, Date of Birth, Driver's License Number, and DL Issue State.

If no existing license is found, the section remains empty (manual entry).

## Acceptance Criteria
- [x] Backend API endpoint returns most recent license data for a driver given their public token/driverId
- [x] Employment application pre-populates License History fields on load: SSN last 4, DOB, DL Number, DL state
- [x] Pre-populated values are editable by the driver
- [x] If no license record exists, section is blank (no error shown)

## Implementation Summary

### FN-528 — Backend (backend/FN-528/license-prefill-endpoint)
Added `GET /public/onboarding/:packetId/license?token=...` to `public-onboarding.js`.

- Token authentication via existing `loadPacketWithToken(packetId, token)` helper
- Rate-limited via `rateLimited` middleware
- Queries `drivers` table for `date_of_birth` and CDL fields via `safeOptionalQuery`
- Queries `driver_licenses` (most recent by `created_at DESC`) for `cdl_number`, `cdl_state`, etc.
- Queries `employment_applications` (most recent) for `applicant_snapshot` (used for SSN last 4)
- Delegates to existing `buildEmploymentPrefill({ driver, license, latestApp })` to normalize fields
- Returns: `{ ssnLast4, dateOfBirth, licenseNumber, licenseState }`
- Errors → `dtLogger.error('public_driver_license_prefill_failed', ...)` + 500 response

**URL note**: Ticket spec mentioned `/api/public/driver-license?token=...` but `/api/public` is proxied to AUTH_USERS_SERVICE_URL (wrong service). Used `/public/onboarding/:packetId/license?token=...` which is correctly proxied to DRIVERS_COMPLIANCE_SERVICE_URL via the existing `/public/onboarding` gateway entry.

### FN-529 — Frontend (frontend/FN-529/license-prefill-ui)
- On component init, calls `GET /public/onboarding/:packetId/license?token=...`
- Pre-populates first `licenses` FormArray entry with `licenseNumber` and `licenseState`
- Silent failure: if API errors or returns null, section remains blank — no error shown to driver
- Pre-populated fields remain fully editable
