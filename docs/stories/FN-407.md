# FN-407: Implement transaction normalization and 1-to-many row splitting in fuel import staging

## Requirement
When a raw vendor transaction has multiple product types (diesel + DEF + reefer), split it into separate normalized batch rows so each product is tracked independently.

## Acceptance Criteria
- [x] Raw row with diesel + DEF + reefer amounts produces 3 normalized batch rows
- [x] Each split row has correct product_type, gallons, amount for its product
- [x] All split rows share the same source_transaction_id
- [x] Single-product rows pass through unchanged with product_type set
- [x] Default product_type = 'diesel' when no product info exists
- [x] raw_payload preserved on all split rows (original raw data)
- [x] Duplicate detection works across split rows (source_transaction_id + product_type)
- [x] Row counts in batch reflect expanded rows (not raw rows)

## Agent
backend

## Implementation Summary
- Added `normalizeRow()` function to `fuel-parser.js` with two strategies:
  - **Multi-column split**: Detects diesel_gallons/def_gallons/reefer_gallons columns in raw data, splits into separate rows per non-zero product
  - **Description-based**: Parses product_type from mapped value, classifies into diesel/def/reefer + category (fuel/maintenance/advance)
- Updated `stageBatch()` in `fuel-service.js` to call `normalizeRow()` after column mapping, iterating over split results
- Duplicate detection uses composite key `source_transaction_id::product_type` for split rows
- Batch `total_rows` reflects expanded row count, not raw file row count
- Each split row preserves original `raw_payload` and shares `source_transaction_id`

## Files Changed
- `backend/packages/goodmen-shared/services/fuel-parser.js` — added `normalizeRow()`, `PRODUCT_COLUMN_PATTERNS`
- `backend/packages/goodmen-shared/services/fuel-service.js` — updated `stageBatch()` loop, import, batch summary

## Key Decisions
- Split detection happens on the raw row keys (before mapping), so it works regardless of column map configuration
- Category inference from product description handles common non-fuel items (oil, maintenance, cash advance)
- PPG is recalculated per split row from the split gallons/amount

## Deployment Handoff
- **Services**: Backend (goodmen-shared)
- **Migrations**: None (depends on FN-405 migration for `category` and `source_transaction_id` columns)
- **Env/config**: None
- **Deploy order**: FN-405 migration must run first, then deploy this
- **Post-deploy verification**: Upload a multi-product fuel CSV — verify split rows appear in staging with correct product types
