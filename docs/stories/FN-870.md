## Key Decisions

### FN-891 (subtask)
- **Scope discipline.** The story doc (line 29) warns that FN-890 + FN-891 were split across two frontend subtasks against intake rules. I still kept this subtask narrow: HTML + SCSS only. HTML and SCSS live in different files from the TS, so merging won't conflict ‚Äî FN-890 can land on its own branch without contention.
- **Template still compiles even with stale TS properties.** Angular only errors when a template references a property that does not exist on the component. Removing the HTML first leaves `showManualModal`, `manualLoadForm`, etc. in the TS as dead but valid members; `manualLoadForm` is still live because the inline new-load entry bar (`*ngIf="showInlineNewLoad"`) uses it.
- **`.modal` flattening.** Once `modal--draft-review` is no longer applied anywhere, `&:not(.modal--draft-review) { display: flex; flex-direction: column; }` reduces to the plain declarations. Inlined them on `.modal` rather than leaving a selector that no longer distinguishes anything.
- **`aiPulse` keyframe retained.** Shared with the bulk-upload modal's `.bulk-ai-pulse`. `aiProgressSlide` was only referenced by the deleted AI modal and is gone.
