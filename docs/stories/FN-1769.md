# FN-1769: AI fault-code diagnosis on Vehicle Tracking telemetry

## Requirement
On the Vehicle Tracking page, when a selected truck has active fault codes, show an AI suggestion of what the issue likely is and whether it needs immediate attention. The telemetry panel already renders raw faults (`{ code, description, severity }`) from the FN-1750 contract (`GET /api/vehicles/:id/telemetry`); this story adds an AI interpretation layer (plain-English diagnosis + urgency call).

Telemetry is currently deterministic MOCK (`source: 'mock'`); the AI diagnosis operates on whatever fault codes the contract returns, so it works identically once a real telematics integration lands.

## AI Contract (defined up front so AI + frontend run in parallel)
```
POST /api/ai/vehicle/fault-diagnosis
Request:  { vehicleId?, unitNumber?, faultCodes: [{ code, description, severity }] }
Response: { summary, immediateAttention, urgency: 'critical'|'high'|'medium'|'low',
            recommendedAction, perFault: [{ code, likelyCause, immediateAttention }] }
```
The frontend owns `api.service.ts` (`getVehicleFaultDiagnosis`); the AI subtask owns only the ai-service handler + router (so the two subtasks don't both edit `api.service.ts`).

### FN-1770 (subtask) ‚Äî AI fault-diagnosis handler + endpoint
- New handler `backend/microservices/ai-service/src/handlers/vehicle-fault-diagnosis-handler.js`, modeled on `work-order-triage-handler.js` (OpenAI `gpt-4.1-mini`, structured JSON output, defensive parsing).
- Registered `POST /vehicle/fault-diagnosis` in `ai-router.js` (gateway exposes `POST /api/ai/vehicle/fault-diagnosis`) with full OpenAPI annotation.
- **No-fault short-circuit:** empty/missing `faultCodes` returns the benign response (`immediateAttention: false`, `urgency: 'low'`, empty `perFault`) **without** a model call.
- **Deterministic safety bias:** any input fault with `high`/`critical` severity whose code/description matches a safety-critical system (engine, oil pressure, brakes, coolant, overheat, temperature, ABS, steering) forces `immediateAttention: true` and `urgency` to at least `high`. The bias only ever escalates ‚Äî it never downgrades a model's `critical` rating.
- **Output normalization:** model JSON is merged onto the validated input fault list; every input fault always gets a `perFault` entry keyed by `code` even if the model omits it. Raw model JSON is never returned to the client. Unparseable model output still yields a valid contract shape.
- Mirrors existing error handling (`502` + `AI_FAULT_DIAGNOSIS_ERROR`) and `logAiInteraction` logging conventions.
- Tests: `src/handlers/__tests__/vehicle-fault-diagnosis-handler.test.js` (standalone `node`, mocked `deps.openai`); added to the ai-service `npm test` chain. All pass.

## Key Decisions
