# FN-1114: Reports — AI narrative panel (Claude Sonnet, prompt-cached)

## Requirement
When a financial report loads, fetch a Claude-generated 2–3 sentence summary that explains the headline movement (what changed, by how much, plausible driver) and render it above the chart. New endpoint `POST /api/ai/reports/:reportKey/narrative` in `ai-service` using Claude Sonnet 4.6 with prompt caching on the system prompt + report schema.

- [ ] Endpoint requires `REPORTS_VIEW`; 403 otherwise

- [ ] FE renders panel above chart; shimmer while loading; gracefully hides on error
- [ ] Narrative call does not block initial chart render
- [ ] Endpoint documented in Swagger
- [ ] No regression on existing 13 reports
- [ ] Screenshots in `docs/stories/evidence/FN-1114/`

## API Contract (defined upfront so FE + AI can run parallel)
**Request** `POST /api/ai/reports/:reportKey/narrative`
```json
{ "cards": [...], "data": [...], "filters": {...}, "priorPeriod": {...} }
```
**Response**
```json
{ "narrative": "Revenue rose 12% MoM, driven by …", "generatedAt": "2026-05-04T..." }
```

## Implementation Summary

### FN-1123 (subtask) — AI narrative endpoint
- New handler `reports-narrative-handler.js` adds `POST /api/ai/reports/:reportKey/narrative`. Returns `{ narrative, generatedAt, meta }`.
- Anthropic call uses Claude Sonnet 4.6 (`claude-sonnet-4-6`, override via `ANTHROPIC_NARRATIVE_MODEL`) with `system` as a two-block array; both blocks carry `cache_control: { type: 'ephemeral' }` (system prompt block + per-report schema block keyed by `reportKey`).
- Schema blocks hardcoded for `revenue-by-driver`, `fuel-spend-by-truck`, `load-margin`; unknown keys fall back to a generic schema block.
- RBAC enforced at the handler layer: 403 unless `req.user.role` ∈ `{ admin, manager, owner, dispatcher }`. JWT decoded locally as a defensive fallback when the gateway has not pre-attached `req.user` (mirrors `reporting-service/routes/insights.js#authenticate`).
- 400 on invalid `reportKey` (must match `/^[a-z0-9-]{1,64}$/i`) or malformed body. 413 if body > 256 KB. 502 `AI_UNAVAILABLE` when Anthropic upstream errors. All paths logged via `analytics/logger#logAiInteraction`.
- Swagger block added at the route in `ai-router.js`. The gateway already proxies `/api/ai/*` → ai-service (`gateway/index.js:547`), so no gateway changes were needed.

## Key Decisions
- **Cache strategy:** ephemeral `cache_control` on system prompt and per-report schema; user message stays uncached. Cache is keyed implicitly per `reportKey` via the schema block's content, so different reports get independent cache entries.
- **No fallback narrative:** unlike loads-nlq which returns `{ fallback: true }` on parse failure, narrative has no useful fallback so the handler returns 502 and lets the FE hide the panel (per FN-1114 AC: "gracefully hides on error").
- **RBAC at handler:** chose handler-level role guard (not new middleware) so the change is contained. JWT decode is `lazyRequire`d so `jsonwebtoken` stays optional from the package perspective.
- **Model env override:** `ANTHROPIC_NARRATIVE_MODEL` env var lets ops swap models without a deploy.

## Deployment Handoff
- **Service to redeploy:** `ai-service` (Render service: see `.agent/docs/render_services.md`). No DB migrations, no other services affected.
- **New env vars (optional):**
  - `ANTHROPIC_NARRATIVE_MODEL` — defaults to `claude-sonnet-4-6` if unset.
  - Existing `ANTHROPIC_API_KEY` and `JWT_SECRET` are reused; no changes.
- **Verification after deploy:**
  1. `curl -H 'Authorization: Bearer <admin-jwt>' -H 'Content-Type: application/json' -d '{"cards":[],"data":[],"filters":{},"priorPeriod":{}}' https://<ai-service>/api/ai/reports/revenue-by-driver/narrative` → expect 200 with `narrative` and `generatedAt`.
  2. Repeat the same call within 5 minutes; expect `meta.cacheReadTokens > 0`.
  3. Without bearer, or with a non-allowlisted role → expect 403.
  4. Swagger UI at `/api-docs` should list `POST /api/ai/reports/{reportKey}/narrative`.
- **No feature flags, no cache invalidation, no migrations, no deploy ordering constraints.**
