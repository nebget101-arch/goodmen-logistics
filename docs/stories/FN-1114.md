# FN-1114: Reports — AI narrative panel (Claude Sonnet, prompt-cached)

## Requirement
When a financial report loads, fetch a Claude-generated 2–3 sentence summary that explains the headline movement (what changed, by how much, plausible driver) and render it above the chart. New endpoint `POST /api/ai/reports/:reportKey/narrative` in `ai-service` using Claude Sonnet 4.6 with prompt caching on the system prompt + report schema.

## Acceptance Criteria
- [ ] `POST /api/ai/reports/:reportKey/narrative` exists in `ai-service` and is gateway-routed
- [ ] Endpoint requires `REPORTS_VIEW`; 403 otherwise
- [ ] System prompt + schema block uses `cache_control` ephemeral; second call shows `cache_read_input_tokens > 0`
- [ ] FE renders panel above chart; shimmer while loading; gracefully hides on error
- [ ] Narrative call does not block initial chart render
- [ ] Endpoint documented in Swagger
- [ ] No regression on existing 13 reports
- [ ] Screenshots in `docs/stories/evidence/FN-1114/`

## API Contract (defined upfront so FE + AI can run parallel)
**Request** `POST /api/ai/reports/:reportKey/narrative`
```json
{ "cards": [...], "data": [...], "filters": {...}, "priorPeriod": {...} }
```
**Response**
```json
{ "narrative": "Revenue rose 12% MoM, driven by …", "generatedAt": "2026-05-04T..." }
```

## Implementation Summary

### FN-1123 (subtask) — AI narrative endpoint
- New handler `reports-narrative-handler.js` adds `POST /api/ai/reports/:reportKey/narrative`. Returns `{ narrative, generatedAt, meta }`.
- Anthropic call uses Claude Sonnet 4.6 (`claude-sonnet-4-6`, override via `ANTHROPIC_NARRATIVE_MODEL`) with `system` as a two-block array; both blocks carry `cache_control: { type: 'ephemeral' }` (system prompt block + per-report schema block keyed by `reportKey`).
- Schema blocks hardcoded for `revenue-by-driver`, `fuel-spend-by-truck`, `load-margin`; unknown keys fall back to a generic schema block.
- RBAC enforced at the handler layer: 403 unless `req.user.role` ∈ `{ admin, manager, owner, dispatcher }`. JWT decoded locally as a defensive fallback when the gateway has not pre-attached `req.user` (mirrors `reporting-service/routes/insights.js#authenticate`).
- 400 on invalid `reportKey` (must match `/^[a-z0-9-]{1,64}$/i`) or malformed body. 413 if body > 256 KB. 502 `AI_UNAVAILABLE` when Anthropic upstream errors. All paths logged via `analytics/logger#logAiInteraction`.
- Swagger block added at the route in `ai-router.js`. The gateway already proxies `/api/ai/*` → ai-service (`gateway/index.js:547`), so no gateway changes were needed.

### FN-1121 (subtask) — FE narrative panel
- New `<app-report-narrative>` declared in `ReportsModule` and rendered above the KPI cards / chart in `report-view.component.html`.
- `ReportsService.getNarrative(reportKey, payload)` POSTs to `${apiUrl}/ai/reports/:reportKey/narrative` (gateway-routed to ai-service per the API contract above).
- New types `ReportNarrative` and `ReportNarrativeRequest` added to `reports.models.ts`.
- Narrative fetch fires in parallel with chart rendering: it kicks off as soon as report data resolves (so the chart paints first and is never blocked by the narrative request) and keeps its own loading/failed state.
- States: panel hidden when no narrative is available and not loading; shimmer placeholder while loading; panel hides silently on error (no toast spam); shows AI summary text plus a "generated at" timestamp on success. Honors `prefers-reduced-motion`.
- Theme: only CSS custom properties from `src/styles.css` (`--card-background`, `--text-dark`, `--text-light`) plus `rgba(...)` derived from existing palette accents — no new hardcoded hex.
- Unit spec covers: renders text on emit, shimmer + `aria-busy=true` while loading, panel hidden on error, panel hidden when idle/empty; plus a sanity check for the formatted timestamp.

## Key Decisions

### FN-1123 (subtask)
- **Cache strategy:** ephemeral `cache_control` on system prompt and per-report schema; user message stays uncached. Cache is keyed implicitly per `reportKey` via the schema block's content, so different reports get independent cache entries.
- **No fallback narrative:** unlike loads-nlq which returns `{ fallback: true }` on parse failure, narrative has no useful fallback so the handler returns 502 and lets the FE hide the panel (per FN-1114 AC: "gracefully hides on error").
- **RBAC at handler:** chose handler-level role guard (not new middleware) so the change is contained. JWT decode is `lazyRequire`d so `jsonwebtoken` stays optional from the package perspective.
- **Model env override:** `ANTHROPIC_NARRATIVE_MODEL` env var lets ops swap models without a deploy.

### FN-1121 (subtask)
- **Render order — narrative above KPI cards (above chart)**: per the parent story AC ("renders panel above chart"), placed `<app-report-narrative>` above the existing `.cards` block so it sits above both the KPI cards and the table/chart area in the report view.
- **Parallel fetch semantics**: the narrative endpoint contract requires `cards`/`data` in the request body, so the FE cannot literally start the narrative request before the report response. To honor "does not block initial chart render", we set `isLoading=false` (chart paints) before kicking off `fetchNarrative()`; the narrative panel manages its own loading state and never delays chart paint.
- **Hide-on-error, no toasts**: `failed` flag flips to `true` when the request errors; the component's `*ngIf` removes the entire panel from the DOM. No global toast/snackbar fires, matching the AC and avoiding noise when ai-service is unavailable.
- **CSS extension `.css`** (not `.scss`): matches the existing convention of every other component in `reports/components/` (kpi-cards, report-chart, report-table, report-filters all use `.css`). The subtask description listed `.scss` but the convention takes precedence.
- **Theme tokens**: used existing `:root` custom properties (`--card-background`, `--text-dark`, `--text-light`). The teal/blue accent gradient/border uses `rgba(45, 212, 191, …)` and `rgba(59, 130, 246, …)` — values that match other AI-themed surfaces in the app (daily-briefing, control-center) rather than introducing new colors. FN-1191 will sweep the rest of Reports SCSS to fully tokenize accent colors.
