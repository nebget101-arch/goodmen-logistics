# FN-273: Update driver form date pickers to match Loads Dashboard design

## Requirement
The date picker inputs in the Add New Driver and Edit Driver modals need to match the dark-theme calendar design used on the Loads Dashboard (pickup/delivery date filters).

## Acceptance Criteria
- [x] Add New Driver modal: DOB, Hire Date, CDL Expiry, Medical Cert Expiry date pickers match Loads Dashboard design
- [x] Edit Driver modal: DOB, Hire Date, CDL Expiry, Medical Cert Expiry date pickers match the same design
- [x] Calendar popup renders with dark navy background, cyan today highlight, clean grid
- [x] Input fields have dark background, cyan focus border, and styling matching the Loads reference
- [x] Date picker inputs visually consistent with other form inputs in driver modals
- [x] No regression in date value handling (YYYY-MM-DD format preserved)

## Dependencies
- None

## Agent
frontend

## Implementation Summary
Added CSS overrides in `drivers.component.css` to style `app-ai-date-picker` within `.dqf-field` context. The overrides harmonize the date picker input fields with the surrounding `.form-input` styling (matching background gradient, border color, and dimensions) while keeping the cyan focus accent from the Loads Dashboard reference design.

The calendar popup was already correctly styled via global `ai-datepicker-overlay` panel class in `styles.css` — no changes needed there.

### Changes Made
- Matched background gradient to `.form-input`: `rgba(15, 23, 42, 0.98)` with blue radial gradient
- Matched border color: `rgba(148, 163, 184, 0.8)` (was `rgba(71, 85, 105, 0.95)`)
- Kept cyan focus border from Loads design: `rgba(56, 189, 248, 0.95)` with glow
- Matched font size (13px), placeholder color, toggle icon sizing
- Same border-radius (10px) and dimensions (38px height) as form inputs

## Files Changed
- `frontend/src/app/components/drivers/drivers.component.css` — Added ~40 lines of CSS overrides for `app-ai-date-picker` within `.dqf-field` context

## Key Decisions
- Used CSS overrides in the component stylesheet rather than modifying the shared `ai-date-picker` component to avoid affecting other pages
- Kept cyan focus color (matching Loads Dashboard reference) rather than green (used by `.form-input:focus`) — date pickers have a distinct interactive element (calendar toggle) that benefits from the cyan accent
- No HTML template changes needed — the `app-ai-date-picker` component configuration is already correct

## Deployment Handoff
- **Services to restart**: Frontend only (Angular build)
- **DB migrations**: None
- **Env/config changes**: None
- **Cache invalidation**: Browser cache clear recommended for CSS changes
- **Post-deploy verification**: Open Drivers page → Add New Driver → verify date fields match design; Edit any driver → verify same
