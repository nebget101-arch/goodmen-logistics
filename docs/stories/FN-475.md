# FN-475: Build FMCSA Inspections View in Safety Module

## Requirement
New `/safety/fmcsa/inspections` route showing FMCSA inspection records stored from the FN-474 backend, with summary cards, filterable data grid, click-to-detail panel, and manual match capability.

## Acceptance Criteria
- [x] Summary cards: Total, OOS Count, Clean Count, Unmatched Count
- [x] Data grid: Date, State, Level, Vehicle, Driver, Violations, OOS badge, Match Status badge
- [x] Click row → detail panel with violations table and vehicle list
- [x] Match Vehicle/Driver modal for unmatched/partial records
- [x] Filters: date range, match status dropdown, OOS only toggle
- [x] Rematch Unmatched button triggers bulk rematch endpoint

## Implementation Summary

Single-page component at `/safety/fmcsa/inspections` (standalone route, not under SafetyShell). Uses `ChangeDetectionStrategy.OnPush` with manual `cdr.markForCheck()`. Split layout: scrollable table on the left, sticky detail panel on the right when a row is selected.

## Files Changed
- `frontend/src/app/safety/fmcsa-safety.service.ts` — added `FmcsaInspection`, `FmcsaInspectionDetail` interfaces + `getInspections`, `getInspectionDetail`, `matchInspection`, `rematchInspections` methods
- `frontend/src/app/safety/fmcsa-inspections/fmcsa-inspections.component.ts` (NEW)
- `frontend/src/app/safety/fmcsa-inspections/fmcsa-inspections.component.html` (NEW)
- `frontend/src/app/safety/fmcsa-inspections/fmcsa-inspections.component.css` (NEW)
- `frontend/src/app/safety/safety.module.ts` — registered `FmcsaInspectionsComponent`
- `frontend/src/app/safety/safety-routing.module.ts` — added `fmcsa/inspections` route

## Key Decisions
- Reuses `FmcsaSafetyService` (already `providedIn: 'root'`) — no new service file needed
- Uses `forkJoin` to load vehicles + drivers in parallel for match modal dropdowns
- `computeSummary()` derives KPI card values from the current page of rows (not a separate endpoint)
- Match confidence displayed as a decimal (0.0–1.0) from the backend

1. Navigate to `/safety/fmcsa/inspections`
2. Verify KPI cards render (zeros if no data yet)
