# FN-1406: Locations admin list — AI dark-theme redesign + responsive columns

## Requirement
Redesign the Locations admin list (`/admin/locations`) to match the AI dark-theme polish of recent list redesigns (FN-1352 loads, FN-1383 trucks, FN-1393 drivers). Today's layout stacks `NAME / CODE`, `TYPE`, `CITY / STATE` headers in a single column producing a cramped, vertically dense look inconsistent with the rest of the app.

## Acceptance Criteria
- [x] Each header (`Name / Code`, `Type`, `City / State`, `Phone`, `Bins`, `Users`, `Status`, `Actions`) sits in its own column with sensible widths — no stacked headers in a single cell.
- [x] SCSS pulls AI dark-theme tokens (CSS variables) instead of hardcoded hex values; matches the visual rhythm of `loads-list`/`trucks-list`/`drivers-list`.
- [x] Type badges match chip styling from trucks (ownership chip) / loads (status chip).
- [x] Active/Inactive pill matches the shared pattern used elsewhere.
- [x] Action icons have consistent sizing and tooltips.
- [x] At ≤1024px low-priority columns (Phone, Bins, Users) collapse gracefully.
- [ ] No regressions: search, type/status filters, pagination, sort, row click → Edit Location modal, Add Location, Refresh. _(QA — FN-1410)_

## Implementation Summary

### FN-1409 (subtask)
Branch: `frontend/FN-1409/locations-list-redesign`

**HTML (`locations-list.component.html`)**
- Each `<th>` already had its own column; the perceived "stacked header" was a CSS bug — `display: flex` was set on `.sortable` (a `<th>`), which forced flex layout on table cells and made adjacent cells visually collapse. Wrapped each sortable header's text + sort icon in an inner `<span class="th-content">` so the `<th>` keeps native table layout while the inner span owns the flex.
- Added per-column class hooks (`col-name`, `col-type`, `col-city`, `col-phone`, `col-count`, `col-status`, `col-actions`) and tagged Phone / Bins / Users with `col-secondary` so the responsive rules can hide them as a group.
- Wrapped the header copy in `.header-left` for cleaner flex alignment, added `aria-label` to the four icon-only action buttons (parity with FN-1393 dispatch list), and renamed the table class to `ai-table` to mirror the shared naming pattern referenced in the ticket.

**SCSS (`locations-list.component.scss`)**
- Re-tokenized: hardcoded greys (`#e5e7eb`, `#94a3b8`, `#cbd5e1`, `#475569`, `#9ca3af`) routed to `var(--text-dark)`, `var(--text-light)`, or rgba slate variants used in FN-1352/FN-1383. Hardcoded brand cyans (`#38bdf8`, `#67e8f9`) routed to rgba accents (`rgba(56, 189, 248, *)`, `rgba(103, 232, 249, *)`) consistent with `vehicles.component.css` and `loads-dashboard.component.scss`.
- Type chips: switched from light pastel backgrounds (`#e8f4fd`, `#f0fdf4`, `#fef3c7`, …) to dark-theme rgba accents with matching border + soft text per chip pattern used in `trucks-list` (ownership) / `loads-list` (status). Pill shape (`border-radius: 999px`) matches existing chip rhythm.
- Status pill: kept Active green / Inactive slate but tightened to match `vehicles-dashboard .badge-success` rgba accent (border + soft text).
- Filter bar inputs: replaced custom dark fill with the same `radial-gradient + rgba(15,23,42,0.96)` background used in `.vehicles-dashboard .search-input` / `.filter-select`, plus the shared `--fn-select-chevron` for `<select>` chevrons.
- Action buttons: bumped icon-button to 32×32 (was 30×30), 8 px radius (was 6 px), focus-visible outline ring, and danger hover that matches `btn-icon-danger` rhythm in vehicles.
- Primary "Add Location" button: adopted the radial-gradient teal/cyan pill from FN-1383 trucks header (the only "primary CTA" pattern in the recent redesigns) so the affordance is consistent across admin lists.
- Responsive: `@media (max-width: 1024px)` hides `.col-secondary` (Phone, Bins, Users); `@media (max-width: 768px)` stacks the header, tightens row padding, and reduces `min-width` so the table stays usable on phone widths via horizontal scroll (no card view — neither loads/trucks/drivers redesigns introduced one).
- Sticky `<thead>` so the header stays visible when scrolling long lists (matches FN-1383 trucks behavior).

**TS (`locations-list.component.ts`)**
- No changes. All template bindings (`getTypeBadgeClass`, `getTypeLabel`, `sortIcon`, `onSort`, …) are already present and unchanged.

## Key Decisions
- **Bug fix vs. structural rewrite of the table.** The existing `<th>` markup was already one cell per header — the symptom in the ticket ("crams the first three under one column") was caused by `display: flex` on the `<th>` itself, which collapses normal table layout. Fixed via the `.th-content` wrapper rather than restructuring columns; lower diff, no behavior change.
- **No card view at narrow widths.** None of the three reference redesigns (FN-1352, FN-1383, FN-1393) introduce a card layout below 768 px — they all keep the table and rely on horizontal scroll. Followed that precedent rather than inventing a new responsive mode.
- **No `LocationsListComponent` spec exists in the repo.** Verified via `find …/locations-list -name '*.spec.ts'` → none. No new spec added; visual + interaction validation belongs to FN-1410 QA.
- **Tokens stayed pragmatic.** `styles.css` exposes only a small palette (`--text-dark`, `--text-light`, `--border-color`, `--success-color`, …). The reference redesigns themselves layer rgba accents directly (e.g. `rgba(56, 189, 248, 0.15)` for cyan tints) rather than introducing new CSS variables. Followed that precedent — the ticket's "tokens" are the global vars *plus* the established rgba accent palette, not a fresh design token system.
