# FN-550: Add Equipment Owner Percentage to Pay Configuration

## Requirement
Currently the pay config only stores Driver percentage. The system assumes the remainder goes to the Equipment Owner — this is incorrect because the company also retains a portion.

Add an explicit Equipment Owner Percentage field to the pay configuration. Company % is auto-calculated as `100 - Driver% - EO%` (read-only).

## Acceptance Criteria
- [ ] Pay Configuration section shows three rows: Driver % (editable), Equipment Owner % (editable), Company % (read-only, auto-calculated)
- [ ] Company % = 100 - Driver% - EO%, updates in real-time as user types
- [ ] Validation: Driver% + EO% cannot exceed 100%; show error if they do
- [ ] For Owner Operators: only Driver % shown (they are their own EO); Company % = 100 - OO%
- [ ] Backend stores `equipment_owner_percentage` on `driver_compensation_profiles` (new DECIMAL 5,2 column, nullable)
- [ ] Migration: backfill existing records where `equipment_owner_percentage IS NULL` as `100 - percentage_rate`
- [ ] Settlement calculation uses explicit EO% instead of `gross - driverPay` remainder fallback
- [ ] Backend save endpoint validates EO% and returns 400 if Driver% + EO% > 100

## Implementation Summary

### FN-554 (database)
Added `equipment_owner_percentage DECIMAL(5,2) NULLABLE` to `driver_compensation_profiles` with:
- Idempotent `hasColumn` guard (safe to run multiple times)
- Backfill: sets `100 - percentage_rate` for non-owner_operator rows where `percentage_rate` is set
- Owner operator rows left NULL (they are their own equipment owner)
- `down()` drops the column

### FN-555 (backend)
- **drivers.js**: POST and PUT driver endpoints now accept `equipmentOwnerPercentage` in the request body. Validates `percentage_rate + equipment_owner_percentage <= 100`, returns 400 if violated. Stores value on `driver_compensation_profiles.equipment_owner_percentage`.
- **settlement-calculation.js**: `computeLoadPay` now accepts `equipmentOwnerPercentage`. When set (non-null, > 0), uses it to compute `additionalPayeePay = gross * equipmentOwnerPercentage / 100`. Falls back to `additionalPayeeRate`, then to remainder (legacy).
- **settlement-service.js**: All three `computeLoadPay` call sites updated to pass `equipmentOwnerPercentage` from profile snapshot.

### FN-556 (frontend)
- `dispatch-drivers.component`: Added Equipment Owner % input and Company Retains (read-only) fields to Pay Configuration section.
- Real-time Company% = 100 - Driver% - EO% computed on input.
- Owner Operator: EO% field hidden; Company% = 100 - OO%.
- Validation error shown if Driver% + EO% > 100; save blocked.
- Payload includes `equipment_owner_percentage` on save; loaded from profile on open.

## Key Decisions
- Owner operator rows backfill to NULL, not `100 - percentage_rate`, because the EO% concept does not apply to them
- Backfill uses `percentage_rate IS NOT NULL` guard so flat-pay profiles (no percentage_rate) are not backfilled to a nonsensical value
