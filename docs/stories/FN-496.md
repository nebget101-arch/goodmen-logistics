# FN-496: Equipment Owner on Vehicles & Pay Config Auto-Population

## Requirement
Auto-populate the Equipment Owner on a driver's pay configuration when a truck is assigned, so accounting doesn't need to manually look up truck ownership.

## Acceptance Criteria
- [x] Vehicle edit form (trucks only): new "Equipment Owner" field — plain text input for `equipment_owner_name`
- [x] When a driver's assigned truck changes → frontend reads `truck.equipment_owner_name` → auto-fills the Equipment Owner field
- [x] If truck change results in a DIFFERENT Equipment Owner → confirmation dialog shown before overwriting
- [x] Equipment Owner field on Pay Configuration is read-only when auto-populated from vehicle
- [x] Pay Configuration: renamed "Additional Payee" section header/tab to "Equipment Owner"
- [x] `GET /api/vehicles?equipment_owner_id=X` filter added to backend

## Dependencies
- FN-495 (Schema — `equipment_owner_name` column on `vehicles`)

## Agent
frontend + backend

## Implementation Summary

### Backend (`vehicles.js`)
Added `equipment_owner_id` query-param filter to `GET /api/vehicles`. Guarded by `vehicleColumns.has('equipment_owner_id')` so it's safe before FN-495 migration runs.

### Frontend — Vehicle Form (`vehicle-form.component.*`)
- Added `equipment_owner_name` to `VehicleFormData` interface
- Added "Ownership" section (trucks only, hidden for trailers) with a plain text input for `equipment_owner_name`
- Included in `loadFormData()` and `onSubmit()` payload; `null` when empty
- `PUT /:id` on the backend already accepts any column present in `vehicleColumns` — no backend route change needed

### Frontend — Dispatch Drivers (`dispatch-drivers.component.*`)
- Renamed "Additional payee" tab → **"Equipment Owner"** (icon changed to `business_center`)
- Renamed "Additional payee" label → **"Equipment Owner"** in both the Rates tab payee inline section and the Equipment Owner tab
- Added `equipmentOwnerAutoPopulated: boolean` flag; when `true`, the input is read-only (disabled + styled)
- Added `pendingOwnerChange` state; when a new truck has a different owner than the current selection, a yellow confirmation banner appears with "Update Owner" / "Keep Current" buttons
- `onTruckSelected(truckId)` reads `truck.equipment_owner_name` from the already-loaded `this.trucks` array — no additional HTTP call
- Clear button dismisses auto-populated state, allowing manual override
- Added CSS for `.auto-populated-badge` (indigo pill), `.owner-change-dialog` (amber confirmation banner)

## Key Decisions
- Equipment Owner on vehicle is stored as a plain text name (`equipment_owner_name`), not a FK, because the `contacts` table doesn't exist yet (FN-495 adds the nullable FK column `equipment_owner_id` which links to contacts when available)
- Auto-population reads from the already-loaded `trucks` array — no extra HTTP call needed
- The `PUT /api/vehicles/:id` route already accepts any column present in `vehicleColumns` via the dynamic update pattern, so no backend route change was needed for saving `equipment_owner_name`

- **Deploy order**: Run FN-495 migration → redeploy frontend + drivers-compliance-service

  - Change truck to one with different owner → confirmation dialog appears
