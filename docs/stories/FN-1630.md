# FN-1630: DQF CDL extract — fetch failed (AI_SERVICE_URL missing on drivers-compliance-service)

## Requirement
Restore the FN-1625 CDL upload → AI-extract → prefill flow on dev. Currently `POST /api/dqf/cdl-extract` returns `{success:false, reason:'ai_unavailable'}` on every request because the `drivers-compliance-service` runtime has no `AI_SERVICE_URL` env var, so `cdl-extraction-service.js` falls back to `http://localhost:4100` and undici raises `TypeError: fetch failed`.

## Root Cause
`render.yaml` defines `AI_SERVICE_URL` on `fleetneuron-logistics-gateway` (line 35) and `fleetneuron-logistics-service` (line 341), but **not** on `fleetneuron-drivers-compliance-service` — the service that actually mounts `dqf.js` and calls the CDL extraction helper.

## Fix
Add to `render.yaml` inside the `fleetneuron-drivers-compliance-service` envVars block (alongside the existing R2/PG env vars, before `NODE_ENV: production`):

```yaml
      - key: AI_SERVICE_URL
        value: https://fleetneuron-ai-service.onrender.com
```
