## Key Decisions

- Supply rules loaded on source change via `GET /inventory/supply-rules?locationId=...` (`api.getLocationSupplyRules()`)
- Destination dropdown falls back to all locations if no supply rules are found (prevents empty dropdown for new warehouses)
- Suggested quantities = `max(1, reorder_level − on_hand_qty)`; panel only shown when destination has `auto_replenish=true` rule and at least one low-stock item
- `getInventory(toLocationId)` reused to fetch shop stock for suggestion panel
