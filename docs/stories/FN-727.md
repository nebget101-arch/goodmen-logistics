# FN-727: Load Templates, Cloning & Return Load

## Requirement
Support three load-reuse patterns on the Loads page:
- **Templates** — named reusable snapshots of a load
- **Clone** — duplicate a load with new dates/number, status=DRAFT
- **Return Load** — flip origin/destination for the backhaul trip

### FN-755 (subtask) — Frontend — Load Templates admin page

**Branch**: `frontend/FN-755/load-templates-admin` (merge target for the story)

**Scope delivered**
- New route `/loads/templates` rendering an admin list of templates with columns **Name · Description · Broker · Route · Last Used · Actions**
- `Templates` button in the loads-dashboard header navigates to the list
- **Save as Template** button on an open saved load (edit modal footer) opens a prompt for name + description and calls `POST /api/load-templates`
- **Use Template** button on the list navigates back to `/loads` via router state; loads-dashboard pre-fills the inline new-load form with the template's broker, stops, rate, driver/truck/trailer, and notes (dates and PO are intentionally cleared) and status is set to `DRAFT`
- `markUsed(templateId)` (`POST /:id/use`) is called on Use to bump `last_used_at`
- **Edit** opens a modal to update name/description only (template_data is immutable from the UI)
- **Delete** confirms then removes the template

**Files changed**
- `frontend/src/app/services/load-templates.service.ts` (new) — typed HTTP client for the template API
- `frontend/src/app/components/loads-dashboard/load-templates/load-templates.component.ts|html|scss` (new) — admin list page at `/loads/templates`
- `frontend/src/app/components/loads-dashboard/loads-dashboard.module.ts` — declares the new component and adds the `templates` child route
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.ts` — adds `LoadTemplatesService`/`Router` injection, `navigateToTemplates()`, Save-As-Template modal state + submit, `applyTemplateToManualForm()`, and router-state detection in `ngOnInit` (`applyUseTemplateFromRouterState()`)
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.html` — Templates button in header, Save-as-Template button in edit modal footer, Save-As-Template prompt modal
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.scss` — styles for Templates button and Save-As-Template modal

**Key decisions**
- **Separate route** (`/loads/templates`) rather than a tab, because the existing loads-dashboard component is already ~2800 lines; a child route keeps concerns separate and is cheaper to maintain.
- **Use Template delivers via router state + inline new-load form**, not the not-yet-implemented 4-step wizard (FN-732 placeholder). When the wizard is wired up in FN-733 through FN-736, `applyTemplateToManualForm()` can be retargeted to seed wizard state instead, without changing the URL-level contract.
- **Dates and PO are not templated** — templates are reusable across trips; seeding a date would almost always be wrong.
- **Edit is name/description only.** Template_data edits are out of scope — users re-snapshot by re-saving a load.

**API contract consumed (owned by FN-753)**
- `GET /api/load-templates` → `{ success, data: LoadTemplateListItem[] }`
- `GET /api/load-templates/:id` → `{ success, data: LoadTemplateDetail }` (includes `template_data`)
- `POST /api/load-templates` body `{ load_id, name, description? }`
- `PATCH /api/load-templates/:id` body `{ name?, description? }`
- `DELETE /api/load-templates/:id`
- `POST /api/load-templates/:id/use` — bumps `last_used_at`

The list response is expected to include flattened display columns: `broker_name`, `first_pickup_city/state`, `last_delivery_city/state`, `last_used_at`.
