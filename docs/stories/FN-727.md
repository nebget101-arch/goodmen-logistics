## Implementation Notes

### FN-752 — Migration: load_templates
**Branch**: `database/FN-752/load-templates-table`
**Migration file**: `backend/packages/goodmen-database/migrations/20260418150000_create_load_templates.js`

**Schema:**
- `id UUID PK` (default `uuid_generate_v4()`)
- `tenant_id UUID NOT NULL`
- `name TEXT NOT NULL`
- `description TEXT NULL`
- `template_data JSONB NOT NULL` — full snapshot consumed by the load wizard
- `created_by UUID NOT NULL` → FK `users(id)` `ON DELETE RESTRICT`
- `last_used_at TIMESTAMPTZ NULL`
- `created_at TIMESTAMPTZ DEFAULT now()`
- `updated_at TIMESTAMPTZ DEFAULT now()`

**Constraints / indexes:**
- `UNIQUE (tenant_id, name)` → `uq_load_templates_tenant_name` — prevents duplicate template names within a tenant.
- `INDEX (tenant_id)` → `idx_load_templates_tenant_id` — list/scope queries are always tenant-scoped.

### FN-753 — Backend load templates CRUD endpoints
**Branch**: `backend/FN-753/load-templates-route`

Added `backend/packages/goodmen-shared/routes/load-templates.js` with six handlers, all gated by `auth(['admin', 'dispatch'])` and scoped to `tenant_id` via `tenantContextMiddleware`. Route mounted on the logistics service and proxied via the gateway at `/api/load-templates`.

**`POST /api/load-templates`** — `{ name, description?, load_id? | template_data? }`. If `load_id` is supplied, snapshots the load + stops (tenant + operating-entity scoped) into `template_data`. Records `source_load_id`. Duplicate `(tenant_id, name)` → 409.

**`GET /api/load-templates`** — list scoped to tenant. Optional `?q=` ILIKE on name/description. Paginated. Ordered by `COALESCE(last_used_at, updated_at, created_at) DESC`.

**`GET /api/load-templates/:id`** — 404 if not in tenant.

**`PATCH /api/load-templates/:id`** — partial update of name, description, template_data. Duplicate-name → 409.

**`DELETE /api/load-templates/:id`** — hard delete within tenant scope.

**`POST /api/load-templates/:id/use`** — stamps `last_used_at = NOW()` and returns the template so the wizard can pre-fill.

### FN-754 — Backend `/clone` and `/return-load` endpoints
**Branch**: `backend/FN-754/load-clone-return-endpoints`

Added two POST handlers to `backend/packages/goodmen-shared/routes/loads.js`. Gated by `requireRole(['admin', 'dispatch'])`, use existing `getLoadDetail(client, loadId, req.context)` (tenant + OE scoped), and **never persist** — both return a wizard-ready draft payload.

Helper `buildDraftFromLoad(sourceLoad, sourceStops, options)` centralizes draft construction. Options: `clearBroker`, `clearPo`, `clearRate`, `newLoadNumber`, `flipStops`. Draft shape matches what `POST /api/loads` already accepts (camelCase) so the wizard can round-trip without mapping.

**`POST /api/loads/:id/clone`** — draft with: new `loadNumber` via `generateLoadNumber`, `status='DRAFT'`, `billingStatus='PENDING'`, `poNumber=null`, rate/broker/driver/truck/trailer/notes copied, stops copied 1:1 with `sequence` preserved and all date fields (`date|stopDate|stop_date`) cleared. 404 if source load is out of tenant/OE scope.

**`POST /api/loads/:id/return-load`** — same as clone with: `rate` cleared, `stops[]` reversed and each `stopType` flipped (`PICKUP` ↔ `DELIVERY`). Broker/driver/truck/trailer kept.

### FN-755 — Frontend Load Templates admin page

## Key Decisions

### FN-752
- `onDelete('RESTRICT')` on `created_by` — `created_by` is NOT NULL; `SET NULL` would violate the constraint.
- `uuid_generate_v4()` default — consistent with sibling tables.
- `hasTable` guard on `up` — idempotent (matches FN-741 pattern).

### FN-753
- Tenant + operating-entity scoping on the source load for POST snapshots.
- Two creation modes: `load_id` for Save-as-Template, `template_data` for wizard drafts.
- Explicit `POST /:id/use` (not side-effect on load creation) — clean separation from FN-754's non-persisting endpoints.
- `COALESCE(last_used_at, updated_at, created_at)` list ordering — recent usage bubbles to top.
- Role gating `admin + dispatch` (not drivers — templates are an administrative surface).

### FN-754
- Draft shape matches `POST /api/loads` camelCase so frontend round-trips without mapping.
- Reuse `getLoadDetail` for reads (tenant/OE scoping).
- Server-side `generateLoadNumber` — wizard opens with a real unique number.
- Return-load flip: reverse array AND toggle each `stopType` (`PICKUP`/`DELIVERY`).
- Clear all three date aliases (`date|stopDate|stop_date`) to prevent stale leaks.
- `source_load_id` in response — useful for "Cloned from #XXXX" UI header.
