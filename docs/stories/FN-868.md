# FN-868: S8 — Wire View flow as read-only wizard with Edit toggle

## Requirement
Wire `<app-load-wizard>` into view mode. Given `[mode]="'view'"` + `[loadId]`, prefill all four step FormGroups from `getLoad`, disable all inputs, hide Submit, and render a footer with Back / Next / Close. A header Edit button (permission-gated on `loads.update`) flips `mode` to `'edit'` while preserving the current step.

- [ ] Footer shows Back / Next / Close (no Submit).
- [ ] Edit button in wizard header flips mode to `'edit'`; current step preserved.
- [ ] Edit button hidden for users without `loads.update` permission.
- [ ] Attachments step renders read-only list with download links (no dropzone, no delete).

## Key Decisions
- **Permission code**: AC specified `loads.update` but the app's canonical permission constant is `PERMISSIONS.LOADS_EDIT = 'loads.edit'` (see `frontend/src/app/models/access-control.model.ts:81`). Used the canonical code so the gate integrates with the existing RBAC map without introducing a new alias.
- **Disable strategy**: Called `this.form.disable()` from the parent on init instead of relying on each step child's `applyModeState()`. This matters because step children mount lazily via `*ngSwitchCase`, so an unmounted step would otherwise stay enabled. The parent-side disable ensures a user navigating between steps in view mode never sees an editable control before the child re-asserts state.
- **Edit-button placement**: Put the Edit button inside `<div class="load-wizard-panel">` as an absolute-positioned top-right header, not inside `<app-wizard-shell>`. The shared shell is consumer-agnostic (used by other wizards too); this keeps view-mode semantics co-located with the load wizard.
- **Footer change is shared**: Moved the Back/Next/Close footer logic into `app-wizard-shell` itself (not load-wizard-only). View mode is a generic wizard concept, and future wizards (fuel, IFTA, etc.) should inherit the same read-only footer contract.
- **Mode as mutable input**: The `@Input() mode` is mutated directly by `onEditClick()`. Angular re-runs child `ngOnChanges` when the bound value changes, which causes each step's `applyModeState()` to re-enable its FormGroup — no explicit re-bind needed.
