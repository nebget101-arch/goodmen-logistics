# FN-728: Enhanced AI Extraction with Confidence-Based Auto-Approval, Vision Fallback & Caching

## Requirement
Enhance `load-ai-extractor` to support per-field and overall confidence scoring, auto-approval logic, vision fallback for scanned PDFs, and PDF-hash caching so repeated uploads skip the AI call entirely.

## Acceptance Criteria
- [ ] Confidence per field + overall_confidence in response
- [ ] Auto-approval logic drives create status (`overall_confidence >= 0.90 && mode === 'bulk'`)
- [ ] Vision fallback triggers on scanned PDFs
- [ ] Prompt extracts all new fields (multi-stop, appointment times, contacts, commodity, special instructions)
- [ ] Cache hit returns immediately with no OpenAI API call
- [ ] Cache TTL enforced (7 days)

## Implementation Summary

### FN-738 — Confidence Scoring (branch: `ai/FN-738/confidence-scoring`)

Enhanced `backend/packages/goodmen-shared/services/load-ai-extractor.js`:

**Threshold constants:**
- `CONFIDENCE_TIER_GREEN = 0.95` (>95% → green)
- `CONFIDENCE_TIER_YELLOW = 0.80` (80–95% → yellow; <80% → red)
- `AUTO_APPROVE_THRESHOLD = 0.90`

**New helpers:**
- `confidenceTier(score)` → `'green' | 'yellow' | 'red'`
- `computeOverallConfidence(conf)` → `Math.min(...requiredFields)` (broker, rate, pickup, delivery)

**Success return now includes:**
```js
{
  confidence:              { brokerName, poNumber, rate, pickup, delivery },
  confidence_tiers:        { brokerName, poNumber, rate, pickup, delivery },
  overall_confidence:      0.0–1.0,
  overall_confidence_tier: 'green'|'yellow'|'red',
  auto_approve:            true|false  // mode==='bulk' && overall >= 0.90
}
```

**Error returns** all use a `ZERO_CONFIDENCE` spread for a consistent response shape.

### FN-741 — PDF Hash Cache (branch: `database/FN-741/load-ai-extractions-cache`)

**Migration:** `backend/packages/goodmen-database/migrations/20260418130000_create_load_ai_extractions.js`

Table `load_ai_extractions`:
- `id UUID PK` (uuid_generate_v4)
- `tenant_id UUID NOT NULL`
- `pdf_hash TEXT NOT NULL` (SHA-256 hex of raw PDF bytes)
- `extracted_data JSONB NOT NULL`
- `extraction_method TEXT` (`'pdf-parse'` | `'pdftotext'` | `'vision'`)
- `created_at TIMESTAMPTZ DEFAULT now()`
- UNIQUE `(tenant_id, pdf_hash)`
- INDEX `idx_load_ai_extractions_created_at`

**Cache logic in `load-ai-extractor.js`:**

`extractLoadFromPdf(buffer, filename, opts = {})` — new `opts` fields:
- `opts.tenantId` — enables cache lookup + store when provided
- `opts.skipCache` — bypass cache even when tenantId is set (force re-extraction)

New helpers:
- `computePdfHash(buffer)` → SHA-256 hex string
- `getCachedExtraction(tenantId, pdfHash)` → row or null (respects 7-day TTL, swallows DB errors)
- `storeCachedExtraction(tenantId, pdfHash, extractedData, extractionMethod)` → upserts, swallows errors
- `_getDb()` → lazy knex require (returns null if DB unavailable — graceful degradation)

Cache hit response includes `cache_hit: true` added to the returned object.

Cache store uses `INSERT … ON CONFLICT DO UPDATE` to refresh `created_at` on re-upload.

## Key Decisions
1. **`Math.min` for overall_confidence** — weakest required field drives the overall score.
2. **`ZERO_CONFIDENCE` spread** — consistent response shape on all failure paths.
3. **`opts.mode` / `opts.tenantId`** — non-breaking; existing callers pass `(buffer, filename)` and get defaults.
4. **Lazy `_getDb()`** — `load-ai-extractor` works in no-DB environments; cache is silently skipped.
5. **`INSERT … ON CONFLICT DO UPDATE`** — re-uploading the same PDF refreshes the TTL instead of erroring.
6. **`opts.skipCache`** — lets callers force re-extraction without changing function signature (useful for QA).

## Deployment Handoff
- **Services to restart**: `goodmen-shared` is a shared package; restart `ai-service`, `logistics-service` (loads), and any bulk-upload service after merge.
- **Migrations to run**: `20260418130000_create_load_ai_extractions.js` — run before deploying services.
- **Env/config changes**: None.
- **Feature flags**: Cache is activated automatically when `opts.tenantId` is passed by the calling route. If `tenantId` is not passed, cache is silently skipped.
- **Deploy order**: Run migration first, then restart backend services.
- **Post-deploy verification**:
  1. Upload a clear PDF with `tenantId` → confirm `cache_hit` absent (first upload).
  2. Upload same PDF again → confirm `cache_hit: true` in response with no OpenAI API call logged.
  3. Check `load_ai_extractions` table directly: `SELECT count(*) FROM load_ai_extractions WHERE tenant_id = '<id>'`.
  4. Force TTL test: manually update `created_at` to 8 days ago → confirm next upload re-extracts.
  5. Pass `opts.skipCache = true` → confirm fresh extraction even for a cached hash.
