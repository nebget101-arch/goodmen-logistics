## Implementation Notes

### FN-739 (subtask)
**Vision fallback — detect scanned PDFs and use Claude Vision API**

Added to `backend/packages/goodmen-shared/services/load-ai-extractor.js`:

1. **`isScannedPdf(text)`** — heuristic detection:
   - Text < 100 chars → scanned
   - Alphanumeric ratio < 20% → scanned
   - Catches image-only PDFs that pdf-parse/pdftotext can't extract text from

2. **`extractWithVision(buffer, filename)`** — Claude Vision API fallback:
   - Converts PDF pages to PNG using `pdftoppm` (Poppler, 200 DPI, max 5 pages)
   - Sends page images to Anthropic Claude Vision API (claude-sonnet-4) with the same LOAD_EXTRACTION_PROMPT
   - Parses JSON response (strips markdown fences if present)
   - Returns same schema as text-based extraction
   - Uses `ANTHROPIC_API_KEY` env var (separate from OPENAI_API_KEY)
   - 90-second timeout for vision API call
   - Full temp file cleanup in finally block

3. **Modified `extractLoadFromPdf()`**:
   - After text extraction, checks `isScannedPdf()` or `looksGarbled()` or empty text
   - If any trigger: calls `extractWithVision()` before returning empty result
   - Returns `extraction_method: 'vision'` + `vision_fallback_reason` when vision used
   - Returns `extraction_method: 'text'` for normal text-based extraction
   - Falls back to original empty warning only if vision also fails

## Deployment Handoff

- **Services to redeploy**: `fleetneuron-ai-service` or any service that imports load-ai-extractor
- **DB migrations**: None
- **Environment/config changes**:
  - `ANTHROPIC_API_KEY` — required for vision fallback (already set for other AI features)
  - `ANTHROPIC_VISION_MODEL` — optional, defaults to `claude-sonnet-4-20250514`
- **System dependency**: `pdftoppm` (part of Poppler) must be installed on the server (already available for pdftotext)
- **Post-deploy verification**: Upload a scanned rate con PDF → verify extraction returns data with `extraction_method: 'vision'`
