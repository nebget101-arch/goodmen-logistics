## Key Decisions
1. **`Math.min` for overall_confidence** â€” weakest required field drives the overall score; a high-confidence broker name doesn't compensate for a zero-confidence rate.
2. **`ZERO_CONFIDENCE` spread** â€” all error early-returns use a single const so callers always receive a consistent object shape regardless of failure mode.
3. **`opts.mode` parameter** â€” adding `mode` to `opts` (not the top-level signature) is non-breaking; all existing callers pass `(buffer, filename)` and get `auto_approve: false` by default.
4. **Exported helpers** â€” `confidenceTier` and `computeOverallConfidence` exported so consumers (frontend display logic, tests) can apply the same thresholds without duplicating them.
