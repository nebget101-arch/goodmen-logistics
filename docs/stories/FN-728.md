## Implementation Summary

### FN-738 тАФ Confidence Scoring (branch: `ai/FN-738/confidence-scoring`)

**Threshold constants:**
- `CONFIDENCE_TIER_GREEN = 0.95` (>95% тЖТ green)
- `CONFIDENCE_TIER_YELLOW = 0.80` (80тАУ95% тЖТ yellow; <80% тЖТ red)
- `AUTO_APPROVE_THRESHOLD = 0.90`

**New helpers:**
- `confidenceTier(score)` тЖТ `'green' | 'yellow' | 'red'`
- `computeOverallConfidence(conf)` тЖТ `Math.min(...requiredFields)` (broker, rate, pickup, delivery)

**Success return now includes:**
```js
{
  confidence:              { brokerName, poNumber, rate, pickup, delivery },  // 0тАУ1 per field
  confidence_tiers:        { brokerName, poNumber, rate, pickup, delivery },  // tier strings
  overall_confidence:      0.0тАУ1.0,   // min of the 4 required fields
  overall_confidence_tier: 'green'|'yellow'|'red',
  auto_approve:            true|false  // mode==='bulk' && overall >= 0.90
}
```

`opts.mode` drives `auto_approve` (`'single'` default, `'bulk'` for batch upload).
All error returns use `ZERO_CONFIDENCE` spread for consistent response shape.

### FN-739 тАФ Vision Fallback (branch: `ai/FN-739/vision-fallback`)

- **`isScannedPdf(text)`** тАФ heuristic: text < 100 chars OR alphanumeric ratio < 20%
- **`extractWithVision(buffer, filename)`** тАФ Claude Vision fallback:
  - Converts PDF pages to PNG via `pdftoppm` (Poppler, 200 DPI, max 5 pages)
  - Calls Anthropic Claude Vision API (`ANTHROPIC_API_KEY`) with same extraction prompt
  - 90-second timeout, full temp file cleanup
- Vision path returns `extraction_method: 'vision'` + `vision_fallback_reason`
- Text path returns `extraction_method: 'text'`
- Env vars: `ANTHROPIC_API_KEY` (required), `ANTHROPIC_VISION_MODEL` (optional, defaults to `claude-sonnet-4-20250514`)
- System dep: `pdftoppm` (Poppler тАФ already available)

### FN-740 тАФ Prompt Improvements (branch: `ai/FN-740/extraction-prompt-v2`)

### FN-741 тАФ PDF Hash Cache (branch: `database/FN-741/load-ai-extractions-cache`)

**Migration:** `20260418130000_create_load_ai_extractions.js`

Table `load_ai_extractions`: `id UUID PK`, `tenant_id UUID NOT NULL`, `pdf_hash TEXT NOT NULL`, `extracted_data JSONB NOT NULL`, `extraction_method TEXT`, `created_at TIMESTAMPTZ`.
UNIQUE `(tenant_id, pdf_hash)` + INDEX on `created_at`.

**Cache logic in `extractLoadFromPdf(buffer, filename, opts = {})`:**
- `opts.tenantId` тАФ activates cache lookup + store
- `opts.skipCache` тАФ bypass cache (force re-extraction)
- `computePdfHash(buffer)` тЖТ SHA-256 hex
- `getCachedExtraction(tenantId, pdfHash)` тАФ 7-day TTL, swallows DB errors
- `storeCachedExtraction(...)` тАФ UPSERT refreshes TTL on re-upload
- `_getDb()` тАФ lazy knex require (graceful no-DB degradation)
- Cache hit returns `{ ...extractedData, cache_hit: true }`

## Key Decisions
1. **`Math.min` for overall_confidence** тАФ weakest required field drives the overall score.
2. **`ZERO_CONFIDENCE` spread** тАФ consistent shape on all failure paths.
3. **`opts.mode` / `opts.tenantId` / `opts.skipCache`** тАФ non-breaking additive opts.
4. **Lazy `_getDb()`** тАФ extractor works in no-DB environments; cache silently skipped.
5. **`INSERT тАж ON CONFLICT DO UPDATE`** тАФ re-uploading refreshes TTL rather than erroring.
6. **Vision fallback before giving up** тАФ FN-739 logic supersedes the old "no text" / "garbled" hard stops.
