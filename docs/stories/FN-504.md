# FN-504: Add Pre-Hire Risk Assessment Card to DQF & Cross-Module Risk Badge

## Requirement
Add two missing UI features from FN-480: a Pre-Hire Risk Assessment card on the DQF modal, and color-coded risk badges on the Driver list, DQF header, and Dispatch Board.

## Acceptance Criteria
- [x] Pre-Hire Assessment Card appears on DQF page after MVR/PSP analysis
- [x] Card shows score, risk level, hire recommendation, red flags (category breakdown bars)
- [x] Risk badge appears on Driver list with correct color coding
- [x] Risk badge appears on DQF driver header
- [x] Risk badge appears on Dispatch Board driver cards
- [x] Badge tooltip shows risk level and top factor
- [x] Pages without risk data show no badge (graceful fallback)

## Implementation Summary

### Backend: `safety-risk-engine.js`
- Extended `GET /api/safety/driver-risk-scores/fleet-summary` response with `all_scores` array containing all scored drivers (score, risk_level, trend per driver_id), not only high/critical drivers.
- The underlying SQL already fetched all scored drivers; `all_scores` simply exposes the full set for badge rendering without extra queries.

### Frontend: `safety-risk.service.ts`
- Added `DriverRiskBadge` interface (`driver_id`, `score`, `risk_level`, `trend`)
- Added `all_scores?: DriverRiskBadge[]` to `FleetRiskSummary` interface

### `drivers.component.ts`

- `driverRiskMap: Map<string, DriverRiskBadge>` populated via `getFleetSummary()` after driver list loads
- `preHireRisk: DriverRiskScore | null` loaded via `getDriverScore(driverId)` when DQF modal opens
- `loadPreHireRisk()`, `recalculateRisk()`, `getHireRecommendation()`, `getRiskBadgeClass()`, `getRiskBadgeTooltip()` helper methods

### `drivers.component.html`

- Added Pre-Hire Risk Assessment card at top of DQF modal body (above pre-employment gate), showing: score circle, risk level, hire recommendation badge, trend, event count, 4-category breakdown bars (MVR/PSP/FMCSA/Incidents), Re-analyze button

## Key Decisions
- `getFleetSummary()` is used for batch badge loading (single HTTP call) instead of per-driver requests; the existing SQL already returns all scored drivers
- Badge rendering is graceful: `driverRiskMap.get(driver.id)` returns undefined for unscored drivers ‚Üí no badge rendered
- Pre-hire card only renders when `preHireRisk?.current` is non-null; loading state shown while fetching
- `recalculateRisk()` re-fetches score after recalculation completes
- Hire recommendation derived client-side from risk_level: low‚ÜíRecommended, medium/high‚ÜíConditional, critical‚ÜíNot Recommended
