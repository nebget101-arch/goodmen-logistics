## Implementation Notes

### FN-821 (subtask) — Frontend loads-list polish

Branch: `frontend/FN-821/loads-list-polish`

**New components**

- `frontend/src/app/components/loads-dashboard/loading-skeleton/` — the
  `LoadsLoadingSkeletonComponent` renders 10 placeholder rows inside a
  `<table>` whose `<col>` widths mirror the real table's colgroup. Each bar
  is a 200%-wide linear-gradient background whose `background-position`
  animates via a `loads-skeleton-shimmer` keyframe. Rows stagger the shimmer
  by 80ms each so the highlight reads as a single wave. `OnPush` change
  detection and a stable `trackByIndex` keep CD out of the hot path while
  the animation runs.

- `frontend/src/app/components/loads-dashboard/empty-state/` — the
  `LoadsEmptyStateComponent` encapsulates every non-data state on the
  list: `filtered`, `no-loads`, `smart-filter-celebrate`, `api-error`,
  `permission-denied`. Mode drives the icon, copy, and available actions;
  the parent handles the side-effects (clearing filters, opening the
  wizard, triggering a hidden file input for bulk-PDF import, retrying
  the failed fetch).

**Parent wiring** (`loads-dashboard.component.{ts,html,scss}`)

- `loadLoads()` now sets/clears `loadError`, `permissionDenied`, and
  `loadErrorDetail` on every fetch. It also flips `filterPulseActive` for
  ~320ms so the list viewport plays its fade-in animation whenever
  filters/sort/smart-filters change.
- A new `emptyStateMode` getter maps the current component state to the
  right variant. Smart-filters whose "0 matches" reads as good news
  (`overdue`, `needs_review`, `missing_pod`) route to the celebratory
  variant; everything else falls through to `filtered`.
- The `cdk-virtual-scroll-viewport`'s `[itemSize]` binds to the computed
  `rowHeightPx`, which reads from the active density option. Padding and
  line-height scale alongside via the `.loads-dashboard--compact/--comfortable/--spacious`
  modifier classes on the host.
- `UserPreferencesService`'s `LoadsDashboardPrefs` interface gains a
  `density?` field. `loadUserPreferences()` restores the saved mode;
  `setDensity()` persists changes with `patchLoadsDashboard({ density })`.
- A window `scroll` `@HostListener` flips `showScrollTop` once the user
  scrolls past 300px. `scrollToTop()` uses `window.scrollTo({ behavior })`
  and falls back to `behavior: 'auto'` under `prefers-reduced-motion`.
- A hidden `<input type="file" #emptyStateBulkInput>` backs the empty-state
  "Import from PDF" action — clicking the button routes through the
  existing `onHeroBulkPdfs()` flow, so PDFs land in the bulk extraction
  modal without duplicating upload plumbing.

**Animations shipped, wiring deferred to FN-790**

Three effects are defined in CSS but are gated on class names added by the
WebSocket integration (`row-just-created`, `row-just-deleted`, `status-pill--pulse`).
FN-790's `websocket.service.ts` already applies those classes in the client
subscriptions (see `docs/stories/FN-790.md`). Once FN-790 merges, the
animations "come alive" with no further FE work. Same story for the
`wsDisconnected` banner, which simply needs one line wiring
`websocket.status$` to the flag.

**Reduced-motion coverage**

Every keyframe introduced in this change (`loads-skeleton-shimmer`,
`loads-empty-fade-in`, `fn821-banner-slide`, `fn821-row-slide-in`,
`fn821-row-fade-out`, `fn821-status-pulse`, `fn821-viewport-fade`) has a
matching `@media (prefers-reduced-motion: reduce)` override. The
scroll-to-top button also opts out of `behavior: 'smooth'` under the
same query.

**Post-deploy verification**

1. Open `/loads` as a dispatcher. Before the request returns, confirm the
   shimmer skeleton renders (10 rows, column widths match the header).
2. Apply any filter → observe the skeleton reappear briefly, then the
   list fade in.
3. Apply a filter combination that returns zero rows → the filtered
   empty-state card appears with a "Clear filters" CTA that works.
4. Toggle a smart-filter chip known to return zero (e.g. Overdue on a
   healthy tenant) → celebratory variant shows.
5. In DevTools Network, throttle `Offline` and reload `/loads` → the
   api-error empty-state shows with a working "Try again" button. Restore
   the network; retry should succeed.
6. Click the Density toolbar button → cycle Compact / Comfortable /
   Spacious; row heights visibly change and persist across a refresh.
7. Scroll the page past the intelligence panel → the floating
   scroll-to-top button appears; clicking it returns smoothly to the top.
8. In OS settings, enable "Reduce motion" → reload; the shimmer freezes
   but placeholders remain visible, row hover lift stops, the scroll-to-top
   jump becomes instant.

**Rollback**: revert the subtask branch or the merged story PR. No
backend or DB state to undo.
