# FN-498: Recurring Deductions — applies_when & Configurable Sharing

## Requirement
As a fleet accountant, I need to configure when recurring deductions apply (always vs only when loads exist) and how they're split between Driver and Equipment Owner, so settlements calculate correctly for idle periods and shared expense arrangements.

## Acceptance Criteria
- [x] Recurring deduction form adds `applies_when` toggle: "Always (even if no loads)" / "Only when loads exist"
- [x] Sharing configuration when expense_responsibility = "Shared": Split type (Percentage or Fixed Amount) with configurable values
- [x] Backend validates percentage splits sum to 100%
- [x] Backend validates fixed amount splits sum to total deduction amount
- [x] List view shows applies_when and split configuration per rule

## Implementation Summary
1. **Migration**: Added `expense_responsibility`, `split_type`, `driver_share`, `owner_share` columns to `recurring_deduction_rules` table
2. **Backend**: Updated POST and PATCH routes in `settlements.js` with sharing split validation — percentage splits must sum to 100%, fixed amount splits must sum to total deduction amount
3. **Frontend**: Updated `scheduled-deductions` component:
   - Added `expense_responsibility` dropdown (Company/Driver/Owner/Shared)
   - Conditionally shows split config (split type, driver share, owner share) when "Shared" is selected
   - Client-side validation mirrors backend validation
   - List view now shows "Applies when", "Responsibility", and "Split" columns
   - Updated `appliesWhenOptions` labels to match AC wording

## Key Decisions
- Split columns added directly to `recurring_deduction_rules` (not a separate join table) for simplicity
- `expense_responsibility` field on each rule lets users mark Company/Driver/Owner/Shared independently of the expense_responsibility_profiles table
- When expense_responsibility changes away from "shared", backend clears split fields to null
