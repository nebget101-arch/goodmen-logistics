## Implementation Summary

### FN-765 (subtask)
Global keyboard shortcut system with a help modal that lists every shortcut active on the current view.

- **`KeyboardShortcutsService`** (`frontend/src/app/shared/services/keyboard-shortcuts.service.ts`)
  - A single `window`-level `keydown` listener dispatches to bindings registered via `register()` / `registerAll()`.
  - Input-field guard: bindings are skipped when focus is inside `<input>`, `<textarea>`, `<select>`, or a `contentEditable` element unless the binding opts in with `allowInInput: true` (used for Cmd/Ctrl+S which must save from within the form).
  - Listener runs outside Angular's zone to avoid change-detection churn on unrelated keystrokes; matched handlers are re-entered into the zone.
  - Exposes `bindings$` and `helpOpen$` observables so the help modal updates reactively.
  - Static `formatKey(binding)` helper renders mac glyphs (⌘/⇧/⌥) on macOS and `Ctrl+…` / `Shift+…` elsewhere.
- **`KeyboardShortcutsHelpComponent`** (`frontend/src/app/shared/components/keyboard-shortcuts-help/`)
  - Rendered once at app root (`app.component.html`) so it works on every view.
  - Registers the `?` (Shift+/) global shortcut that opens / toggles itself and unregisters it on destroy.
  - Groups bindings by `ShortcutBinding.group` (e.g. "Loads", "Load wizard", "Global") and shows formatted keys + descriptions.
  - Closes on Esc (captured at document level with `stopPropagation()` so no other Esc handler runs), the ✕ button, or a backdrop click.
- **Loads list wiring** (`loads-dashboard.component.ts`)
  - Registers `N` (New load → `openLoadWizard()`), `Cmd/Ctrl+K` (quick search), and `/` (focus search) on `ngOnInit`, releases all three in `ngOnDestroy` via the single callback returned by `registerAll()`.
  - Added `@ViewChild('searchInput')` + `focusSearch()` so the header search input can be targeted.
- **Load wizard migration** (`load-wizard.component.ts`)
  - Removed the inline `@HostListener('document:keydown')` and re-registered its four shortcuts (`Esc`, `Cmd/Ctrl+S`, `Cmd/Ctrl+Shift+S`, `Enter`) through the service so they appear in the help modal *only while the wizard is mounted*. This cleanly scopes "Save", "Save & New", and "Close" to the wizard context.

### Key Decisions (FN-765)
- **Window-level listener, not document-level.** `window.addEventListener('keydown')` catches events earlier and consistently across focus states. Existing component-level `HostListener('document:keydown')` usages are unaffected because `preventDefault()` doesn't block other listeners, only browser defaults.
- **No-bind-collision rule.** Service bindings are matched exactly on modifier flags, so `Cmd+S` and `Cmd+Shift+S` can coexist — Shift must be on/off exactly as declared, not truthy-or-falsy. The earlier inline HostListener used an explicit shift check first to avoid shadowing; encoding it into the matcher removes the ordering pitfall.
- **Wizard owns its shortcuts.** Rather than registering wizard shortcuts globally (visible even when the wizard is closed), the wizard component registers them in its own lifecycle. The help modal therefore only advertises what is actually live.
- **`allowInInput` flag.** `Cmd/Ctrl+S` and `Cmd/Ctrl+Shift+S` must work from inside form fields (that's the whole point — save what you were editing). `N`, `/`, `?` must NOT fire while typing, otherwise a dispatcher typing "notes" would open the wizard. Per-binding opt-in handles both.
- **Runs outside the Angular zone.** The keydown listener is registered in `runOutsideAngular` so background typing doesn't trigger change detection across the app. We re-enter the zone via `zone.run(handler)` only when a binding actually matches.

- **Post-deploy verification**:
  1. Log in, open the **Loads** page → press `?` → help modal opens listing "New load (N)", "Quick search loads (⌘K / Ctrl+K)", "Focus search bar (/)", and "Show keyboard shortcuts (?)".
  2. Press `N` → load wizard opens. Inside the wizard press `?` again → help modal now also lists the wizard shortcuts (Save, Save & New, Close, Next). Close the wizard → those wizard rows disappear from the help modal next time it's opened.
  3. Click a text field, type — pressing letters should not trigger shortcuts. Press `Cmd/Ctrl+S` inside the wizard → save still fires.
  4. `/` with focus in the body → search input focuses. `/` while already typing in a field → character types normally.
  5. `Esc` inside the wizard with unsaved edits → unsaved-changes warning appears; `Esc` again dismisses the warning without closing the wizard.
