## Implementation Summary

### FN-764 (subtask)
Smart defaults now prefill every new load opened via the 4-step wizard:

- **Status = `DRAFT` / Billing = `PENDING`** — applied in `LoadsDashboardComponent.defaultBasics()` so every new wizard session starts in a safe, editable state rather than advancing the lifecycle (previously defaulted to `BOOKED`/`UNBILLED`).
- **Dispatcher = current user** — `applySmartDefaults()` reads `dispatcherName` (already fetched via `LoadsService.getCurrentUser()`) and seeds `wizardBasics.dispatcher`. The field remains read-only in `step-basics`.
- **Pickup = today, Delivery = today + 1** — `applySmartDefaults()` seeds the default pickup/delivery stops with local-time-safe `YYYY-MM-DD` values via `formatLocalDate()`. If the wizard is later opened from clone/return (stops already populated), existing `stop_date` values are preserved.
- **Previous driver suggested** — new `UserPreferencesService.getRecentDriverId(dispatcherId)` is called during `applySmartDefaults()`. When the user selects/changes a driver in the wizard (`onWizardDriverChange`), the same service writes the id back, keyed per dispatcher so multiple users on one browser get the right suggestion.
- **Broker history → PO format + rate range** — `StepBasicsComponent.loadBrokerSuggestion()` fires on `selectBroker()` (and on init when a brokerId was pre-filled). It calls `LoadsService.listLoads({ brokerId, pageSize: 20, sortBy: 'created_at', sortDir: 'desc' })` and computes:
  - PO format: most common non-digit prefix + typical digit length rendered as `PO-XXXXX`
  - Rate stats: min / avg / max across non-zero rates
  - Hints appear as subtle italic text under the PO and Rate fields with an info icon; they are purely informational and do not auto-fill the fields.

### Files changed (FN-764)
- `frontend/src/app/services/user-preferences.service.ts` — **new** (localStorage-backed per-dispatcher preferences).
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.ts` — `defaultBasics()` flipped to DRAFT/PENDING; `openLoadWizard()` → `applySmartDefaults()`; `onWizardDriverChange()` remembers recent driver; new `formatLocalDate()` helper; added `UserPreferencesService` to the DI list.
- `frontend/src/app/components/loads-dashboard/load-wizard/step-basics/step-basics.component.ts` — `BrokerSuggestion` interface, `loadBrokerSuggestion()`, `poHint`/`rateHint` getters, `computeBrokerSuggestion()` aggregator; suggestions cleared on `clearBroker()`.
- `frontend/src/app/components/loads-dashboard/load-wizard/step-basics/step-basics.component.html` — `<small class="field-hint">` blocks under PO and Rate fields bound to `poHint`/`rateHint`.
- `frontend/src/app/components/loads-dashboard/load-wizard/step-basics/step-basics.component.css` — `.field-hint` + `.hint-icon` styles.

### Key Decisions (FN-764)
- **Hints, not auto-fill, for broker-derived suggestions.** Auto-filling the PO and rate would mask genuine negotiation differences between loads from the same broker; showing a hint lets dispatchers use them as guardrails without trusting stale data.
- **Recent-driver keyed by dispatcher id, not user session.** Previous implementation (`step-driver` component, `fn_recent_driver_ids`) tracked the last 5 recent drivers globally per browser. FN-764 specifically wants *per-dispatcher* memory, so we chose a new service + key namespace rather than mutating the existing recent-list behaviour.
- **Local-date formatting.** Used `formatLocalDate()` (Y-M-D built from local getters) instead of `toISOString().slice(0, 10)` because the latter shifts to UTC and causes off-by-one defaults for users east of GMT after ~17:00 local.
- **Scope bounded to the wizard.** The inline new-load form (`manualLoadForm`) and clone/return flows are intentionally untouched — they already compute their own defaults, and the AC scopes FN-764 to the wizard path.

  4. Open Dev Tools → Application → Local Storage → confirm a `fn_recent_driver_by_dispatcher:<userId>` key is written on driver selection.
