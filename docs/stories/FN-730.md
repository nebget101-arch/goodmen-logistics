## Implementation Summary

### FN-764 (subtask) — Smart defaults in the Load Wizard
Smart defaults now prefill every new load opened via the 4-step wizard.

- **Status = `DRAFT` / Billing = `PENDING`** — applied in `LoadsDashboardComponent.defaultBasics()` so every new wizard session starts in a safe, editable state rather than advancing the lifecycle (previously defaulted to `BOOKED`/`UNBILLED`).
- **Dispatcher = current user** — `applySmartDefaults()` reads `dispatcherName` (already fetched via `LoadsService.getCurrentUser()`) and seeds `wizardBasics.dispatcher`. The field remains read-only in `step-basics`.
- **Pickup = today, Delivery = today + 1** — `applySmartDefaults()` seeds the default pickup/delivery stops with local-time-safe `YYYY-MM-DD` values via `formatLocalDate()`. If the wizard is later opened from clone/return (stops already populated), existing `stop_date` values are preserved.
- **Previous driver suggested** — `UserPreferencesService.getRecentDriverId(dispatcherId)` is called during `applySmartDefaults()`. When the user selects/changes a driver in the wizard (`onWizardDriverChange`), the same service writes the id back, keyed per dispatcher so multiple users on one browser get the right suggestion. Storage is localStorage (best-effort) via key `fn_recent_driver_by_dispatcher:<dispatcherId>`.
- **Broker history → PO format + rate range** — `StepBasicsComponent.loadBrokerSuggestion()` fires on `selectBroker()` (and on init when a brokerId was pre-filled). Calls `LoadsService.listLoads({ brokerId, pageSize: 20, sortBy: 'created_at', sortDir: 'desc' })` and computes:
  - PO format: most common non-digit prefix + typical digit length rendered as `PO-XXXXX`.
  - Rate stats: min / avg / max across non-zero rates.
  - Hints appear as subtle italic text under the PO and Rate fields with an info icon; they are purely informational and do not auto-fill the fields.

### FN-765 (subtask) — Keyboard shortcuts service + help modal
Global keyboard shortcut system with a help modal that lists every shortcut active on the current view.

- **`KeyboardShortcutsService`** (`frontend/src/app/shared/services/keyboard-shortcuts.service.ts`)
  - A single `window`-level `keydown` listener dispatches to bindings registered via `register()` / `registerAll()`.
  - Input-field guard: bindings are skipped when focus is inside `<input>`, `<textarea>`, `<select>`, or a `contentEditable` element unless the binding opts in with `allowInInput: true`.
  - Listener runs outside Angular's zone; matched handlers are re-entered into the zone.
  - Exposes `bindings$` and `helpOpen$` observables so the help modal updates reactively.
  - Static `formatKey(binding)` helper renders mac glyphs (⌘/⇧/⌥) on macOS and `Ctrl+…` / `Shift+…` elsewhere.
- **`KeyboardShortcutsHelpComponent`** — rendered once at app root, groups bindings by `group`, closes on Esc/✕/backdrop.
- **Loads list wiring** — registers `N` / `Cmd/Ctrl+K` / `/` on mount; unregisters on destroy.
- **Load wizard migration** — inline `@HostListener` replaced with service registration for `Esc`, `Cmd/Ctrl+S`, `Cmd/Ctrl+Shift+S`, `Enter`.

### FN-766 (subtask) — Virtual scrolling for loads list (Angular CDK)
Introduced `@angular/cdk/scrolling` virtual scrolling around the loads table body so render cost stays flat regardless of `filteredLoads.length`. Existing server-side pagination (`pageSize = 25`) is untouched.

- Registered `ScrollingModule` in `LoadsDashboardModule`.
- Split the dashboard table into two `<table>` elements sharing identical `<colgroup>` percentage widths and `table-layout: fixed`.
- Viewport sized via `.loads-viewport { height: calc(100vh - 420px); min-height: 320px; }`.
- Empty state moved out of tbody into a sibling `.empty-state` div.
- `itemSize="48"`, `minBufferPx="400"`, `maxBufferPx="800"`.

### FN-767 (subtask) — Column visibility + saved filter views + inline status change
Branch: `frontend/FN-767/loads-saved-views`

- **Persistence layer.** Added a JSONB `preferences` column on `users` (migration `20260419300000_add_preferences_to_users.js`).
- **Backend route.** New shared route `backend/packages/goodmen-shared/routes/user-preferences.js` exposes `GET /api/user-preferences` and `PUT /api/user-preferences`. PUT shallow-merges the body at the top level into `users.preferences`. Payload capped at 32 KB. Mounted in the auth-users-service and proxied through the gateway.
- **Frontend service.** `user-preferences.service.ts` provides a `BehaviorSubject`-backed cache with `load()`, `patch()`, and namespace helpers `getLoadsDashboardPrefs()` / `patchLoadsDashboard()`. **Merged with FN-764's localStorage-backed recent-driver helpers** in the same service, so the component has a single injection point for both server-persisted and local prefs.
- **Column visibility.** Loads-dashboard carries a `columnDefs` metadata array (13 columns, two flagged `alwaysVisible`: Load # and Actions). "Columns" menu toolbar above the table toggles checkboxes and persists to `loadsDashboard.columnVisibility`.
- **Saved filter views.** "Views" menu lists persisted views and applies them. Users can save the current filter combo. Three starter views (My Drafts, This Week, Unpaid) are seeded in-memory.
- **Inline status change.** Status cell replaces read-only pill with a button-wrapped pill + chevron → dropdown → optimistic PATCH via existing `PUT /api/loads/:id`.

### FN-768 (subtask) — Bulk actions on loads list
Multi-select bulk operations on the loads list, backed by two new transactional endpoints.

- **Backend** (`backend/packages/goodmen-shared/routes/loads.js`)
  - `POST /api/loads/bulk-update` — whitelisted to `status`, `billingStatus`, `driverId`, `truckId`. Validates every id belongs to the caller's `tenant_id` + `operating_entity_id` *before* writing; any stranger ids roll back the whole transaction and are returned in `notFound`. Mirrors single-PUT behaviour by auto-filling `completed_date` when a bulk transition to DELIVERED/COMPLETED is applied. Hard cap 500 ids per request.
  - `POST /api/loads/bulk-delete-drafts` — same tenant scoping + transactional semantics, refuses to delete unless *every* selected load is currently `DRAFT`. Non-drafts returned in `nonDraft`.
  - Both endpoints gated by `requireRole(['admin', 'dispatch'])`.
- **Frontend service** (`frontend/src/app/services/loads.service.ts`) — `bulkUpdate(ids, changes)` and `bulkDeleteDrafts(ids)`.
- **Frontend UI** — per-row checkbox + header select-all (indeterminate state for partial selection). Bulk actions toolbar renders above the table when `selectedCount > 0`:
  - Change Status / Assign Driver / Assign Truck (dropdowns → POST bulk-update)
  - Approve drafts (filters to DRAFT, POST bulk-update `{status: 'NEW'}`)
  - Delete drafts (filters to DRAFT, POST bulk-delete-drafts)
  - Export CSV (client-side — from loaded rows)
  - Toolbar hidden entirely in driver-role views; action buttons disable while a bulk request is in flight.

## Key Decisions

### FN-764
- **Hints, not auto-fill, for broker-derived suggestions.** Auto-filling the PO and rate would mask genuine negotiation differences between loads from the same broker; hints let dispatchers use them as guardrails.
- **Recent-driver keyed by dispatcher id, not user session.** Tracks last driver *per-dispatcher* rather than the earlier global last-5 list.
- **Local-date formatting.** Used `formatLocalDate()` (Y-M-D built from local getters) instead of `toISOString().slice(0, 10)` to avoid UTC-shift off-by-one for users east of GMT after ~17:00 local.
- **Scope bounded to the wizard.** Inline `manualLoadForm` + clone/return flows are untouched.

### FN-765
- **Window-level listener, not document-level.**
- **No-bind-collision rule** — exact-modifier matching so `Cmd+S` and `Cmd+Shift+S` coexist.
- **Wizard owns its shortcuts** — scoped lifecycle registration.
- **`allowInInput` opt-in** — `Cmd/Ctrl+S` works in inputs; `N`/`/`/`?` do not.
- **Outside Angular zone** — avoids change-detection churn on unrelated keystrokes.

### FN-766
- **Keep pagination.** Virtual scroll and server pagination are orthogonal.
- **Two-table split** — static header + virtualized body keeps filter inputs pinned.
- **Fixed `itemSize`** — all rows share padding/font.
- **`trackBy: trackByLoadId`** — required to avoid re-creating child components on every scroll tick.
- **Viewport `calc(100vh - 420px)`** with `min-height: 320px`.

### FN-767
- **Extend `users.preferences` JSONB vs. separate `user_preferences` table.** JSONB column avoids a join on every dashboard mount.
- **Top-level shallow merge in PUT.** Namespaced by feature key.
- **Seed default views in-memory only** until the user interacts.
- **`alwaysVisible` for Load # and Actions.**
- **Optimistic inline status update with rollback.**
- **Single service for all prefs surfaces** — FN-764's localStorage recent-driver and FN-767's server-persisted prefs live in the same `UserPreferencesService`, so the component has one injection point.

### FN-768
- **Whitelist, don't passthrough.** `bulk-update` explicitly enumerates the four columns it accepts (`status`, `billingStatus`, `driverId`, `truckId`).
- **Pre-flight SELECT then UPDATE inside the same transaction.** Aborts before any write if any id is unknown; deterministic error messages.
- **Two endpoints instead of a single multiplexed one.** Splitting keeps the delete path obviously scoped to drafts.
- **Delete/approve apply only to drafts in the selection.** UI filters client-side so users don't curate twice; backend still enforces "drafts only".
- **CSV export is client-side** — no duplicate server endpoint; rows already tenant-scoped.
- **Hard cap at 500 ids.** Keeps worst-case response time bounded.
