## Implementation Summary

### FN-765 (subtask) — Keyboard shortcuts service + help modal
Global keyboard shortcut system with a help modal that lists every shortcut active on the current view.

- **`KeyboardShortcutsService`** (`frontend/src/app/shared/services/keyboard-shortcuts.service.ts`)
  - A single `window`-level `keydown` listener dispatches to bindings registered via `register()` / `registerAll()`.
  - Input-field guard: bindings are skipped when focus is inside `<input>`, `<textarea>`, `<select>`, or a `contentEditable` element unless the binding opts in with `allowInInput: true` (used for Cmd/Ctrl+S which must save from within the form).
  - Listener runs outside Angular's zone to avoid change-detection churn on unrelated keystrokes; matched handlers are re-entered into the zone.
  - Exposes `bindings$` and `helpOpen$` observables so the help modal updates reactively.
  - Static `formatKey(binding)` helper renders mac glyphs (⌘/⇧/⌥) on macOS and `Ctrl+…` / `Shift+…` elsewhere.
- **`KeyboardShortcutsHelpComponent`** (`frontend/src/app/shared/components/keyboard-shortcuts-help/`)
  - Rendered once at app root (`app.component.html`) so it works on every view.
  - Registers the `?` (Shift+/) global shortcut that opens / toggles itself and unregisters it on destroy.
  - Groups bindings by `ShortcutBinding.group` (e.g. "Loads", "Load wizard", "Global") and shows formatted keys + descriptions.
  - Closes on Esc (captured at document level with `stopPropagation()` so no other Esc handler runs), the ✕ button, or a backdrop click.
- **Loads list wiring** (`loads-dashboard.component.ts`)
  - Registers `N` (New load → `openLoadWizard()`), `Cmd/Ctrl+K` (quick search), and `/` (focus search) on `ngOnInit`, releases all three in `ngOnDestroy` via the single callback returned by `registerAll()`.
  - Added `@ViewChild('searchInput')` + `focusSearch()` so the header search input can be targeted.
- **Load wizard migration** (`load-wizard.component.ts`)
  - Removed the inline `@HostListener('document:keydown')` and re-registered its four shortcuts (`Esc`, `Cmd/Ctrl+S`, `Cmd/Ctrl+Shift+S`, `Enter`) through the service so they appear in the help modal *only while the wizard is mounted*.

### FN-766 (subtask) — Virtual scrolling for loads list (Angular CDK)
Introduced `@angular/cdk/scrolling` virtual scrolling around the loads table body so render cost stays flat regardless of `filteredLoads.length`. The existing server-side pagination (`pageSize = 25`) is untouched — CDK virtualization makes the component safe if pagination is later relaxed or if aggressive header-filter narrowing produces large local result sets.

**Shape of the change**
- Registered `ScrollingModule` in `LoadsDashboardModule` (`loads-dashboard.module.ts`).
- Split the single dashboard table into two `<table>` elements sharing identical `<colgroup>` percentage widths and `table-layout: fixed`:
  - **`.table--header`** — colgroup + thead (column labels + header-filter row). Stays outside the viewport so filter inputs remain interactive while scrolling rows below.
  - **`.table--body`** — colgroup + tbody, placed **inside** `cdk-virtual-scroll-viewport` with `*cdkVirtualFor` on the data `<tr>` and `trackBy: trackByLoadId` for stable row recycling.
- Viewport sized via `.loads-viewport { height: calc(100vh - 420px); min-height: 320px; }` — the container needs a definite height for virtualization to engage.
- Empty state moved out of the tbody into a sibling `.empty-state` div.
- `itemSize="48"` — matches current td padding + font metrics. `minBufferPx="400"` / `maxBufferPx="800"` give ~8 rows above/below the visible window.
- Added `trackByLoadId(_i, load) => load.id`.

## Key Decisions

### FN-765
- **Window-level listener, not document-level.** `window.addEventListener('keydown')` catches events earlier and consistently across focus states.
- **No-bind-collision rule.** Service bindings are matched exactly on modifier flags, so `Cmd+S` and `Cmd+Shift+S` can coexist — Shift must be on/off exactly as declared.
- **Wizard owns its shortcuts.** Rather than registering wizard shortcuts globally, the wizard component registers them in its own lifecycle. The help modal therefore only advertises what is actually live.
- **`allowInInput` flag.** `Cmd/Ctrl+S` and `Cmd/Ctrl+Shift+S` must work from inside form fields. `N`, `/`, `?` must NOT fire while typing.
- **Runs outside the Angular zone.** Background typing doesn't trigger change detection; we re-enter the zone via `zone.run(handler)` only when a binding matches.

### FN-766
- **Keep pagination.** Virtual scroll and server pagination are orthogonal — one protects render cost, the other protects API/DB cost.
- **Two-table split over `display: contents` hack.** Header/body table split is boring, safe, and keeps column alignment via `table-layout: fixed` + identical colgroup percentages.
- **Fixed `itemSize`.** All loads rows have the same `<td>` padding/font, so fixed-size virtualization beats `AutoSizeVirtualScrollStrategy`.
- **`trackBy: trackByLoadId`.** Without it, virtual recycling + default identity tracking re-creates child components on every scroll tick.
- **Viewport height via `calc(100vh - 420px)`.** `min-height: 320px` prevents the viewport from collapsing on short screens.
