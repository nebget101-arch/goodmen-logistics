# FN-543: vehicles-maintenance-service does not run knex.migrate.latest() on startup — all_vehicles view missing in dev DB

## Problem
`vehicles-maintenance-service` never called `knex.migrate.latest()` on startup. As a result, the `all_vehicles` PostgreSQL view (and any other schema objects added by recent migrations) did not exist in its database. The `GET /api/equipment` endpoint silently returned `{ "success": true, "data": [] }`.

Confirmed from Render logs:
```
[equipment] query error — code=42P01 message=relation "all_vehicles" does not exist
```

## Root Cause
`logistics-service` was the only microservice calling `knex.migrate.latest()` on startup. `vehicles-maintenance-service/server.js` used a synchronous `app.listen()` callback with no migration step, so its DB schema was never brought up to date.

## Fix
Updated `app.listen()` in `server.js` to `async`, matching the `logistics-service` pattern exactly:

```js
app.listen(PORT, async () => {
  try {
    await knex.migrate.latest();
    console.log('✅ Database migrations applied');
  } catch (err) {
    console.error('⚠️  Migration error (non-fatal):', err.message);
  }
  console.log(`🛠️ Vehicles maintenance service running on http://localhost:${PORT}`);
});
```

`knex` was already imported — no new dependencies needed.
