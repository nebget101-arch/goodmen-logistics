# FN-422: Toll-to-Settlement Integration

## Overview

Integrates toll transaction consumption into the settlement processing pipeline so toll costs are automatically reflected as deductions in driver settlements.

## Subtasks

### FN-447: BE: Add toll consumption logic to settlement processing pipeline

**Status**: Done
**Branch**: `backend/FN-447/settlement-toll-integration`

#### Changes

**`backend/packages/goodmen-shared/services/settlement-service.js`**
- Added `consumeTollsForSettlement()` function that:
  - Queries unlinked toll transactions (`settlement_link_status = 'none'`) for the driver within the payroll period date range
  - Looks up `expense_responsibility_profiles` for the driver's `toll_responsibility` setting
  - Handles three responsibility modes:
    - `company` -- skip, no deduction from driver
    - `driver` -- full deduction (100% of toll amount)
    - `shared` -- applies split percentage from `custom_rules.toll_split_percentage` (defaults to 50% if not configured)
  - Creates `settlement_adjustment_items` with `source_type = 'imported_toll'`, `charge_party` based on responsibility
  - Links toll transactions by setting `settlement_id`, `settlement_adjustment_item_id`, and `settlement_link_status = 'linked'`
- Integrated `consumeTollsForSettlement` into `createDraftSettlement()` -- runs after recurring deduction insertion, before recalc
- Exported `consumeTollsForSettlement` for use by other routes

**`backend/packages/goodmen-shared/routes/tolls.js`**
- Added `POST /api/tolls/transactions/:id/post-to-settlement` endpoint for manually linking a toll transaction to a settlement
  - Validates toll exists and is not already linked
  - Validates settlement exists, is not void, and belongs to same tenant
  - Applies expense responsibility profile rules for deduction amount
  - Creates adjustment item and links the toll transaction
  - Recalculates settlement totals
- Added `POST /api/tolls/transactions/:id/unlink-from-settlement` endpoint for reversing a manual link
  - Validates toll is currently linked
  - Prevents unlinking from approved/void settlements
  - Removes adjustment item and resets toll settlement fields
  - Recalculates settlement totals

#### Database Fields Used (existing)
- `toll_transactions.settlement_id` -- FK to settlements
- `toll_transactions.settlement_adjustment_item_id` -- FK to settlement_adjustment_items
- `toll_transactions.settlement_link_status` -- 'none' | 'linked'
- `expense_responsibility_profiles.toll_responsibility` -- 'company' | 'driver' | 'shared'
- `expense_responsibility_profiles.custom_rules` -- JSON with `toll_split_percentage` for shared mode
