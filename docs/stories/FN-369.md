# FN-369: Show previous delivery city and state in Driver Position for empty miles context

## Overview
Display the previous load's last delivery city and state in the Driver Position area so dispatchers understand the empty miles origin.

## Subtasks

| Key | Summary | Agent | Branch | Status |
|-----|---------|-------|--------|--------|
| FN-375 | Return previous delivery city/state from computeTripMetrics | backend | `agent/backend/FN-375/prev-delivery-city-state` | Done |
| FN-376 | Display previous delivery city/state in Driver Position UI | frontend | `agent/frontend/FN-376/driver-position-prev-delivery` | Done |
| FN-377 | QA: Validate Driver Position shows previous delivery city/state | qa | — | Done |

### FN-375 (subtask) — Backend: Return prev_delivery_city/state

**Changes:**
- Expanded `computeTripMetrics()` SQL query to `SELECT s.zip, s.city, s.state` (was only `s.zip`)
- Added `prevCity` and `prevState` variables parsed from query result
- Added `prev_delivery_city` and `prev_delivery_state` to the returned metrics object
- These fields flow through `getLoadDetail()` via the existing `...tripMetrics` spread

**Files Changed:**
- `backend/packages/goodmen-shared/routes/loads.js` — `computeTripMetrics()` (~line 70–116)

**Key Decisions:**
- Reused the existing previous delivery query (no new DB query needed — just expanded SELECT columns)
- Null fallback: if no previous delivery exists, both fields return `null`

### FN-376 (subtask) — Frontend: Display prev delivery city/state in Driver Position

**Changes:**
- Added `prev_delivery_city` and `prev_delivery_state` to `LoadDetail` interface
- Updated `driverPositionDisplay` getter to check prev delivery fields as second priority (after `driver_position_city/state`, before current load's last delivery fallback)
- When prev delivery data is available, displays as "Last delivery: City, ST" to clearly distinguish from GPS position
- Graceful fallback: if no prev delivery exists, falls back to current load's last delivery stop

**Files Changed:**
- `frontend/src/app/models/load-dashboard.model.ts` — added `prev_delivery_city`, `prev_delivery_state` to `LoadDetail`
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.ts` — updated `driverPositionDisplay` getter
