# FN-620: Angular Frontend Observability with OpenTelemetry Browser SDK (Phase 3)

## Requirement
Add client-side observability to the Angular frontend so API call traces connect to backend traces, client errors are captured, and page performance is measured.

## Acceptance Criteria
- [ ] OTel Browser SDK initialized in `main.ts` before Angular bootstrap
- [ ] All HttpClient API calls auto-instrumented with traceparent propagation
- [ ] Page load performance captured (FCP, LCP, TTFB)
- [ ] Angular ErrorHandler sends unhandled errors to SigNoz
- [ ] Custom UI spans for settlement create, fuel import, load edit, report generate
- [ ] Environment configs updated with OTel endpoint settings
