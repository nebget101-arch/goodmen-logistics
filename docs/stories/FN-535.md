# FN-535: [Bug] Applicant Certification section doesn't auto-populate from form data

## Problem
The **Applicant Certification & Signature** section at the bottom of the employment application had only `applicantPrintedName` and `applicantSignature` — and these were blank when the driver arrived at the section. All data was already entered earlier in the form (Applicant Information + License History) but was never reflected in the certification section.

## Expected Behavior (source mapping)
| Certification Field | Source |
|---|---|
| Full Name (Applicant Printed Name) | `applicant.firstName` + `applicant.middleName` + `applicant.lastName` |
| Date of Birth | `applicant.dateOfBirth` |
| SSN Last 4 | Last 4 digits of `ssnRawValue` |
| Driver's License Number | `licenses[0].licenseNumber` |
| State of Issue | `licenses[0].state` |

Population is reactive — updates as the driver types, not just on submit. Fields remain editable.

## Fix

### TS (`employment-application.component.ts`)
1. **Added 4 new fields** to the `certification` FormGroup: `dateOfBirth`, `ssnLast4`, `driversLicenseNumber`, `stateOfIssue`
2. **Added `setupCertificationSync()`** — called in `ngOnInit` after `addLicense()`. Subscribes to:
   - `applicant.firstName/middleName/lastName` → patches `applicantPrintedName` + `applicantSignature`
   - `applicant.dateOfBirth` → patches `certification.dateOfBirth`
   - `licenses.at(0).licenseNumber` → patches `certification.driversLicenseNumber`
   - `licenses.at(0).state` → patches `certification.stateOfIssue`
3. **Updated `onSsnInput()`** — after computing `ssnRawValue`, patches `certification.ssnLast4` with the last 4 digits

### HTML (`employment-application.component.html`)
Added 4 new fields to the certification section grid:
- Date of Birth (type="date")
- SSN Last 4 (maxlength=4)
- Driver's License Number
- State of Issue

All show placeholder text indicating they're auto-filled, and all remain editable by the driver.
