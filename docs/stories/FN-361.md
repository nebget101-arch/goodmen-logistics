# FN-361: Unify filter field sizes — dropdowns & date pickers must match text inputs; center placeholder text

## Requirement
Dropdown (`app-ai-select`) and date picker (`app-ai-date-picker`) filter fields are visually taller than adjacent text input filters. Placeholder text is not centered. All filter bar fields must render at the same height with centered placeholder text.

## Acceptance Criteria
- [x] All filter bar fields (text inputs, dropdowns, date pickers) render at the same visual height across all pages
- [x] Placeholder text in dropdowns and date pickers is vertically centered
- [x] Placeholder text is horizontally centered in all filter fields
- [ ] No regression in filter functionality (selections, date picking, clearing) — QA pending
- [ ] Consistent appearance on pages: Dispatch Board, Fuel Transactions, Reports, Safety (Accidents/Claims/Tasks), Settlements, Vehicles — QA pending

## Implementation Summary

### FN-362 (subtask)
Fixed both shared components (`app-ai-select` and `app-ai-date-picker`) to match text input filter height (36px) and center placeholder text.

**Changes to `ai-select.component.scss`:**
- Set explicit `height: 36px` on `.mat-mdc-text-field-wrapper` with `display: flex; align-items: center`
- Reduced `.mat-mdc-form-field-infix` min-height from 38px to 36px, padding from 8px to 6px
- Added `display: flex; align-items: center` to infix for vertical centering
- Hidden `.mat-mdc-form-field-subscript-wrapper` to prevent extra height
- Added `text-align: center` to `.mat-mdc-select-value` and `.mat-mdc-select-placeholder`
- Set placeholder color to `#94a3b8` for consistency

**Changes to `ai-date-picker.component.scss`:**
- Set explicit `height: 36px` on `.mat-mdc-text-field-wrapper` with `display: flex; align-items: center`
- Reduced `.mat-mdc-form-field-infix` min-height from 38px to 36px, padding from 8px to 6px
- Added `display: flex; align-items: center` to infix for vertical centering
- Hidden `.mat-mdc-form-field-subscript-wrapper` to prevent extra height
- Added `text-align: center` to `.mat-mdc-input-element` and `::placeholder`
- Reduced datepicker toggle button from 36x36 to 28x28 to fit inside 36px wrapper
- Updated inline and external-label variants with same flex centering

## Files Changed
- `frontend/src/app/shared/ai-select/ai-select.component.scss`
- `frontend/src/app/shared/ai-date-picker/ai-date-picker.component.scss`

## Key Decisions
- Target height: 36px (matches `.search-input` and `.btn-icon` on Dispatch Board)
- Used `height: 36px` on wrapper (not just `min-height`) for strict enforcement
- Hidden subscript wrapper entirely since these components use `subscriptSizing="dynamic"` and don't show validation errors in filter contexts
