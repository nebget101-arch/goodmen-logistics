# FN-497: Compensation Type Cleanup & Expense Responsibility Rework

## Status: In Progress
**Branch:** `frontend/FN-497/compensation-expense-rework`
**Parent Epic:** FN-494 (Settlement Engine V2)
**Depends on:** FN-495 (schema changes -- included via cherry-pick)

## Summary

Simplifies compensation types from three options (Company Driver, Owner Operator, Hired Driver for Owner) down to two (Driver, Owner Operator). Reworks the expense responsibility section to show only when pay model is Percentage, with three modes: Driver, Equipment Owner, and Shared. Shared mode provides per-category config with percentage sliders for variable expenses and dollar inputs for fixed expenses.

## Changes

### Frontend (`frontend/src/app/components/dispatch-drivers/`)

- **Compensation type chips**: Removed "Hired Driver for Owner", renamed "Company Driver" to "Driver"
- **driverTypeOptions**: Updated to `['driver', 'owner_operator']`
- **responsibilityOptions**: Updated to `['driver', 'equipment_owner', 'shared']` (removed 'company')
- **Expense Responsibility section**: Now conditionally shown only when `payModel === 'percentage'`
  - Owner Operator: auto-sets all expenses to Equipment Owner at 100%
  - Driver type shows three chip options: Driver, Equipment Owner, Shared
  - Shared mode: per-category config panel with:
    - Variable expenses (fuel, tolls, repairs): percentage sliders
    - Fixed expenses (insurance, ELD, trailer rent): dollar input pairs (driver/owner)
  - Non-shared modes: simple read-only label showing assigned party
- **resetForm()**: Resets new shared expense state, default driverType now 'driver'
- **saveDriver()**: Normalizes legacy 'company'/'company_driver' to 'driver'
- **getDriverTypeLabel()**: Handles backward-compatible label mapping
- **CSS**: Added styles for shared-expense-config panel, sliders, fixed-dollar inputs

### Backend (`backend/packages/goodmen-shared/routes/`)

- **drivers.js**: Updated `profile_type` mapping from `'company_driver'` to `'driver'` in both create and update driver flows
- **settlements.js**: Updated default `profile_type` from `'company_driver'` to `'driver'`; added `split_type`, `driver_percentage`, `driver_fixed_amount`, `owner_fixed_amount` columns to expense responsibility profile insert

### Database (cherry-picked from FN-495)

- **Migration `20260329100000_settlement_v2_schema.js`**: Adds `split_type`, `driver_percentage`, `driver_fixed_amount`, `owner_fixed_amount` to `expense_responsibility_profiles`; migrates `company_driver` to `driver` in `driver_compensation_profiles`; removes `hired_driver_for_owner` rows
