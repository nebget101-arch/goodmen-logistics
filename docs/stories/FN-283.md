# FN-283: Split Address field into Address, City, State, Zip with zip code auto-populate

## Requirement
Split the single Address field on the DQF/Driver page into four separate fields: Street Address, City, State, and Zip Code. When a user enters a zip code, City and State should auto-populate via an external zip code lookup API (Zippopotam.us).

## Acceptance Criteria
- [ ] Address field is split into: Street Address, City, State, Zip Code
- [ ] Entering a valid zip code auto-populates City and State via API lookup
- [ ] All four fields are saved separately in the database
- [ ] Existing single-address data is migrated (best-effort parse or left in street address)
- [ ] Edit Driver modal and DQF page both use the new split fields
- [ ] Form validation: zip code format (5-digit or ZIP+4), state as 2-letter code
- [ ] Works on both Add Driver and Edit Driver flows

## Dependencies
- None (standalone enhancement)
