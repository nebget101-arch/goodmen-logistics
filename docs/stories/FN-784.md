## Implementation Summary

### FN-794 (subtask)
Intelligence Panel shell ships with four metric cards and a pill period selector above the loads list.

- **New `IntelligencePanelComponent`** (`frontend/src/app/components/loads-dashboard/intelligence-panel/`)
  - Presentational / dumb. Accepts `[period]`, `[metrics]`, `[loading]`, `[needsAttentionActive]`; emits `(periodChange)` + `(needsAttentionClick)`.
  - Pill selector: Today / Week / Month / All — one active pill at a time.
  - Four cards: **Gross** (sum of rates, $-formatted), **Delivered** (count of DELIVERED/COMPLETED), **In Transit** (count of IN_TRANSIT / EN_ROUTE / PICKED_UP / DISPATCHED, with a pulsing green dot in the header), **Needs Attention** (clickable `<button>`, active highlight when the filter is on).
  - Trend arrows render only when `previous` is supplied on the metric — otherwise the card hides the trend row so we never show a misleading 0%. Direction inferred from a ±0.5% dead-band. Colors: green up / red down / slate flat. For the Needs Attention card the semantics invert (more attention = worse), so the CSS flips the red/green assignment via `data-direction`.
- **Wiring in `LoadsDashboardComponent`**
  - New state: `intelligencePeriod`, `intelligenceMetrics`, `needsAttentionActive`, plus a private `prevIntelligenceAggregate` cache.
  - `onIntelligencePeriodChange(period)` updates the pill, maps to the existing `grossPeriod` values (today, this_week, this_month, all), reloads the current page, and fires a background fetch for the previous equivalent period used to compute trends.
  - `recomputeIntelligenceMetrics()` assembles the four cards from the currently loaded rows and the cached previous-period aggregate. Runs after every `loadLoads()` success.
  - `isNeedsAttention(load)` — composite predicate: DRAFT, `needs_review === true`, or a delivery date in the past with a non-terminal status. Powers both the card count and the client-side filter.
  - Client-side `filteredLoads` gains a `needsAttentionActive` guard so toggling the card instantly narrows the table; no extra server request.
  - Panel embedded in the template just below the alerts, above the existing `.status-summary-bar`, and hidden for driver-role views.
- **Module**: `IntelligencePanelComponent` declared in `LoadsDashboardModule` (it's only used here, so it stays out of `SharedModule`).

### Key Decisions (FN-794)
- **Dumb component, smart parent.** The panel receives metrics — it does not fetch them. When FN-793 ships `/api/loads/ai-insights`, the parent can swap its aggregate source without touching the panel.
- **Trend hidden until previous is known.** Rendering a "0.0% vs prev." on first paint would suggest stable performance even when we simply haven't fetched prior-period data yet. `null previous → no trend row` keeps the UI honest.
- **Client-side Needs Attention filter.** The server has no composite "drafts + overdue + needs_review" filter, and the three components are cheap to check on the currently loaded page. This mirrors how the existing `needs_review` chip narrows results client-side.
- **Previous-period fetch caps pageSize at 500.** The existing list endpoint doesn't expose period aggregates, so we do a second listLoads call bounded to 500 rows for trend. That's enough for realistic trucking fleets and bounded enough to avoid accidentally paging the entire dataset. When FN-793 lands, the fetch will shrink to a single `/ai-insights` call.
- **Existing `grossPeriod` retained.** Rather than splitting the period state, the Intelligence pills drive `grossPeriod` via a small mapping so the existing status summary bar + list filters keep working while we ship the new panel. FN-795 removes / collapses the old bar as part of its scope.

  2. Click **Week** → panel pills switch, list reloads for the current week, Needs Attention count updates. Trend arrows appear on Gross / Delivered / In Transit / Needs Attention when the previous-week fetch returns (may briefly be absent during the network round-trip — expected).

  4. Click **All** → period shows "All", trend arrows disappear (no previous-period comparison available for "all").
  5. Driver role login → Intelligence Panel must not render (it's hidden alongside the other dispatcher-only controls).
