## Implementation Summary

### FN-793 (subtask — backend)

**Branch:** `backend/FN-793/loads-ai-insights`

**What shipped**

- `GET /api/loads/ai-insights?period=today|week|month|all` added to
  `backend/packages/goodmen-shared/routes/loads.js` (registered before
  `GET /:id` so Express routes `ai-insights` correctly).
- Role-gated to `admin` + `dispatch` via the existing `requireRole` helper.
- Tenant + operating-entity scoped through `req.context` (populated by
  `tenant-context-middleware`), mirroring the patterns used elsewhere in
  the file.

**Response shape**

```json
{
  "success": true,
  "period": "week",
  "window": { "from": "…", "to": "…", "prev_from": "…", "prev_to": "…" },
  "metrics": {
    "gross":           { "value": 0, "previous": 0, "delta_pct": 0 },
    "delivered":       { "value": 0, "previous": 0, "delta_pct": 0 },
    "in_transit":      { "value": 0, "previous": 0, "delta_pct": 0 },
    "needs_attention": { "value": 0, "previous": null, "delta_pct": null }
  },
  "insights": [
    { "type": "drafts_ready", "severity": "info", "count": 3,
      "message": "…", "action_url": "/loads?needsReview=true",
      "action_label": "Review drafts" }
  ]
}
```

**Insight rules (all rule-based SQL, no LLM calls)**

| Type | Trigger | Severity |
|---|---|---|
| `drafts_ready` | `status = DRAFT`, `needs_review = true`, `created_at < now() − 2h` | info |
| `overdue` | Latest `DELIVERY` stop_date in the past AND status ∉ delivered/cancelled/draft/TONU | critical |
| `rate_anomaly` | `rate < 70%` of the broker's last-30d average (broker must have ≥3 prior loads) | warn |
| `missing_documents` | Delivered without `PROOF_OF_DELIVERY` attachment OR picked-up/in-transit without `BOL` | warn |
| `driver_idle` | `drivers.status = 'active'`, no active (non-terminal) load updated in last 24h | info |
| `high_margin` | `SUM(fuel_transactions.amount) / loads.rate < 0.15` on period-window loads with fuel | info |
| `low_margin` | `SUM(fuel_transactions.amount) / loads.rate > 0.35` on period-window loads with fuel | warn |

The margin query gracefully degrades to `{ high: 0, low: 0 }` if the
`fuel_transactions` table is unavailable at request time.

**Metric totals**

- `gross` / `delivered` — summed/counted over loads in `DELIVERED`/`COMPLETED`
  whose effective date (completed_date → first pickup stop_date → created_at)
  falls in the window; previous-period comparison uses the same logic shifted
  back one window length.
- `in_transit` — count of loads whose status is
  `EN_ROUTE | PICKED_UP | IN_TRANSIT | DISPATCHED`, same windowing rules.
- `needs_attention` — `drafts_ready + overdue + missing_documents` counts
  (no previous-period comparison).

**Period windows**

- `today` → UTC midnight today (1 day); prev = yesterday.
- `week` → last 7 days; prev = 7 days before that.
- `month` → last 30 days; prev = 30 days before that.
- `all` → no date bound (previous-period deltas become null).

### FN-794 (subtask — frontend shell)

Intelligence Panel shell ships with four metric cards and a pill period selector above the loads list.

- **New `IntelligencePanelComponent`** (`frontend/src/app/components/loads-dashboard/intelligence-panel/`)
  - Presentational / dumb. Accepts `[period]`, `[metrics]`, `[loading]`, `[needsAttentionActive]`; emits `(periodChange)` + `(needsAttentionClick)`.
  - Pill selector: Today / Week / Month / All — one active pill at a time.
  - Four cards: **Gross** (sum of rates, $-formatted), **Delivered** (count of DELIVERED/COMPLETED), **In Transit** (count of IN_TRANSIT / EN_ROUTE / PICKED_UP / DISPATCHED, with a pulsing green dot in the header), **Needs Attention** (clickable `<button>`, active highlight when the filter is on).
  - Trend arrows render only when `previous` is supplied on the metric — otherwise the card hides the trend row so we never show a misleading 0%. Direction inferred from a ±0.5% dead-band. Colors: green up / red down / slate flat. For the Needs Attention card the semantics invert (more attention = worse), so the CSS flips the red/green assignment via `data-direction`.
- **Wiring in `LoadsDashboardComponent`**
  - New state: `intelligencePeriod`, `intelligenceMetrics`, `needsAttentionActive`, plus a private `prevIntelligenceAggregate` cache.
  - `onIntelligencePeriodChange(period)` updates the pill, maps to the existing `grossPeriod` values (today, this_week, this_month, all), reloads the current page, and fires a background fetch for the previous equivalent period used to compute trends.
  - `recomputeIntelligenceMetrics()` assembles the four cards from the currently loaded rows and the cached previous-period aggregate. Runs after every `loadLoads()` success.
  - `isNeedsAttention(load)` — composite predicate: DRAFT, `needs_review === true`, or a delivery date in the past with a non-terminal status. Powers both the card count and the client-side filter.
  - Client-side `filteredLoads` gains a `needsAttentionActive` guard so toggling the card instantly narrows the table; no extra server request.
  - Panel embedded in the template just below the alerts, above the existing `.status-summary-bar`, and hidden for driver-role views.
- **Module**: `IntelligencePanelComponent` declared in `LoadsDashboardModule` (it's only used here, so it stays out of `SharedModule`).

### FN-795 (subtask — frontend insights)
Ships the AI insights list (with 60-second polling) and collapses the Status / Billing chip rows into a default-hidden breakdown.

- **New `IntelligenceInsightsComponent`** (`frontend/src/app/components/loads-dashboard/intelligence-panel/intelligence-insights.component.{ts,html,scss}`)
  - Polls `LoadsService.getAiInsights(period)` on mount, on every `[period]` change, and every 60 seconds (`timer(0, refreshMs)`). The Rx stream is cancelled on destroy.
  - Renders each insight as a card with an icon (type-derived or service-supplied), title, optional subtitle, and an optional right-aligned action button. Severity-based colored left border (blue / amber / red).
  - Supports an optional `href` for navigation via `routerLink` and an `(action)` output for app-level handlers.
  - Empty state: after the first fetch completes with `[]`, the panel shows a check_circle + "everything looks healthy" message. Before the first fetch it says "Loading insights…".
  - Shows a small inline spinner while a refresh is in flight.
- **Shared insight contract** (`loads.service.ts`)
  - New types `AiInsight`, `AiInsightType`, `AiInsightSeverity` locked to what the panel consumes. FN-793's backend endpoint fills this shape.
  - `getAiInsights(period)` degrades gracefully when the endpoint is missing (catches 404/error → `{ success: false, data: [] }`).
- **Collapsible Status/Billing chip rows**
  - `status-summary-bar` gets a "Show/Hide breakdown" toggle (`aria-expanded`).
  - `chipRowsOpen` defaults to `false`, so only period pills + TOTAL render on first paint.
  - When expanded: two sub-sections — Status breakdown and a new **Billing breakdown** with its own color palette (`billingColorMap`) and `billingBarSegments` getter.
  - Billing chip click toggles `filters.billingStatus` via the existing `setBillingFilter()` helper.
- **Module registration**: `IntelligenceInsightsComponent` declared in `LoadsDashboardModule` next to the panel shell.

## Key Decisions

### FN-793
- Rule-based SQL only (no LLM calls) for determinism and zero marginal cost.
- Margin calculation gracefully degrades if `fuel_transactions` is unavailable.
- Previous-period window mirrors the primary window exactly so comparisons are apples-to-apples.

### FN-794
- **Dumb component, smart parent.** The panel receives metrics — it does not fetch them.
- **Trend hidden until previous is known.** `null previous → no trend row` keeps the UI honest.
- **Client-side Needs Attention filter.** Cheap predicate over loaded rows mirrors how the existing `needs_review` chip narrows results.
- **Previous-period fetch caps pageSize at 500** until FN-793's aggregate endpoint is consumed end-to-end.
- **Existing `grossPeriod` retained.** Intelligence pills drive `grossPeriod` so the legacy bar keeps working during the transition.

### FN-795
- **Polling via `timer(0, 60_000)`.** Fires immediately and every 60s; period changes call `_restartPolling()` so the user never waits up to 60s for fresh data.
- **Graceful 404 degradation.** `catchError(() => of({ …empty }))` means the insights panel shows an empty state rather than an error banner when FN-793 hasn't deployed yet.
- **Component stays out of `SharedModule`.** Intelligence pieces are only used on `/loads`.
- **Manual `*ngIf` toggle vs `<details>`.** Keeps styling + animation consistent with the dark theme.
- **Billing breakdown added as a sibling of Status.** Uses existing `summaryTotals.byBilling` data that was already populated but never rendered.
