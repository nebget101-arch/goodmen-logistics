```json
    "gross":           { "value": 0, "previous": 0, "delta_pct": 0 },
    "delivered":       { "value": 0, "previous": 0, "delta_pct": 0 },
    "in_transit":      { "value": 0, "previous": 0, "delta_pct": 0 },

      "action_label": "Review drafts" }

```

**Insight rules (all rule-based SQL, no LLM calls)**

| Type | Trigger | Severity |
|---|---|---|
| `drafts_ready` | `status = DRAFT`, `needs_review = true`, `created_at < now() − 2h` | info |
| `overdue` | Latest `DELIVERY` stop_date in the past AND status ∉ delivered/cancelled/draft/TONU | critical |
| `rate_anomaly` | `rate < 70%` of the broker's last-30d average (broker must have ≥3 prior loads) | warn |
| `missing_documents` | Delivered without `PROOF_OF_DELIVERY` attachment OR picked-up/in-transit without `BOL` | warn |
| `driver_idle` | `drivers.status = 'active'`, no active (non-terminal) load updated in last 24h | info |
| `high_margin` | `SUM(fuel_transactions.amount) / loads.rate < 0.15` on period-window loads with fuel | info |
| `low_margin` | `SUM(fuel_transactions.amount) / loads.rate > 0.35` on period-window loads with fuel | warn |

The margin query gracefully degrades to `{ high: 0, low: 0 }` if the
`fuel_transactions` table is unavailable at request time.

**Metric totals**

- `gross` / `delivered` — summed/counted over loads in `DELIVERED`/`COMPLETED`
  whose effective date (completed_date → first pickup stop_date → created_at)
  falls in the window; previous-period comparison uses the same logic shifted
  back one window length.
- `in_transit` — count of loads whose status is
  `EN_ROUTE | PICKED_UP | IN_TRANSIT | DISPATCHED`, same windowing rules.
- `needs_attention` — `drafts_ready + overdue + missing_documents` counts
  (no previous-period comparison).

**Period windows**

- `today` → UTC midnight today (1 day); prev = yesterday.
- `week` → last 7 days; prev = 7 days before that.
- `month` → last 30 days; prev = 30 days before that.
- `all` → no date bound (previous-period deltas become null).
