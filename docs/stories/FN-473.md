# FN-473: Create FMCSA Inspection, Violation, Risk Scoring & Risk Event Tables

## Requirement
Create database tables to support the FMCSA driver safety intelligence system: store inspection records, violations, composite risk scores, MVR-extracted data, and a full lifecycle event log.

## Acceptance Criteria
- [ ] fmcsa_inspections table with fleet matching fields (VIN, CDL, plate, match_status)
- [ ] fmcsa_violations table linked to inspections with BASIC categories and severity
- [ ] fmcsa_driver_risk_scores table with composite 0-100 scores and trend tracking
- [ ] mvr_extracted_records table for AI-extracted violations/accidents/suspensions
- [ ] driver_risk_events table for full lifecycle event log
- [ ] All tables tenant-scoped with proper indexes and FKs
- [ ] Migration runs cleanly with hasTable guards

## Agent
database

## Implementation Summary
Single migration `20260328500000_create_fmcsa_risk_tables.js` creating 5 tables:
- **fmcsa_inspections**: FMCSA roadside inspection records with fleet matching (truck_id, driver_id, match_status, match_confidence, match_method), VIN/CDL/plate matching fields, OOS flags, severity weight
- **fmcsa_violations**: Individual violations per inspection with BASIC category classification (unsafe_driving, hos_compliance, driver_fitness, etc.)
- **fmcsa_driver_risk_scores**: Composite risk scores per driver per day (0-100), component breakdowns (inspection, oos, violation, mvr, accident), trend tracking
- **mvr_extracted_records**: AI-extracted events from MVR/PSP documents — violations, accidents, suspensions, with confidence score
- **driver_risk_events**: Full lifecycle event log from onboarding through ongoing monitoring, with score delta tracking

## Files Changed
- `backend/packages/goodmen-database/migrations/20260328500000_create_fmcsa_risk_tables.js`

## Key Decisions
- composite_score is 0-100 (100 = highest risk) for intuitive alerting
- Unique index on (tenant_id, driver_id, score_date) — one score entry per driver per day, replaced on recalculation
- fmcsa_inspections.report_number is UNIQUE to prevent duplicate imports
- driver_risk_events references both fmcsa_inspections and mvr_extracted_records via nullable FKs for cross-referencing
