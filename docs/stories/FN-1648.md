# FN-1648: Port driver-form behaviors from main → dev (State select + CDL State select + Zip auto-fill)

## Requirement

Restore three driver-form UX behaviors that ship on `main` (from direct-to-main commit `0b46bc29`, 2026-03-24, bundling FN-178–184) but were lost on `dev` during the AI-themed driver redesign (FN-1393, FN-1625, FN-1632).

This is a prerequisite for the `dev → main` promotion. With these behaviors restored on dev, the 6 frontend merge conflicts can be safely resolved by taking dev's version (option A in the 2026-05-31 promotion audit).

Three behaviors to restore:

1. **FN-181** — Add Driver / Edit Driver: `State` field becomes `<app-ai-select>` with 51 US states (currently a plain 2-char `<input>`).
2. **FN-182** — CDL State field: same treatment (`<app-ai-select>` with 51 options).
3. **FN-183** — Zip code field: `(blur)` handler calls `https://api.zippopotam.us/us/{zip}` and auto-fills `city` + `state`.

## Implementation Summary

Restored three driver-form behaviors on dev that ship on `main` (from commit `0b46bc29`, 2026-03-24, bundling FN-178–184) but were lost during the AI-themed driver redesign (FN-1393, FN-1625, FN-1632):

1. **State dropdown** — `driver-edit` page + Add Driver modal + Edit Driver inline section now use `<app-ai-select>` bound to a shared 51-entry US state list (50 states + DC).
2. **CDL State dropdown** — same `<app-ai-select>` pattern across all three surfaces.
3. **Zip → City/State auto-fill** — added `onZipBlur()` to `driver-edit.component.ts` that calls `https://api.zippopotam.us/us/{zip}` on blur and silently tolerates 404/network failure. The pattern mirrors `loads.component.ts` `lookupPickupZip()`. The drivers component already had a similar `onZipCodeBlur('new'|'edit')` handler (uses `ApiService.lookupZipCode`) wired into the templates — left untouched.

Hoisted the state list to `frontend/src/app/shared/constants/us-states.ts` (`US_STATE_OPTIONS`) so both components consume one source of truth. Pre-existing US-state lists in `locations-admin/location-edit-dialog` and `onboarding-packet/consent-form` were left as-is to avoid scope creep; they can migrate to the shared constant in a follow-up.

AI-prefill chip wrappers (`[class.dqf-field--ai]="isAiPrefilled('state')"` / `isAiPrefilled('cdlState')`) from FN-1625 are preserved on the Add Driver modal — only the inner `<input>` was swapped for `<app-ai-select>`.

## Key Decisions

- **`stateOptions` is a stable `readonly` class field** bound directly to `[options]`. Never a getter — getter-based `[options]` bindings caused the FN-317 infinite change-detection loop, and the constraint is documented in repo memory (`feedback_no_getter_options_bindings.md`).
- **Element type is `UsStateOption` (mutable `{ value: string; label: string }`)** rather than `readonly UsStateOption[]`. Angular's `app-ai-select` declares `@Input() options: AiSelectOption<T>[]` as a mutable array; `ReadonlyArray<…>` triggers TS4104 ("readonly cannot be assigned to mutable") at template-type-check time. The "stability" guarantee in the FN-317 RCA refers to **reference identity** (the constant is exported once, never reassigned, never mutated), not to TypeScript's `readonly` keyword.
- **DC included, territories excluded** — the ticket requires "51 US states." Used 50 states + DC, matching the convention from `onboarding-packet/consent-form/consent-form.component.ts`. (The `locations-admin` list also has PR/GU/VI; left out here to match the spec text.)
- **Zip handler error swallowing is intentional and silent** — no `console.error`, no toast, no UI state change on 404/network. This matches the `loads.component.ts` reference pattern. A bad zip just leaves city/state where they were.
- **Did not migrate `locations-admin` / `onboarding-packet` consumers** to `US_STATE_OPTIONS` — out of scope for FN-1648 (driver surfaces only) and those lists have different shapes (string codes / `{code, name}`). A follow-up cleanup can consolidate.
- **Drivers component zip handler left untouched** — the dev branch already had `onZipCodeBlur('new'|'edit')` wired in templates + TS, using `ApiService.lookupZipCode` (which itself proxies zippopotam.us). No change needed.
