# FN-455: [Bug] Logistics service missing AI_SERVICE_URL env var

## Requirement
The fuel AI preprocess route (`goodmen-shared/routes/fuel.js`) reads `AI_SERVICE_URL` but runs inside the logistics service, which didn't have the env var configured. It fell back to `http://localhost:4100`, causing all AI preprocess calls to fail on Render.

## Acceptance Criteria
- [x] `AI_SERVICE_URL` added to `fleetneuron-logistics-service` in `render.yaml`
- [x] `AI_SERVICE_URL` added to `fleetneuron-logistics-service-dev` in `render-dev.yaml`
- [x] `AI_SERVICE_URL` added to logistics service in `docker-compose.yml` and `docker-compose.fast.yml`
- [ ] After deploy, fuel AI preprocess connects to the real AI service (no more localhost fallback)

## Agent
devops

## Implementation Summary
Added `AI_SERVICE_URL` env var to the logistics service in all 4 deployment config files:
- `render.yaml` → `https://fleetneuron-ai-service.onrender.com`
- `render-dev.yaml` → `https://fleetneuron-ai-service-dev.onrender.com`
- `docker-compose.yml` → `http://ai-service:4100`
- `docker-compose.fast.yml` → `http://ai-service:4100`

## Files Changed
- `render.yaml` — logistics service envVars section
- `render-dev.yaml` — logistics service envVars section
- `docker-compose.yml` — logistics-service environment section
- `docker-compose.fast.yml` — logistics-service environment section

## Key Decisions
- Placed `AI_SERVICE_URL` before `NODE_ENV` in Render configs (consistent with gateway ordering)
- Docker compose uses internal service name `http://ai-service:4100` (consistent with gateway config)

- **Service to redeploy**: `fleetneuron-logistics-service` (prod) and `fleetneuron-logistics-service-dev` (dev)
- **DB migrations**: None

- **Deploy order**: No dependencies, single service redeploy
- **Post-deploy verification**: Trigger a fuel import with AI preprocessing and confirm the log shows the correct AI service URL instead of `http://localhost:4100`
