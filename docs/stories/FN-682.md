### FN-698 (subtask)
**Locations Add/Edit dialog ‚Äî Details tab with all fields**

**Branch:** `frontend/FN-698/locations-details-tab`

1. **`frontend/src/app/models/location.model.ts`** (new on this branch):
   - `LocationType` union, `OperatingHoursDay`, `OperatingHours`, `LocationListItem`, `LocationBin`, `LocationUser`, `LocationSupplyRule`, `Location`, `LocationListResponse`, `LocationFormValue`

2. **`frontend/src/app/components/locations-admin/location-edit-dialog/location-edit-dialog.component.ts`** (new):
   - `@Input() location: Location | null` ‚Äî null = Add mode; Location = Edit mode
   - `@Input() isOpen` ‚Äî controls visibility via `*ngIf`
   - `@Output() saved`, `@Output() close` EventEmitters
   - Tab navigation: `details | bins | users | supply_rules`
   - `showBinsTab` and `showSupplyRulesTab` getters: true only for SHOP/WAREHOUSE
   - `resetForm()` called on `ngOnChanges` when `isOpen` opens ‚Äî populates from `location` or clears
   - `buildDefaultHours()` ‚Äî Mon‚ÄìFri 08:00‚Äì17:00 open, Sat‚ÄìSun closed
   - `validateForm()` ‚Äî name required, type required, email format if provided
   - `onSave()` ‚Äî calls `api.createLocation()` or `api.updateLocation()` based on mode
   - OnPush with explicit `cdr.markForCheck()` after async ops

3. **`frontend/src/app/components/locations-admin/location-edit-dialog/location-edit-dialog.component.html`** (new):
   - Tab bar: Details, Bins (disabled if not SHOP/WAREHOUSE), Users, Supply Rules (disabled if not SHOP/WAREHOUSE)
   - **Details tab**: Name*, Code, Type* dropdown (6 options with descriptions), Active toggle, Address block (line1/2, city+state+zip), Contact block (name, email with validation, phone), Settings (timezone dropdown ‚Äî 9 US zones), Operating hours grid (7 days, closed toggle, open/close time inputs)
   - Bins/Users/Supply Rules tabs: placeholder content with message
   - Footer: Cancel + Save/Create primary button with spinner

4. **`frontend/src/app/components/locations-admin/location-edit-dialog/location-edit-dialog.component.scss`** (new):
   - Dark fleet theme matching existing modals
   - `.led-*` class namespace
   - Backdrop blur, blue border glow, responsive via media query

5. **`frontend/src/app/services/api.service.ts`** (modified):
   - Added `listLocations()`, `getLocationById()`, `createLocation()`, `updateLocation()`, `deleteLocation()`
   - Added `getLocationBins()`, `createLocationBin()`, `updateLocationBin()`, `deleteLocationBin()`
   - Added `getLocationUsers()`, `assignUsersToLocation()`, `removeUserFromLocation()`, `getUserLocations()`

6. **`frontend/src/app/app.module.ts`** (modified):
   - Imported and declared `LocationEditDialogComponent`

**Key decisions:**
- Bins and Supply Rules tabs disabled (not just hidden) for non-SHOP/WAREHOUSE types so users understand the feature exists but needs the right type
- Operating hours stored as JSONB ‚Äî default is Mon‚ÄìFri open (08:00‚Äì17:00), weekends closed
- Toggle uses native `<input type="checkbox">` hidden + styled `.led-toggle-track/thumb` to avoid Material CD issues
- `addressLine2` is kept in the form but not yet in the backend model ‚Äî stored in `address` field after merge (future migration can add address_line2 column)
