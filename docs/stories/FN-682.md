### FN-702 (subtask)
**Delete flow ‚Äî dependency detection dialog (soft vs hard delete)**

**Branch:** `frontend/FN-702/locations-delete-flow`

1. **`frontend/src/app/models/location.model.ts`** (new on this branch):
   - Added `LocationDependencies` interface: `{ work_orders, inventory_items, users, vehicles }`
   - All other types same as prior subtasks

2. **`frontend/src/app/components/locations-admin/delete-location-dialog/delete-location-dialog.component.ts`** (new):
   - `@Input() location: LocationListItem | null`, `@Input() isOpen`
   - `@Output() deleted` (after hard delete), `@Output() markedInactive` (after soft delete), `@Output() close`
   - **Three dialog phases**: `confirm` ‚Üí `has_deps` ‚Üí `done`
   - `onConfirmDelete()` ‚Äî attempts DELETE; on **200** emits `deleted`, moves to `done`; on **409** extracts `err.error.dependencies` and pivots to `has_deps`; on **404** treats as already-deleted success
   - `onMarkInactive()` ‚Äî calls PATCH `{ active: false }` on the location; on success emits `markedInactive`, moves to `done`
   - `depKeys/depNames/depIcons` lookup tables ‚Äî static arrays for the 4 dependency types
   - `cityState(location)` helper ‚Äî formats city + state for the confirm phase

3. **`frontend/src/app/components/locations-admin/delete-location-dialog/delete-location-dialog.component.html`** (new):
   - **Phase 1 (confirm)**: location name, type/city pill, "Delete" (red) button ‚Üí triggers delete attempt
   - **Phase 2 (has_deps)**: 2√ó2 dependency grid showing counts for work_orders/inventory_items/users/vehicles; "Mark Inactive" (amber) button + Cancel
   - **Phase 3 (done)**: success state with different copy for hard vs soft delete
   - All phases: spinner on in-flight, inline error within the relevant phase
   - Color-coding: danger (red) for hard delete, warning (amber) for soft delete, success (green) for done

4. **`frontend/src/app/components/locations-admin/delete-location-dialog/delete-location-dialog.component.scss`** (new):
   - `.dld-*` namespace; colored left-border on header per phase
   - Dependency grid cards with amber highlight when count > 0

5. **`frontend/src/app/services/api.service.ts`** (modified):
   - Added `getLocationById`, `createLocation`, `updateLocation` (used for soft-delete), `deleteLocation`
   - JSDoc on `deleteLocation` documents the 409 response shape

6. **`frontend/src/app/app.module.ts`** (modified):
   - Declared `DeleteLocationDialogComponent`

**Key decisions:**
- The "Delete" button in phase 1 triggers the actual DELETE request ‚Äî there is no separate "check" endpoint. This means if the location has no dependencies, clicking Delete immediately removes it (correct UX ‚Äî the dialog IS the confirmation).
- 409 from the backend carries `{ dependencies: { work_orders, inventory_items, users, vehicles } }` ‚Äî we show all 4 counts even if some are 0 (amber highlight only on non-zero ones)
- Soft-delete is a PATCH with `{ active: false }` ‚Äî the location stays in the DB but is hidden from active lists
- 404 on delete is treated as success (location was already removed)
