# FN-1790: Adapter — Lease-to-Own driver agreement signing

> Epic: **FN-1786**. Blocked by **FN-1787 + FN-1788** (uses the generic engine).

## Requirement
Wire the engine into the existing **Lease-to-Own** module so a lease-to-own agreement can be **sent to the driver to e-sign**, reusing existing `lease_agreements` fields (`status`, `sent_for_signature_at`, `signed_at`, `document_storage_key`). From the lease-financing dashboard/detail, the internal user uploads/prepares the agreement and sends it to the driver; the driver signs via the public web interface (or existing `driver-lease-view`); on signature the agreement flips `pending_signature → active`, persisting `signed_at` + signed-PDF `document_storage_key`.

## Acceptance Criteria
- [ ] "Send for signature" from lease-financing creates a signature request for the agreement's driver via the engine (`sent_for_signature_at` set, `status=pending_signature`).
- [ ] Driver signs through the public e-sign interface; `driver-lease-view` surfaces/links the signing.
- [ ] On signature: `status → active`, `signed_at` set, signed PDF in `document_storage_key`.
- [ ] Reuses generic engine — no duplicate signing logic; existing lease lifecycle respected.
