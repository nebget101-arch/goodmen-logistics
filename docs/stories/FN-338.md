### FN-344: Convert fuel-transactions filter bar to inline header filters

**Implementation Summary:**
- Removed the separate `.filter-bar` div with 4 filter groups (date from, date to, provider, matched status)
- Added `<tr class="header-filters">` row inside `<thead>` with filters mapped to appropriate columns:
  - Date column: two stacked `<input type="date">` for from/to range
  - Provider column: text input
  - Matched column: native `<select>` dropdown (replaced `app-ai-select` to avoid CD loops)
  - Actions column: clear filters button
- Updated CSS to match loads-dashboard inline filter styling (radial gradient background, 11px font, 28px min-height)
- Removed unused `AiSelectOption` import and `matchedStatusOptions` array from TS

**Files Changed:**
- `frontend/src/app/fuel/fuel-transactions/fuel-transactions.component.html` — removed filter-bar, added header-filters row
- `frontend/src/app/fuel/fuel-transactions/fuel-transactions.component.css` — replaced filter-bar CSS with header-filters CSS
- `frontend/src/app/fuel/fuel-transactions/fuel-transactions.component.ts` — removed unused import and options array

**Key Decisions:**
- Used native `<select>` instead of `app-ai-select` for matched status filter (avoids infinite CD loop per FN-317 RCA)
- Used native `<input type="date">` instead of `app-ai-date-picker` for compact inline sizing
- Stacked from/to date inputs vertically in the Date column to fit the compact header row
