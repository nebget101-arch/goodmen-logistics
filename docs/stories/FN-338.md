# FN-338: Convert separate filter bars to inline table header filters across all data pages

## Requirement
The loads-dashboard uses inline filter controls in a second `<tr class="header-filters">` row inside `<thead>`. This pattern should be applied to all other data pages that currently use separate filter bars above their tables.

## Acceptance Criteria
- [ ] All listed pages have inline filter controls inside a `<tr class="header-filters">` row in `<thead>`
- [ ] Separate filter bars removed (replaced by inline header filters)
- [ ] All filter dropdowns use `<app-ai-select>` (not raw `<select>`)
- [ ] Filter functionality preserved (two-way binding, auto-refresh on change)
- [ ] Consistent styling with loads-dashboard filter row
- [ ] loads-dashboard raw `<select>` elements converted to `<app-ai-select>`
- [ ] Dark theme styling consistent across all pages
