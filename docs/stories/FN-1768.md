# FN-1768: Vehicle Tracking polish — fix header icon + Google Maps link on position

## Requirement
Two small frontend fixes to the Vehicle Tracking page (`frontend/src/app/components/vehicle-tracking/`, shipped by FN-1750/FN-1751):
1. **Header bug** — `vehicle-tracking.component.html:6` uses `class="material-icons"` with the `my_location` ligature, but `my_location` is a **Material Symbols** glyph, so the literal text "my_location" renders in the header. Fix: use `material-symbols-outlined` (as `geofences`/`tracking-map`/`public-roadside` already do).
2. **Google Maps link** — the "Position & Speed" block renders `positionDisplay()` as plain text; make it a link to `https://www.google.com/maps?q=lat,lng` opening in a new tab when a position exists.

Both touch the same template/component, so they are intentionally one story (one branch) rather than two branches racing on the same file.

## Acceptance Criteria
- [x] Header shows the `my_location` map-pin icon, not the literal text.
- [x] When a selected truck has a position, the Position value links to `https://www.google.com/maps?q=lat,lng` in a new tab (`rel="noopener"`).
- [x] When a truck has no position, Position renders `—` with no broken link.
- [x] AI dark-theme tokens only; no new hex; build passes.

## Implementation Summary
1. **Header icon** — changed the title icon's class from `material-icons` to `material-symbols-outlined` so the `my_location` ligature resolves to the map-pin glyph instead of rendering literal text. The `.vt-title-icon` SCSS selector is class-based (`.vt-title-icon`, not scoped to `.material-icons`), so the existing 2rem / teal styling carries over unchanged — no SCSS change needed for this fix.
2. **Google Maps link** — added a `googleMapsUrl(): string | null` helper that returns `https://www.google.com/maps?q=<lat>,<lng>` from the raw (untruncated) `latitude`/`longitude` when `hasPosition(telemetry)` is true, else `null`. In the template the Position `<dd>` now uses `*ngIf="googleMapsUrl() as mapsUrl; else plainPosition"` to render an `<a target="_blank" rel="noopener noreferrer">` wrapping `positionDisplay()` plus an `open_in_new` icon; the `#plainPosition` template renders the plain `—`/text when there is no position, so there is never a broken link. Styled with `.vt-maps-link` / `.vt-maps-icon` reusing the existing teal accent token `rgba(45, 212, 191, …)` — no new hex.

## Key Decisions
- Used the raw lat/lng for the Maps URL (per AC) so the pin is exact, while still showing the 4-decimal `positionDisplay()` as the link text.
- Used an `*ngIf … as … else` template rather than putting `googleMapsUrl()` in the `[href]` of an always-rendered `<a>`, so the no-position case is plain text with no anchor at all (guarantees "no broken link").
- No SCSS change for the icon fix because `.vt-title-icon` was never scoped to `.material-icons`.
