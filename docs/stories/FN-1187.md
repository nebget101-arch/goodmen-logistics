# FN-1187 — Roadside v2 Story 1.2: drivers-compliance triage handler + persistence

**Epic:** FN-1140 — Roadside AI v2 — Epic 1: AI Triage Engine  
**Status:** In Progress  
**Integration Branch:** `integration/FN-1187`

## User Story

As a dispatcher, I need triage results persisted and queryable so I can see triage rationale on the dispatcher board and audit/rollback model versions.

## Acceptance Criteria

- [ ] `services/drivers-compliance` exposes `POST /incidents/:id/triage` that calls ai-service triage and persists the result
- [ ] New table `incident_triage` with columns: `incident_id`, `tenant_id`, `severity`, `category`, `urgency`, `vendor_skills` (jsonb), `rationale`, `prompt_version`, `model_name`, `latency_ms`, `cache_read_tokens`, `cache_creation_tokens`, `created_at`
- [ ] All queries scoped by `tenant_id`
- [ ] `GET /incidents/:id/triage` returns latest triage record
- [ ] Idempotent: re-triage on the same incident creates a new row, never updates in place (audit trail)
- [ ] Telemetry: triage call count, p50/p95 latency, error rate
