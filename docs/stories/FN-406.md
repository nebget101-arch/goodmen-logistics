# FN-406: [AI] Build AI-assisted fuel import preprocessing

## Summary
Added AI-assisted preprocessing step for fuel CSV/XLSX imports. The AI service analyzes raw file headers and sample rows to produce column mapping suggestions with confidence scores, product type inference, row split proposals, and skip/flag detection for Money Code and card-less rows.

## Implementation Summary

### New Files
- `backend/microservices/ai-service/src/handlers/fuel-preprocess-handler.js` — AI handler that builds a structured prompt from headers + sample rows, calls OpenAI, parses and validates the response

### Modified Files
- `backend/microservices/ai-service/src/ai-router.js` — Added `POST /fuel/preprocess` route
- `backend/packages/goodmen-shared/routes/fuel.js` — Added `POST /import/ai-preprocess` endpoint that parses the file and proxies to AI service
- `frontend/src/app/fuel/fuel.model.ts` — Added `AiPreprocessResult` and related interfaces; added `category` to `FUEL_NORMALIZED_FIELDS`
- `frontend/src/app/fuel/fuel.service.ts` — Added `aiPreprocess()` method

### Key Decisions
- AI preprocessing runs in the existing `ai-service` microservice using OpenAI (gpt-4.1-mini) with `json_object` response format
- The fuel route endpoint (`/api/fuel/import/ai-preprocess`) accepts the file upload, parses it server-side, and proxies extracted headers + sample rows to the AI service — keeping file parsing in the logistics service
- Temperature set to 0.1 for deterministic mapping results
- Max 20 sample rows sent to AI to balance quality vs. token usage
- AI response includes per-field confidence scores (0.0-1.0) so the frontend can highlight low-confidence mappings
- Added `category` field to FUEL_NORMALIZED_FIELDS for downstream FN-405 compatibility
