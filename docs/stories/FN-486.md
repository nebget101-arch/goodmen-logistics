# FN-486: Toll Device Driver Auto-Sync

## Requirement
When a truck's driver assignment changes (driver reassigned, new driver created with truck, or driver removed), automatically update the `driver_id` on toll devices assigned to that truck -- unless the device has a direct driver override (`is_driver_override = true`).

## Acceptance Criteria
- [ ] Changing a truck's driver auto-updates toll devices assigned to that truck
- [ ] Direct driver overrides (`is_driver_override = true`) are NOT affected by truck driver changes
- [ ] `assign-vehicle` auto-resolves and sets driver with `is_driver_override = false`
- [ ] `assign-driver` sets `is_driver_override = true`
- [ ] `remove-vehicle` clears driver only if `is_driver_override = false`

### FN-488 (subtask)

**New service:**
- `backend/packages/goodmen-shared/services/toll-device-driver-sync.js` -- `syncTollDeviceDrivers()` utility that updates `driver_id` on toll devices for a given truck where `is_driver_override = false`. Supports both knex and raw pg client callers.

**Updated routes:**
- `backend/packages/goodmen-shared/routes/tolls.js`:
  - `POST /devices/:deviceId/assign-vehicle` -- auto-resolves driver from active driver on the truck, sets `is_driver_override = false`
  - `POST /devices/:deviceId/assign-driver` -- sets `is_driver_override = true` (manual override)
  - `POST /devices/:deviceId/remove-vehicle` -- clears `driver_id` only when `is_driver_override = false`
- `backend/packages/goodmen-shared/routes/drivers.js`:
  - `POST /` (create driver) -- syncs toll devices when new driver is assigned to a truck
  - `PUT /:id` (update driver) -- syncs toll devices on both old and new trucks when `truck_id` changes
