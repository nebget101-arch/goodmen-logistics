# FN-714: Restructure Work Order Form into 5 Tabs

## Summary
Refactored the monolithic Work Order component (3,136 LOC) into a tabbed interface with 5 child tab components and a status bar, reducing the parent shell to 395 LOC.

## Architecture

### Parent Shell (`work-order.component`)
- Owns the `workOrder` data model (single source of truth)
- Manages tab state via `activeTab` property
- Passes data to child tab components via `@Input()` bindings
- Handles save/load and API calls
- Renders status bar + tab navigation + active tab content
- **395 LOC** (TS file), down from 1,734

### Tab Components
| Tab | Component | Content |
|-----|-----------|---------|
| 1 - Basics | `WoBasicsTabComponent` | Customer search/select, vehicle search/select, WO number, title, type, priority, location, assigned mechanic, dates |
| 2 - Service Details | `WoServiceDetailsTabComponent` | Category, description, problem reported, root cause, work performed, safety issue, road call, estimated duration, AI suggest button |
| 3 - Work | `WoWorkTabComponent` | Parts table with add/remove, part lookup, barcode scanning, phone bridge, labor table with mechanic lookup |
| 4 - Financials | `WoFinancialsTabComponent` | Credit info, estimated/actual/tax/total costs, billing status, workflow tracking, compliance fields |
| 5 - Notes | `WoNotesTabComponent` | Internal/mechanic/driver/customer notes, file attachments, document upload, audit info (collapsed) |

### Status Bar (`WoStatusBarComponent`)
Fixed header visible across all tabs displaying WO number, title, status badge, and action buttons (Save, Cancel, Generate Invoice).

## Data Flow
- Parent `workOrder` object is passed to tabs via `@Input()` bindings
- Tabs modify the shared object reference directly (two-way binding via shared reference)
- Parent handles all save/load API calls
- Child tabs emit events via `@Output()` for actions that require parent coordination (reload, invoice generation)
- Each tab injects `ApiService` directly for tab-specific API calls

## Key Decisions
- Used `*ngIf` for tab switching to keep form state across tabs
- Applied `ChangeDetectionStrategy.OnPush` on all child tab components
- Used `::ng-deep` in parent CSS to propagate dark theme styles to child components
- Status bar is always visible regardless of active tab
- Audit section in Notes tab is collapsed by default with toggle
