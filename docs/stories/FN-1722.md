# FN-1722: Story K — Live map polish: US styling, street zoom, follow-unit, open-in-Maps

## Requirement
Polish the MapLibre live map (FN-1717) for real-world demo use. It currently uses the minimal CARTO `dark_all` basemap — no state borders, blurry when zoomed, no per-unit focus. Make it US-legible, street-level capable, and interactive.

## Acceptance Criteria
- [ ] US state borders + place labels render (map reads as a US map; keep AI dark theme — e.g. `dark_matter` *with labels* or an admin-boundary vector layer). No empty dark void.
- [ ] Zooming in resolves to crisp street detail (raise `maxZoom`; street-level tile source).
- [ ] Follow-the-unit: clicking a vehicle eases the camera to it and re-centers on each new ping until deselected/panned, with a clear exit affordance.
- [ ] Vehicle side panel shows lat/lng + an "Open in Google Maps" link (`https://www.google.com/maps?q=<lat>,<lng>`, new tab, `rel="noopener"`).
- [ ] AI dark-theme tokens only; `[options]` = readonly class fields (FN-317); geofence overlay + filters preserved.

## Implementation Summary

### FN-1723 (subtask) — frontend
Polished the `tracking-map` MapLibre component on all four AC axes, scoped entirely to the component (no new services, deps, or schema):

- **US states + labels (crisp street zoom):** swapped the inline CARTO `dark_all` **raster** style for CARTO's key-less **`dark-matter` GL vector** style (`https://basemaps.cartocdn.com/gl/dark-matter-gl-style/style.json`). Vector ships US state/admin boundaries + place labels and re-rasterises at every zoom, so the map reads as a US map and stays sharp at street level — solving the "no borders / blurry when zoomed" AC pair in one change. Raised `maxZoom` to 18. The 3D camera (pitch/rotate/tilt) from FN-1720 rides on top unchanged.
- **Follow-the-unit:** clicking a vehicle now `easeTo`s the camera to it (framing at ≥ z11) and re-centers on every new ping. A "Following _Unit_ · Stop" chip overlays the map (top-left); follow exits on the chip's Stop button, on panel close/deselect, on a filter that drops the selection, or on a user **pan** (`dragstart` — programmatic `easeTo` never fires it). The camera glide runs outside the Angular zone (no CD thrash).
- **Open in Google Maps:** the side panel's Position row is joined by an "Open in Maps → Google Maps" link to `https://www.google.com/maps?q=<lat>,<lng>` (`target="_blank" rel="noopener"`), built by the `mapsUrl()` helper.
- **Constraints preserved:** AI dark-theme tokens only (no new hex beyond the established palette); `[options]` bindings remain readonly class fields (FN-317); geofence overlay + driver/status/geofence filters + breadcrumb trails untouched.
- **Tests:** added spec coverage for `mapsUrl()` (URL build + null/empty guards) and the follow lifecycle (`enterFollow` sets target/label, `stopFollow`/`closePanel` clear it).

## Key Decisions
- **Vector style over augmenting the raster:** a single key-less vector style (CARTO `dark-matter` GL) satisfies both "US states + labels" and "crisp street zoom" cleanly — vector stays sharp at any zoom, where raster blurs past native tile resolution. The AC explicitly permits "or a vector style". Same CARTO host as the prior raster, so no new external origin to allowlist.
- **Follow tied to selection, exits on user pan:** `dragstart` distinguishes a real user pan from the programmatic `easeTo` re-centers, so re-centering never self-cancels follow. First follow bumps to ≥ z11 so the unit is actually framed; live re-centers keep the user's zoom/pitch/bearing.
- **`mapsUrl()` as a method (not a getter `[options]`):** it returns a string for an `[href]`, unrelated to the FN-317 getter-bound `app-ai-select` option-array pitfall.
