# FN-1702: Story A — user_sessions schema

## Requirement
Create the `user_sessions` table backing single-active-session enforcement. One row per active session, with device/connection details for the "you're already signed in on …" message and a revocation flag for takeover/logout.

## Acceptance Criteria
- [ ] Knex migration `user_sessions`: `id` uuid PK (`gen_random_uuid()`), `user_id` uuid FK→users(id) ON DELETE CASCADE, `tenant_id` uuid FK→tenants(id) nullable, `session_token_hash` varchar(128) UNIQUE NOT NULL, `user_agent` varchar(512), `ip_address` varchar(45), `device_label` varchar(255), `created_at` ts default now, `last_seen_at` ts default now, `revoked_at` ts null, `revoked_reason` varchar(100) null.
- [ ] Indexes: `(user_id, revoked_at)`, `session_token_hash`, `tenant_id`.
- [ ] `YYYYMMDDhhmmss_` naming (latest existing `20260427120000`); working `down`; uuid PK/FK throughout; hash approach mirrors `password_reset_tokens`.
