### FN-1668 (subtask) â€” geofence_events table
Migration `20260603100300_create_geofence_events.js` creates the `geofence_events`
table that the FN-1669 Bull worker writes to (one row per detected crossing) and
that load-status automation reads from.

Schema:
- `id` UUID PK
- `geofence_id` UUID NOT NULL â†’ FK `geofences.id` **ON DELETE CASCADE**
- `vehicle_id` UUID NOT NULL â€” **no hard FK** (copied from the triggering ping; mirrors `vehicle_position_pings`)
- `event_kind` TEXT NOT NULL CHECK `('enter','exit','dwell')`
- `ts` TIMESTAMPTZ NOT NULL â€” crossing time (= triggering ping's `ts`)
- `ping_id` UUID NOT NULL â€” **soft reference** to `vehicle_position_pings.id`, no hard FK
- `load_id` UUID NULL â†’ FK `loads.id` **ON DELETE SET NULL** (FK wired only when `loads` exists)
- `created_at` TIMESTAMPTZ DEFAULT now()

Constraints / indexes:
- `UNIQUE (ping_id, geofence_id, event_kind)` â€” idempotency net so the Bull worker can safely retry/reprocess a ping (one enter/exit/dwell per ping+geofence)
- `INDEX (vehicle_id, ts DESC)` â€” worker lookup: latest events per vehicle
- `INDEX (geofence_id, ts DESC)` â€” worker lookup: events per geofence
- `INDEX (load_id) WHERE load_id IS NOT NULL` â€” load-status automation reads events by load

Key decisions:
- **`ping_id` has no hard FK by design.** `vehicle_position_pings` (FN-1660) is RANGE-partitioned by `ts` with a composite PK `(id, ts)`. PostgreSQL forbids a FK that references only part of a partitioned table's key, and adding `ts` to the FK would couple this table to the partition key. `ping_id` is stored as a plain UUID soft reference, consistent with how the pings table itself treats `vehicle_id` (no FK on the write-optimized ingest path).
- **`vehicle_id` has no hard FK**, matching the pings ingest pattern; the value is copied straight off the triggering ping.
- **`geofence_id` uses CASCADE**: an event log row is meaningless once its geofence is gone. Geofences are soft-disabled via `is_active` (FN-1664), so hard deletes are rare.
- **`load_id` uses SET NULL**: events are a historical record and should survive a load deletion.

Verified: ran the full migration chain (211 migrations) on a scratch Postgres â€” table, FKs (`geofences` CASCADE, `loads` SET NULL), check constraint, unique constraint, and all four indexes created as specified. Rollback (`down` = `dropTableIfExists`) confirmed.
