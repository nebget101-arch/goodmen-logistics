# FN-818 — Frontend — AI sparkle/badge components + list row & drawer integration

**Parent story:** [FN-789 — Phase 2 — AI Confidence Badges, Sparkle Indicators & ai_metadata Persistence](https://nebget101.atlassian.net/browse/FN-789)
**Branch:** `frontend/FN-818/ai-sparkle-badge` (merges into the FN-789 story PR)
**Agent:** `agent:frontend`

## Scope (consolidated FN-818 + FN-819)

Two reusable components in `frontend/src/app/shared/components/`:

- **`app-ai-sparkle`** — a shimmering ✦ marker with teal / yellow / orange tiers
  (≥95 / 80–95 / <80), `prefers-reduced-motion` fallback, tooltip input, and
  `(sparkleClick)` output. Used next to values that originated from an AI
  extraction pipeline.
- **`app-confidence-badge`** — a chip that summarises extraction confidence
  with copy that tracks the tier (`"95% confidence"`, `"88% — review"`,
  `"62% — please verify"`) and a `(chipClick)` output. Used in the loads
  list status cell for draft rows and in the drawer header.

Integration (the rolled-up FN-819 scope):

- Loads list: `app-ai-sparkle` renders next to the Load Number for rows with
  `source in ('ai_extraction' | 'ai' | 'email')`; `app-confidence-badge`
  renders in the status cell for DRAFT/NEW rows that carry
  `ai_metadata.overall_confidence`. Chip click opens the detail drawer with
  `focusLowConfidence=true` so the basics step highlights fields scored <80.
- Drawer header: same sparkle + chip pair appears next to the load number.
- Drawer basics step: per-field sparkles render wherever the API returns
  `ai_metadata.fields[field]`; tooltip exposes the per-field confidence;
  local edits hide the sparkle until the drawer re-fetches (matches the
  "edit clears marker" AC).
- Newly-arrived AI rows get a one-off teal glow (3s) via a diff against the
  previous list snapshot inside `LoadsDashboardComponent.trackNewAiLoads`.

## Key Decisions

1. **Inline components over hover-popover wrapping** — the sparkle's tooltip
   uses the native `[title]` attribute to keep the component leaf-level and
   avoid pulling `app-hover-popover` into every row cell. Parents that need a
   richer popover can wrap it externally.
2. **Tier defaults** — `AiSparkleComponent.tierFor(null)` returns `'high'` so
   AI-sourced content without per-field scores still renders a teal marker.
   `ConfidenceBadgeComponent.tierFor(null)` returns `'medium'` because the
   chip is only rendered with an explicit `*ngIf` guard upstream.
3. **Local edit clearing** — `StepBasicsComponent.locallyEditedFields`
   tracks fields the user has touched and hides their sparkle until the
   drawer re-populates from the API. This keeps the "edit clears marker" AC
   working without coordinating with the (not-yet-shipped) backend clear
   (FN-817).
4. **Glow diff** — `trackNewAiLoads` diffs the ID set across list refreshes
   so the glow only fires for *new* AI rows (no flash on the first page load
   or on subsequent refreshes that simply re-sort existing rows).
5. **Forward-compat `ai_metadata` shape** — the `AiMetadata` interface is
   intentionally loose (`[key: string]: unknown`) because the JSONB column
   may grow additional keys after FN-816/FN-817 land.

## Deployment Handoff

Services to redeploy: **loads-frontend** only (Render: `fleetneuron-frontend`).

- No DB migration here (that lives in FN-816).
- No backend changes here (that lives in FN-817). Frontend is ready for the
  `ai_metadata` payload; if the API does not yet return it, the sparkle/chip
  simply do not render (graceful degradation via `*ngIf`).
- No env vars, feature flags, or cache changes.
- Post-deploy verification:
  1. Load an AI-extracted draft (once FN-816/FN-817 land, or stub
     `ai_metadata` locally) — confirm sparkle + confidence chip render.
  2. Click the chip — drawer opens with low-confidence fields outlined in
     orange.
  3. Edit a highlighted field — sparkle for that field disappears until save
     (which re-fetches and re-evaluates).
  4. Check `prefers-reduced-motion` (macOS: System Settings → Accessibility
     → Display → Reduce motion) — shimmer stops, glow still gives a brief
     tint.
