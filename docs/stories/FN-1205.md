## Implementation Notes (FN-1261)

### IncidentDetailComponent
- Route: `/driver-portal/incidents/:id` (guarded by `AuthGuard`)
- Loads incident via `ApiService.getSafetyIncidentById(id)` — uses safety incidents API
- Builds 4-step stepper: Reported → In Progress → Resolved → Closed
- Uses `OnPush` change detection throughout

### PhotoUploaderComponent
- `[available]` input controls whether upload is enabled (set to `false` when incident is closed)
- When `available=false`: renders a disabled-state placeholder explaining Story 3.1 soft-dependency
- Validates file size ≤ 10 MB client-side before sending
- Calls `ApiService.uploadIncidentImage()` → `POST /api/incidents/:id/images`

### FeedbackFormComponent
- Shown when `incident.status` is RESOLVED or CLOSED AND `driver_feedback` is absent
- Star rating (1–5, optional) + required text ≥ 5 characters
- Submits via `ApiService.updateSafetyIncident()` → `PATCH /safety/incidents/:id`
