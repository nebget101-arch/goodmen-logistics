# FN-319: Backend — Due date calculation and DQF completeness logic for time-sensitive items

## Requirement
Update the backend DQF service to calculate due dates for time-sensitive checklist items, return urgency levels (green/yellow/red), and refine the completeness calculation.

## Acceptance Criteria
- [x] Due dates returned in `/api/dqf/driver/:driverId/status` response for time-sensitive items
- [x] Urgency field (green/yellow/red) calculated per item
- [x] DQF completeness excludes within_30_days items before due date
- [x] DQF completeness excludes annual items before due date
- [x] DQF completeness excludes other category entirely
- [x] annual_clearinghouse_limited_query in ANNUAL_DUE_DATE_KEYS

## Implementation Summary

### New function: `computeDueDateAndUrgency(category, requirementKey, hireDate)`
- **Within 30 Days items**: `due_date = hire_date + 30 days`
  - green: > 15 days remaining
  - yellow: <= 15 days remaining
  - red: past due
- **Annual items** (3 keys): `due_date = next hire anniversary`
  - green: > 30 days remaining
  - yellow: <= 30 days remaining
  - red: past anniversary

### Completeness update
- Added `if (row.category === 'other') return;` to exclude Other Documents entirely

### Status endpoint enrichment
- Each requirement in the response now includes `due_date` (YYYY-MM-DD or null) and `urgency` (green/yellow/red or null)

## Files Changed
- `backend/packages/goodmen-shared/services/dqf-service.js` — new `computeDueDateAndUrgency()`, `ANNUAL_DUE_DATE_KEYS`, exclude other from completeness
- `backend/packages/goodmen-shared/routes/dqf.js` — enrich status endpoint requirements with due_date/urgency

## Key Decisions
- Both `annual_clearinghouse_limited_query` and `annual_clearinghouse_query` DB alias included in ANNUAL_DUE_DATE_KEYS
- Annual due date uses rolling anniversary (next occurrence of hire month/day)
- Urgency is computed server-side so frontend and backend stay consistent

## Deployment Handoff
- **Services**: goodmen-shared (backend)
- **DB migrations**: None (categories already assigned by FN-261 migration)
- **Env/config**: None
- **Post-deploy verification**: `GET /api/dqf/driver/:id/status` — verify `due_date` and `urgency` fields on within_30_days and annual requirements
