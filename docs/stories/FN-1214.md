### FN-1291 (subtask) — [frontend] POC: read-only incident list

**Branch:** `frontend/FN-1291/driver-mobile-poc`

**Approach:** Standalone PWA scaffold (no framework, no build step). Chosen over a Capacitor wrapper for the POC because it runs immediately in any browser without platform tooling — easier stakeholder demo. Full Capacitor wrapping is recommended for the production implementation (see FN-1290 recommendation doc).

**Files created:**
- `experiments/driver-mobile-poc/index.html` — app shell, login + incident list screens
- `experiments/driver-mobile-poc/styles.css` — AI dark theme, touch-optimized (44px targets, WCAG contrast)
- `experiments/driver-mobile-poc/app.js` — login flow (`POST /api/auth/login`), incident list (`GET /api/roadside/calls`), status filter chips, pull-to-refresh
- `experiments/driver-mobile-poc/manifest.json` — PWA manifest (standalone display, theme colour)
- `experiments/driver-mobile-poc/sw.js` — service worker, static shell caching

**API endpoints consumed:**
- `POST /api/auth/login` → JWT token stored in `sessionStorage`
- `GET /api/roadside/calls` → incident list (status, urgency, caller, call_number)

**How to run:**
```
# Point a local HTTP server at the repo root, then open:
http://localhost:<port>/experiments/driver-mobile-poc/
# Or pass ?api=https://your-dev-gateway to point at a deployed instance
```

**WCAG / touch notes:**
- All interactive targets ≥ 44 × 44 px
- Colour contrast ≥ 4.5:1 on body text (AI dark theme tokens)
- `aria-label`, `role="button"`, `aria-live` wired on all interactive / dynamic regions

**Known gaps (intentional for POC scope):**
- Detail view shows a `window.alert` — full slide-up sheet deferred to story implementation
- No real offline data (SW caches only the shell assets, not API responses)
- Icons are SVG inline placeholders (production would use proper icon assets)

### FN-1290 (subtask) — [backend] Spike research: native vs PWA evaluation + recommendation doc

**Branch:** `backend/FN-1290/driver-mobile-spike`

**Approach:** Authored the full architecture spike and recommendation document at
`docs/spikes/driver-mobile-app.md`. Recommendation: **Capacitor (Angular wrapper) for
production; PWA-first as an incremental rollout path.**

**Files created:**
- `docs/spikes/driver-mobile-app.md` — comprehensive recommendation covering stack
  comparison, App Store implications, offline support (SQLite + sync queue pattern),
  push notifications (APNs/FCM via `@capacitor/push-notifications`), camera
  (`@capacitor/camera`), tenant scoping plan, telemetry event catalogue, cost-of-ownership
  estimate, and follow-on epic sizing (10–12 weeks, 3 agents).

**Key decisions:**
- Capacitor over React Native: 95%+ code reuse vs full rewrite (16–24 weeks). No
  functional gap React Native fills for this use case.
- PWA-first recommended: ship `/driver` route as PWA immediately (no App Store gating),
  wrap in Capacitor for production-grade push + camera in the follow-on epic.
- Tenant scoping requires no new work at the auth/API layer — only the `user_push_tokens`
  table needs a `tenant_id` column when push is implemented.
- FN-1292 contributed the detailed App Store account provisioning checklist, CI cost comparison, MDM implications, and code-signing secrets inventory.

### FN-1292 (subtask) — [backend] App Store / Play Store account research + CI cost estimate

**Branch:** `backend/FN-1292/mobile-distribution-research`

**Approach:** Authored the detailed distribution and CI research document at
`docs/spikes/mobile-distribution-research.md` and updated the App Store Implications and
Cost-of-Ownership sections in `docs/spikes/driver-mobile-app.md`.

**Files created/updated:**
- `docs/spikes/mobile-distribution-research.md` *(new)* — detailed research covering: Apple Developer Program requirements ($99/year, D-U-N-S, provisioning profiles), Google Play account ($25 one-time), distribution mechanics (public store vs MDM vs TestFlight vs sideload), MDM implications for fleet tablets (Jamf/Intune/Workspace ONE, UDID limits, kiosk mode), CI cost comparison (GitHub Actions ~$74/month, Bitrise $115/month flat, Codemagic $95/month), and full code-signing secrets inventory for iOS + Android.
- `docs/spikes/driver-mobile-app.md` *(updated)* — replaced FN-1292 placeholder notes in App Store Implications and Cost-of-Ownership sections with concrete data; updated status header to Complete.

**Key findings:**
- **CI choice:** GitHub Actions (hosted macOS) — no new vendor, ~$74/month at 30 iOS builds/month; self-hosted Mac mini pays off above 50 builds/month.
- **Phase 1 distribution:** PWA immediately; Ad Hoc + TestFlight (iOS) / APK sideload (Android) when Capacitor shell is built. No public store listing needed until Phase 2.
- **UDID scaling gate:** Ad Hoc profile caps at 100 devices. Revisit at 80 tablets → Enterprise Program ($299/year) or ABM Managed Distribution.
- **Android keystore:** irreplaceable — must be backed up to org password manager before any release build.
- 5 follow-on devops tickets identified for Capacitor build epic prerequisite work.
