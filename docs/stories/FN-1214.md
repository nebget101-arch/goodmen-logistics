### FN-1291 (subtask) ‚Äî [frontend] POC: read-only incident list

**Branch:** `frontend/FN-1291/driver-mobile-poc`

**Approach:** Standalone PWA scaffold (no framework, no build step). Chosen over a Capacitor wrapper for the POC because it runs immediately in any browser without platform tooling ‚Äî easier stakeholder demo. Full Capacitor wrapping is recommended for the production implementation (see FN-1290 recommendation doc).

**Files created:**
- `experiments/driver-mobile-poc/index.html` ‚Äî app shell, login + incident list screens
- `experiments/driver-mobile-poc/styles.css` ‚Äî AI dark theme, touch-optimized (44px targets, WCAG contrast)
- `experiments/driver-mobile-poc/app.js` ‚Äî login flow (`POST /api/auth/login`), incident list (`GET /api/roadside/calls`), status filter chips, pull-to-refresh
- `experiments/driver-mobile-poc/manifest.json` ‚Äî PWA manifest (standalone display, theme colour)
- `experiments/driver-mobile-poc/sw.js` ‚Äî service worker, static shell caching

**API endpoints consumed:**
- `POST /api/auth/login` ‚Üí JWT token stored in `sessionStorage`
- `GET /api/roadside/calls` ‚Üí incident list (status, urgency, caller, call_number)

**How to run:**
```
# Point a local HTTP server at the repo root, then open:
http://localhost:<port>/experiments/driver-mobile-poc/
# Or pass ?api=https://your-dev-gateway to point at a deployed instance
```

**WCAG / touch notes:**
- All interactive targets ‚â• 44 √ó 44 px
- Colour contrast ‚â• 4.5:1 on body text (AI dark theme tokens)
- `aria-label`, `role="button"`, `aria-live` wired on all interactive / dynamic regions

**Known gaps (intentional for POC scope):**
- Detail view shows a `window.alert` ‚Äî full slide-up sheet deferred to story implementation
- No real offline data (SW caches only the shell assets, not API responses)
- Icons are SVG inline placeholders (production would use proper icon assets)
