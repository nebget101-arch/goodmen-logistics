### FN-879 (subtask)

**Implementation Summary**
- Added `<app-load-wizard-driver-equipment>` at `frontend/src/app/components/load-wizard/steps/driver-equipment/` — standalone, OnPush, bound to the parent wizard's `driverEquipment` FormGroup (owned by FN-862 shell).
- Driver combo is fed by `LoadsService.getActiveDrivers()`; search filters the local list and the dropdown renders a loading/empty state while loading.
- Selecting a driver patches `driverId`, resets `showAllTrucks=false`, and auto-fills `truckId` / `trailerId` from the driver's assigned units when present in the equipment lists. The user can override either without losing the driver selection.
- Truck combo defaults to the driver's assigned truck (filtered pool); a "Show all trucks" toggle (bound to `showAllTrucks` in the FormGroup) swaps the source to the full fleet. A "Show assigned only" toggle reverses it.
- Trailer combo is optional — has a clear button and empty values leave `canProceed` green as long as driver + truck are set.
- Validators: the component installs `Validators.required` on `driverId` and `truckId` in `ngOnInit`, so `driverEquipment.valid` (already wired to `canProceed` in the shell) drives the Next button correctly. Trailer has no required validator.
- View mode: `applyModeState()` disables the entire FormGroup on `mode='view'` and re-enables for other modes; inputs also use `[disabled]` to show the disabled visual state while data is loading.
- Styling mirrors the legacy `<app-wizard-step-driver>` component so the two wizards look consistent while they co-exist (the legacy component is scheduled for removal in S10).

**Key Decisions**
- **Required validators added from the child component** (not in the shell). Keeps the wizard's FormGroup contract as a pure scaffold and lets each step decide its own gating — matches the pattern FN-875 uses for basics (adding `loadNumber` required + rate numeric/min validators on mount).
- **No "Last Used" chips port from legacy.** Scope of FN-865 AC didn't include it; the legacy component's localStorage driver-recency can be re-added in a follow-up if UX asks for it. Keeping the first cut small.
- **Reused the legacy styles.** Copied the classnames/colors from the legacy `step-driver.component.scss` rather than redesigning — visual consistency while both wizards are mounted, and a future cleanup can consolidate into a shared mixin.
