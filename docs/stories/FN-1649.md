# FN-1649: Roadside board polish — filter dropdown overflow fix + CTA button restyle

## Requirement
Post-launch polish on the redesigned Roadside Service AI board (`/roadside`, shipped in FN-1643 / FN-1646). Two user-reported UI issues, both pure frontend styling/layout in `roadside-board.component.{css,html}`:

1. **Status filter dropdown overflows its container.** The Urgency + Status `<app-ai-select>` filters sit in `<aside class="ai-panel rail"><div class="rail-filters">` (a tight `grid-template-columns: 1fr 1fr`). The Status select crosses the rail panel's border / its open dropdown is clipped by the panel's `overflow: hidden`. Enlarge the box that holds the fields so both selects fit cleanly and the dropdown is fully visible.
2. **CTA buttons aren't user-friendly / catchy.** `New Call` (`.btn-primary`), `Start triage` (`.btn-primary`), and `Create public link` (`.btn-secondary`) look cramped/under-emphasised. Restyle into friendlier, inviting CTAs (comfortable padding/height, clear icon gap, stronger primary affordance) — AI theme tokens only.

## Implementation Summary
_Filled by implementing agent (FN-1650)._

Both fixes landed as a **CSS-only** change in `roadside-board.component.css`; the template (`.html`) needed no edits because both issues were styling/box-model problems, not markup problems.

### Issue 1 — Status filter crossing the rail border
Root cause: the two `<app-ai-select>` filters live in a `grid-template-columns: 1fr 1fr` grid. CSS grid items default to `min-width: auto`, so a `1fr` track refuses to shrink below its content's min-content width. The Status select's intrinsic min-content width pushed its track wider than its share of the rail, so the field bled past the panel's right border.

Fix: `min-width: 0` on `.rail-filters .form-field` (and `.rail-filters app-ai-select`) lets each select shrink to its track, so both sit cleanly inside the rail at every width. Bumped the grid gap 10px → 12px and added `align-items: end` so the labelled fields baseline-align.

The "open dropdown clipped" symptom is a non-issue with this component: `<app-ai-select>` wraps Angular Material `mat-select`, whose panel renders in a **CDK overlay attached to `<body>`** (`panelClass="ai-select-panel"`), outside any ancestor `overflow`. No `overflow` rule was changed (none existed on `.ai-panel`/`.rail`).

### Issue 2 — CTAs cramped / under-emphasised
Root cause: the CTAs use `class="btn-primary"` / `class="btn-secondary"` **without** the base `.btn` class. In global `styles.css`, `.btn-primary`/`.btn-secondary` supply only the gradient surface + `border-color`; the entire box model (`display:inline-flex`, padding, gap, radius, min-height, `border-width/style`) lives on `.btn`. So the CTAs rendered as default inline buttons with a gradient and no icon gap.

Fix: scoped CTA layout in the component CSS (`.roadside-board .btn-primary, .roadside-board .btn-secondary`) — `inline-flex`, 44px min-height, `0 20px` padding, 9px icon→label gap, `border-width/style` so the global `border-color` shows, `:focus-visible` ring (`0 0 0 3px rgba(14,165,233,.5)`), and `:disabled`. Primaries get a stronger teal/cyan glow (box-shadow) that lifts on hover; the secondary stays quieter — clearly distinct. All colors reuse existing palette `rgba()` tokens already in the file → **zero new hex**.

## Key Decisions
- **CSS-only, template untouched** → guarantees no functional regression (all click handlers/bindings unchanged) and keeps the diff minimal.
- **`min-width:0` over widening the rail** — fixes the overflow at its source (grid min-content) and preserves the 320px rail + the `<1120px` stacked (`1fr`) behavior, rather than masking it with extra width.
- **Scoped CTA layout rather than adding `.btn` in HTML** — the global `.btn` padding (10px 18px) is smaller than the "friendlier" target; scoping in-component lets us hit 44px min-height + glow precisely and keeps the change inside the two ticket-scoped files. Specificity `.roadside-board .btn-primary` (0,2,0) cleanly beats global `.btn-primary` (0,1,0).
- **No getter-based `[options]`** introduced (FN-317 RCA respected) — bindings untouched; this was a CSS-only change.
