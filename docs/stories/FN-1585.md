# FN-1585: Spreadsheet import ‚Äî AI handler (column mapping + status/billing enum normalization + multi-stop pattern detection)

## Requirement
Build the AI service handler that takes a spreadsheet's headers + sample rows and returns a confident column mapping to FleetNeuron's `loads` schema, plus enum normalization for status/billing_status and a detected multi-stop pattern. **One Claude call per file** (Phase 1 strategy A ‚Äî column mapping only, no per-row AI).

This story is upstream of FN-1584 (the backend depends on the AI response shape). Lock the response JSON shape in this doc before either story cuts code.

## Implementation Summary

The AI handler `loads-spreadsheet-handler.js` exposes `POST /api/ai/loads/spreadsheet-import` and turns the headers + first 20 sample rows of a CSV/XLSX upload into the locked JSON contract above. Implementation strictly mirrors the existing `loads-nlq-handler.js` pattern: lazy Anthropic SDK init, fallback envelopes instead of 5xx, and `deps.anthropic` test seam. Prompt caching is wired on the system prompt block so the (large, stable) loads-schema description hits cache after the first request per 5-minute window. Cache lookup against `load_ai_extractions` (FN-741) runs before any model call.

## Key Decisions

1. **Reuse `load_ai_extractions` table (FN-741) for the spreadsheet cache.** The FN-741 table was originally PDF-keyed but its schema (`tenant_id`, `pdf_hash` text, `extracted_data` jsonb, `extraction_method` text) is generic enough to fit. The new handler stores `extraction_method = 'loads-spreadsheet-mapping'` so spreadsheet imports never collide with PDF rate-con extractions. This avoided creating a redundant cache table ‚Äî DB scope stays untouched in this story.
2. **Stable JSON for the cache key.** A naive `JSON.stringify({headers, sampleRows})` would produce different bytes for the same logical input depending on object key order. The handler's `stableStringify` walks the value recursively and sorts keys at every level so the SHA-256 fingerprint is deterministic. The unit test asserts this invariant.
3. **System prompt is locked-stable; volatile content goes after the breakpoint.** Per the canonical Claude API caching rules, the schema description and enum sets live in the system block (with `cache_control: { type: 'ephemeral' }`); the spreadsheet-specific headers + sample rows go in the user message. This is the prefix-match invariant ‚Äî any byte change in the system block invalidates the cache, so dynamic context is kept out of it.
4. **Validator always returns the full 22-field shape.** Even if the model omits a field, the validator backfills `{ sourceHeader: null, confidence: 0 }`. Confidences out of `[0, 1]` are clamped, not rejected. Unknown enum values are dropped. This eliminates downstream `if (mapping.field?.confidence ?? 0) > X` defensive code in FN-1584.
5. **Fallback envelope, not HTTP 5xx, on every recoverable failure.** Unparseable JSON, Anthropic rate limits, or network errors all return `{ success: true, fallback: true, meta: { reason } }`. The frontend can detect `fallback: true` and prompt the user to map columns manually rather than seeing a generic 5xx.
6. **`deps.db` test seam.** The handler's cache plumbing originally hard-required `@goodmen/shared/internal/db`. Made it overridable through `deps.db` so tests can stub the cache layer cleanly without monkey-patching `Module._resolveFilename`. Production callers omit `deps.db` and the lazy require is preserved.

### FN-1592 (subtask)

- **System prompt design.** The prompt enumerates the 22 FN load fields explicitly inside the JSON-schema example so the model returns a fully-shaped object. Both enum sets (`LOAD_STATUSES`, `BILLING_STATUSES`) are interpolated as JSON arrays in the system prompt ‚Äî the validator on the receive side enforces them again. Concrete worked examples are provided for the four `multiStopPattern` values plus common status/billing synonyms (`Paid Out -> PAID`, `On Hold -> DRAFT`, `Awaiting BOL -> PENDING`, `Sent to Factor -> SENT_TO_FACTORING`).
- **Validator logic.** `validateAiResult()`:
  1. Backfills any missing FN field with `{ sourceHeader: null, confidence: 0 }`.
  2. Clamps every confidence to `[0, 1]` (out-of-range coerced, non-numeric -> 0).
  3. Drops `statusEnumMapping` / `billingStatusEnumMapping` entries whose target is not in the canonical enum set, and uppercases otherwise valid targets.
  4. Coerces unknown `multiStopPattern` to `'single'`.
  5. Filters `extraStopColumns` to non-empty strings only.
  6. Normalizes `warnings` to `{ code, message }` shape.
- **Fallback strategy.** Three fallback exits: (a) `parse failure` after a successful Claude call ‚Üí `meta.reason = 'unparseable_model_output'`; (b) `client.messages.create` throws ‚Üí `meta.reason = 'ai_upstream_error'`; (c) request-shape validation hits before any AI call ‚Äî those return 400 (NOT a fallback, they're caller bugs). Tests cover all three paths.
- **Cache lookup behavior.** Hash is computed from `stableStringify({headers, sampleRows})`, scoped by `tenant_id` and `extraction_method`, with a 7-day TTL via `created_at` cutoff. On hit the AI client is never invoked; the cached `extracted_data` is returned with `cacheHit: true`. On write the handler uses `INSERT ... ON CONFLICT DO UPDATE` so a re-upload refreshes `created_at`. DB unavailability is silently tolerated ‚Äî `getDb()` returns `null` and the handler proceeds to a live AI call.
- **Fixtures.** Three recorded `messages.create` responses live under `__tests__/fixtures/loads-spreadsheet/`:
  - `clean-headers.json` ‚Äî dispatch-software-style export, all 22 FN fields map at ‚â•0.92 confidence; `multiStopPattern: 'single'`.
  - `abbreviated-headers.json` ‚Äî heavily-abbreviated TMS export (LD, PO, Brkr, Stat, etc.); confidences in 0.65‚Äì0.90 range; warning `ABBREVIATED_HEADERS`; status/billing synonym mappings (`Del -> DELIVERED`, `Pd -> PAID`).
  - `multi-row-stops.json` ‚Äî order-grouped multi-stop export; `multiStopPattern: 'multi_row'`, `groupByColumn: 'Order Number'`; exercises every status/billing synonym from the FN-1593 acceptance list (`On Hold`, `Awaiting BOL`, `Sent to Factor`, `Paid Out`).
