## Implementation Summary

### FN-1173 (subtask) ÔÇö Decision: ADD `?variant=long`

**Outcome:** existing endpoint (FN-1123) is suitable as a base, but the 2ÔÇô3 sentence default is too short for a 1+ page branded PDF. Added an opt-in `?variant=long` query parameter that doubles the token budget (400 ÔåÆ 900) and extends the narrative to 5ÔÇô8 sentences across 1ÔÇô2 paragraphs. The on-screen panel (FN-1146) keeps the original short variant by omitting the query param.

Also fixed a real server-to-server gap: `reports-narrative-handler.js#tryRequireJwt` falls back to `null` when `jsonwebtoken` isn't installed, returning 403 to any caller without `req.user` (which is exactly what reporting-service forwards). `jsonwebtoken@^9.0.3` was missing from `ai-service/package.json` even though every other microservice carries it. Added it as a direct dependency so the JWT verification path works end-to-end.

**Variant mechanism:**
- `resolveVariant(req)` reads `req.query.variant`, normalises (case-insensitive, trims, accepts first element of an array), and falls back to `short` for any unknown value. Never errors.
- The variant directive is delivered via the **per-call user message** (`form: "long"|"short"`) ÔÇö the cached system blocks stay byte-identical between variants, so the prompt cache hit-rate is preserved across both call sites.
- `SYSTEM_PROMPT` describes both variants once; the runtime decision lives in the user payload's `form` key.
- `max_tokens` is the only Anthropic-call argument that varies per variant (400 vs 900).
- Response now includes `meta.variant` so callers can verify which length was applied.

**Server-to-server validation:**
- Reporting-service forwards the user's `Authorization: Bearer <jwt>` header verbatim. `req.user` is NOT pre-attached because the request bypasses the gateway.
- `resolveUser()` already supports this ÔÇö it reads `req.headers.authorization`, verifies via `JWT_SECRET`, and surfaces the payload as `req.user`. Same `JWT_SECRET` env var must be set on both services (already true on Render).
- No IP allowlist / mTLS needed: AUTH is via the user's own JWT, which is shorter-lived and scoped to the same user that triggered the PDF export. No service-to-service shared secret to leak.

**Prompt-caching end-to-end:**
- The two cached system blocks (system prompt + per-report schema) are byte-identical between short/long calls ÔÇö the `form` key only changes the per-call user message which is uncached.
- Therefore the second PDF export for the same `reportKey` will report `meta.cacheReadTokens > 0`, satisfying the AC.

## Key Decisions

### FN-1173 (subtask)
- **`?variant=long` over splitting endpoints**: a query-string variant is the smallest possible API surface change.
- **Variant lives in the user message, not the system block**: keeping `SYSTEM_PROMPT` and the schema block byte-identical between variants preserves prompt-cache hits across both call sites.
- **Token budget doubled (400 ÔåÆ 900), not tripled**: 5ÔÇô8 sentences across 1ÔÇô2 paragraphs comfortably fits in ~700 output tokens; 900 leaves headroom.
- **Unknown variants silently degrade to `short`** instead of returning 400.
- **Added `jsonwebtoken` to ai-service**: the handler relied on it for the server-to-server path but the dep was missing ÔÇö every other microservice already carries `^9.0.3`.
- **No IP allowlist or shared-secret scheme**: the user's own JWT is forwarded and verified with the same `JWT_SECRET` both services already share.

### FN-1160 (subtask)
- **Same endpoint, new flag** rather than a new branded URL.
- **No global toast service introduced.** Reuses the inline `<p class="error">` element.
- **Per-format export flags** (`{ csv, pdf }`) instead of a single boolean ÔÇö the two formats can race independently.

### FN-1167 (subtask)
- **Co-located new services with the route in `goodmen-shared/services/`** rather than under reporting-service to avoid a backwards dependency from a shared package into a microservice.
- **Anomalies fetched in parallel with the narrative on the PDF path.** The original ticket only mentioned narrative s2s, but the AC explicitly lists "anomaly list" as a required PDF section, so we fetch them through the same client.
- **`?narrative=off` is the only knob.** The FN-1160 FE sends `includeNarrative=true` (already the default), so the branded path applies uniformly. `narrative=off` exists as a non-AI escape hatch for batch exports or QA.
- **`pickTableColumns` caps at 6 columns and uses ellipsis truncation;** full data remains accessible via the unchanged CSV export.
- **The render function returns a promise that resolves on `stream.finish`** so the route handler awaits completion, surfacing pdfkit errors via the existing `try/catch`.

- **Service**: `ai-service` redeploy. The new `jsonwebtoken` dep needs to land via `npm install` during the Render build, so a redeploy (not a hot restart) is required.
- **Migrations**: none.

  - `JWT_SECRET` must be identical on `ai-service` and `reporting-service`.

  1. Two narrative calls (default + `?variant=long`) return the expected lengths.
