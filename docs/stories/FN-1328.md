## Implementation Summary

### FN-1345 (subtask)
Final responsive SCSS pass for the redesigned Control Center. Introduced a shared breakpoint token file (`frontend/src/styles/_breakpoints.scss`) so the 1280 / 1024 / 768 / 375 breakpoints used across the dashboard are defined in one place; refactored kpi-strip and added new mobile-rhythm media queries to control-center, action-queue, and the dashboard shell.

Key behaviors:
- KPI strip wraps to **3-up under 1280px** and **2-up under 768px** (was 720px — re-aligned to the 768 breakpoint declared by the acceptance criterion).
- `.control-center__layout` is reserved for a future two-column main+rail layout and explicitly collapses to a single column under 1280px so any sibling rail introduced later stacks inline below the main column.
- Mobile rhythm: `.control-center` and `.control-center__grid` tighten gap; quick-actions panel reduces padding so its buttons stay reachable without horizontal scroll.
- Action Queue tightens padding, list gap, and details padding under 768px; filter selects flex to 50%/row under 375px so all three filters fit without overflow.
- Dashboard shell reduces inline padding and grid gap under 768px and adds `min-width: 0` to `.dashboard-grid` and `.dashboard-column` so children can shrink below their content width without forcing horizontal scroll.
- Drag-reorder unaffected — only gap and padding values changed; CDK transform animations remain on `.control-center__card`.

## Key Decisions

### FN-1345 (subtask)
- **Shared tokens via SCSS partial, not CSS variables.** The breakpoints are needed inside `@media` queries; CSS custom properties can't drive `@media` rules in any current browser. Plain SCSS variables + mixins are the right tool, and the partial is named `_breakpoints.scss` (leading underscore) so it's `@use`-only.
- **Re-aligned 720 → 768.** The KPI strip was using 720px for the "wrap to 2-up" point. The story acceptance criterion explicitly calls for 768px; re-alignment keeps the strip in sync with the dashboard's standard tablet breakpoint and the QA validation matrix.
- **Reserved `.control-center__layout` even though no right rail exists today.** The story description ("collapse the right rail inline under 1280px") and parent story acceptance criteria reference a rail that may be added by a future story. The class is opt-in (the existing template still uses `.control-center__grid`), so this is forward-only — no template change needed in this subtask, and the rail-collapse rule is already in place when the layout is adopted.
- **`min-width: 0` on grid/flex children.** Without this, deeply nested CSS-grid + flex content (KPI strip → KPI card row, action queue list) can force horizontal scroll on narrow viewports. Adding it at the dashboard-grid + dashboard-column + control-center level is the standard fix and has zero visual cost above the breakpoint.
