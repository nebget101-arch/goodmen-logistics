# FN-1038: karate-core: maven-jar-plugin version not pinned in parent pluginManagement

## Requirement
Maven was emitting a "build stability" warning when building the Karate test reactor because `maven-jar-plugin` was declared in `tests/karate/karate-core/pom.xml` without a version, and the parent `pluginManagement` block did not pin it. The fix: pin the plugin version in the parent pom so it matches the management pattern already used for `maven-compiler-plugin`, `maven-surefire-plugin`, and `maven-resources-plugin`.

## Acceptance Criteria
- [x] Parent `pluginManagement` pins `maven-jar-plugin` to a specific version via a property.
- [x] Running `mvn -pl karate-gateway -am test-compile` from `tests/karate/` no longer emits the "build.plugins.plugin.version is missing" warning.
- [x] Build still produces the `karate-core` test-jar consumable by per-service modules.
- [x] Evidence: terminal output captured under `docs/stories/evidence/FN-1038/`.

## Dependencies
- None (single-file fix layered on top of FN-908 / FN-909 which are already on `dev`).

## Agent
qa

## Implementation Summary
Two additive edits to `tests/karate/pom.xml`:

1. Added a new property `<maven.jar.plugin.version>3.4.1</maven.jar.plugin.version>` alongside the existing `maven.surefire.plugin.version`, `maven.compiler.plugin.version`, and `maven.resources.plugin.version` properties.
2. Added a `maven-jar-plugin` entry to the parent `<pluginManagement>` block that references `${maven.jar.plugin.version}`.

`tests/karate/karate-core/pom.xml` was left as-is — it continues to declare the `maven-jar-plugin` plugin with the `test-jar` execution (no version, by design) and now inherits the pinned version from the parent. Service modules added in the future (`karate-auth-users`, `karate-logistics`, etc.) get the same pinned version automatically.

## Key Decisions
- **Version pinned to `3.4.1`** — current stable maven-jar-plugin release. Keeps parity with the other build plugins (compiler 3.13.0, surefire 3.2.5, resources 3.3.1) which are also on recent stable versions.
- **Did not touch `karate-core/pom.xml`** — leaving the child plugin declaration empty of version is the canonical Maven pattern when `pluginManagement` provides the version. Modifying the child would have made the version pin redundant and brittle.
- **Did not address the unrelated env warnings** — `sun.misc.Unsafe`, native-access, and `--release` vs `-source/-target` are JDK/Maven distribution concerns separate from this bug. Filing them separately if needed; out of scope here.
