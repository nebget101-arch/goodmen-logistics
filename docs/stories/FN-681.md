## Implementation Notes

### FN-691 (subtask)

**Branch:** `backend/FN-691/locations-crud-enhanced`

**File:** `backend/packages/goodmen-shared/routes/locations.js`

**What changed:**
1. **Removed `supportsLocations` subscription gate** — all endpoints now accessible on any plan.
2. **`getTenantContext` simplified** to `getTenantId` — plan lookup removed entirely.
3. **`GET /api/locations`** — added `type`, `active`, `search` query filters; pagination (`page`, `pageSize` max 200); returns `{ data: [...], meta: { page, pageSize, total } }`.
4. **`GET /api/locations/:id`** — returns full detail columns (city, state, zip, phone, email, contact_name, timezone, operating_hours, settings) plus three nested arrays:
   - `bins` — all `location_bins` rows for this location
   - `users` — users assigned via `user_locations` join (graceful fallback if table absent)
   - `supply_rules` — `warehouse_shop_supply_rules` rows where this location is either side (graceful fallback)
5. **`POST /api/locations`** — accepts all new columns from FN-686 migration; validates `location_type` against `VALID_TYPES` enum (`SHOP|YARD|DROP_YARD|WAREHOUSE|OFFICE|TERMINAL`), returns 400 if invalid.
6. **`PATCH /api/locations/:id`** — new partial-update endpoint using COALESCE on all fields; validates `location_type` if provided.
7. **`PUT /api/locations/:id`** — backward-compatible alias that delegates to `handleLocationUpdate` (same behaviour as PATCH).
8. **`DELETE /api/locations/:id`** — checks for hard dependencies (work_orders, inventory, user_locations, vehicles); returns **409** with structured `{ dependencies: { work_orders, inventory_items, users, vehicles } }` if any exist; hard-deletes (CASCADE removes bins + supply_rules) if clear.

**COLS split:**
- `LIST_COLS` — lean set for the list endpoint (excludes JSONB operating_hours, settings)
- `DETAIL_COLS` — full column set for create/update/detail responses
