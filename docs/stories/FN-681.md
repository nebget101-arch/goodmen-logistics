## Implementation Notes

### FN-692 (subtask)
**Location Bins CRUD + bulk pattern create endpoint**

Created `backend/packages/goodmen-shared/routes/location-bins.js` with 5 endpoints:
- `GET /api/locations/:locationId/bins` — list bins with optional `active` filter, includes inventory_count
- `POST /api/locations/:locationId/bins` — create single bin with validation (bin_type enum, unique bin_code → 409)
- `POST /api/locations/:locationId/bins/bulk` — pattern-based bulk create supporting:
  - Range pattern: `"A-1..A-20"` → creates A-1 through A-20 (supports zero-padding)
  - Zone+rows pattern: `{ zone: "X", rows: ["R1","R2"] }` → creates X-R1, X-R2
  - Max 500 bins per request, ON CONFLICT DO NOTHING for duplicates
- `PATCH /api/locations/:locationId/bins/:binId` — partial update with enum validation
- `DELETE /api/locations/:locationId/bins/:binId` — smart delete: soft (active=false) if inventory references bin, hard delete otherwise

Mounted in `inventory-service/server.js` with authMiddleware + tenantContextMiddleware.
