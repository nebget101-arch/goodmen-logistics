## Implementation Notes

### FN-695 (subtask)
**RBAC — Add locations.* permissions to catalog + role grants**

1. **Seed update** (`01_rbac_seed.js`):
   - Added `EXTRA_PERMISSIONS` array with 3 sub-module permissions: `locations.bins.manage`, `locations.users.manage`, `locations.supply_rules.manage`
   - Appended to `buildPermissions()` output so they're generated alongside the standard MODULE.ACTION grid
   - Granted all 3 + `locations.manage` to `shop_manager` and `parts_manager` roles

2. **Migration** (`20260416190000_add_location_submodule_permissions.js`):
   - Idempotently inserts the 3 new permissions
   - Grants them to `super_admin`, `shop_manager`, `parts_manager` via `role_permissions`
   - Down migration removes grants then permissions

3. **Route middleware** (`locations.js`):
   - Replaced legacy `auth(['admin', 'fleet'])` with broader role list + `loadUserRbac`
   - Added `requirePermission('locations.view')` on GET endpoints
   - Added `requirePermission('locations.manage')` on POST/PUT/DELETE endpoints

Note: Middleware wiring on location-bins.js, warehouse-supply-rules.js, and user-locations.js will be done when those branches merge — those files don't exist on dev yet.
