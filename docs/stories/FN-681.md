### FN-694 (subtask)

**Branch:** `backend/FN-694/user-locations-route`

**Files:**
- `backend/packages/goodmen-shared/routes/user-locations.js` (new)
- `backend/microservices/logistics-service/server.js` (mount added)

**What was created:**
New `user-locations.js` route mounted at `/api` in logistics-service, exposing 4 endpoints:

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/locations/:locationId/users` | List users assigned to a location (joined with `users` for name/email/role/assigned_at) |
| `POST` | `/api/locations/:locationId/users` | Bulk idempotent assign тАФ body `{ user_ids: [...] }`. Uses `ON CONFLICT DO NOTHING`. Returns `{ assigned, already_assigned, invalid_user_ids, total_requested }` |
| `DELETE` | `/api/locations/:locationId/users/:userId` | Unassign one user from a location. 404 if no such assignment. |
| `GET` | `/api/users/:userId/locations` | Reverse lookup тАФ all locations a user is assigned to, scoped to tenant. |

**All endpoints:** tenant-scoped (location must belong to caller's tenant, users must belong to same tenant), protected by `auth(['admin', 'fleet'])`.
