## Implementation Summary
The bottom `panel-actions` chip strip on the dashboard duplicated the role-aware Quick Actions component already mounted at the top. We removed the duplicate surface and migrated its action set into the global Cmd/Ctrl+K palette so the same shortcuts are reachable from every view, not only `/dashboard`.

The palette itself was already mounted at the app root (`app.component.html` line 269) and bound to Cmd/Ctrl+K via `KeyboardShortcutsService`. The remaining work was to (a) make the canonical action set explicit and permission-gated, and (b) drop the duplicate dashboard surface.

### FN-1335 (subtask)
Branch: `frontend/FN-1335/quick-actions-and-palette`

- Created `frontend/src/app/shared/components/command-palette/command-actions.ts` exporting `COMMAND_ACTIONS` (Drivers, Vehicles, Loads, HOS, Audit, Reports, Roadside, "Generate Audit Report") plus a `COMMAND_ACTION_PATHS` set used to dedupe nav-link entries.
- Updated `command-palette.component.ts` to source the "Quick Actions" group from `COMMAND_ACTIONS`, filtered through `AccessControlService` (`canSee(tab)`, `hasFeatureAccess(featureFlag)`, `canAccessUrl(path)`); the existing "Show keyboard shortcuts" helper is preserved.
- Removed the `panel-actions` block from `dashboard.component.html` (lines 160–187) and the matching `.panel-actions / .actions-grid / .action-chip / .action-chip-emphasis / .chip-title / .chip-sub` rules from `dashboard.component.css`.
- The palette continues to register Cmd/Ctrl+K through `KeyboardShortcutsService`, which means the binding shows up in the shortcuts help modal and won't double-bind with another listener — no platform-specific conflict either.

## Key Decisions
- **Single source of truth for the action set.** Hard-coding the Quick Actions list inside the palette component was avoided; an exported `COMMAND_ACTIONS` array makes future role-aware reordering and additions a one-file change and gives QA + e2e tests something to import.
- **Dedupe via `COMMAND_ACTION_PATHS`.** Loads / Reports / Drivers / Roadside / HOS / Vehicles also appear in the sidebar nav. Without dedupe they would show up twice in the palette (once under Quick Actions, once under Navigation). The `Set` lookup is O(1) per nav link and keeps both sources independently editable.
- **Audit listed twice on purpose.** The acceptance criteria explicitly enumerate both "Audit" (the workspace) and "Generate Audit Report" as separate items. Both navigate to `/audit`, but the labels signal different intents and either can be the user's first instinct.
- **Permission gating reuses `AccessControlService`.** Same model the sidebar uses, so "what the user sees in the palette" is consistent with "what the user sees in the sidebar".
- **Did not touch the top Quick Actions component** (`components/control-center/quick-actions/`) — it's already FN-1130 role-aware and outside this subtask's scope.
