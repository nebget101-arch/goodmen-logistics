# FN-1297 (Bug): AI Insights panel renders empty — frontend service expects res.data but API returns res.insights

## Requirement
Loads dashboard "AI Insights" panel always renders the empty state ("No insights for this period — everything looks healthy") even when `/api/loads/ai-insights` returns insights. Reproduced with `period=all` returning 3 insights (overdue=18, missing_documents=28, driver_idle=5). Fix the frontend so it consumes the actual backend response shape produced by FN-793.

## Acceptance Criteria
- [ ] AI INSIGHTS panel renders one card per backend insight when API returns a non-empty `insights` array.
- [ ] Each card shows the backend `message` as its title.
- [ ] Card click navigates to `action_url`.
- [ ] Severity-based left-border color renders for `critical` (red), `warn` (amber), `info` (blue).
- [ ] Each backend `type` value (`drafts_ready`, `overdue`, `rate_anomaly`, `missing_documents`, `driver_idle`, `high_margin`, `low_margin`) gets a non-default icon.
- [ ] Period pill change triggers a refetch and the cards update.
- [ ] Empty-state still shows when `insights: []`.
- [ ] Existing `loads-ai-insights.test.js` still passes (no backend changes).
- [ ] Manual screenshot in `docs/stories/evidence/FN-1297/` showing populated panel.

## Root Cause Summary
Three contract drifts between `backend/packages/goodmen-shared/routes/loads.js:2452-2463` (FN-793) and `frontend/src/app/services/loads.service.ts:360-371` + `intelligence-insights.component.ts` (FN-795):

1. **Wrapper key**: backend → `insights`; frontend reads → `data`. Service falls back to `[]` and component shows empty state.
2. **Item field names**: backend → `message` / `action_url` / `action_label`; frontend → `title` / `href` / `action.{label,event}` / `id`.
3. **Enum values**:
   - `severity`: backend `'critical' | 'warn' | 'info'` vs frontend type `'low' | 'medium' | 'high'` (also breaks the `[data-severity]` SCSS selectors).
   - `type`: backend uses `missing_documents`, `drafts_ready`, `rate_anomaly`, `high_margin`, `low_margin`, `driver_idle`; frontend `iconFor()` switch uses `missing_docs` (different spelling) and other values that the backend never emits.

## Fix Direction (Frontend Only)
Backend is source of truth (shipped, tested, documented in FN-784 story doc). Update:
- `loads.service.ts` `getAiInsights` — read `res?.insights`; map each item to UI shape (synthesize `id` for `trackBy`; `title ← message`; `href ← action_url`; collapse the redundant `action` button per HTML simplification below).
- `loads.service.ts` types — `AiInsightSeverity = 'critical' | 'warn' | 'info'`; `AiInsightType` = the seven backend values.
- `intelligence-insights.component.ts:76-87` `iconFor()` — handle all seven types.
- `intelligence-insights.component.html` — pick one click target (recommend keeping the title-link form for whole-row navigation; drop the redundant action button).
- `intelligence-insights.component.scss` — update `[data-severity="critical"|"warn"|"info"]` selectors.

## Implementation Summary
Re-aligned the frontend AI Insights contract with the backend `/api/loads/ai-insights` response shape (FN-793). The service now reads `res.insights` (was `res.data`) and maps each backend item (`message`, `action_url`, `severity: 'critical'|'warn'|'info'`, `type: drafts_ready|overdue|rate_anomaly|missing_documents|driver_idle|high_margin|low_margin`) into the UI `AiInsight` (`title`, `href`, severity, type). The component icon switch covers all seven backend types. The card template was simplified to a single click target — the title link — per the bug's recommended UX (whole-row navigation); the redundant action button was dropped along with the now-unused `subtitle` / `icon`-override / `action` / `generatedAt` fields. SCSS `[data-severity]` selectors updated to the new severity values.

## Files Changed
- `frontend/src/app/services/loads.service.ts` — replaced `getAiInsights` mapper to read `res.insights` and project to UI shape; tightened `AiInsight` interface; added internal `AiInsightApiItem` / `AiInsightsResponse` types matching the backend contract; new severity union (`critical|warn|info`) and type union (the seven backend-emitted values).
- `frontend/src/app/components/loads-dashboard/intelligence-panel/intelligence-insights.component.ts` — `iconFor()` switch updated for all seven types (`drafts_ready→rate_review`, `overdue→schedule`, `rate_anomaly→trending_down`, `missing_documents→description`, `driver_idle→local_shipping`, `high_margin→trending_up`, `low_margin→trending_down`); removed unused `@Output() action`, `onAction()`, and the `EventEmitter` / `Output` / `interval` imports.
- `frontend/src/app/components/loads-dashboard/intelligence-panel/intelligence-insights.component.html` — collapsed title-link + action-button into a single `<a [routerLink]>` row; removed the `*ngIf` href fallback (backend always emits `action_url`) and the unused subtitle line.
- `frontend/src/app/components/loads-dashboard/intelligence-panel/intelligence-insights.component.scss` — replaced `[data-severity="low|medium|high"]` selectors with `[data-severity="info|warn|critical"]` (info→blue, warn→amber, critical→red); removed dead `.intel-insights__action`, `.intel-insights__subtitle`, `.intel-insights__body`, and `.intel-insights__item-title` rules.

## Key Decisions
- **Backend is source of truth.** Frontend was the side that drifted (no consumer of the planned `subtitle` / `icon` / `action` / `generatedAt` fields ever shipped). Type was tightened to exactly what the backend emits — fewer optional fields, no dead-code paths in the template.
- **Single click target.** Per the bug's open question, kept the row-as-link form for accessibility (whole row navigates) and dropped the right-aligned action button. Backend's `action_label` is currently unused in the UI; the `message` text already conveys the action.
- **Synthesized `id`.** Backend doesn't supply a stable identifier per insight — `${type}-${period}-${index}` is sufficient for `trackBy` since the list re-renders fully on each refetch and order is stable per backend rule sequence.
- **No defensive fallbacks.** Backend always emits `action_url` and `message`; the previous `*ngIf="insight.href; else plainTitle"` template fork was defensive against a contract that never existed and is removed.
