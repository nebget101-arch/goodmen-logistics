# FN-1553: [Work Order] Financials tab — date pickers render without labels (Approval Date, Next Service Due Date)

## Requirement
On the Financials tab, two `app-ai-date-picker` instances render with no visible label: (1) Approval Date in Status/Workflow Tracking, (2) Next Service Due Date in Compliance/Fleet Maintenance. Templates supply `label="..."` correctly, but the rendered output is empty. Likely a CSS/scoping bug or a `floatLabel="auto"` interaction with the dark theme.

- [x] Next Service Due Date label renders on Financials → Compliance/Fleet Maintenance

## Implementation Summary
The bug is global to every `app-ai-date-picker` consumer that passes `label="..."`, not just Financials.

Root cause: `ai-date-picker.component.scss` sets `display: none !important;` on `.mdc-notched-outline` (lines 49–53) so the dark-theme custom border replaces Material's outline. With `appearance="outline"`, Material renders `<mat-label>` *inside* `.mdc-notched-outline__notch` — hiding the notched outline therefore hides the label. The Basics tab is unaffected because it uses `ariaLabel="..."` plus an external `<label>` in the parent template (no `<mat-label>` is rendered).

Fix: change the date-picker template to render an external `<label class="ai-date-picker-label" [attr.for]="inputId">` above the form field whenever `[label]` is provided, and drop `<mat-label>` entirely. The component now consistently uses the external-label layout (`--external-label` class, no float). All consumers using `label="..."` (Financials WO tab, WO Notes tab, Settlements wizard, Settlements scheduled deductions, Safety Claims, Safety Accident Detail, IFTA Quarterly when applicable) immediately gain a visible label without any per-call-site change.

## Files Changed
- `frontend/src/app/shared/ai-date-picker/ai-date-picker.component.html` — render external `<label>` when `[label]` is set; remove `<mat-label>`; drop `floatLabel` (no internal floating label remains).
- `frontend/src/app/shared/ai-date-picker/ai-date-picker.component.scss` — add `.ai-date-picker-label` styling (12px / 500 / `rgba(203, 213, 225, 0.9)` / 6px bottom margin) matching the WO shell's `:host ::ng-deep label` aesthetic.
- `frontend/src/app/shared/ai-date-picker/README.md` — update the `label` description to "Visible label rendered above the field"; clarify the visible-label guidance in the a11y note.

## Key Decisions
- **External `<label>` over re-styling Material's floating label.** Reviving `<mat-label>` would require either un-hiding `.mdc-notched-outline` (which discards the dark-theme custom border that several FN tickets shipped — high blast radius) or absolutely re-positioning `.mdc-floating-label` outside the hidden notch (fragile against Angular Material upgrades). Rendering an external `<label>` is one HTML element and a tiny CSS rule, and it matches the dominant pattern used by the rest of the codebase (Basics tab, IFTA, Safety Reports filters).
- **Fix in the component, not at every call site.** Bug AC scopes the fix to Financials, but ~15 other call sites have the same invisible label and the user just hadn't noticed. Fixing the shared component closes them all in one change. No call site behavior changes for consumers that already use `ariaLabel` + external `<label>`.
- **Drop `floatLabel` rather than keep `'never'`.** With no `<mat-label>` rendered, `floatLabel` has no effect. Material's typings reject `'never'` (the original bypassed via `$any`), and removing the attribute lets the default apply.
- **Left dead `.mdc-floating-label` color rules in SCSS** (lines 94–102). Removing them is out of scope for this bug fix and risks overlap with any other consumer that might still bring its own `<mat-label>` in the future.
