# FN-552: Scheduled Deduction smart defaults — auto-populate amounts from expense responsibility config

## Requirement
The New Scheduled Deduction dialog should auto-populate amounts and defaults based on the driver's expense responsibility configuration:

- **Fixed categories** (Insurance, ELD, Trailer Rent): auto-fill dollar amount from expense responsibility config for the selected "Apply to" party
- **Variable categories** (Fuel, Tolls, Repairs): auto-set Amount Type to "Percentage" and pre-fill split % from expense responsibility config
- **Apply To change**: amount/percentage updates to match the newly selected party's configured value; if user has manually overridden, prompt before resetting

## Acceptance Criteria
### Fixed expense auto-populate (Insurance, ELD, Trailer Rent)
- [x] Selecting category reads driver's expense responsibility config for that category
- [x] Amount auto-fills with driver's configured dollar amount when "Apply to" = Driver
- [x] Amount auto-fills with EO's configured dollar amount when "Apply to" = Equipment Owner
- [x] Auto-populated amount is editable (user can override)
- [x] No auto-fill if no expense responsibility config exists for that category (defaults to $0.00)

### Variable expense defaults (Fuel, Tolls, Repairs)
- [x] Selecting variable category auto-sets Amount Type to "Percentage"
- [x] Percentage auto-fills from expense responsibility split (e.g., Fuel Driver 50%)
- [x] Label shows "Split %" instead of "Amount $" when Amount Type is Percentage
- [x] Code comment added: variable deductions will be superseded by actual transaction data in Settlement V2

### Apply To change handler
- [x] Switching "Apply to" between Driver and Equipment Owner updates pre-filled amount/percentage
- [x] For fixed: switches between driver's dollar amount and EO's dollar amount
- [x] For variable: switches between driver's % and EO's %
- [x] If user has manually edited the amount, prompt "Reset to configured amount?" before overwriting

## Implementation Summary

### FN-561 (fixed auto-populate)
- Added `expenseProfile: any = null` property to store raw expense responsibility API response
- On driver load, stores full API response in `this.expenseProfile`
- `resetForm()` clears `this.expenseProfile = null`
- Extended `onRecurringExpenseCategoryChange()`: insurance/eld/trailerRent categories auto-fill `amount` from `expenseProfile.driver_fixed_amount` (primary) or `expenseProfile.owner_fixed_amount` (additional)

### FN-562 (variable % defaults)
- Extended `onRecurringExpenseCategoryChange()`: fuel/tolls/repairs categories set `amount_type = 'percentage'`
- Auto-fills `amount` from `sharedExpensePercentages[category]` (driver share for primary, `100 - driverShare` for additional)
- HTML: dynamic label `{{ newRecurringDeduction.amount_type === 'percentage' ? 'Split %' : 'Amount' }}`
- Added Settlement V2 code comment

### FN-563 (Apply To change handler)
- `onDeductionTargetChange()` re-runs amount auto-fill when Apply To selection changes
- Dirty-check: if user has manually edited the amount, shows confirm dialog before overwriting

## Key Decisions
- Fixed categories use `driver_fixed_amount`/`owner_fixed_amount` from the raw expense profile API response
- Variable categories use `sharedExpensePercentages[category]` (same source as expense config panel)
- Dirty check for Apply To change prevents silently overwriting a user's manual override
