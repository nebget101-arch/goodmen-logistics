# FN-552: Scheduled Deduction Smart Defaults

## Requirement
The New Scheduled Deduction dialog should auto-populate amounts and defaults based on the driver's expense responsibility configuration.

## Acceptance Criteria
- [ ] Selecting fixed category (Insurance, ELD, Trailer Rent) auto-fills amount from driver's expense responsibility config
- [ ] Amount auto-fills with driver's configured dollar amount when Apply To = Driver
- [ ] Amount auto-fills with EO's configured dollar amount when Apply To = Equipment Owner
- [ ] Auto-populated amount is editable (user can override)
- [ ] No auto-fill if no expense responsibility config exists (defaults to $0.00)
- [ ] Selecting variable category (Fuel, Tolls, Repairs) auto-sets Amount Type to "Percentage"
- [ ] Percentage auto-fills from expense responsibility split %
- [ ] Label shows "Split %" instead of "Amount $" when Amount Type is Percentage
- [ ] Switching Apply To updates pre-filled amount/percentage
- [ ] If user has manually edited amount, prompt "Reset to configured amount?" before overwriting

## Implementation Summary

### FN-561 (frontend)
- Added `expenseProfile: any = null` property to store the raw expense responsibility API response
- When loading a driver's expense responsibility (`getExpenseResponsibility`), the full response is stored in `this.expenseProfile`
- `resetForm()` clears `this.expenseProfile = null`
- Extended `onRecurringExpenseCategoryChange()`: when category is `insurance`, `eld`, or `trailerRent`, auto-fills `newRecurringDeduction.amount` from `expenseProfile.driver_fixed_amount` (target=primary) or `expenseProfile.owner_fixed_amount` (target=additional)
- If no profile or amount is 0, amount stays null (shows placeholder $0.00)
