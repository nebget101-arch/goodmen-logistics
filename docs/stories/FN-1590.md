## Endpoints Added
| Method | Path | Notes |
|--------|------|-------|
| POST   | `/api/loads/import/preview`         | multer (memory, 15 MB, csv/xls/xlsx/txt). Hashes file → R2 → cache lookup → optional AI call → inserts `load_import_batches` row in 'pending' status. |
| POST   | `/api/loads/import/stage`           | Re-parses the file from R2, applies `columnMapping` deterministically, writes per-row outcomes, transitions batch to 'staged'. |
| POST   | `/api/loads/import/commit/:batchId` | Single transaction; inserts `loads` + `load_stops` for `ok` rows; marks `duplicate` per-tenant by `load_number`; threshold-based DRAFT/NEW routing. Idempotent. |

| GET    | `/api/loads/import/batches/:id`     | Batch detail with row breakdown. |

All routes are gated by `requireRole(['admin', 'dispatch'])` — driver role is
explicitly excluded.

## Behavior Summary

**Preview** is a single multipart POST. It always:
1. Parses headers + rows; if `> 500` rows, returns 413 with the Phase 2 message.
2. SHA-256 hashes the file and uploads to R2 under `loads/import/{tenantId}/…`.
   R2 failure is logged but non-fatal (the batch still records).
3. Looks up `load_ai_extractions` keyed `(tenant_id, pdf_hash)` with a 7-day
   created_at window. The legacy `pdf_hash` column re-stores the spreadsheet
   hex; `extraction_method='spreadsheet-import'` distinguishes from PDF rows.
4. On cache miss, calls `${AI_SERVICE_URL}/api/ai/loads/spreadsheet-import`
   with a 15 s abort timeout. Any failure flips `aiUnavailable: true` and
   keeps `columnMapping: null` — the wizard can fall back to manual mapping.
5. Inserts a `load_import_batches` row in 'pending' status and returns
   `{ batchId, headers, sampleRows, totalRows, fileHash, cacheHit, aiUnavailable, columnMapping, … }`.

**Stage** re-downloads the R2 buffer (so a different gateway worker can run
stage even if preview happened on another instance), reparses, walks every
row applying `columnMapping`, normalizes status / billing-status enums, and
writes per-row records to `load_import_rows`. Validation outcomes:
- `ok` — has `load_number` (or `po_number` fallback) and any enum mappings resolved
- `needs_review` — at least one enum value couldn't be resolved
- `error` — missing both `load_number` and `po_number`

**Commit** opens one transaction, walks `ok` and `needs_review` rows in
`source_row_index` order:
- Pre-checks `loads.load_number` collision per tenant. Hits add to `duplicates`
  and update the row to `validation_status='duplicate'` — no insert, no throw.
- Calls `fuzzy.matchBroker / matchDriver / matchVehicle` (broker is
  tenant-global per FN-205; driver and vehicle are tenant + soft op-entity
  scoped). The matcher returns the best candidate even below threshold; the
  caller applies `LOADS_IMPORT_AUTO_THRESHOLD` to decide auto vs DRAFT routing.
- Inserts `loads` + `load_stops`. Insert errors (CHECK / FK / etc.) update the
  row to `validation_status='error'` and append the message to `error_messages`
  but do NOT roll back the whole batch.
- Stamps `loads.ai_metadata` with `{ source, batchId, sourceRowIndex,
  confidences, matchedOn, thresholdApplied }`.
- Sets `load_import_rows.resulting_load_id` and `validation_status='ok'`.

After the row loop, the batch row gets `status='committed'`,
`committed_at=NOW()`, and `result_summary={ created, duplicates, errors }`.
A second POST sees `status='committed'` and returns the cached summary with
`idempotent: true` — no re-insert.
