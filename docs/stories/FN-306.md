### FN-308 (subtask)
**Changes:**

Converted 14 raw `<select>` elements to `<app-ai-select>` across 3 settlement module files:

1. **scheduled-deductions.component.html** (10 selects converted):
   - Filter: Driver dropdown, Status/Enabled dropdown
   - Backfill: Driver dropdown
   - Modal: Rule scope, Driver ID, Payee ID, Amount type, Frequency, Applies when, Source/Expense type

2. **settlement-wizard.component.html** (3 selects converted):
   - Payroll period selector
   - Driver selector
   - Date basis selector

3. **equipment-owners.component.html** (1 select converted):
   - Settlement template type selector

**TypeScript changes:**
- `scheduled-deductions.component.ts`: Added `enabledFilterOptions` array, `driverOptions` and `payeeOptions` computed getters
- `settlement-wizard.component.ts`: Added `dateBasisOptions` array, `payrollPeriodOptions` and `driverOptions` computed getters
- `equipment-owners.component.ts`: Removed empty placeholder option from `settlementTemplateTypes` (now handled by `allowEmpty`/`emptyLabel`)

**Pattern used:**
- `(change)` events replaced with `(ngModelChange)` for ai-select compatibility
- Options arrays already in `{value, label}` format — no restructuring needed
- Dynamic driver/payee lists use computed `get` properties to transform API data to options format
- `[allowEmpty]="true"` with `emptyLabel` used for "All" / "Select..." placeholder options
