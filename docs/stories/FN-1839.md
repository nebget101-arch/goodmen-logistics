# FN-1839 — Field Placement Editor: PDF preview fails to load (R2 presigned-URL CORS)

**Type:** Bug · **Parent epic:** FN-1786 (Agreements / e-sign engine) · **Status:** In Progress
**Integration branch:** none (Shape A — standalone bug, branch `backend/FN-1839/r2-pdf-proxy` off `dev`)
**Agent:** backend (with a frontend touch to repoint the fetch)

## Problem

In the Field Placement Editor (`/agreements/:id/placement`) the document preview
never loads — the page shows *"Document preview unavailable…"* and DevTools logs:

> Failed to load resource: Origin `https://dev.fleetneuron.ai` is not allowed by
> Access-Control-Allow-Origin. Status code: 200.

## Root cause

`agreement-placement.component.ts` loaded the PDF **directly from the Cloudflare
R2 presigned URL** (`res.sourceDownloadUrl`) via `pdfjs.getDocument({ url })`.
pdf.js issues a cross-origin XHR to the R2 host, which has no CORS policy
allowing the app origin — R2 returns the bytes (200) but the missing
`Access-Control-Allow-Origin` header makes the browser reject the response. The
component then fell back to `previewUnavailable = true`, so box editing kept
working over a blank page but the page itself never rendered.

## Fix — auth-gated backend proxy (recommended option)

Stream the source bytes through the existing auth-gated API (which already
returns the right CORS headers via the gateway) instead of letting the browser
touch R2. **Important nuance:** the app's API runs on the gateway host (a
different origin than the frontend), and auth is a **bearer token attached by
Angular's `HttpInterceptor`**. pdf.js's internal fetch does *not* run through
HttpClient, so it would neither carry the token nor benefit from the
interceptor. Therefore the fix fetches the bytes with `HttpClient`
(`responseType: 'arraybuffer'`) and hands the `ArrayBuffer` to pdf.js via
`getDocument({ data })` — token attached, CORS-clean, no presigned URL in the
browser.

## Acceptance Criteria

- [x] Opening `/agreements/:id/placement` renders the actual PDF pages (no "preview unavailable" banner) for a template whose source is in R2.
- [x] No CORS error — the PDF is fetched through the auth-gated backend, not the R2 presigned URL.
- [x] Access is tenant-scoped/auth-gated — `getTemplateSource` resolves the storage key only for the caller's own template (`id` + `tenant_id`), so a user cannot fetch another tenant's source PDF; the route inherits `authMiddleware + tenantContextMiddleware + requireActiveSubscription`.
- [x] The blank-page fallback still works if the source genuinely can't be loaded (fetch error → `openPdf(null)` → `previewUnavailable`).

## Files Changed

**Backend (`goodmen-shared`, runs in logistics-service):**

- `storage/r2-storage.js` — new `getObjectStream(key)` returns `{ body, contentType, contentLength }` (AWS SDK v3 `GetObject`), streaming the object without buffering the whole file in memory.
- `services/agreement-service.js` — new `getTemplateSource({ templateId, tenantId, db })`: tenant-scoped lookup (`where { id, tenant_id }`) returning `{ storageKey, name }` or `null`. Exported.
- `routes/agreements.js` — new `GET /templates/:id/source`: resolves the key tenant-scoped (404 if missing), streams the bytes via `getObjectStream` with `Content-Type` (falls back to `application/pdf`), `Content-Disposition: inline`, and `Cache-Control: private, no-store`. Stream errors → 502 (or socket destroy if headers already sent).
- `services/agreement-service.test.js` — 4 new unit tests for `getTemplateSource` (happy path, `id`+`tenant_id` scoping, no-match → null, null storage key).

**Frontend:**

- `agreements/agreement.service.ts` — new `getTemplateSource(id): Observable<ArrayBuffer>` hitting the proxy with `responseType: 'arraybuffer'`.
- `agreements/agreement-placement/agreement-placement.component.ts` — `load()` now calls a new `loadSource()` which fetches bytes via the proxy; `openPdf()` takes `ArrayBuffer | null` and uses `getDocument({ data })` instead of `{ url }`. Fetch failure degrades to the blank-page editor.

`sourceDownloadUrl` is **still returned** by `GET /templates/:id` and is still used
by the "open original" `<a href target=_blank>` links in agreement-review and the
signer page — those are user-initiated navigations, not CORS-bound, so they are
left unchanged.

## Key Decisions

- **Fetch bytes via HttpClient, not point pdf.js at the URL.** pdf.js bypasses
  Angular's interceptor, so a `{ url }` pointed at the auth-gated route would be
  unauthenticated (401). `{ data: ArrayBuffer }` is the only approach that is
  both authenticated and CORS-clean.
- **Stream, don't buffer.** `getObjectStream` pipes `response.Body` straight to
  the HTTP response so a 25 MB source isn't held in service memory; the existing
  `downloadBuffer` (full read into a Buffer) was left for its current callers.
- **Did not drop `sourceDownloadUrl`** to avoid breaking the unrelated
  download-original links; the proxy simply supersedes it for the pdf.js preview.
