# FN-1787: Agreement upload + AI field & signature detection with role assignment

> Epic: **FN-1786** — Customer-uploaded agreements: AI field detection + in-house e-signature engine

## Requirement
An internal user uploads an agreement PDF (often a **scanned image — no text/AcroForm layer**, confirmed with the sample Motor Carrier Lease Agreement). **AI vision** detects every fillable field + signature block, classifies each field's type, and **suggests a fill-role** (`internal` = our staff vs `signer` = external customer/driver). The user reviews and **confirms/edits role assignment**, finalizing a reusable template (source doc in R2 + field map) consumed by the Sign engine (FN-1788) and the adapters (FN-1789/1790).

## Field-detection contract (fixed up front — AI/backend/frontend build in parallel)
`POST /api/ai/agreements/detect-fields` — `{ fileUrl }` or `{ base64, contentType }` → `200`
```json
{
  "documentType": "lease_agreement|generic",
  "pageCount": 6,
  "fields": [
    { "key": "lessee_name", "label": "Lessee Name", "type": "text",
      "page": 1, "bbox": [x, y, w, h], "suggestedRole": "internal",
      "suggestedValue": null, "confidence": 0.82 },
    { "key": "driver_signature", "label": "Driver Signature", "type": "signature",
      "page": 6, "bbox": [x, y, w, h], "suggestedRole": "signer", "confidence": 0.91 }
  ]
}
```
`type ∈ {text,date,number,checkbox,signature,initials}`, `role ∈ {internal,signer}`. Persisted shape: `agreement_templates` + `agreement_template_fields`, with the user's confirmed `role` overriding `suggestedRole`.

## Acceptance Criteria
- [ ] AI handler detects fields + signature blocks from scanned PDF/image; per-field confidence; hallucination-guarded (server re-validates type/role enums).
- [ ] Schema stores source-doc R2 key + ordered field map (page/bbox/type/role/confidence).
- [ ] Backend: upload endpoint (R2), persist template, invoke AI, CRUD field map/role assignments; tenant-scoped.
- [ ] Frontend: upload UI + field-mapping review (per-field role toggle internal↔signer); low-confidence fields flagged; confirm to finalize (`status=ready`).
