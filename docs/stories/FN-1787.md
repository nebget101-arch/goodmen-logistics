# FN-1787 — Agreement upload + AI field & signature detection with role assignment

**Epic:** FN-1786 — Customer-uploaded agreements: AI field detection + in-house e-signature engine
**Integration branch:** `integration/FN-1787` (subtasks branch off this; story PR merges to `dev`)

## Summary

First story of the epic: an internal user uploads an agreement PDF (often a scanned
image with no text/AcroForm layer). AI vision detects every fillable field and
signature block, classifies field type, and suggests a fill-role (`internal` vs
`signer`). The user reviews/edits role assignment and finalizes the template. The
finalized template (source doc in R2 + ordered field map) is reused by the Sign
engine (story 2) and the adapters.

## Field-detection contract (authoritative — AI + backend + frontend build against this)

`POST /api/ai/agreements/detect-fields` — body `{ fileUrl }` or `{ base64, contentType }` → `200`

```json
{
  "documentType": "lease_agreement|generic",
  "pageCount": 6,
  "fields": [
    { "key": "lessee_name", "label": "Lessee Name", "type": "text",
      "page": 1, "bbox": [x, y, w, h], "suggestedRole": "internal",
      "suggestedValue": null, "confidence": 0.82 },
    { "key": "driver_signature", "label": "Driver Signature", "type": "signature",
      "page": 6, "bbox": [x, y, w, h], "suggestedRole": "signer", "confidence": 0.91 }
  ]
}
```

- `documentType` ∈ `{ lease_agreement, generic }` (invalid → `generic`).
- `type` ∈ `{ text, date, number, checkbox, signature, initials }`.
- `suggestedRole` ∈ `{ internal, signer }`.
- `bbox` is `[x, y, w, h]` **normalized to 0..1** relative to the page (top-left origin).
- `suggestedValue` is a string only when the blank is pre-printed, else `null`.
- `confidence` ∈ `[0, 1]`.

Persisted template (`agreement_templates` + `agreement_template_fields`) mirrors this,
with the user's confirmed `role` overriding `suggestedRole`.

**Hallucination guard (server-side):** the AI handler re-validates `type` and `suggestedRole`
against the enums above. An invalid value is coerced to a safe default (`type → text`,
`suggestedRole → internal`) and that field's `confidence` is forced to `0` so the review UI
flags it. Invalid `documentType` → `generic`. `bbox` is sanitized to four numbers clamped to
`[0,1]`; malformed boxes become `[0,0,0,0]`.

## Implementation Notes

### FN-1791 (subtask) — AI field & signature detection handler

New AI-service handler `agreement-fields-handler.js` exposing `POST /api/ai/agreements/detect-fields`, registered in `ai-router.js`. Modeled on `invoice-extractor-handler.js` (the closest existing Claude Vision/Document handler).

- **Input**: accepts either `{ fileUrl }` (http/https URL → Anthropic `url` source) or `{ base64, contentType }`. Scanned PDFs are sent as a `document` block (Claude 4.x reads inline PDF, no pre-rendering); images go as `image` blocks. Supported content types: JPEG/PNG/WebP/GIF/HEIC/HEIF + `application/pdf`.
- **Prompt caching**: the system prompt is split into two `ephemeral` cache blocks — a persona block and a schema/rules block — so repeated agreement uploads within a session re-use the cached prefix (matches the invoice-extractor caching strategy). Usage (`cache_read_input_tokens`, etc.) is surfaced in `meta.usage`.
- **Size cap**: base64 payloads over `MAX_FILE_BYTES` (12 MiB, matching the gateway upload cap) are rejected `400 AI_FILE_TOO_LARGE` before any model call.
- **Retry-once-then-422**: a parse failure triggers one retry with a stricter "JSON only" reinforcement user message; a second failure returns `422 AI_PARSE_ERROR`. Upstream/model errors return `502 AI_UPSTREAM_ERROR`.
- **Hallucination guard** (`normalizeDetection`): re-validates each field's `type` and `suggestedRole` against the contract enums. Invalid `type` → `text`, invalid `suggestedRole` → `internal`, and the field's `confidence` is forced to `0`. Invalid `documentType` → `generic`. `bbox` is coerced to four `[0,1]`-clamped numbers (malformed → `[0,0,0,0]`). `confidence` is clamped to `[0,1]`. Field `key`s are slugified to `snake_case`, de-duplicated, and back-filled (`field_N`) when missing. `pageCount` falls back to the highest detected field page when the model under-reports.
- **Privacy logging**: `logAiInteraction` records counts only — `fields=N pages=N guarded=N size=bytes type=...`. Never logs document bytes, field labels, or extracted values.
- **Model**: default `claude-sonnet-4-20250514` (capable vision/bbox), overridable via `AI_AGREEMENT_MODEL` or `ANTHROPIC_VISION_MODEL`.

Response shape: `{ success, data: { documentType, pageCount, fields[] }, meta: { model, processingTimeMs, usage } }`.

**Files changed (FN-1791):**
- `backend/microservices/ai-service/src/handlers/agreement-fields-handler.js` (new) — handler, prompt builders, source-block builder, `normalizeDetection` hallucination guard, `validateBody`, exports.
- `backend/microservices/ai-service/src/handlers/__tests__/agreement-fields-handler.test.js` (new) — 17 standalone `node` assertions (no jest): guard coercion, bbox sanitation, confidence clamping, key de-dup, pageCount fallback, validateBody, happy path, markdown-fence stripping, retry-once-then-success, retry-then-422, upstream-502, bad-request-400, enum exports.
- `backend/microservices/ai-service/src/ai-router.js` — import + `router.post('/agreements/detect-fields', ...)` with full OpenAPI doc block.

**Key decisions (FN-1791):**
- **Followed `invoice-extractor-handler.js` over `psp-report-vision-handler.js`.** Both were cited in the ticket; the invoice extractor is the only existing handler that already implements all four required patterns (prompt caching, `fileUrl`/base64 source, size cap, retry-once-then-422), so reusing its structure keeps the AI service consistent.
- **Prompt kept inline in the handler (not split into `src/prompts/**`).** The ticket made the split optional; the gold-standard reference builds its cached blocks inline via `buildSystemBlocks()`.
- **Confidence forced to 0 on any guard hit.** A field with a hallucinated type/role is exactly the field a human must double-check, so zeroing confidence makes the frontend's low-confidence flag (FN-1794) fire on it automatically.
- **`pageCount` trusts the model only when ≥ the highest detected field page**, otherwise derives from the fields.
- **Default model `claude-sonnet-4-20250514`** rather than Haiku: agreement detection needs bbox localization and nuanced internal/signer reasoning across multi-page scans, where Sonnet's vision is materially stronger. Overridable via env for cost tuning.

### FN-1792 (subtask) — Database schema

Two additive, backward-compatible Knex migrations (style follows
`20260316013000_create_lease_to_own_financing.js` with `hasTable` guards):

- **`20260613120000_create_agreement_templates.js`**
  - `agreement_templates` — `id` (uuid pk), `tenant_id`, `operating_entity_id` (nullable),
    `name`, `document_type` (default `generic`), `source_storage_key` (R2 key),
    `page_count`, `status` (default `draft`; `draft|ready`), `created_by`
    (FK `users` SET NULL), `created_at`/`updated_at`.
    Indexes: `tenant_id`, `operating_entity_id`, `status`, `document_type`.
  - `agreement_template_fields` — `id` (uuid pk), `template_id` (FK
    `agreement_templates` CASCADE), `field_key`, `label`, `field_type`
    (default `text`; `text|date|number|checkbox|signature|initials`), `page`,
    `bbox` (jsonb `[x,y,w,h]`), `role` (default `signer`; `internal|signer`),
    `suggested_role`, `suggested_value`, `confidence` (decimal 5,4), `sort_order`,
    timestamps.
    Indexes: `template_id`, `role`, composite `(template_id, page, sort_order)`.
- **`20260613120100_add_agreement_permissions.js`** — RBAC grants
  (mirrors `add_lease_financing_permissions.js`): `agreement.templates.{view,create,edit,delete}`
  in module `agreement_templates`, assigned to `super_admin`, `admin`, `company_admin`,
  `finance_manager` (full) and `accounting`, `carrier_accountant` (view-only).
  Idempotent upserts; guarded by `hasTable` on `permissions`/`roles`/`role_permissions`.

- **Services to restart/redeploy:** `goodmen-ai-service` (mounts `ai-router.js`).
- **DB migrations:** none in this subtask (schema is FN-1792).
- **Env / config changes:** optional `AI_AGREEMENT_MODEL` (defaults to `claude-sonnet-4-20250514`,
  then falls back to `ANTHROPIC_VISION_MODEL`). Uses the existing `ANTHROPIC_API_KEY` — no new secret.
- **Feature flags / cache:** none.
- **Deploy order:** independent; endpoint is additive (`POST /api/ai/agreements/detect-fields`).
- **Post-deploy verification:** `POST /api/ai/agreements/detect-fields` with `{ fileUrl }` to a sample
  scanned lease → expect `200` with `data.fields[]` including at least one `signature` field, each
  carrying `type`, `suggestedRole`, `bbox` (4 numbers in [0,1]), and `confidence`. Confirm
  `meta.usage` reports cache read tokens on a second call.

### FN-1792 (database)
- **DB migrations to run** (on the database backing the API services):
  - `20260613120000_create_agreement_templates.js`
  - `20260613120100_add_agreement_permissions.js`
  - Run via the standard Knex migrate step (`knex migrate:latest`) during deploy of the
    backend/AI services that own the agreement feature.
- **Services to restart/redeploy:** none required for the schema alone; the migrations run
  as part of the normal release.
- **Env/config changes:** none.
- **Cache invalidation:** new RBAC permissions take effect on next permission load; if roles are
  cached, clear the permissions cache after migrating.
- **Deploy order:** run migrations before deploying backend code that queries the new tables.
- **Post-deploy verification:**
  - `\d agreement_templates` and `\d agreement_template_fields` show the expected columns/indexes.
  - `SELECT code FROM permissions WHERE module = 'agreement_templates';` returns the 4 codes.
  - Migration is reversible (`down` drops fields then templates; permission codes removed).
