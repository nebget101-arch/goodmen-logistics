# FN-1325: AI Briefing + Predictive Insights empty-state cleanup

## Requirement
Hide AI Briefing and Predictive Insights cards when they have no baseline data; when partial, replace generic "Not enough data yet" copy with first-baseline-by-{date} messaging.

## Acceptance Criteria
- [ ] Daily Briefing card hidden when briefing-generator returns "no baseline".
- [ ] Predictive Insights card hidden when no insights have meaningful data.
- [ ] When partial, empty-state strings are replaced with "First baseline ready by {ISO date}".
- [ ] Per-user dismissed-cards flag persists through FN-1130 layout system.
- [ ] "Show hidden cards" affordance available in the layout-reset menu.

### FN-1337 (subtask)
- Added `hasBaseline` / `firstBaselineEta` optional fields to `DailyBriefingResponse` and `TrendsResponse` (backward-compatible — server can omit and existing behavior is preserved).
- `DailyBriefingComponent` and `PredictiveInsightsComponent` expose a `(visibilityChange)` `EventEmitter<{hasBaseline, firstBaselineEta}>` fired on every successful fetch. Predictive Insights derives `hasBaseline` from the response when the field is absent (`hasBaseline = any series has any non-null actual/predicted point`).
- `DashboardLayoutService` now PUTs `{ cards, hidden }` and reads `data.layout.hidden` from GET; `DashboardLayout.hidden: WidgetId[]` is plumbed through.
- `ControlCenterComponent` listens to children's `(visibilityChange)`; when `hasBaseline=false`, the widget id is appended to `hidden` and persisted (no flash on reload). When `hasBaseline=true` and the id was previously hidden, it is restored.
- Header gains a "Show hidden cards (n)" affordance next to "Reset to default" — toggles a session-only override that surfaces dismissed cards (with a dashed outline). "Reset to default" clears `hidden` server-side via the existing DELETE.
- Predictive Insights per-cell empty copy switches to "First baseline ready by {ISO date}" when `firstBaselineEta` is set, otherwise keeps the existing "Not enough data yet — collecting baseline." Daily Briefing shows a top-level "First baseline ready by {ISO date}" panel when `hasBaseline=false && firstBaselineEta` (only visible when the user surfaces the hidden card).
- `onDrop` was rewritten to translate visible-list indices back to the full `widgets` order so reordering still works correctly when some cards are hidden.

## Implementation Summary
- Empty-baseline AI cards (Daily Briefing + Predictive Insights) collapse out of the Control Center grid and persist that state through the FN-1130 dashboard-layout API as a per-user `hidden: WidgetId[]` array.
- Partial-baseline empty states are replaced with "First baseline ready by {ISO date}" when the AI service supplies an ETA.
- Users can surface hidden cards via "Show hidden cards (n)" in the header; "Reset to default" restores the server default.

## Key Decisions
- **Server tolerates missing `hidden`.** `LayoutEnvelope.data.layout.hidden` is optional on both read and write; older clients PUT `{ cards }` only and the server should treat absence as `[]`. Forward-compatible with the existing `auth-users-service` route.
- **Visibility is component-driven, not service-driven.** Each card decides whether it has a baseline (using the response shape it already understands). The control-center is a dumb listener — avoids leaking AI-service shape into the layout service.
- **`hasBaseline` derivation falls back to "true" for back-compat.** Daily Briefing assumes `hasBaseline=true` unless the server explicitly says `false`. Predictive Insights derives from data when the field is absent.
- **Persisted hide is sticky across reloads but defeated by Reset.** Matches the AC ("dismissed-cards flag persists through layout system") while keeping a clear escape hatch.
