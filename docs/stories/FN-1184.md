## Implementation Summary (FN-1215)

Implemented the Claude-backed triage service for roadside incidents in `backend/microservices/ai-service/`:

### Files Added

- **`src/triage/triage.service.js`** — Core service: calls Anthropic Claude with ephemeral prompt caching on system + policy blocks. Validates structured output (severity, serviceCategory, urgency, vendorSkills, rationale, safetyRisk). Returns audit fields (prompt_version, model_name, cache_read_tokens, cache_creation_tokens, latency_ms).
- **`src/triage/triage.controller.js`** — Express HTTP handler for `POST /api/ai/roadside/triage`. Validates tenantId + description, passes vehicleType/location through.
- **`src/routes/triage.routes.js`** — Sub-router mounted at `/api/ai/roadside/triage` via ai-router.js.
- **`src/triage/redactor.js`** — PII scrubber: strips phone numbers, email addresses, SSNs, credit card numbers, and named driver references before the model call.
- **`src/telemetry/triage.telemetry.js`** — Telemetry wrappers: emitSuccess (logs cache hit/miss, latency, model) and emitFailure (logs error code).
- **`src/triage/prompts/triage.system.md`** — System prompt: defines triage fields, valid enum values, and classification rules. Cached with `cache_control: ephemeral`.
- **`src/triage/prompts/triage.policy.md`** — Policy block: escalation rules (fire/injury/hazmat → CRITICAL), category disambiguation heuristics, vendor skill defaults. Cached with `cache_control: ephemeral`.
- **`src/handlers/__tests__/triage/triage.service.spec.js`** — 10 unit specs covering parseAndValidate, buildUserMessage, and triageIncident with mocked Anthropic client.
- **`src/handlers/__tests__/triage/triage.controller.spec.js`** — 7 unit specs covering 400 validation, 200 happy path, 502 parse error, 502 upstream error, whitespace trimming, optional field forwarding.

### Modified

- **`src/ai-router.js`** — Added `router.use('/roadside/triage', buildTriageRouter(deps))`.

### Key Decisions

- Prompt caching applied to two static blocks (system + policy), keeping per-request data (incident description + context) in the uncached user message only.
- PII scrubbing happens before the model call using regex patterns; redaction counts are included in the triage record for audit.
- Temperature 0 for deterministic classification.
- Model configurable via `ANTHROPIC_TRIAGE_MODEL` env var; defaults to `claude-sonnet-4-6`.
