# FN-1195: Roadside v2 Story 3.1 — Image Upload + Tenant-Scoped Storage

## Requirement

As a driver, I want to upload damage photos that are stored securely and associated with my incident.

## Acceptance Criteria

- [ ] `POST /api/incidents/:id/images` accepts multipart upload with size (≤10 MB) + format (jpg/png/heic) validation
- [ ] Images stored in R2 under tenant-scoped prefix `tenants/{tenantId}/incidents/{incidentId}/`
- [ ] Pre-signed `GET` URLs returned for retrieval
- [ ] Metadata row: `id`, `incident_id`, `tenant_id`, `s3_key`, `mime_type`, `size_bytes`, `uploaded_by`, `uploaded_at`
- [ ] `GET /api/incidents/:id/images` returns all images with signed URLs
- [ ] `GET /api/incidents/:id/images/:imageId` returns single image with signed URL
- [ ] Telemetry: upload count, average size, rejection reasons

- Uploading a GIF returns 400 with `unsupported_format` reason

## Implementation Summary

### FN-1231 (subtask — backend)

**Service** (`backend/packages/goodmen-shared/services/incident-images.service.js`):
- `uploadImage(incidentId, file, uploadedBy, context)` — validates size/MIME, writes to R2 under `tenants/{tenantId}/incidents/{incidentId}/`, inserts metadata row into `incident_images`, emits telemetry.
- `listImages(incidentId, context)` — fetches all `incident_images` rows for the incident (tenant-scoped) and decorates each with a short-lived signed GET URL.
- `getImage(incidentId, imageId, context)` — single image retrieval with signed URL.
- Uses existing `backend/packages/goodmen-shared/storage/r2-storage.js` (`uploadBuffer`, `getSignedDownloadUrl`).
- R2 key format: `tenants/{tenantId}/incidents/{incidentId}/{ISO-timestamp}-{originalName}`

**Route** (`backend/packages/goodmen-shared/routes/incident-images.js`):
- `POST /:id/images` — multer `memoryStorage`, 10 MB limit, MIME filter (jpg/png/heic), delegates to `incidentImagesService.uploadImage`.
- `GET /:id/images` — delegates to `incidentImagesService.listImages`.
- `GET /:id/images/:imageId` — delegates to `incidentImagesService.getImage`.
- Mounted at `/api/incidents` in `drivers-compliance-service/server.js` alongside `incidentsTriageRouter`.

**Tests** (`backend/packages/goodmen-shared/services/incident-images.service.test.js`):
- 10 hermetic tests covering: size rejection, MIME rejection, missing incident 404, successful jpg/png/heic upload, list, get, missing image 404.
- All pass: `node --test services/incident-images.service.test.js` → 10/10.

### FN-1232 (subtask — database)

**Migration** (`backend/packages/goodmen-database/migrations/20260611120000_create_incident_images.js`):
- Creates `incident_images` table with UUID PK (uuid_generate_v4), logical cross-service FKs for `incident_id` and `tenant_id`, R2 metadata columns (`s3_key`, `mime_type`, `size_bytes`, `original_file_name`), nullable `uploaded_by` UUID, and `uploaded_at` timestamptz defaulting to now().
- Indexes: `idx_incident_images_tenant_incident (tenant_id, incident_id)` — primary list/scope query; `idx_incident_images_incident_id (incident_id)` — admin/global fallback.
- Idempotent: guarded with `hasTable` check.
- Integrated into `integration/FN-1195` via ff-merge.
