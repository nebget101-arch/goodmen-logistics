# FN-545: loads-dashboard: selecting a driver does not auto-fill truck/trailer in edit or create load modal

## Problem
In the Load Edit and Create Load modals rendered by `loads-dashboard.component`, selecting a driver did not auto-populate the Truck and Trailer dropdowns. FN-538 implemented auto-fill in `loads.component.ts` but the production dashboard uses `loads-dashboard.component.ts`, which had no such logic.

- **Create modal** — `selectInlineDriver()` only patched `driverId`; no truck/trailer fill.
- **Edit modal** — `populateFormFromDetail()` set `truckId: detail.truck_id` which is blank for loads that were never assigned equipment; no driver fallback.

## Fix

### `loads.service.ts`
Extended `DriverOption` interface with optional `truckId` and `trailerId` fields so the API response values flow through the type system.

### `loads-dashboard.component.ts`

**1. `drivers` array type** — extended to include `truckId`/`trailerId`.

**2. `loadDropdownData()` driver mapping** — added `truckId: driver.truckId || null` and `trailerId: driver.trailerId || null` to the mapped objects. The `GET /api/drivers` list returns `truckId`/`trailerId` UUIDs via `SELECT d.*` + `transformRow()` (confirmed in FN-539).

**3. `selectInlineDriver()`** — after patching `driverId`, looks up the driver in `this.drivers` and patches `truckId`/`trailerId` if the driver has assigned equipment. Also syncs `truckSearch`/`trailerSearch` labels so the inline search inputs show the correct unit label.

**4. `populateFormFromDetail()`** — `truckId` and `trailerId` now fall back to the driver's cached assignment when the load's own `truck_id`/`trailer_id` are null. After the form reset, a label-sync block updates `driverSearch`, `truckSearch`, and `trailerSearch` from the resolved IDs.

## Acceptance Criteria
- [x] Selecting a driver in the Create Load modal auto-fills Truck and Trailer dropdowns
- [x] Selecting a driver in the Edit Load modal auto-fills Truck and Trailer dropdowns
- [x] Opening the Edit Load modal for a load with no truck assigned auto-fills from the driver's current dispatch assignment
- [x] If the driver has no assigned truck/trailer, dropdowns remain blank (no error)
