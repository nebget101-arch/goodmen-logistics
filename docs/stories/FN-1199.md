#### Key Decisions

- **RealtimeService** is a new `providedIn: 'root'` service that wraps `WebsocketService.on()` and exposes `incidentTimeline$(incidentId)`. This keeps incident-specific event names co-located with the service and avoids coupling components to raw WS event strings.
- **StatusTimelineComponent** is `standalone: true` + `OnPush` so it can be imported directly into any feature module or standalone component without re-declaring in `SharedModule`.
- **Deduplication by id**: a `Set<string>` of seen event IDs prevents double-renders when an HTTP-preloaded event is also broadcast over WS.
- **Vendor-ETA placeholder**: always shown until a `vendor_eta_confirmed` event arrives ‚Äî satisfies the Story 4.3 soft-dependency requirement without blocking this story.
- **Event types** (`INCIDENT_WS_EVENTS`): `incident:status_changed` and `incident:timeline_event`. Backend (Story 4.1) broadcasts both to the tenant room; clients filter by `incident_id` in the payload.
