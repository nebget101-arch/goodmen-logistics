# FN-1788 — Fill fields, send secure e-sign link, capture signature, generate signed PDF

Story 2 of the agreements/e-sign epic. Builds on the agreement templates + field map from
story 1 (FN-1787 / FN-1792 et al). Internal user fills `internal`-assigned fields, system sends
a secure tokenized e-sign link to the signer, signer fills their fields + applies an in-house
signature (typed name + IP/UA capture), system generates a signed PDF stored in R2 and
transitions the request to `signed`/`completed`.

**Integration branch:** `integration/FN-1788` (created from `origin/dev`; subtasks branch off it; story PR → `dev`).

### FN-1796 (subtask) — Database schema

Two additive, `hasTable`-guarded Knex migrations under
`backend/packages/goodmen-database/migrations/`, timestamped after the FN-1792
`agreement_templates` migration so the FK target exists when migrations run in order:

- `20260613130000_create_signature_requests.js`
- `20260613130100_add_signature_permissions.js`

**Tables**

- **`signature_requests`** — one row per agreement sent for signature.
  `id`, `tenant_id` (notNull), `operating_entity_id` (null), `template_id` → `agreement_templates`
  (`SET NULL`, FK attached only when the table exists), `document_type` (default `generic`),
  `status` (default `draft`; `draft|sent|viewed|signed|completed|expired|voided`), `signer_name`,
  `signer_email`, `signer_phone`, `signer_role` (label), `signed_pdf_storage_key`, `sent_at`,
  `viewed_at`, `signed_at`, `expires_at`, `created_by` → `users` (`SET NULL`), `timestamps`.
  Indexes: `tenant_id`, `operating_entity_id`, `template_id`, `status`.
- **`signature_request_fields`** — per-field values (internal-filled + signer-filled).
  `id`, `request_id` → `signature_requests` (`CASCADE`), `field_key`, `role` (default `signer`;
  `internal|signer`), `value` (text / JSON-encoded), `filled_by` → `users` (`SET NULL`),
  `filled_at`, `timestamps`. Unique `(request_id, field_key)`. Indexes: `request_id`, `role`.
- **`signatures`** — captured signature + tokenized signing-link credential, mirroring
  `employer_investigation_tokens` / `consent_records`. `id`, `request_id` → `signature_requests`
  (`CASCADE`), `token_hash` (varchar(64), unique), `expires_at`, `signer_name`, `signature_value`,
  `ip_address`, `user_agent`, `signed_at`, `consent_text_snapshot`, `timestamps`. Indexes:
  `request_id`, `token_hash`.

**RBAC** — `agreement.signatures.{view,create,void}` permissions (module `agreement_signatures`),
granted to `super_admin`, `admin`, `company_admin`, `finance_manager` (full) and
`accounting`, `carrier_accountant` (view). Mirrors `add_agreement_permissions.js`. Public signer
routes are token-gated (no RBAC). Upsert-style and reversible.

**Key decisions**
- `template_id` FK is conditionally attached (guarded by `hasTable('agreement_templates')`) and
  uses `SET NULL` so the schema applies cleanly even if story 1 hasn't merged to this branch yet,
  and signed requests survive template deletion.
- `token_hash`/`expires_at`/IP/UA live on `signatures` to reuse `token-service` hashing/expiry
  semantics (mirror of `employer_investigation_tokens`); the canonical lifecycle `status` lives on
  `signature_requests`.
- `value` is `text` (JSON-encode structured field values in the backend) to match the
  `agreement_template_fields.suggested_value` convention.

### FN-1797 (subtask) — Backend (fill, send, public sign, signed PDF)

Branch: `backend/FN-1797/signature-send-sign`. All routes live in `goodmen-shared` and are
served by `logistics-service`; the gateway proxies both the authed `/api/agreements/*` and the
public `/public/agreements/*` paths to it.

**Files changed**

- `backend/packages/goodmen-shared/services/signature-service.js` (new) — full lifecycle
  orchestration: `createSignatureRequest`, `getRequestById`, `getSignerView`, `submitSignature`,
  `generateAndStoreSignedPdf`, `sendSignerLink`, `resolveToken`, plus DB-free helpers
  (token expiry, field-value encode/decode, signer-link, placement merge, row mappers).
- `backend/packages/goodmen-shared/routes/agreements.js` — added internal
  `POST /api/agreements/:templateId/requests` (fill internal fields, mint+send tokenized link,
  → `sent`) and `GET /api/agreements/requests/:id` (status + signed-PDF download URL).
- `backend/packages/goodmen-shared/routes/public-agreements.js` (new) — token-gated,
  rate-limited `GET|POST /public/agreements/sign/:token` (mirrors `public-employer-investigations.js`).
- `backend/packages/goodmen-shared/services/pdf.service.js` — added `overlaySignedAgreementPdf`
  (overlays field values + signature onto the source PDF via `pdf-lib`, appends a signature
  certificate / audit page; resilient to missing/garbage source bytes).
- `backend/microservices/logistics-service/server.js` — require + mount `publicAgreementsRouter`
  at `/public/agreements` (no auth, ahead of the authed groups).
- `backend/gateway/index.js` — proxy `/public/agreements` → logistics.
- `backend/packages/goodmen-shared/test/signature-service.test.js` (new) — 25 unit tests
  (pure helpers + signed-PDF overlay), all green under Node 20 `node --test`.

**Contract delivered** (matches the story contract the FE builds against)

- `POST /api/agreements/:templateId/requests` → `{ requestId, signerLink, token, status, send }`.
  Body `{ fieldValues: { fieldKey: value }, signer: { name, email, phone, role }, expiresInDays? }`.
  Requires a signer email or phone. Snapshots the template field map into
  `signature_request_fields` (internal values filled now; signer fields empty), creates a
  `signatures` token row, sends via Twilio/SendGrid, advances to `sent` only when a channel delivers.
- `GET /api/agreements/requests/:id` → request + fields + `signedPdfUrl` (signed R2 URL when present).
- `GET /public/agreements/sign/:token` → `{ status, document, consentText, fields, expiresAt }`;
  internal field values are `readOnly: true`, signer fields editable. First open flips `sent → viewed`.
- `POST /public/agreements/sign/:token` → body `{ fieldValues, signerName, signatureValue, consent }`
  → validates token (unused/unexpired/not-voided), persists only signer-role field values, records
  the `signatures` row (name/IP/UA/timestamp + consent snapshot), generates the signed PDF, stores it
  in R2 (`agreements/signed/<tenant>/<request>-<ts>.pdf`), → `signed`, returns `{ status, signedPdfUrl }`.
  **Idempotent**: a second submit on an already-signed/expired/voided request returns the existing
  signed state without re-generating.

**Key decisions (backend)**

- Token credential reuses `token-service` (sha256 hash + timing-safe compare) and lives on the
  `signatures` row (`token_hash`, `expires_at`), mirroring `employer_investigation_tokens`. The
  canonical lifecycle `status` lives on `signature_requests`.
- Signer can never overwrite `internal`-role fields — the POST only updates fields whose row role
  is `signer` (server-side allowlist), independent of what the client sends.
- Signed-PDF generation is **non-fatal**: the signature is committed first; a PDF/R2 failure leaves
  the request `signed` with no `signed_pdf_storage_key` (retryable) rather than failing the signing.
- bbox/page come from the template field map at sign time (joined via `template_id`); when the
  template is gone (`SET NULL`) the overlay still emits a certificate page listing every value, so
  nothing is lost.
- Per-route RBAC is intentionally not added on the internal endpoints (parity with the existing
  `agreements.js` routes, which rely on the `/api/agreements` auth+tenant mount); the
  `agreement.signatures.*` permissions from FN-1796 are available for FE menu gating / future use.

### FN-1798 (subtask) — Frontend (internal fill/send + public signer + signature capture)

Branch: `frontend/FN-1798/internal-fill-public-sign`. Builds on the FN-1787 `agreements` lazy
module (template + field map), reusing its `AgreementField` / `AgreementTemplateDetail` DTOs
rather than duplicating types. Built green against the FN-1797 contract (`ng build`, Node 20;
both lazy chunks emit).

**Files changed**

- `frontend/src/app/agreements/signature-request.model.ts` (new) — contract DTOs
  (`CreateSignatureRequestPayload`, `SignatureRequest`, `SignerView`, `SignerField`,
  `SignSubmission`, `SignResult`) + helpers (`isSignable`/`isComplete`/`statusLabel`,
  `DEFAULT_CONSENT_TEXT`).
- `frontend/src/app/agreements/agreement.service.ts` — added `createRequest(templateId, payload)`
  (`POST /api/agreements/:templateId/requests`) and `getRequest(id)`
  (`GET /api/agreements/requests/:id`).
- `frontend/src/app/agreements/agreement-send/**` (new) — internal **fill & send** screen
  (route `agreements/:id/send`, `AuthGuard`). Renders `internal`-role fields as a form (seeded
  from AI `suggestedValue`), captures signer contact (name + email **or** phone), POSTs the
  request, then shows the signer link (copy) + live status pill (`Refresh status`) and the
  signed-PDF download once available. Blocks send on a still-`draft` template.
- `frontend/src/app/agreements/agreement-review/agreement-review.component.html` — added a
  "Fill & send for signature" button shown once the template is `ready`.
- `frontend/src/app/public-sign/**` (new lazy module) — **public, unauthenticated** signer flow.
  - `signer-page` — renders document details, internal values read-only, the signer's editable
    fields, the signature capture, consent, submit → confirmation w/ signed-PDF download.
    Graceful loading / error / missing-token / **expired** / **voided** / **already-signed**
    (idempotent) states. Self-contained light theme (no `SharedModule`).
  - `signature-capture` — reusable tabbed **Type** (typed legal name = signature, default) /
    **Draw** (`<canvas>`, pointer + touch → PNG data-URL) e-signature; emits `{ value, type }`.
  - `public-sign.service` — token `GET`/`POST /public/agreements/sign/:token` (strips the
    `/api` suffix like the consent / employer-investigation public clients).
- `frontend/src/app/app-routing.module.ts` — mounted the public module at **`agreements/sign`**
  (child `:token`), declared **before** the authed `agreements` lazy route so
  `/agreements/sign/<token>` resolves to the public page.
- Specs: `signer-page.component.spec.ts`, `signature-capture.component.spec.ts`.

**Contract alignment with FN-1797** — the public route is `agreements/sign/:token` to match the
backend-generated link `<PUBLIC_APP_URL>/agreements/sign/<token>` (the ticket's `/sign/:token`
was only an example). GET consumes `{ document, fields (internal `value` read-only), status,
consentText, expiresAt }`; POST sends `{ fieldValues (signer-role only), signerName,
signatureValue, signatureType, consent }` and reads `{ status, signedPdfUrl }`. Signed-PDF
generation is non-fatal backend-side, so the confirmation tolerates a missing `signedPdfUrl`.

**Key decisions (frontend)**
- The field map has no per-field `required` flag, so every placed field except a checkbox
  (unchecked = a valid answer) is treated as required for submit gating.
- `signature`/`initials` fields are excluded from the editable forms — they're fulfilled by the
  signature-capture component / backend overlay, not a text input.
- Touch drawing uses feature detection (`ev.touches`) rather than `instanceof TouchEvent`
  (undefined on some desktop browsers).

  - **Post-deploy (UI):** finalize a template → "Fill & send" → open the `signerLink` in a private
