# FN-1355: Ask Neuron FAB — collision-aware positioning, minimized state, and safe-area contract

## Requirement
The Ask Neuron floating chat button overlaps the Loads table pagination row in the user's screenshot (2026-05-05). Same risk applies anywhere a sticky bottom-right control exists (drawer footers, paginators, mobile bottom nav). Add a minimized state and a `--neuron-fab-safe-bottom` CSS-variable contract so the FAB never covers page content.

## Implementation Summary

The FAB launcher lives in `frontend/src/app/app.component.html` (`.ai-chat-launcher` button, styles in global `frontend/src/styles.css`) — there is no `components/ai-chat/**` directory. The implementation introduces:

1. **`AiChatLauncherService`** holds the launcher display state (`'default' | 'minimized'`) with `BehaviorSubject` and persists to `localStorage` under key `neuronFabState`. On first load with no saved preference, it picks `'minimized'` when `window.matchMedia('(max-width: 639px)')` matches, else `'default'`.
2. **`FabSafeAreaService`** is a registry for active "safe bottom" claims. Each consumer registers a pixel value; the largest active claim is published as `--neuron-fab-safe-bottom` on `document.documentElement`. When all claims are released, the var is removed (FAB falls back to default `0px`). Composing via max keeps simultaneous claims safe (paginator + drawer footer at the same time).
3. **`FabSafeAreaDirective`** (`appFabSafeArea="64"`) is a standalone attribute directive that registers a claim on attach and releases on destroy. Exported through `SharedModule` so any component already importing `SharedModule` can use it.
4. **Launcher markup** in `app.component.html` is now a `*ngIf` between two presentations:
   - **Pill (default)**: a `.ai-chat-launcher--pill` wrapper containing a main button (opens chat) and a separate × button (`.ai-chat-launcher__minimize`) that calls `minimizeAiLauncher()`. The × stops propagation so it doesn't open the chat.
   - **Orb (minimized)**: a single circular `.ai-chat-launcher--orb` button that calls `expandAiLauncher()` to switch back to pill state. Tapping the orb does not open the chat — that matches the spec ("click to expand on the orb").
5. **CSS contract** in `styles.css`: `:root { --neuron-fab-safe-bottom: 0px; }` plus `.ai-chat-launcher { bottom: calc(var(--neuron-fab-safe-bottom, 0px) + 26px); }` (and 20px on mobile, 76–82px for the chat panel). A code comment near the var declaration documents the contract.
6. **Consumers wired**: `appFabSafeArea="64"` on `.pagination` in `loads-dashboard.component.html` and on `.load-drawer__footer` in `load-detail-drawer.component.html`. Both reserve 64px so the FAB pill (~42px tall) and orb (48px) sit above the obstructing element with comfortable gutter.
7. **Unit specs** cover service persistence (default desktop, mobile default, junk-value fallback, toggle, persistence side effects), the safe-area service (single claim, max across multiple, release/recompute, negative clamp), and the directive (publish on attach, update on input change, parse string values, clear on destroy).

## Key Decisions

- **CSS variable propagation strategy.** CSS custom properties cascade only to descendants, but the FAB and the paginator are siblings (both inside `<main>`). To make the contract work without coupling the FAB to the paginator's DOM position, the directive writes the var on `document.documentElement`. The registry (max of all active claims) keeps overlapping claims sane.
- **Pill structure: nested buttons.** The pill is a `<div>` with two `<button>` children rather than a single button with an inner span, so each click target is a separate semantic button with its own ARIA label. The × button calls `event.stopPropagation()` and `event.preventDefault()` so a click on × never falls through to "Open Ask Neuron".
- **Orb tap = expand, not open chat.** The story's "click to expand on the orb" is taken literally. A tap on the orb switches the launcher state to `default` (pill); the user then taps the pill to open the chat. Two taps on mobile is acceptable because the user opted into the minimized state and that affordance reads more naturally than "tapping the orb teleports past the pill state".
- **64px claim default.** The pill is 42px tall + 26px gutter ≈ 68px from the viewport bottom; the orb is 48px + 20–26px gutter ≈ 68–74px. A 64px safe-bottom claim moves the FAB so its bottom edge clears any 64px-tall sticky element with a tiny 4–10px breathing margin. If a future consumer is taller, it can pass its own number.
- **Module registration via `SharedModule`.** `LoadsDashboardModule` and `AppModule` already import `SharedModule`, so adding `FabSafeAreaDirective` to `SharedModule.exports` makes both consumers work with no further plumbing. The directive is `standalone: true` so it can also be imported directly by future standalone components.

### `--neuron-fab-safe-bottom` contract (for follow-up authors)
- **Declaration:** `:root { --neuron-fab-safe-bottom: 0px; }` in `frontend/src/styles.css`.
- **Consumer pattern:** add `appFabSafeArea="<px>"` to any sticky bottom element. The largest active claim wins. Direct CSS writes (`element.style.setProperty('--neuron-fab-safe-bottom', '...')`) are also valid for advanced cases but bypass the max-of-claims registry.
- **Already wired:** `.pagination` in loads-dashboard, `.load-drawer__footer` in load-detail-drawer.

### Sticky-element follow-ups (deferred — out of scope this story)
Per the story's "audit drawer footer + sticky action bars" step, these candidates were spotted but **not** wired (none cause the regression in the original screenshot, and wiring all of them risks scope creep):
- Modal action footers (`.onboarding-modal-footer`, `.modal-footer` variants in load wizard / mileage / etc.) — modals are typically centered overlays that already cover the FAB visually with a backdrop, so the contract is moot. Deferred unless a specific overlap is reported.
- Mobile bottom nav (none currently exists in the app shell). Add when introduced.
- Future sticky save bars on long edit forms. Add `appFabSafeArea` when those land.
