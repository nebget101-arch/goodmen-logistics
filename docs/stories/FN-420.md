# FN-420: AI-Powered Toll Invoice Image Extraction

## Overview
Allow fleet managers to upload toll invoice images and have AI extract structured transaction data using Claude Vision.

## Subtasks

| Key | Summary | Agent | Branch | Status |
|-----|---------|-------|--------|--------|
| FN-440 | AI: Claude Vision toll invoice extraction handler | ai | `agent/ai/FN-440/invoice-vision-handler` | Done |
| FN-441 | BE: POST /import/invoice-image endpoint | backend | `agent/backend/FN-441/invoice-image-endpoint` | Done |
| FN-442 | FE: Invoice upload button and preview dialog | frontend | `frontend/FN-442/invoice-upload-ui` | Done |
| FN-443 | QA: Validate AI invoice extraction | qa | _manual_ | Done |

### FN-440 — AI Vision handler
- Created `toll-invoice-vision-handler.js` with Claude Vision API integration
- Extracts invoice metadata + array of toll transactions from images
- Handles common formats: EZPass, SunPass, FasTrak, Illinois Tollway, etc.

### FN-441 — Backend endpoint
- POST /import/invoice-image with multer (up to 10 images, 10MB each)
- Forwards to AI service, auto-matches plates to devices, checks duplicates

### FN-442 — Frontend upload UI
- Upload button + InvoicePreviewDialogComponent with editable extraction table
- Warning badges for unmatched plates/duplicates, batch confirm

## Deployment Handoff
- **Services**: Frontend + Backend (goodmen-shared) + AI Service
- **Migrations**: None
- **Env/config**: AI_SERVICE_URL must be configured
- **Post-deploy verification**: Upload a toll invoice image, verify AI extraction and preview dialog
