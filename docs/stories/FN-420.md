### FN-441 (subtask) — Backend: POST /import/invoice-image endpoint

**Changes:**
- Added `multer` import and `invoiceUpload` middleware (memory storage, 10MB limit, JPG/PNG/PDF/WebP)
- New `POST /import/invoice-image` endpoint that:
  - Accepts up to 10 image files via `images` field
  - Converts each file to base64 and forwards to AI service (`/api/ai/tolls/invoice-vision`)
  - Auto-matches extracted license plate to `toll_devices.plate_number` → resolves `truck_id`, `driver_id`
  - Checks for duplicate transactions (same date + amount + plate)
  - Returns enriched transactions array with match status and duplicate flags per image
  - 30s timeout for AI service calls
- Error handling: AI timeout, AI unreachable, AI extraction failure — each image processed independently

**Files Changed:**
- `backend/packages/goodmen-shared/routes/tolls.js` — added multer, invoiceUpload middleware, POST /import/invoice-image

## Deployment Handoff

### Services to redeploy
- `goodmen-shared` (backend)

### DB migrations
- None

### Environment/config changes
- `AI_SERVICE_URL` must be set (defaults to `http://localhost:4100`)

### Deploy order
1. AI service first (FN-440 provides the vision handler)
2. Backend second (this endpoint calls the AI service)

### Post-deploy verification
1. POST multipart form to `/api/tolls/import/invoice-image` with a toll invoice image
2. Verify response contains extracted transactions with plate matching
3. Verify duplicate detection flags existing transactions
