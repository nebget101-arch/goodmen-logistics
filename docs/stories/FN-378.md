### FN-379 (subtask)
**Branch**: `backend/FN-379/fix-dashboard-stats-sql`

**Implementation Summary**:
- Replaced the monolithic SELECT with ~20 subqueries with 6 isolated query groups: drivers, vehicles, loads, billing, HOS, compliance
- Each group runs as a separate SQL query using `Promise.allSettled()` so failures are independent
- Used PostgreSQL `COUNT(*) FILTER (WHERE ...)` for cleaner single-table aggregation instead of scalar subqueries
- Per-group fallback: if a group fails, only that group's metrics are zeroed; others remain accurate
- Failed groups are logged with group name and error details via `dtLogger.error()`
- Response includes `degraded: true` and `degradedGroups: [...]` array when partial failure occurs
- Removed the old monolithic fallback catch block (was duplicating vehicle-only queries)

**Files Changed**:
- `backend/packages/goodmen-shared/routes/dashboard.js` — refactored `/stats` route handler

**Key Decisions**:
- Used `COUNT(*) FILTER` instead of scalar subqueries — more efficient single-pass aggregation per table
- Grouped by data source (drivers, loads, vehicles) to minimize per-group queries while maintaining isolation
- Kept `Promise.allSettled()` rather than individual try/catch for cleaner parallel execution
