# FN-378: [Bug] Dashboard shows all zeros — silent degraded fallback activated by failing SQL query

## Summary
The dashboard stats endpoint's main SQL query fails silently, returning zeros for all non-vehicle metrics. The frontend never checks the `degraded` flag, so users see zeros with no indication of incomplete data.

### FN-380 (subtask) — Frontend: Show degraded-mode banner

**Implementation Summary:**
Added a degraded-mode warning banner to the dashboard that appears when the API returns `stats.degraded === true`. The banner uses amber/warning colors consistent with existing FleetNeuron alert patterns and optionally displays which metric groups are affected.

**Changes:**
- `dashboard.component.ts`: Added `isDegraded` and `degradedGroups` properties, set from API response in `loadDashboard()`
- `dashboard.component.html`: Added `degraded-banner` div between loading state and dashboard grid, conditionally shown via `*ngIf="!loading && isDegraded"`
- `dashboard.component.css`: Added `.degraded-banner` styles using existing amber/warning design tokens

**Files Changed:**
- `frontend/src/app/components/dashboard/dashboard.component.ts`
- `frontend/src/app/components/dashboard/dashboard.component.html`
- `frontend/src/app/components/dashboard/dashboard.component.css`
