# FN-320: Frontend — Due date fields, color-coded badges, tab reorganization, updated completeness

## Requirement
Update the DQF checklist UI to show due dates, color-coded status indicators (green/yellow/red), reorganize tabs, and update the frontend completeness calculation.

## Acceptance Criteria
- [x] Readonly due date field on time-sensitive checklist items
- [x] Checklist rows show green/yellow/red based on urgency from backend
- [x] Tab badge counters show green/yellow/red (not just red/green)
- [x] Other Documents tab excluded from DQF completeness
- [x] DQF completeness excludes items not yet past due date
- [x] Yellow warning highlight shown for items approaching due date

## Implementation Summary

### New methods in drivers.component.ts:
1. `getCategoryBadgeClass(cat)` — returns badge-complete/safe/warning/overdue/incomplete
2. `getUrgencyChipClass(req)` — returns chip class based on req.urgency with fallback
3. `getUrgencyLabel(req)` — returns "Overdue", "Due Soon", "On Track", "Missing", "Complete"

### Updated recalculateDqfCompleteness():
- Excludes `other` category entirely
- Excludes within_30_days and annual items where urgency is green or yellow

### HTML changes:
- Tab badges use `getCategoryBadgeClass()` instead of binary complete/incomplete
- Status chips conditionally use urgency-based class/label when `req.urgency` is present
- Due date span shown when `req.due_date` is present

### CSS additions:
- `.badge-safe` (green), `.badge-warning` (amber), `.badge-overdue` (red)
- `.dqf-chip-safe`, `.dqf-chip-overdue`
- `.dqf-due-date` for due date display
