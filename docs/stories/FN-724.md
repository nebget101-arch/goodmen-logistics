## Key Decisions
1. **`COALESCE(l.needs_review, false)`** — safe against rows created before migration runs.
2. **Boolean toggle, not text filter** — `needsReview` bypasses the generic `LoadFilters` string-based params; the service's `Object.entries` loop passes it only when truthy.
3. **Amber/yellow badge** — visually distinct from status pills (cyan/teal) to draw attention without being alarming red.
4. **`clearFilterChip` return guard** — added explicit `return` before the new `needs_review` branch so existing driver-filter handling didn't fall through.
