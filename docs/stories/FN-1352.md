# FN-1352: Loads list — AI-themed redesign with hybrid row layout, route cell, view toggle & actionable AI Insights

## Requirement
Replace the flat 12-column Loads table with a denser-but-friendlier AI-themed layout that keeps power-user efficiency while raising visual hierarchy and personality. Top-of-page hero (Upload, Forward Email, Create Manually, Clone, KPIs, AI Insights) stays as-is — only the list region below changes.

## Implementation Summary

### FN-1353 (subtask)

Frontend-only implementation of the Loads list redesign. Five new presentational primitives under `loads-dashboard/primitives/` plus a new `viewMode` state ('table' | 'cards' | 'kanban') in `LoadsDashboardComponent` that swaps the rendering path via `*ngSwitchCase`. Cards is the default view; localStorage key `loads.list.view` persists per-user choice. The pre-existing virtualized table (Table view) is unchanged — only its visibility is now gated by `viewMode === 'table'`.

The Intelligence Insights list now emits `(insightApply)` instead of routing on click; the dashboard maps the three insight types — `overdue` → smart filter `overdue`, `missing_documents` → `missing_docs`, `driver_idle` → new `idle_drivers` chip — and scrolls the `.table-card` region into view after the filter applies. Other insight types fall back to `Router.navigateByUrl(insight.href)`.

Smart-filter chips wrap into a two-row container (time-range row above, semantic chips below) with a cyan glow on the `.smart-filter-chip--active` state. New `idle_drivers` chip added to `SMART_FILTER_KEYS` (backend count = 0 until backend support lands; toggle is purely client-side for now).

## Key Decisions

- **Cards as the default view (not Table)**: matches the brief's "Cards view (default)" AC. Existing-user persistence may flip them back to Table on first visit since localStorage is empty for first-time users — accepted as intentional under the redesign.
- **`idle_drivers` ships without backend wiring**: count is 0 until backend support lands. The chip and the AI-insight click path are functional client-side; this avoids blocking FN-1353 on a backend coordination cycle. Will need a follow-up subtask if FN-1354 QA flags the empty count.
- **Token-only theming via `--row-attention-accent`**: aliased to `--ai-accent-warning` to keep the color contract single-sourced. No raw hex literals were introduced.
- **Track driver overflow item routes to `/load-map?loadId=…`** (same as View on map) until a driver-specific page exists. Easy retarget when one ships.
- **Kanban grouping is read-only** by design — drag-to-update is explicitly out of scope this story.
