## Role Defaults

**Dispatcher**: Daily Briefing → Smart Alerts (load) → Predictive Insights (load volume + on-time %) → Quick Actions
**Safety**: Daily Briefing → Smart Alerts (HOS + fatigue) → Predictive Insights (compliance) → Quick Actions
**Maintenance**: Daily Briefing → Smart Alerts (inspection + maintenance) → Predictive Insights (maintenance windows + fuel) → Quick Actions
**Owner**: Daily Briefing → Predictive Insights (all 4) → Smart Alerts (top severity)

## Implementation Summary

### FN-1174 (subtask)
Added knex migration creating `user_dashboard_layouts` to back the per-user layout persistence consumed by FN-1172 (layout endpoints) and FN-1171 (Control Center drag-reorder + reset).

- One row per user (`user_id` PK), so reads/writes are point lookups, not range scans.
- `tenant_id` is a NOT NULL FK + secondary index for tenant-scoped reads and ON DELETE CASCADE cleanup paths.
- `layout_json jsonb` defaults to `'{}'` so GETs before the user has reordered anything return an empty object instead of NULL — backend can fall back to the role default cleanly.
- FKs cascade on user/tenant delete, matching the surrounding `users` / `tenants` cleanup contract used by `user_alert_dismissals` (FN-1165) and `user_notifications` (FN-507).

## Key Decisions

### FN-1174 (subtask)
- **`user_id` as PK, not a surrogate `id` UUID** — FN-1172 will upsert per user (one layout per user). A composite/unique constraint on `user_id` would force a separate PK; using `user_id` as PK is simpler and what the ticket specifies.
- **Skipped the README append** — the ticket lists `backend/migrations/README.md` as a touched file, but the actual migrations directory is `backend/packages/goodmen-database/migrations/` and there is no per-folder migrations README in this codebase (recent migrations like FN-1165, FN-507 don't update one). Following established convention.
- **`layout_json` default `'{}'`, not NULL** — keeps the FN-1172 endpoint contract trivial (always returns a JSON object).

## Deployment Handoff

### FN-1174 (subtask)
- **Migration to run:** `20260504200000_create_user_dashboard_layouts.js` (knex `migrate:latest` from backend, or `migrate:prod` on deploy).
