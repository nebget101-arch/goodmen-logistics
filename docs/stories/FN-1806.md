# FN-1806: Visual bbox field-placement editor (drag fields onto document preview)

> Epic: **FN-1786**. Follow-up enhancement. Blocked by **FN-1787**; shares coordinate space with **FN-1797**.

## Requirement
Uploaded agreements are **scanned images**, and signed-PDF generation overlays values/signatures at the AI-detected `bbox`. Placement is only as good as detection, so give internal users a **visual editor**: render each source-doc page as a preview and let them **drag, resize, add, and delete** field boxes directly on the page, persisting corrected `page` + `bbox` onto the template's field map. Complements FN-1794 (which only *flags* low-confidence fields — this lets the user *fix* geometry).

## Acceptance Criteria
- [ ] Document pages render as previews (reuse source-doc signed URL from FN-1793).
- [ ] Each field shows as a draggable/resizable box at its `bbox`; low-confidence boxes distinguished.
- [ ] User can move/resize, add a new box, or delete; role/type/label editing carried over from FN-1794.
- [ ] Save persists `page` + `bbox` + adds/deletes; reopened template shows corrected geometry.
- [ ] Coordinate space matches the FN-1797 signed-PDF overlay (box drawn here lands in the same place on the signed PDF).
