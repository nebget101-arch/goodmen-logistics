# FN-1806 — Visual bbox field-placement editor (drag fields onto document preview)

**Type:** Story · **Parent epic:** Agreements / e-sign engine · **Status:** In Progress
**Integration branch:** `integration/FN-1806` (Shape C — multi-agent)

## Requirement

Uploaded agreements are scanned images; signed-PDF generation overlays
values/signatures at AI-detected `bbox` coordinates, so placement is only as
good as detection. Give internal users a visual editor to correct field
placement: render each page of the source doc as a preview and let the user
drag, resize, add, and delete field boxes directly on the page, persisting the
corrected `page` + `bbox` back onto the template's field map. This raises overlay
accuracy and complements the confidence-flag review in FN-1794 (which only flags
— this lets the user fix geometry).

## Acceptance Criteria

- [ ] Document pages render as previews (reuse the source-doc signed URL from FN-1793).
- [ ] Each detected field shows as a draggable/resizable box positioned by its `bbox`; low-confidence boxes visually distinguished.
- [ ] User can move/resize a box, add a new field box, or delete one; role/type/label still editable (reuse FN-1794 controls).
- [x] Save persists `page` + `bbox` (and adds/deletes) via the field-map endpoint; a re-opened template shows the corrected geometry. *(backend — FN-1808)*
- [x] Coordinate space is consistent with the signed-PDF overlay (FN-1797) so a box drawn here lands in the same place on the signed PDF. *(canonical convention documented — FN-1808)*

### FN-1808 (subtask) — Backend: field-map endpoint geometry edits/adds/deletes

Extends `PATCH /api/agreements/templates/:id/fields` (from FN-1793) so the visual
editor can persist geometry corrections.

**What changed**

- **PATCH contract extended** — body is now `{ fields, adds, deletes, finalize }`:
  - `fields[]`: `{ id, role?, label?, page?, bbox? }` — existing edits plus the
    new geometry columns `page` + `bbox`. `sort_order` is preserved (geometry
    edits never reorder).
  - `adds[]`: `{ fieldType, page, bbox, label?, role?, fieldKey? }` — user-drawn
    boxes, persisted with `confidence: null` (manual, no AI), appended after the
    current max `sort_order`.
  - `deletes[]`: `[fieldId]` or `[{ id }]` — fields the user erased.
- **Validation** — `validatePage(page, pageCount)` enforces `1 ≤ page ≤
  page_count`; `validateBbox(bbox)` enforces a 4-number non-negative `[x,y,w,h]`.
  Invalid page/bbox is dropped from edits and rejects an add — never persisted.
- **Tenant scoping** — the template is verified against the caller's tenant up
  front; every field write is additionally constrained by `template_id`, so a
  tenant can only touch its own template's fields. All three mutation sets run
  in one transaction (delete → edit → add → finalize).
- **Canonical coordinate convention documented** —
  `docs/design/agreements-bbox-coordinates.md`: `bbox = [x, y, w, h]` in
  top-left PDF points, shared by the AI detector, the FN-1807 editor, and the
  FN-1797 signed-PDF overlay (which is the only place the y-axis flips).

**Files changed**

- `backend/packages/goodmen-shared/services/agreement-service.js` — `validateBbox`,
  `validatePage`, `sanitizeNewField`, `normalizeDeleteIds`, `fieldUpdateToRow`;
  `sanitizeFieldUpdate` now accepts `page`/`bbox`; `updateTemplateFields` handles
  `updates`/`adds`/`deletes`; canonical-convention header block.
- `backend/packages/goodmen-shared/routes/agreements.js` — PATCH handler reads
  `adds`/`deletes`; updated OpenAPI doc.
- `backend/packages/goodmen-shared/services/agreement-service.test.js` — 26 new
  unit tests (geometry edits, bbox/page validation, manual adds, delete-id
  normalization, jsonb row mapping). 59/59 pass under Node 20.

**Key decisions**

- bbox stored as a top-left PDF-points `[x,y,w,h]` JSONB array (matches what the
  detector already emits — detected boxes need no transform). The y-axis flip to
  PDF bottom-left space is isolated to the FN-1797 overlay.
- Manual adds carry `confidence: null` so the FE's low-confidence flagging
  (FN-1794) treats them as user-authored, not low-confidence AI output.
- Geometry edits preserve `sort_order`; adds append at the end. No reordering.

**Deployment handoff**

- **Service to redeploy:** `fleetneuron-logistics-service` (mounts the agreements
  router and bundles `goodmen-shared`). No gateway change.
- **DB migrations:** none — `agreement_template_fields.page` / `.bbox` columns
  already exist (FN-1792). No schema change in this subtask.
- **Env / config:** none new.
- **Cache / flags:** none.
- **Post-deploy verification:** `PATCH /api/agreements/templates/:id/fields` with
  a `bbox`/`page` edit, an `adds[]` box, and a `deletes[]` id; re-`GET` the
  template and confirm the geometry persisted, the new field has `confidence:
  null`, and an out-of-bounds `page`/malformed `bbox` is rejected. Tenant
  scoping: a cross-tenant template id returns 404.
