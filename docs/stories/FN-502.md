# FN-502: Financial Reports ā€” Direct Load Profit & Fully Loaded Profit

## Requirement
Fleet owners need Direct Load Profit and Fully Loaded Profit reports to understand per-load and per-truck profitability including all direct and period costs.

## Acceptance Criteria
- [x] Per-load: Load Rate - (fuel + tolls + driver pay attributed to that load)
- [x] Columns: Load#, Date, Driver, Truck, Rate, Driver Pay, Fuel, Tolls, Direct Profit, Margin %
- [x] Filters: date range, driver, truck
- [x] Summary row with totals (KPI cards)
- [x] Direct Load Profit minus allocated period costs (insurance, ELD, maintenance prorated across loads)
- [x] Additional columns: Insurance Allocation, ELD Allocation, Maintenance Allocation, Other Allocations, Fully Loaded Profit, Fully Loaded Margin %
- [x] Toggle: Per Load / Per Truck / Per Driver grouping (groupBy query param)
- [x] P&L Enhancement: Revenue, Direct Costs, Gross Margin, Operating Expenses, Net Profit cards
- [x] Routes at `/reports/direct-load-profit` and `/reports/fully-loaded-profit`
- [x] Nav links in reports shell sidebar

## Dependencies
- FN-499 (Settlement Engine V2 ā€” settlement_adjustment_items used for driver pay attribution)

## Agent
backend + frontend

## Implementation Summary

### Backend (`reports.js`)
- Extended `parseV2Filters` to support `truckId` and `groupBy` params
- **`buildDirectLoadProfit`** (NEW): Per-load CTE query joining `loads`, `drivers`, `vehicles`; subquery sums for `settlement_adjustment_items` (driver pay), `fuel_transactions`, `toll_transactions` per load; returns rows + KPI cards + summary totals
- **`buildFullyLoadedProfit`** (NEW): Calls `buildDirectLoadProfit` for base data; queries `work_orders.total_amount` for period maintenance; queries non-load `settlement_adjustment_items` for insurance/ELD/overhead; prorates period costs per load count; supports `groupBy: load | truck | driver` aggregation
- **`buildProfitLoss`** (UPDATED): Enhanced with direct cost breakdown (driver pay + fuel + tolls from loads), maintenance from `work_orders`, overhead from settlement adjustments; now returns 5 KPI cards: Revenue, Direct Costs, Gross Margin, Operating Expenses, Net Profit
- Registered `'direct-load-profit'` and `'fully-loaded-profit'` in `v2Builders` ā€” auto-wired to REST endpoint and CSV/PDF export

### Frontend
- `reports.models.ts`: Added `'direct-load-profit'` and `'fully-loaded-profit'` to `ReportKey` type; added `truckId` and `groupBy` to `ReportFilters` interface
- `report-view.component.ts`: Added `REPORT_CONFIG` entries for both new reports (columns: Load#, Date, Driver, Truck, Rate, Driver Pay, Fuel, Tolls, Direct Profit, Margin % / fully-loaded columns); updated `profit-loss` config columns to reflect enhanced P&L breakdown
- `reports-shell.component.ts`: Added nav items + icon map entries (`receipt_long` / `account_balance_wallet`) for both new reports
- `reports-routing.module.ts`: Added child routes `direct-load-profit` and `fully-loaded-profit`
