# FN-480 — Driver Risk Dashboard and Timeline UI

## Summary

Builds the frontend for the Driver Risk Scoring module in the Safety section. Adds a Fleet Risk Dashboard (`/safety/risk-scores`) showing fleet-wide risk KPIs, a visual risk level distribution bar, and a high-risk drivers table with score/trend/badge columns. Adds a per-driver Risk Timeline (`/safety/risk-scores/:driverId`) showing the score circle, category breakdown bars, and a chronological risk events timeline with pagination and an on-demand Recalculate action.

## Routes

| Route | Component |
|-------|-----------|
| `/safety/risk-scores` | `RiskDashboardComponent` |
| `/safety/risk-scores/:driverId` | `DriverRiskTimelineComponent` |

Both routes are guarded by `AuthGuard` + `PermissionGuard` with `anyPermission: SAFETY_PERMS`.

## API Endpoints Consumed

All under `/api/safety/driver-risk-scores`:

- `GET /fleet-summary` — fleet KPIs and high-risk driver list
- `GET /:driverId` — current score + history
- `GET /:driverId/events?page&pageSize` — paginated risk events
- `POST /:driverId/recalculate` — trigger score recalculation

## Deployment Handoff

- **Frontend-only** change — no migrations, no backend changes required.
- Rebuild the `fleetneuron-logistics-ui-dev` Render service after merging.
- Backend scoring engine (`/api/safety/driver-risk-scores`) must be deployed (FN-479) before these routes return real data.

## Notes

- `category_scores` JSON string guard: parsed with `JSON.parse` if backend returns a string.
- Score color scale: 0–25 green, 26–50 yellow, 51–75 red, 76–100 critical.
- Distribution bar uses `[style.flex]` proportional to raw counts — no getter-based bindings.
- All components use `ChangeDetectionStrategy.OnPush` + `cdr.markForCheck()` on every async update.
