# FN-477: AI-Powered MVR Report Analysis at Onboarding

## Requirement
Analyze uploaded MVR/PSP reports using Claude Vision to extract driving records and generate pre-hire risk assessments.

## Acceptance Criteria
- [x] Claude Vision handler extracts violations, accidents, suspensions, endorsements
- [x] Risk assessment with red flags and hire recommendation (recommend/caution/decline)
- [x] Violation severity classification (minor/major/serious per FMCSA standards)
- [x] mvr_extracted_records table stores AI-extracted data per driver
- [x] AI route registered at POST /safety/mvr-vision

## Implementation Summary

### AI Handler (`mvr-vision-handler.js`)
- Accepts base64 MVR/PSP report image
- Structured Claude Vision prompt extracts:
  - Driver info: name, DOB, license number/state/class/status, endorsements, restrictions
  - Moving violations: date, description, code, severity, points, disposition
  - Accidents: date, description, severity, at-fault status
  - Suspensions: dates, reason, type, reinstatement status
- Auto-generates risk assessment:
  - Severity classification per FMCSA CDL standards
  - Red flags for major violations, DUI, suspensions
  - Risk level: low/medium/high/critical
  - Hire recommendation: recommend/caution/decline with reason
- Registered at `POST /safety/mvr-vision`

### Migration (`mvr_extracted_records`)
- Stores per-driver MVR extraction: violations, accidents, suspensions as JSONB
- Risk assessment fields: risk_level, hire_recommendation, red_flags
- AI metadata: confidence, warnings, processing time
- Unique constraint on document_id for dedup
