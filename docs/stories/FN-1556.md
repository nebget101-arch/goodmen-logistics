# FN-1556: [Parts Catalog] Make manufacturer field optional — fixes Save no-op for AI-uploaded parts (FN-1541 follow-up)

## Requirement

User reports the FN-1541 Save no-op bug is **still happening** for parts uploaded via the AI invoice / photo flow. After manual testing, user confirmed cause and chose the fix:

> "I just tested it by adding the manufacturer and when it is empty that's why the save button [is] not working. Once I add the manufacturer I was able to save. Can you update it to make the manufacturer as not required field?"

## Confirmed root cause

- `frontend/src/app/components/parts-catalog/parts-catalog.component.ts:60` defines `manufacturer: ['', Validators.required]`.
- AI/invoice-uploaded parts have empty `manufacturer` in the API payload.
- `openForm(part)` patches the empty value → form is invalid → `savePart()` short-circuits at line 190 with no visible toast → user sees a dead Save button.

## Fix

Make `manufacturer` an **optional** field on the Parts form. One-line FE change.

## Acceptance Criteria

- [ ] `manufacturer` is no longer required on the Parts form (no `Validators.required`, no `*` label marker)
- [ ] Editing an AI-uploaded part with empty manufacturer → change Price → Save → persists

- [ ] Editing a manual-created part still works (no regression)
- [ ] Backend accepts null/empty manufacturer on POST + PUT (verified during FN-1557; no BE change needed unless rejection observed)

## Implementation Summary

### FN-1557 (subtask)

Branch: `agent/frontend/FN-1557/manufacturer-optional` (off `origin/dev`)

- Dropped `Validators.required` from the `manufacturer` form control in `parts-catalog.component.ts` (line ~176, inside the `partForm` FormGroup definition). Updated the inline FN-1093 comment to record FN-1557's reason for making the field optional.
- Removed the `*` marker from the `<label for="manufacturer">` in `parts-catalog.component.html` (line ~537). The label now reads `Manufacturer` (no required marker). The adjacent `app-confidence-badge` is unchanged.
- The form input is the existing `app-master-typeahead`; it carries no `[required]` / `aria-required` attribute, so no other template change was needed.
- Backend acceptance verified by code-read of `backend/packages/goodmen-shared/services/parts.service.js`:
  - `createPart` (line 236) only requires `sku` and `name`.
  - `resolveManufacturerVendor` (lines 49–83) explicitly handles empty/blank manufacturer text — when no `manufacturer_id` is supplied and the `manufacturer` string is empty, no patch keys are added and the column stays untouched. No backend change is required.
- Existing `parts-catalog.component.spec.ts` was inspected; no test asserts manufacturer-required validity, so no spec update was needed.
