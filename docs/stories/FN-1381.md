# FN-1381: Add/Edit Vehicle form redesign + explicit OO vs Company-owned classification

## Requirement
Restructure the Add/Edit Vehicle modal into a 5-section layout shared by trucks and trailers, and introduce a first-class **Ownership Type** segmented control (Company | OO | Leased) with conditional fields. Persist as a structured `vehicles.ownership_type` enum so settlements, work-order routing, and maintenance billing can rely on it.

## Acceptance Criteria
- [ ] Add Truck and Add Trailer modals share the same 5-section layout (Basic Info, Ownership Type, Compliance & Maintenance, Status, Documents) plus a collapsible Trailer Details accordion.
- [ ] Segmented Ownership control writes `ownership_type` and shows/hides conditional fields.
- [ ] OO save persists `equipment_owner_name`, `operating_entity_id`, `equipment_owner_percentage`.
- [ ] Leased save persists `lessor_name`, `lease_date`, `lease_payment_amount`.
- [ ] Edit modal pre-populates segmented control from `ownership_type`.
- [ ] Validation: OO requires Equipment Owner; Leased requires Lessor.
- [ ] No regressions in VIN decode, document upload, trailer-type chip-search.
- [ ] `company_owned` boolean kept for settlements backward compatibility.

## Implementation Summary
_To be filled by remaining agents (FN-1386, FN-1387)._

### FN-1385 (subtask)
- Added `vehicles.ownership_type` as a nullable-free `text` column (NOT NULL, DEFAULT `'company'`) backed by a `CHECK (ownership_type IN ('company','oo','leased'))` constraint. Chose CHECK + text over a Postgres native `CREATE TYPE … AS ENUM` because the rest of the codebase uses CHECK-style "enums" (e.g., `customer_billing`, `compensation_profiles`) — values can be added later with `ALTER TABLE … DROP CONSTRAINT … ADD CONSTRAINT …`, no `ALTER TYPE` ceremony.
- Backfill runs in two `UPDATE`s in priority order:
  1. `'oo'` for `company_owned = false`
  2. `'leased'` for `trailer_details->>'ownership' = 'leased'` (overrides `'oo'` when both apply, since a leased trailer driven by an OO is still a leased asset for billing).
- Composite index `(tenant_id, ownership_type)` added for the tenant-scoped ownership filter introduced in FN-1383.
- `all_vehicles` view recreated to expose `ownership_type` on the fleet branch; the shop-client (`customer_vehicles`) branch returns `NULL::text` because customer-owned vehicles aren't fleet-classified. View shape otherwise matches the post-FN-1317 baseline.
- Verified with synthetic rows on local Postgres: 4-bucket backfill (company / oo / leased-trailer / leased-OO) all classify correctly and `down()` cleanly drops index + constraint + column and restores the prior view.

## Key Decisions
- **CHECK constraint over native PG ENUM** — matches existing convention, avoids `ALTER TYPE … ADD VALUE` migration friction if a 4th ownership category is added later.
- **`'leased'` wins over `'oo'`** in backfill — chosen because leased equipment carries a lessor-payment obligation regardless of who drives it, and downstream settlements/maintenance code keys on lease vs. OO billing in that priority order.
- **Shop-client vehicles get `NULL` ownership_type** in the view — the column models fleet-side ownership classification only; customer vehicles aren't fleet equipment.
- **`company_owned` preserved** — explicit AC requirement; FN-1386 will write both columns until settlements migrates off `company_owned`.
