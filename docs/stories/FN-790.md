## Implementation Notes

### FN-811 (subtask) — Backend WS infra + load CRUD emits

Branch: `backend/FN-811/websocket-infra`

**Architecture**

- **Gateway owns the Socket.IO server.** `backend/gateway/index.js` now wraps the
  Express app in an `http.Server`, attaches Socket.IO on path `/socket.io`, and
  authenticates each handshake with the existing JWT secret. On connect the
  socket joins `tenant:<tenantId>`.
- **JWT now carries `tenant_id`.** `routes/auth.js` login resolves the default
  tenant for the user (via `tenantContextService.getDefaultContextForUser`) and
  includes it in the signed JWT. This removes the need for the gateway to hit
  the database during the WS handshake.
- **Shared emit helper.** `backend/packages/goodmen-shared/services/websocket.service.js`
  exposes `emitToTenant({ tenantId, event, payload })`. It fire-and-forgets a
  POST to the gateway's `/internal/ws/emit` endpoint (authed by
  `INTERNAL_WS_SECRET`). The helper never throws, never blocks the caller.
- **Internal emit endpoint on gateway.** `POST /internal/ws/emit` validates the
  `X-Internal-Token` header and broadcasts the body to `tenant:<tenantId>`.
  Rejects with 401 on bad token, 503 when WS is not initialized.

**Load CRUD events (rolled up from FN-812)**

`routes/loads.js` now emits after successful commits:

| Endpoint | Event | Payload |
|---|---|---|
| `POST /api/loads` | `load:created` | full load detail |
| `PUT /api/loads/:id` | `load:updated` | full load detail |
| `PUT /api/loads/:id` (status changed) | `load:status_changed` | `{ id, previousStatus, status, load }` |
| `DELETE /api/loads/:id` | `load:deleted` | `{ id }` |
| `PATCH /api/loads/:id/approve-draft` | `load:status_changed` + `load:updated` | `{ id, previousStatus: DRAFT, status: DISPATCHED, load }` |
| `POST /api/loads/bulk-update` | `load:updated` (per id) + `load:status_changed` (per id when status changed) | `{ id, changes }` / `{ id, status }` |
| `POST /api/loads/bulk-delete-drafts` | `load:deleted` (per id) | `{ id }` |

All emits happen **after** the commit path; a rolled-back transaction emits nothing.

**Envelope shape** (all events):

```json
{
  "tenantId": "<uuid>",
  "event": "load:created",
  "payload": { /* load or { id } */ },
  "emittedAt": "2026-04-23T00:00:00.000Z"
}
```

## Deployment Handoff — FN-811

**Services to redeploy** (both required — gateway and logistics):

1. `fleetneuron-logistics-gateway` (source `backend/gateway`) — hosts the Socket.IO server.
2. `fleetneuron-logistics-service` (source `backend/microservices/logistics-service`) — emits load events via shared helper.
3. `fleetneuron-auth-users-service` (source `backend/microservices/auth-users-service`) — login now issues JWTs that include `tenant_id`.

**Deploy order**

1. `fleetneuron-auth-users-service` first (new JWT shape — backward-compatible; old middleware still works).
2. `fleetneuron-logistics-gateway` next (new WS endpoint).
3. `fleetneuron-logistics-service` last (starts emitting once gateway is reachable).

**New env vars** (set on Render):

| Service | Var | Value |
|---|---|---|
| `fleetneuron-logistics-gateway` | `INTERNAL_WS_SECRET` | generated with `openssl rand -hex 32` |
| `fleetneuron-logistics-service` | `INTERNAL_WS_SECRET` | same value |
| `fleetneuron-logistics-service` | `INTERNAL_WS_EMIT_URL` | `https://fleetneuron-logistics-gateway.onrender.com/internal/ws/emit` |

`JWT_SECRET` is already set on both auth-users and gateway; no change needed.

**Migrations**: none.

**Feature flags**: none.

**Cache invalidation**: users must re-login (or wait for their 8h JWT to expire) to pick up the new `tenant_id` claim. Existing JWTs without `tenant_id` will fail WS handshake only — HTTP requests continue to work.

**Post-deploy verification**

1. `curl https://<gateway>/health` still returns 200.
2. `curl -X POST https://<gateway>/internal/ws/emit -H 'X-Internal-Token: <secret>' -H 'Content-Type: application/json' -d '{"tenantId":"...","event":"load:created","payload":{}}'` returns `{ delivered: true }` (or `503` if no clients are connected — still counts as success for the endpoint).
3. Browser: log in → open DevTools Network → confirm `/socket.io/?EIO=...` upgrade succeeds.
4. Create a load → check logistics logs for `ws_emit_failed` (should be absent) and confirm any connected client receives `load:created`.

**Rollback**: revert `backend/gateway` and `backend/microservices/logistics-service` to the pre-merge commits. No DB changes to undo.
