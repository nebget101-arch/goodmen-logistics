# FN-716: Story 3 — Workflow Buttons Replace Status Dropdown (with Validation + Timeline)

## Requirement
Replace the manual status dropdown with context-aware workflow buttons and validated transitions. Add a status timeline and server-side transition validation.

## Acceptance Criteria
- [x] Status dropdown removed from UI (replaced with read-only display in basics tab)
- [x] Workflow buttons render based on current status
- [x] Client-side validation blocks invalid transitions with inline errors
- [x] Server-side validation in work-orders-hub.js rejects disallowed transitions (409)
- [x] Status timeline visible at top of status bar
- [x] Cancel requires reason text input
- [ ] Manager approval gate for CLOSED when approval_required = true
- [x] Existing work orders in any status load correctly with appropriate buttons

## Agent
frontend + backend

## Implementation Summary

### Frontend — Workflow Buttons Component
- `WoWorkflowButtonsComponent` — context-aware buttons per status:
  - DRAFT: "Start Work" (green)
  - IN_PROGRESS: "Waiting for Parts" (amber), "Mark Complete" (green)
  - WAITING_PARTS: "Resume Work" (blue)
  - COMPLETED: "Close Work Order" (green)
  - CLOSED/CANCELED: read-only badge
  - Any non-terminal: "Cancel" (red) with reason dialog
- Client-side validation before transitions with inline error display
- Cancel reason modal with textarea

### Frontend — Status Timeline Component
- `WoStatusTimelineComponent` — horizontal step indicator
- Pipeline: DRAFT → IN_PROGRESS → COMPLETED → CLOSED
- Checkmarks for past, highlight for current, gray for future
- WAITING_PARTS shown as amber branch off IN_PROGRESS
- CANCELED shown as separate badge

### Frontend — Status Bar Updates
- Removed status dropdown, replaced with timeline + workflow buttons
- Status bar emits `statusChange` event to parent
- Basics tab shows read-only status input

### Backend — Transition Validation
- Added `ALLOWED_TRANSITIONS` map and `validateStatusTransition()` function
- DRAFT → IN_PROGRESS: requires customer, vehicle, title, type, priority
- IN_PROGRESS → COMPLETED: requires ≥1 part or labor line item
- Any → CANCELED: requires cancel_reason
- Returns 409 with `{ error, message, validationErrors }` on invalid transitions
- Added to PATCH /:id/status handler in work-orders-hub.js
