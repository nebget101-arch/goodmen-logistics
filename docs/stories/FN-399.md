### FN-401 (subtask) — Auto-set completed_date on status transition

**Implementation Summary:**
Added auto-set logic in the `PUT /:id` endpoint so that when a load's status transitions to DELIVERED or COMPLETED, `completed_date` is automatically populated if it's currently NULL. Also created a Knex migration to backfill existing DELIVERED/COMPLETED loads with NULL completed_date.

**Changes:**
- `backend/packages/goodmen-shared/routes/loads.js`: Added auto-set query after field updates/stops, before COMMIT. Uses `COALESCE(completed_date, delivery_date, CURRENT_DATE)` and only updates rows where `completed_date IS NULL` (preserves existing values).
- `backend/packages/goodmen-database/migrations/20260328200000_backfill_completed_date_for_delivered_loads.js`: One-time backfill migration using `COALESCE(delivery_date, updated_at::date, created_at::date)` for existing DELIVERED/COMPLETED loads with NULL completed_date.

**Files Changed:**
- `backend/packages/goodmen-shared/routes/loads.js`
- `backend/packages/goodmen-database/migrations/20260328200000_backfill_completed_date_for_delivered_loads.js` (NEW)

**Key Decisions:**
- Only the PUT endpoint needs auto-set (approve-draft transitions to DISPATCHED, not DELIVERED)
- Backfill uses `delivery_date` first, then `updated_at::date`, then `created_at::date` as fallback
- Migration down is a no-op — cannot distinguish auto-set from manually-set values
