## Implementation Summary

### FN-1435 (subtask)

Added a dedicated NLQ search bar at the top of `dispatch-board.component.html`. It calls `POST ${apiUrl}/loads/search/nlq` with body `{ query }` — the same endpoint and shape the global command palette already uses (see `frontend/src/app/shared/components/command-palette/command-palette.component.ts:348`). On submit the dispatch grid is hidden and an inline results panel renders the returned loads as clickable cards; clicking a row navigates to `/loads?loadId=<id>` (matching the existing `goToLoad` pattern). The bar exposes a `/` global shortcut to focus and `Esc` to clear; a "Back to dispatch board" affordance returns to the week grid. Errors render an inline bottom-right toast (matching the tolls-accounts toast pattern) for 4 seconds.

The story description mentioned `GET /api/loads/search/nlq?q=…`, but the production endpoint and the command palette's call are POST `/loads/search/nlq` with `{ query }`. Reused the live shape rather than the doc's sketch (keeps a single contract; otherwise the bar would have diverged from the palette's call).

## Key Decisions

- **POST, not GET.** The story description specified GET, but the existing production endpoint and command palette use POST `{ query }`. Matching the palette avoided creating a divergent contract or a duplicated AI service method.
- **Replace the grid, not filter it.** The NLQ response is a flat list with no driver/day context — projecting it back onto the week-grid (driver rows × 7 day columns) would have required server-side joins or client-side fabrication. A dedicated results panel that swaps for the grid is cleaner, gives a clear "Back to dispatch board" path, and is the same affordance the command palette uses.
- **Local toast vs. global service.** No global toast service exists in the codebase; matched the `tolls-accounts` inline-toast pattern (component-owned `toast`/`toastType` with a 4s timer) rather than introducing a new dependency.
- **Global `/` shortcut is component-scoped.** Implemented as a HostListener on `document:keydown`, which only registers while the dispatch board is mounted — won't collide with `/` typed in other pages or other inputs (skips when target is INPUT/TEXTAREA/contentEditable).

4. Press `Esc` while focused — search clears and the grid returns.
5. With network DevTools, force a 500 on `/api/loads/search/nlq` — verify the bottom-right error toast appears and clears after ~4s.
