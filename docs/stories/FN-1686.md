# FN-1686: Story A — Stripe production configuration & go-live

## Requirement
Make Stripe production-ready. All `STRIPE_*` keys are currently **absent from `render.yaml`** and exist only as placeholders in `.env.example`. Provision them, create live Products/Prices for all 4 plans + the extra-seat add-on, register the webhook endpoint, document the dashboard steps, and add a backend config-validation safety net so a misconfigured deploy fails loudly instead of silently disabling billing.

## Acceptance Criteria
- [ ] `render.yaml` declares (sync:false) for `auth-users-service`: `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `STRIPE_PRICE_BASIC`, `STRIPE_PRICE_MULTI_MC`, `STRIPE_PRICE_END_TO_END`, `STRIPE_PRICE_ENTERPRISE`, `STRIPE_PRICE_EXTRA_USER_SEAT`; frontend gets `STRIPE_PUBLISHABLE_KEY`.
- [ ] `docs/deploy/stripe-go-live.md` documents creating Products/Prices per plan + extra-seat, copying price IDs into env vars, registering `/api/stripe/webhook`, and capturing `STRIPE_WEBHOOK_SECRET`.
- [ ] Backend startup validation + admin-only `GET /api/billing/config-status` verifying `STRIPE_SECRET_KEY` and every plan→price-ID mapping (`PLAN_PRICE_MAP` pattern); logs missing keys.
- [ ] No secrets committed; placeholders only in `.env.example`.

## Key Decisions
- **No frontend/`.env.example` edits.** Both already satisfied the AC on `dev`; editing them would be scope creep. Verified placeholders are present and contain no real secrets.
- **All Stripe vars on `auth-users-service` only.** That service owns the billing routes and webhook handler; no other service references `STRIPE_*`.
- **`sync: false`, not blueprint values.** Keeps every secret and environment-specific price ID out of git; values are entered in the Render dashboard per the runbook.

## Deployment Handoff
- **Service to redeploy:** `fleetneuron-auth-users-service` (Render).
- **Manual env vars to set in Render (Environment tab)** before/with deploy — all `sync: false`, so the deploy will not populate them automatically:
  - `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`
  - `STRIPE_PRICE_BASIC`, `STRIPE_PRICE_MULTI_MC`, `STRIPE_PRICE_END_TO_END`, `STRIPE_PRICE_ENTERPRISE`, `STRIPE_PRICE_EXTRA_USER_SEAT`
- **Frontend:** set `STRIPE_PUBLISHABLE_KEY` in the matching `environment*.ts` (prod=live key, dev=test key) and redeploy `fleetneuron-logistics-ui`. (Build-time, not a Render env var.)
- **Stripe dashboard prerequisites:** create live Products/Prices + extra-seat, register the webhook endpoint, capture the signing secret — see `docs/deploy/stripe-go-live.md`.
- **No DB migrations.** No cache invalidation. No feature flags.
- **Deploy order:** set Render env vars → redeploy `auth-users-service` → register webhook → verify via `GET /api/billing/config-status` (FN-1692).
- **Post-deploy verification:** confirm the `[stripe] STRIPE_SECRET_KEY is not configured` warning is absent from `auth-users-service` logs; send a Stripe test webhook event and confirm a `2xx`.
