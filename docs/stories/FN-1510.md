# FN-1510: [goodmen-shared/auth.js] swagger-jsdoc YAML parse errors pollute every service boot log

## Requirement
The `@openapi` JSDoc block for `/api/auth/forgot-password` in `backend/packages/goodmen-shared/routes/auth.js` is malformed (a stray line of JS code was pasted inside the comment). swagger-jsdoc emits two YAML errors (`YAMLSyntaxError`, `YAMLSemanticError`) every time any service whose swagger glob includes `goodmen-shared/routes/*.js` boots — that is 7 microservices: `auth-users-service`, `inventory-service`, `logistics-service`, `drivers-compliance-service`, `vehicles-maintenance-service`, `reporting-service`, `integrations-service`. Real boot errors are harder to spot in the noise, and the bad block also causes the `/api/auth/forgot-password` endpoint to be silently dropped from the generated swagger spec.

## Implementation Summary
Replaced a single stray line in the `@openapi` JSDoc block at lines 117–138 of `backend/packages/goodmen-shared/routes/auth.js`. Line 129 contained `    const isGlobalAdmin = roles.includes('super_admin');` — a JS line accidentally pasted into the JSDoc comment, replacing the YAML `*             type: object` declaration. The stray line broke YAML indentation and (because it was missing the `*` comment marker) was treated by swagger-jsdoc as the start of new YAML content from the next non-comment JS line, producing a second cascading error. Fix replaces line 129 with the canonical `*             type: object` line, mirroring the shape of the adjacent `/api/auth/reset-password` block (line 225). No functional change to the `forgot-password` request handler — the stray JS line referenced variables (`roles`) that don't exist in that scope and was pure dead text.

### Reproduction & Verification
Ran `swagger-jsdoc` against the file before and after the fix using a minimal Node harness. Before: 2 YAML errors (`YAMLSyntaxError`, `YAMLSemanticError`), 4 paths in spec (forgot-password silently missing). After: 0 YAML errors, 5 paths (login, forgot-password, reset-password, contact, me — all 5 `@openapi` blocks parse).

## Files Changed
- `backend/packages/goodmen-shared/routes/auth.js` — replaced stray JS line at 129 with `*             type: object`
- `docs/stories/FN-1510.md` — new

## Key Decisions
- Single-line replacement, not a structural rewrite. The block was 99% correct; only line 129 was wrong, and its surrounding indentation (`*             ...`) was already aligned for the `type:` line that should have been there.
- No swagger schema redesign. Other `@openapi` blocks in the file already use the same shape (`schema: { type: object, required: [...], properties: {...} }`); the fix matches that pattern exactly, so existing API contracts are unchanged.
