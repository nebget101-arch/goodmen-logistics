### FN-388 (subtask) — Replace filter select dropdowns with compact text inputs

**Implementation Summary:**
Replaced 4 `app-ai-select` dropdown components in the loads dashboard filter row with compact text `<input>` fields. Filter logic now uses case-insensitive substring matching (matching the pattern of existing text filters like broker, PO, pickup, delivery).

**Changes:**
- `loads-dashboard.component.html`: Replaced `app-ai-select` for Driver, Status, Billing, Attachments columns with `<input type="text">` bound to `headerFilters.driver`, `headerFilters.status`, `headerFilters.billingStatus`, `headerFilters.attachmentType`
- `loads-dashboard.component.ts`:
  - Added `driver`, `status`, `billingStatus` to `headerFilters` object and `headerFilterLabels`
  - Added client-side substring matching for driver (by `driver_name`), status, and billingStatus in `filteredLoads` getter
  - Updated attachmentType filter from exact match to substring matching (joins all types, matches against search text)
  - Updated `clearAllFilters()` to include new header filter fields

**Files Changed:**
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.html`
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.ts`

**Key Decisions:**
- Kept server-side `filters.status`/`filters.billingStatus`/`filters.driverId` for URL query param routing and deep linking. The new text inputs are additional client-side filters on the current page, matching the pattern of existing text filters.
- Status/billing text matching normalizes underscores to spaces so "del" matches "DELIVERED" and "in transit" matches "IN_TRANSIT"
