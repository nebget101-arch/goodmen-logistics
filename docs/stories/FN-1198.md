# FN-1198: Roadside v2 Story 4.1 — Real-time WebSocket + SMS for no-ETA state changes

## Requirement

As a driver/dispatcher/customer, I want to receive real-time notifications when an incident state changes (intake started, triage complete, on-site, complete), without having to refresh.

## Acceptance Criteria

- [ ] WebSocket gateway in `backend/gateway` (port 4000) brokers per-tenant channels; clients authenticate with their JWT
- [ ] Backend publishes `incident.state_changed` events from `backend/microservices/drivers-compliance-service`
- [ ] SMS opt-in stored per recipient with timestamp; outbound SMS via Twilio
- [ ] Idempotent delivery: events keyed by (incident_id, state, version)
- [ ] Telemetry: delivery success per channel, latency
- [ ] Excludes vendor-ETA event (Story 4.3)

## Implementation Summary

### FN-1240 (subtask — backend)

The gateway (`backend/gateway`) already had Socket.IO with JWT auth and per-tenant rooms from FN-811. FN-1240 adds the incident-specific layer:

**Gateway side:**
- `backend/gateway/services/incident-broadcaster.js` — `buildIncidentBroadcaster` follows the `alerts-ws.js` pattern; wraps the existing `makeSocketIoEmitter` to broadcast `incident.state_changed` events to `tenant:<id>` rooms. Wired into `index.js` alongside the alerts broadcaster.

**Drivers-compliance side:**
- `backend/microservices/drivers-compliance-service/services/incident-event-publisher.js` — idempotent HTTP POST to `POST /internal/ws/emit` on the gateway. Deduplicates via the `event_log` table (FN-1241); gracefully degrades (fires anyway) if the table doesn't exist yet. Telemetry included per publish: `published`, `duplicate`, `latency_ms`.
- `backend/microservices/drivers-compliance-service/services/incident-sms-notify.js` — SMS notifications for incident state changes; checks `sms_optin` table (FN-1241); wraps `notification-service.js`'s `sendSms`. Includes canonical message templates per state.
- `backend/microservices/drivers-compliance-service/services/incident-events.js` — combined dispatcher calling WS publish + SMS in parallel; emits structured telemetry JSON to stdout.
- `backend/microservices/drivers-compliance-service/routes/roadside-realtime.js` — thin Express router mounted BEFORE the shared `roadsideRouter`; intercepts `PATCH /api/roadside/calls/:id/status` to fire-and-forget `dispatchIncidentStateChanged` after response is sent. Maps roadside `status` values to canonical incident states.

**Event flow:**
```
PATCH /api/roadside/calls/:id/status
  → roadside-realtime.js handler
    → roadsideService.setStatus()  (shared DB update)
    → res.json(row)                (response sent immediately)
    → dispatchIncidentStateChanged (fire-and-forget)
        ├── publishIncidentStateChanged → POST /internal/ws/emit (gateway)
        │       → io.to('tenant:<id>').emit('incident.state_changed', payload)
        └── notifyIncidentStateChanged  → sendSms via Twilio (if opted-in)
```

## Key Decisions

1. **No new WS infrastructure** — FN-811 already ships Socket.IO with per-tenant rooms and an internal emit endpoint. FN-1240 adds incident-specific modules on top; no duplication.
2. **Fire-and-forget dispatch** — the `PATCH /status` response is sent synchronously; WS publish + SMS fire after `res.json()`. Keeps API latency tight even if the gateway or Twilio is slow.
3. **Graceful degradation for FN-1241 tables** — `incident-event-publisher` and `incident-sms-notify` silently continue if `event_log` / `sms_optin` tables don't exist yet. FN-1241 can land independently.
4. **Version = unix-seconds of updated_at** — avoids an extra DB column while giving a monotonically increasing idempotency key.

- None from this subtask. FN-1241 must run its migrations (`sms_optin`, `event_log`) before idempotent dedup and SMS opt-in enforcement are active. Code degrades gracefully without these tables.

### Environment / config
- `INTERNAL_GATEWAY_URL` — URL of the gateway service (default: `http://localhost:4000`). Must be set on `fleetneuron-drivers-compliance-service` to point at `https://fleetneuron-logistics-gateway.onrender.com` (or internal service URL if available).
- `INTERNAL_WS_SECRET` — already required by the gateway's `/internal/ws/emit` endpoint (FN-811). Must be set on both gateway and drivers-compliance. If unset on drivers-compliance, `publishIncidentStateChanged` returns `{ published: false, error: 'INTERNAL_WS_SECRET not configured' }`.
- `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_PHONE_NUMBER` — already used by `notification-service.js` for onboarding SMS; no new env vars.

### Deploy order
1. Deploy `fleetneuron-drivers-compliance-service` (new routes + services)
2. Deploy `fleetneuron-logistics-gateway` (new broadcaster wiring — non-breaking)
3. Run FN-1241 migrations to enable idempotent dedup + SMS opt-in checks
4. (FN-1242) Set `INTERNAL_GATEWAY_URL` on drivers-compliance if not already set

### Post-deploy verification

4. Verify SMS not sent for recipient without `sms_optin` row; SMS sent for opted-in recipient.
