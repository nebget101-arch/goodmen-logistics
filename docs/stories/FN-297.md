# FN-297: Auto-set unit number from last 4 digits of VIN instead of timestamp

## Requirement
Change the vehicle form to auto-populate the unit number field with the last 4 digits of the VIN instead of using a TRK/TRL-timestamp format. The unit number field should be editable and not overwritten when the user manually changes it.

## Acceptance Criteria
- [ ] VIN entry auto-populates unit_number with last 4 VIN digits
- [ ] Unit number field is editable (not readonly)
- [ ] Manual edits to unit number are preserved when VIN changes
- [ ] Works for both truck and trailer forms
- [ ] Edit mode doesn't overwrite existing unit numbers

### FN-301 (subtask)
**Changes:**

1. Added `unitNumberManuallyEdited` flag to track manual input
2. `onVinChange()` now sets `unit_number = vin.slice(-4)` when VIN >= 4 chars and not manually edited
3. Replaced `getNextUnitNumber()` with `onUnitNumberInput()` handler
4. Removed `readonly` from truck unit number field, added `(input)="onUnitNumberInput()"`
5. Added `(input)` handler to trailer unit number field too
6. Edit mode sets `unitNumberManuallyEdited = true` to protect existing values
7. Form reset clears the flag

**Files changed:**
- `frontend/src/app/components/vehicles/vehicle-form/vehicle-form.component.ts`
- `frontend/src/app/components/vehicles/vehicle-form/vehicle-form.component.html`
