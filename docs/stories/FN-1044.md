# FN-1044: Pickup/Delivery date displays full UTC timestamp instead of date only

## Type
Bug — Severity: Medium

## Symptom
On the Stops tab, the pickup/delivery date column shows `"2026-04-15T00:00:00.000Z"` instead of `"2026-04-15"`. Expanding the card shows the date input empty even though a date is stored.

## Confirmed Root Cause
- DB column `load_stops.stop_date` is type DATE (`backend/packages/goodmen-database/migrations/20260228193000_add_loads_tables.js` line 143).
- `pg` driver deserializes DATE → JS Date → ISO timestamp via `JSON.stringify`.
- `stop-card.component.ts:96–99` `dateLabel` returns the raw value.
- `stop-card.component.html:101` binds `[value]="stop.stop_date"` to `<input type="date">` — only accepts `YYYY-MM-DD`.

## Implementation Summary
Added a `toYmd()` helper on `StopCardComponent` that normalizes any `stop_date`
value (`Date`, ISO string, `YYYY-MM-DD` string, or null) to a calendar-only
`YYYY-MM-DD` string by slicing the first 10 chars of the ISO representation.
The `dateLabel` getter now formats through `toYmd()`, and the expanded date
input binds `[value]="toYmd(stop.stop_date)"` so the native `<input type="date">`
populates correctly. No timezone offsets are applied — the string the user sees
round-trips to the API.

## Key Decisions
- **ISO slice over `Date.getFullYear/Month/Date`.** The pg driver serializes
  DATE columns as UTC-midnight Date objects, so `toISOString().slice(0,10)`
  reliably returns the same calendar date the DB stored. Using local-time
  components on those Date objects can shift the date by one day in negative
  UTC offsets (PT/ET). The Jira description explicitly green-lit slicing as
  preferred for simplicity.
- **No conversion on save.** The native `<input type="date">` already emits a
  `YYYY-MM-DD` string on `input`, so no extra normalization is needed in
  `onFieldChange`. The outgoing payload preserves the calendar string.
- **Stop-card scope only.** `loads-dashboard.component.ts` already has its own
  `normalizeDate` helper; left untouched per ticket scope (FN-1052 is
  stop-card only).
