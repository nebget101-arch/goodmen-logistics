# FN-501: Idle Truck Notification System

## Requirement
Detect trucks with active drivers that have had no recent loads but still accrue recurring deductions. Create alerts and send email notifications at two escalation levels: week 1 (dispatch/accounting/manager notified) and week 2 (admin-only escalation with deactivation suggestions).

## Acceptance Criteria
- [x] Identify trucks idle 7+ days with always-on recurring deductions
- [x] Create `week_1_idle` alert and send email to admin, manager, dispatcher, accounting
- [x] Identify trucks idle 14+ days with no response to week_1 alert
- [x] Create `week_2_no_response` alert and send escalation email to admin
- [x] Avoid duplicate open alerts (idempotent per vehicle per alert_type)
- [x] `POST /api/idle-truck-monitor/run` ŌĆö trigger the check (admin/manager only)
- [x] `GET /api/idle-truck-monitor/alerts` ŌĆö list alerts with filters
- [x] `PATCH /api/idle-truck-monitor/alerts/:id/respond` ŌĆö acknowledge/resolve/escalate

## Implementation Summary

### `idle-truck-monitor-service.js` (NEW)
- `runIdleTruckCheck(knex, userId)` ŌĆö main daily job
  - Fetches all active drivers with a truck assigned (`drivers.truck_id NOT NULL, driver_status = 'active'`)
  - For each: gets `lastLoadDate` (latest of pickup/delivery date from `loads`)
  - Skips if `idleDays < 7` or no always-on recurring deductions
  - **Week 1 (7ŌĆō13 days idle)**: creates `week_1_idle` alert if no open alert exists; emails dispatch/accounting/admin/manager
  - **Week 2 (14+ days idle)**: creates `week_2_no_response` alert if unresponded week_1 alert is >7 days old; emails admin only with deactivation recommendations; back-fills week_1 if missing
- `getAlwaysOnDeductionTotal(knex, driverId, tenantId)` ŌĆö sums `recurring_deduction_rules` where `applies_when = 'always' OR NULL`
- `createAlert()` ŌĆö guarded by `hasTable('idle_truck_alerts')` for safe pre-migration
- `sendAlertEmails()` ŌĆö uses existing `sendEmail` from `notification-service.js`; targets users by role

### `idle-truck-monitor.js` (NEW route)
- `POST /run` ŌĆö admin/manager only; calls `runIdleTruckCheck`; returns `{ stats: { checked, week1Created, week2Created, errors } }`
- `GET /alerts` ŌĆö tenant-scoped; filters: `alert_type`, `response_status`, `vehicle_id`, `driver_id`, `limit`, `offset`; joins vehicles + drivers for display names
- `PATCH /alerts/:id/respond` ŌĆö validates `response_status` Ōłł `[acknowledged, resolved, escalated]`; records `responded_by`, `responded_at`

### `logistics-service/server.js` (MODIFIED)
- Mounted `idleTruckMonitorRouter` at `/api/idle-truck-monitor` with `authMiddleware` + `tenantContextMiddleware`
