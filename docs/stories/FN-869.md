# FN-869: S9 — Wire AI-Extract flow + confidence badges

## Requirement
Wire `<app-load-wizard>` into AI-extract mode. Step 0 is a PDF dropzone; post-upload, `aiExtractFromPdf` runs and its `LoadAiEndpointExtraction` result pre-fills Step 1 Basics and Step 2 Stops. Fields carry a confidence badge (`<app-confidence-badge>`) visualizing per-field confidence tiers: red <0.6, amber 0.6–0.85, none ≥0.85. Submit creates the load via the standard create path and attaches the source PDF automatically.

## Acceptance Criteria
- [ ] Step 0 PDF dropzone present in `mode='ai-extract'` only; calls `aiExtractFromPdf`.
- [ ] On extraction success: Steps 1–2 pre-filled from `LoadAiEndpointExtraction`; wizard focuses Step 1.
- [ ] Per-field confidence badge renders for fields with score < 0.85 (red < 0.6, amber 0.6–0.85).
- [ ] Submit path = `createLoad` + auto-attach source PDF as an attachment.
- [ ] Extraction failure path: inline error with Retry; user inputs preserved on retry.

### FN-887 (subtask)
**Branch**: `frontend/FN-887/confidence-badge-design`

**What shipped**
- `LoadAiEndpointExtraction.fieldConfidences?: Record<string, number>` added to `frontend/src/app/models/load-dashboard.model.ts`. Keys are form field paths (e.g. `brokerName`, `pickup.city`, `stops[0].date`); values are 0–1 scores. Leaves the existing card-level `confidence` object untouched.
- `<app-confidence-badge>` extended with a `variant: 'card' | 'field'` input. Defaults to `'card'` so all existing FN-818 callers are unaffected.
- New `variant="field"` + `score` (0–1) inputs drive the per-field tier per FN-887:
  - `score < 0.6` → **red** pill, copy "Needs review"
  - `0.6 ≤ score < 0.85` → **amber** pill, copy "Verify"
  - `score ≥ 0.85` → nothing rendered (host collapses via `--hidden` class)
- Exposed `FieldConfidenceTier` type + `ConfidenceBadgeComponent.fieldTierFor(score)` static helper for FN-888 to consume when building the `fieldConfidences` → tier mapping.
- Minimal SCSS: inline pill (no button affordance), dot glyph, red/amber tokens, hidden state. Placement tokens (`margin-left: 6px`, `vertical-align: middle`) assume rendering next to an Angular Material `mat-label` / plain `<label>`.
- Tooltip copy defaults: "AI extracted this field at {n}% — please review before submitting." (red) / "… — double-check the value." (amber). Parents can override via `[tooltip]`.
- Tests added covering `fieldTierFor` thresholds and field-variant rendering.

**Hand-off to FN-888**
- Use `<app-confidence-badge variant="field" [score]="extraction.fieldConfidences?.['brokerName']"></app-confidence-badge>` next to each pre-filled field label.
- Badge hides itself when score ≥ 0.85, so callers do **not** need `*ngIf` guards.
- `ConfidenceBadgeComponent.fieldTierFor(score)` is public/static — use it when you need to drive a separate "low-confidence field" summary (e.g. count of amber+red).

### FN-888 (subtask)
**Branch**: `frontend/FN-888/wizard-ai-extract-flow-impl`

**What shipped**
- Step 0 PDF dropzone lives inline in `LoadWizardComponent` — renders only when `mode === 'ai-extract' && !extractionComplete`. Idle state shows a drag-drop zone plus a "Choose file" input; with a selected file we show file-summary + a 4-step progress list that animates while `LoadsService.aiExtractFromPdf` is in flight. Non-PDF selections surface an inline "Only PDF files are supported" error and never hit the backend.
- On extraction success the wizard maps `LoadAiEndpointExtraction` → form:
  - `basics.loadNumber` ← `loadId ?? orderId ?? proNumber`
  - `basics.poNumber` ← `poNumber`
  - `basics.rate` ← `rate`
  - `basics.notes` preserved if user already typed, otherwise seeded from extraction
  - Stops FormArray is rebuilt from `extraction.stops[]` (sorted by `sequence`) when present, else from the legacy `pickup` + `delivery` pair
  - `fieldConfidences` is cached on the wizard and piped into child steps
  - `aiBrokerNameHint` (broker display name) piped into `LoadWizardBasicsComponent` → seeds the broker-search input as a hint (we deliberately do not auto-link a `brokerId`; user still picks/creates the broker record)
- `currentStepId` is set to `'basics'` and `extractionComplete = true`, so the 4-step shell renders with the prefilled form.
- Extraction failure → inline error with **Retry** and **Choose different file** buttons. Retry reuses the same `sourcePdfFile` so the user's original selection is preserved; server-side warnings (`err.error.warning` / `err.error.message`) are surfaced verbatim.
- Confidence badges (field variant) render via `<app-confidence-badge variant="field" [score]="...">` next to:
  - Basics: Load number, Broker, PO number, Rate
  - Stops: Zip, City, State, Address, Date (per row, keys shaped `stops[i].<field>`; `scoreFor(i, field)` is a small helper on `LoadWizardStopsComponent`)
  - Badges auto-hide when score ≥ 0.85 — no `*ngIf` at the call site.
- Submit path (`submitCreate`): before building the payload, the wizard auto-queues `sourcePdfFile` as a `RATE_CONFIRMATION` attachment unless the user already queued the same file reference. The existing `createLoad → uploadQueuedAttachments` pipeline does the rest, so partial-failure reporting and the `created` emit stay unchanged.
- `ConfidenceBadgeComponent` was promoted to `standalone: true` (+ imports `CommonModule`) so standalone step components can consume it directly. `SharedModule` now imports (and continues to export) the badge instead of declaring it, so every module-based consumer sees the same component with no behavioural change.

**Files changed**
- `frontend/src/app/components/load-wizard/load-wizard.component.ts` — extraction state, dropzone/retry handlers, `applyExtraction` mapper, auto-attach in `submitCreate`
- `frontend/src/app/components/load-wizard/load-wizard.component.html` — conditional Step 0 panel + wires `fieldConfidences` / `aiBrokerNameHint` to child steps
- `frontend/src/app/components/load-wizard/load-wizard.component.scss` — Step 0 dropzone / progress / error styles
- `frontend/src/app/components/load-wizard/load-wizard.component.spec.ts` — ai-extract flow tests (prefill, retry, auto-attach, double-queue guard)
- `frontend/src/app/components/load-wizard/steps/basics/basics.component.ts|.html` — `fieldConfidences` + `aiBrokerNameHint` inputs; badges on load number / broker / PO / rate
- `frontend/src/app/components/load-wizard/steps/stops/stops.component.ts|.html` — `fieldConfidences` input + `scoreFor(i, field)` helper; badges on zip / city / state / address / date
- `frontend/src/app/shared/components/confidence-badge/confidence-badge.component.ts` — marked `standalone: true`
- `frontend/src/app/shared/components/confidence-badge/confidence-badge.component.spec.ts` — TestBed now imports the standalone component instead of declaring it
- `frontend/src/app/shared/shared.module.ts` — moved `ConfidenceBadgeComponent` from `declarations` to `imports` (still exported)

**Key decisions**
- **Step 0 lives outside the wizard-shell stepper, not as a 5th `WizardStepDef`.** The 4-step stepper (Basics/Stops/Driver/Attachments) is identical across all four wizard modes; making Step 0 a real wizard step would have leaked AI-Extract branding into create/edit/view. Gating it with `*ngIf="showExtractStep"` keeps the shell definition untouched and lets the user see the familiar 4-step layout the moment extraction completes.
- **Broker name surfaced as a search-input hint, not auto-linked.** `LoadAiEndpointExtraction.brokerName` is a display string; the wizard's basics FormGroup only has `brokerId` (the server linkage). Fuzzy-matching to a broker record would have required pulling the broker list up into the wizard and made silent mispicks possible. Instead we pre-populate the broker search field with the extracted name so the user must click-to-select (or "+ Create new broker") — aligns with the existing broker combo affordance.
- **`ConfidenceBadgeComponent` promoted to standalone.** `SharedModule` can still declare module components, but the new wizard sub-components are all `standalone: true` and mixing declaration-based components into standalone imports was a dead-end. Promoting the badge (and leaving `SharedModule` to import+export it) preserved every existing consumer while unblocking the new ones.
- **Field-confidence keys use `stops[i].<field>` indexing, not a FormGroup-anchored map.** If the user reorders stops after extraction, badges stay at their original row position rather than tracking the row data. Accepted trade-off for FN-888: confidence indicators are a "look at what the AI was unsure about right now" signal, not persistent row metadata. FN-889 QA should call this out if it surprises the user.
