### FN-887 (subtask)
**Branch**: `frontend/FN-887/confidence-badge-design`

**What shipped**
- `LoadAiEndpointExtraction.fieldConfidences?: Record<string, number>` added to `frontend/src/app/models/load-dashboard.model.ts`. Keys are form field paths (e.g. `brokerName`, `pickup.city`, `stops[0].date`); values are 0ÔÇô1 scores. Leaves the existing card-level `confidence` object untouched.
- `<app-confidence-badge>` extended with a `variant: 'card' | 'field'` input. Defaults to `'card'` so all existing FN-818 callers are unaffected.
- New `variant="field"` + `score` (0ÔÇô1) inputs drive the per-field tier per FN-887:
  - `score < 0.6` ÔåÆ **red** pill, copy "Needs review"
  - `0.6 Ôëñ score < 0.85` ÔåÆ **amber** pill, copy "Verify"
  - `score ÔëÑ 0.85` ÔåÆ nothing rendered (host collapses via `--hidden` class)
- Exposed `FieldConfidenceTier` type + `ConfidenceBadgeComponent.fieldTierFor(score)` static helper for FN-888 to consume when building the `fieldConfidences` ÔåÆ tier mapping.
- Minimal SCSS: inline pill (no button affordance), dot glyph, red/amber tokens, hidden state. Placement tokens (`margin-left: 6px`, `vertical-align: middle`) assume rendering next to an Angular Material `mat-label` / plain `<label>`.
- Tooltip copy defaults: "AI extracted this field at {n}% ÔÇö please review before submitting." (red) / "ÔÇĞ ÔÇö double-check the value." (amber). Parents can override via `[tooltip]`.
- Tests added covering `fieldTierFor` thresholds and field-variant rendering.

**Hand-off to FN-888**
- Use `<app-confidence-badge variant="field" [score]="extraction.fieldConfidences?.['brokerName']"></app-confidence-badge>` next to each pre-filled field label.
- Badge hides itself when score ÔëÑ 0.85, so callers do **not** need `*ngIf` guards.
- `ConfidenceBadgeComponent.fieldTierFor(score)` is public/static ÔÇö use it when you need to drive a separate "low-confidence field" summary (e.g. count of amber+red).
