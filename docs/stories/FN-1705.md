# FN-1705: Story D — Login takeover dialog + forced-logout handling

## Requirement
Front the conflict/takeover flow and handle being displaced. On login conflict show a dialog with the other session's high-level details + a "Sign out other session & continue here" action; when this session is displaced, log out cleanly.

## Acceptance Criteria
- [ ] Login conflict → takeover dialog (MatDialog, AI dark theme — reuse the dark dialog pattern e.g. `manual-toll-dialog`) showing `deviceLabel`/browser/OS, masked IP, "last active N min ago".
- [ ] Actions: "Sign out other session & continue" → takeover, store new token, proceed; "Cancel" → back to login.
- [ ] Real-time displacement: subscribe to `session:revoked` (add to `WS_EVENTS`); clear auth + redirect to `/login` with an explanatory message.
- [ ] 401 fallback: `auth.interceptor.ts` handles `401 session_revoked` → clear + redirect (today it does not auto-logout on 401).
- [ ] Wire `POST /api/auth/logout` into `app.component.ts` logout.
- [ ] AI dark-theme tokens only; `[options]` = readonly class fields (FN-317).
