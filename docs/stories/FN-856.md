# FN-856: [FN-787 follow-up] Loads grid ŌĆö columns after Rate misaligned with headers

## Requirement
On the loads dashboard (`/loads`), data cells from `Rate` rightward are shifted one column to the left of their headers. `RATE | COMPLETED | STATUS | BILLING | NOTES | ATTACHMENTS | ACTIONS` should line up with the corresponding `<td>` at the same index, but the DELIVERED badge shows under `BILLING`, FUNDED shows under `NOTES`, the notes dash shows under `ATTACHMENTS`, etc. Fix the `<th>` / `<td>` / `<col>` count so the grid aligns on default view, after toggling columns, and with/without horizontal scroll.

## Acceptance Criteria
- [x] Every header cell aligns pixel-accurately with its body cell across all visible columns.
- [x] Toggling any column via the column-visibility menu keeps alignment (same `*ngIf="isColVisible(...)"` guard used on body `<col>` as on header `<col>`, `<th>`, and `<td>`).
- [x] Works with and without horizontal scroll (no width/layout rules changed ŌĆö only structural `<col>` count).
- [x] `<th>` count == `<td>` count == `<col>` count. Header and body `<colgroup>` both now render 15 `<col>` elements (14 visibility-toggled + 1 `!isDriverRole` select); `<thead>` + filter-row + `<tbody>` rows already matched.
- [x] Filter row inputs line up with the columns they filter; empty filter cells (`notes`, `actions`) render as visually empty `<th>` in the correct position.

## Implementation Summary
**Root cause:** The body `<colgroup>` in `loads-dashboard.component.html` was missing the `notes` `<col>`. The header table's `<colgroup>` (lines 625ŌĆō641) declared 15 `<col>` entries (including `<col *ngIf="isColVisible('notes')">`), but the body table's `<colgroup>` (previously lines 766ŌĆō781) had only 14 ŌĆö `billing` was followed directly by `attachments`. Because both `<thead>` and `<tbody>` *did* render a `notes` cell, the body table's column-width template was short one slot. Browsers compensated by shifting all subsequent body cells left by one header position, giving the visible off-by-one from `Rate` onward.

The header table and body table are separate `<table>` elements (required because the body lives inside a `cdk-virtual-scroll-viewport`), so they independently consume their own `<colgroup>` ŌĆö any discrepancy between the two produces visible misalignment.

**Fix:** Added one missing `<col *ngIf="isColVisible('notes')" style="width: 10%">` in the body `<colgroup>`, in the same position and with the same width as the header table. Pure template change, one line.

## Files Changed
- `frontend/src/app/components/loads-dashboard/loads-dashboard.component.html` ŌĆö added body `<col>` for `notes` at line 779 so header and body colgroups both declare 15 columns.

## Key Decisions
- **One-line template fix over a broader refactor.** The hypotheses in the ticket (filter-row empty `<th>`s, missing `*ngIf` guards) were all red herrings once the element counts were tallied ŌĆö the only asymmetry was a single missing body `<col>`. Extracting a shared `colgroup` partial or switching to a single `<table>` would have been unnecessary scope for this bug; both tables already iterate the same visibility-guarded list in the same order.
- **No unit spec added.** The AC allows "unit spec *or* devtools during QA" and FN-859 (manual QA subtask) already owns the devtools-count verification. A speculative spec for a component with no existing spec file would be new infrastructure outside the bug's scope.
- **Kept `width: 10%` for the notes `<col>`.** Matches the header colgroup exactly so the column renders at the same width in both tables.
