# FN-1600: [P1] [Loads Import] Combined "City, ST" cells in Pickup/Delivery columns not parsed

## Requirement
When a spreadsheet has a single `Pickup` or `Delivery` column with values like `Denton, TX` / `Johnson City, TN`, the AI handler correctly emits `warnings: [{code: "CITY_STATE_COMBINED"}]` and maps the column to `pickup_address1`/`delivery_address1` at 0.75 confidence — but `pickup_city`/`pickup_state`/`pickup_zip`/`delivery_city`/`delivery_state`/`delivery_zip` stay at `confidence: 0` with `sourceHeader: null`. After commit, every saved load has null city/state/zip, and **no stops are created** (because `buildStopsFromRow()` early-returns when all three are null).

There is no util in the codebase that splits "Denton, TX" into `{city, state}`. Add one to `loads-import-mapper.js`, validate the state against the US 50+DC abbreviation set, and wire it into `applyColumnMapping()` so the mapper writes city/state/zip when the warning fires. Multi-word cities ("Johnson City, TN", "Kansas City, MO") must split correctly — split on the **last** comma, not the first.

## Acceptance Criteria
- [ ] When `warnings` includes `CITY_STATE_COMBINED`, the mapper splits the combined cell into `{city, state, zip?}`.
- [ ] Splitter handles `"City, ST"`, `"City, ST 12345"`, multi-word cities, leading/trailing whitespace.
- [ ] State validated against US 2-letter abbreviations (50 + DC). On invalid state, fall back: store raw value in `*_address1`, mark `needs_review: true`, add per-row warning.
- [ ] After parse, `applyColumnMapping()` populates `pickup_city`/`pickup_state`/`pickup_zip` (and delivery equivalents); `buildStopsFromRow()` then creates pickup + delivery stops.
- [ ] Confidence for parsed sub-fields stored in `ai_metadata.confidences`, inherited from parent column.
- [ ] Idempotent — same `fileHash` produces identical parsed values.
- [ ] Unit tests cover: `"Denton, TX"`, `"Johnson City, TN"`, `"Kansas City, MO"`, `"Denton, TX 76201"`, `"Denton"` (no comma), `"Denton, ZZ"` (invalid state), `""`, `"  Denton ,  TX  "` (whitespace).

## Implementation Summary

### FN-1603 (subtask)
Added `parseCombinedCityState(raw)` to `loads-import-mapper.js` that splits a
combined "City, ST" cell on the **last** comma, captures an optional trailing
5-digit zip, validates the state against the US 50 + DC 2-letter set, and
returns `{city, state, zip}` (all `null` on any failure mode).

Extended `applyColumnMapping(rawRow, columnMapping, options)` with a third
optional argument. When `options.warnings` contains a `CITY_STATE_COMBINED`
entry (string code or `{code}` object), the mapper inspects
`pickup_address1` / `delivery_address1` after standard mapping and, when the
sub-fields aren't already populated by direct column mapping, runs the parser
to populate `*_city` / `*_state` / `*_zip`. The original `*_address1` value is
preserved for trail/display. On parse failure (invalid state, no comma, etc.),
the raw value stays in `*_address1` and a `CITY_STATE_PARSE_FAILED:*` marker
is appended to `options.rowWarnings` (when caller provides the array).

`stageBatch` now reads `batch.ai_metadata.warnings` and passes them as
`options.warnings`; per-row parse failures are appended to the row's `errors`
array, which already drives `validation_status = 'needs_review'`. The
function is pure and idempotent — same `{rawRow, columnMapping, warnings}`
input always produces the same output, so re-running the same `fileHash`
produces identical parsed values.

`buildStopsFromRow()` is unchanged — once city/state are populated, it
already emits the PICKUP/DELIVERY stops.

## Key Decisions
- **Parser gating on AI warning, not auto-detection.** We only attempt the city/state extraction when the AI explicitly emitted `CITY_STATE_COMBINED`. This avoids false positives on free-text address lines that happen to contain commas (e.g. `"Suite 12, Building B"`).
- **Don't strip `*_address1` after extraction.** The raw combined value is preserved on the load record; stops use the discrete sub-fields, so this doesn't affect routing.
- **Don't overwrite explicit column mappings.** If the spreadsheet has both a `Pickup` ("City, ST") column and a separate `PU City` column that mapped to `pickup_city`, we keep the explicit value — the warning condition implies one column carries combined data, not that other columns are wrong.
- **Per-row warnings as soft signal.** A parse failure flags the row `needs_review` (via `errors[]`) rather than rejecting it; the user fixes city/state in the wizard before commit.
- **US + DC only for Phase 1.** Matches the Open Item below.
- **Tests live next to the file, not under `__tests__/`.** The Jira description suggested an `__tests__/` directory; the existing repo pattern for this package is `loads-import-mapper.test.js` as a sibling, and `node --test` discovers both. Keeping the existing convention.
