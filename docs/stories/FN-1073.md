## Implementation Summary

### FN-1075 (subtask)
Mirrored the standalone wizard's `onZipBlur` pattern into the shared drawer/legacy stops UI without sharing or importing wizard code:

- **`StopCardComponent`** тАФ added `(blur)="onZipBlur()"` on the ZIP `<input>`, plus `@Output() zipBlur` (emits the trimmed ZIP) and `@Input() loadingZip` (drives a small italic "looking upтАж" indicator next to the ZIP label, using existing dark-theme tokens тАФ no new colors).
- **`StepStopsComponent`** тАФ injected `LoadsService`, maintains a per-row `Set<number> zipLoading`, and added `onStopZipBlur(i, zip)`:
  - Validates ZIP shape (`5` or `5+4` digits) before any HTTP call.
  - Skips the lookup entirely when both `city` and `state` are already non-empty (preserves user input).
  - Patches only empty fields from `res.data.city` / `res.data.state`; leaves any user-typed value untouched.
  - Wraps the `lookupZip` Observable in `finalize(() => zipLoading.delete(i))` so the indicator clears on success **and** error.
  - Swallows errors silently (no toast, no console.error) тАФ meets AC.
  - Emits `stopsChange` only when a field actually changed, so the drawer's dirty flag flips correctly.
- **`StepStopsComponent` template** тАФ bound `[loadingZip]="zipLoading.has(i)"` and `(zipBlur)="onStopZipBlur(i, $event)"` on `<app-stop-card>`.

Both new card inputs/outputs default safely (`loadingZip = false`, `zipBlur` optional) so the only consumer (`StepStopsComponent`) is unaffected by the new contract. The standalone load wizard does not use `StopCardComponent` (it builds rows directly from a FormArray), so its `onZipBlur` continues to work unchanged тАФ no regression.

## Key Decisions
- **Mirror, do not import**: per the bug's design guidance, the wizard's `onZipBlur` was reimplemented in `StepStopsComponent` rather than swapping the drawer to use the wizard's stops component. Keeps the drawer's existing structure, avoids reactive-form/template-driven mismatches, and lets each surface evolve independently.
- **Validation in parent, not card**: the ZIP shape validation lives in `StepStopsComponent.onStopZipBlur`, not `StopCardComponent.onZipBlur`. The card always emits the trimmed ZIP; the parent decides whether to call the API. Keeps the card dumb/reusable.
- **Skip-when-both-filled**: if both `city` and `state` are already non-empty, the lookup is skipped entirely (no spinner flash). Matches the standalone wizard's "do not clobber user input" behavior.
- **`finalize()` over manual try/finally**: ensures the spinner always clears, including on the silent error path.
- **No new SCSS tokens**: `.sc-zip-status` reuses the existing `#64748b` slate-500 muted color and matches the label typography.
