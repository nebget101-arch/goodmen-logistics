## Implementation Summary

### FN-1488 (subtask)
Added the database surface area for invoice attachment on receiving tickets. New Knex migration adds three nullable columns to `receiving_tickets`:
- `invoice_file_url` (varchar) â€” uploaded file URL
- `invoice_extracted_at` (timestamptz) â€” AI extraction timestamp
- `invoice_extracted_data` (jsonb) â€” Claude Vision payload (vendor, reference, invoiceDate, lines[])

Migration is idempotent via `hasColumn` guards on both up and down. Verified locally with a forward â†’ reverse â†’ forward â†’ reverse cycle against `goodmen_logistics`: types correct, all nullable, no data loss, idempotent on repeat runs.

`backend/packages/goodmen-shared/services/inventory.service.js` does not reference `receiving_tickets`, so no downstream column-info refresh is required.

### FN-1489 (subtask)
New AI service endpoint `POST /api/ai/invoice/extract` that takes a vendor invoice (image or PDF) and returns structured `{ vendor, reference, invoiceDate, lines: [{ sku, description, qty, unitCost, match }] }`.

- Accepts `{ fileUrl }` OR `{ base64, contentType }`. Supports `image/jpeg|png|webp|gif|heic|heif` and `application/pdf` (sent as a Claude `document` block â€” no pre-rendering).
- Default model `claude-haiku-4-5-20251001`, override via `AI_INVOICE_MODEL`.
- **Prompt caching**: persona + extraction schema split into two `text` blocks with `cache_control: { type: 'ephemeral' }` so back-to-back uploads reuse the cached prefix. Usage stats (`cacheReadTokens`, `cacheCreationTokens`) bubble up in the response `meta`.
- **Resilience**: one parse-failure retry with stricter "JSON only" reinforcement; 422 on second failure. 502 on Anthropic upstream errors. DB query failures degrade gracefully â€” lines come back with `match: null`.
- **Parts matching**: SKUs from extracted lines are looked up against `parts` table case-insensitively in batches of 200. Matched lines get `match: { partId, sku, name }`.
- **Logging**: only file size + content-type are logged; never invoice bytes, vendor names, or line items.
- **Tests**: 19 cases in `src/handlers/__tests__/invoice-extractor-handler.test.js`. Anthropic SDK and pg pool injected via `deps` for full isolation. Runs as part of `npm test` in the ai-service package.

## Key Decisions

### FN-1488 (subtask)
- **Filename prefix**: ticket spec proposed `20260508_â€¦` but a sibling migration already exists at `20260508120000_â€¦`. Used full timestamp `20260508130000_â€¦` to keep lexical ordering consistent with the rest of the directory and ensure this migration runs after the existing same-day migration.
- **`timestamp` column type**: Knex's `table.timestamp()` produces `timestamp with time zone` in PostgreSQL â€” same convention as the existing `posted_at`/`created_at`/`updated_at` columns on this table.
- **Down migration drops in reverse-add order** for predictability, with `hasColumn` guards so partial state from a failed up doesn't break a re-run.

### FN-1489 (subtask)
- **File layout**: ticket sketched `routes/`/`services/` directories, but the ai-service convention is a flat `handlers/` folder. Followed the existing pattern (handler + test under `handlers/__tests__/`) so future maintainers don't have to context-switch between two layouts.
- **PDF strategy**: Claude 4.x supports inline PDFs as a `document` block, so we pass the file straight through instead of pre-rendering pages. Falls back to the same image-block path only when the caller flags it as image content.
- **Parts matching is best-effort, not authoritative**: case-insensitive `UPPER(sku) IN (â€¦)` keyed by the *input* SKU so the frontend can correlate matches back to the original extracted line, even if the user later overrides. DB failure does not fail the request â€” lines just come back with `match: null`.
- **Test runner**: matched the existing `node:assert/strict` + plain-`node` pattern from `loads-nlq-handler.test.js` instead of pulling in jest, keeping the test surface dependency-free.
- **Dep injection vs lazy singleton**: handler accepts `{ anthropic, pool }` via deps for tests. Production calls fall through to lazy SDK + lazy `@goodmen/shared/config/database` pool. Used `'pool' in deps` (not `||`) so tests can pass `pool: null` to skip DB without inadvertently triggering the real connection.
