# FN-1416: Migrate consumers to fmcsa.* reference data + retire SAFER scraper

## Requirement

Final cutover: switch all live consumers off the FMCSA SAFER/SMS/L&I page scraper onto the new `fmcsa.*` reference dataset, validate parity against current scraped data, then delete the scraper code.

Consumers to migrate:
1. `routes/fmcsa-safety.js` — carrier dashboard, snapshots, L&I, SMS, inspections, crashes
2. `routes/shop-clients.js` — `/fmcsainfo/{dot}` lookup (latency drops from ~5-15s to <50ms)
3. `routes/brokers.js` — populate brokers table from `fmcsa.authorities`, preserving tenant-curated fields

New shared service `fmcsa-reference.js` is the only public read API. After parity QA passes, delete the scraper and drop legacy tables.

- [ ] All 3 routes use `fmcsa-reference` service exclusively
- [ ] No live HTTP calls to `safer.fmcsa.dot.gov`, `li-public.fmcsa.dot.gov`, or `ai.fmcsa.dot.gov` remain (verified by grep)
- [ ] `/api/shop-clients/fmcsainfo/{dot}` p95 latency < 200ms
- [ ] Brokers table populated; tenant-curated fields (`notes`, `payment_rating`, `broker_notes`) preserved on import

- [ ] Legacy tables dropped via revertible migration: `fmcsa_safety_snapshots`, `fmcsa_monitored_carriers`, `fmcsa_scrape_jobs`, `fmcsa_basic_details`, `fmcsa_basic_measures_history`, `fmcsa_violations`, `fmcsa_inspection_history`

## Implementation Summary

### FN-1427 (subtask, pass 1)

`backend/packages/goodmen-shared/services/fmcsa-reference.js` is the new single
read API over the `fmcsa.*` schema. It exposes:

- `getCarrier(dot)`, `getCarrierContacts(dot)`, `getCarrierAuthorities(dot)`,
  `searchCarriers({ q, limit, offset })`
- `getBroker(mc)`, `searchBrokers({ q, limit, offset })`
- `getInspections(dot, { since })`, `getInspectionStats(dot, { since })`
- `getCrashes(dot)`
- `getBasicScores(dot, { latest = true })`

All reads go through `getFmcsaKnex()` (Phase-1 returns the main DB Knex; Phase-2
will swap to a separate connection without touching consumers).

**Consumer migrations:**

- `routes/shop-clients.js` `/fmcsainfo/{dot}` — was making a live HTTP call to
  `mobile.fmcsa.dot.gov`; now reads from fmcsa-reference. Response now also
  surfaces phone/fax/email + the latest BASIC scores. `axios` import removed.
- `routes/brokers.js` — added `POST /api/brokers/sync-from-fmcsa` (admin-only)
  and `syncBrokersFromFmcsa()` exported function. Two SQL passes: INSERT
  active broker authorities not yet in the table (NOT EXISTS guard on
  `mc_number`), then UPDATE rows where `source='fmcsa'` to refresh data.
  Manual rows (`source='manual'`) are NEVER touched. Tenant overrides in
  `tenant_broker_overrides` are structurally untouched.
- `routes/fmcsa-safety.js` — tenant-facing reads (`/my-scores`,
  `/my-scores/:dotNumber/history`) switched to fmcsa-reference. A
  `buildSnapshotShape(dot)` helper assembles a snapshot-shaped row from
  `fmcsa.basic_scores` + `fmcsa.carriers` + `fmcsa.authorities` so the client
  contract stays stable. `safety_rating`, `safety_rating_date`, and
  `out_of_service_date` return null pending an importer enhancement.

  The internal admin endpoints (`/dashboard`, `/carriers` CRUD,
  `/carriers/:id/basic-details*`, `/inspection-details*`, `/jobs`, `/scrape*`
  POSTs) **intentionally still read from / write to legacy tables** in pass 1
  so the scraper can run in parallel for QA parity. Pass 2 retires them.

**New migrations** (`backend/packages/goodmen-database/migrations/`):

- `20260506140000_add_brokers_source_and_synced_at.js` — adds `source` (CHECK
  IN ('manual','fmcsa'), default 'manual') + `fmcsa_synced_at` timestamptz,
  backfills existing rows to 'manual', adds `idx_broker_source` btree index.
- `20260506140100_add_brokers_trgm_index.js` — `CREATE EXTENSION IF NOT EXISTS
  pg_trgm` + GIN trigram indexes on `legal_name`, `dba_name`, `mc_number`.

**Tests:** `services/fmcsa-reference.test.js` is a node:test e2e suite that
seeds fixture rows in `fmcsa.*`, exercises every public API, and tears down.
Skips cleanly when the FMCSA-schema database is unreachable (matches the
existing `fmcsa-knex.test.js` / fmcsa-importer test pattern).

## Key Decisions

- **Two-pass split** instead of one heavy "build + delete" subtask. The QA
  subtask (FN-1428) needs the scraper alive to validate parity, which is
  incompatible with the scraper being deleted in pass 1. Confirmed with
  product owner before starting.
- **Tenant-curated broker fields are structurally protected.** `notes`,
  `payment_rating`, `broker_notes`, `is_blocked`, `is_preferred` already live
  in `tenant_broker_overrides` (keyed by `tenant_id, broker_id`), not on the
  `brokers` table. The `source` column adds a second guard: even
  `brokers.broker_notes` (the legacy unscoped column from
  `20260308000000_add_brokers_extended.js`) is never overwritten because the
  sync only touches rows where `source='fmcsa'`.
- **`mobile.fmcsa.dot.gov` (referenced from `routes/fmcsa.js`) is NOT in the AC's
  forbidden-URL list** (`safer.fmcsa.dot.gov`, `li-public.fmcsa.dot.gov`,
  `ai.fmcsa.dot.gov`). It stays in pass 1; pass 2 will assess.
- **`utils/fmcsa.js` is dead code** (no callers) and contains a
  `safer.fmcsa.dot.gov` URL. Pass 2 deletes it.
- **`fmcsa-scrape-queue.js` will NOT be repurposed** — `fmcsa-import-queue.js`
  is already the importer's queue (FN-1422). Pass 2 deletes scrape-queue.
- **Snapshot-shaped response helper** preserves the existing API contract for
  `/my-scores` so the Angular frontend doesn't need a coordinated change.
  `safety_rating` / `safety_rating_date` / `out_of_service_date` are returned
  as null until the importer adds them.
