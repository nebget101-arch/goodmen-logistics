# FN-410: [Backend] Update import commit to persist product_type, category, source_transaction_id

## Summary
Updated the fuel import commit pipeline to persist `product_type`, `category`, and `source_transaction_id` on fuel_transactions. Updated duplicate detection to account for split rows (same date/amount but different product type). Added product_type filter to transaction list API and frontend.

## Implementation Summary

### Modified Files
- `backend/packages/goodmen-shared/services/fuel-service.js`
  - `commitBatch()`: Now writes `product_type` (default: 'diesel'), `category` (default: 'fuel'), and `source_transaction_id` from normalized_payload
  - `isDuplicateTransaction()`: Includes `product_type` in duplicate check so split rows with different products aren't flagged as duplicates; compound check on `(external_transaction_id, product_type)`
- `backend/packages/goodmen-shared/routes/fuel.js`
  - `GET /transactions`: Added `product_type` and `category` query param filters
- `frontend/src/app/fuel/fuel.service.ts`
  - `getTransactions()`: Added `product_type` and `category` filter params
- `frontend/src/app/fuel/fuel-transactions/fuel-transactions.component.ts`
  - Added `productType` filter property, included in load/clear
  - CSV export includes Product Type and Category columns
- `frontend/src/app/fuel/fuel-transactions/fuel-transactions.component.html`
  - Added Product column header + filter dropdown (Diesel/DEF/Reefer)
  - Added product_type cell in table body
  - Added Product Type + Category to detail drawer

### Key Decisions
- Default product_type to 'diesel' (not null) for backwards compatibility
- Default category to 'fuel' for all imports (maintenance/advance set via AI preprocessing)
- Duplicate detection uses `COALESCE(product_type, 'diesel')` for safe comparison with legacy rows
- Product type filter uses a select dropdown (only 3 options) rather than text input
