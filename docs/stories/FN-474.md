# FN-474: Build FMCSA Inspection Storage & Fleet Matching Engine

## Requirement
Build storage pipeline for FMCSA inspections and a matching engine that auto-links inspections to fleet vehicles/drivers via VIN, plate, CDL, and fuzzy name matching.

## Acceptance Criteria
- [x] POST /inspections/ingest — batch store with dedup by report_number
- [x] GET /inspections — list with filters (carrier, match status, date range)
- [x] GET /inspections/:id — single inspection detail
- [x] PATCH /inspections/:id/match — manual match assignment
- [x] POST /inspections/rematch — re-run matching for unmatched inspections
- [x] Matching chain: VIN exact -> plate match -> CDL exact -> name fuzzy
- [x] Cross-validate vehicle-driver assignment
- [x] Auto-create driver_risk_events on match
- [x] driver_risk_events table with severity scoring

## Implementation Summary

### Migration
- Created `driver_risk_events` table with severity scoring, OOS flags, violation counts, match metadata
- Unique constraint on (driver_id, event_source, source_id) prevents duplicate events
- Added match columns to `fmcsa_inspection_history`: match_status, match_method, match_confidence, matched_driver_id, matched_vehicle_id

### Matching Service (`fmcsa-matching-service.js`)
- **VIN match** (confidence 1.0): Exact VIN lookup against vehicles table
- **Plate match** (confidence 0.9): Exact plate_number lookup
- **CDL match** (confidence 0.95): Exact license_number lookup via driver_licenses
- **Name match** (confidence 0.85/0.75): Exact name or reversed name (Last, First) lookup
- Cross-validates vehicle-driver assignment consistency
- Creates driver_risk_events with severity based on OOS status and violation count

### Routes (added to `fmcsa-safety.js`)
- `POST /inspections/ingest`: Batch ingest with dedup, auto-match on ingest
- `GET /inspections`: Paginated list with carrier_id, match_status, date filters
- `GET /inspections/:id`: Single record + detailed report from fmcsa_inspection_details
- `PATCH /inspections/:id/match`: Manual match with risk event creation
- `POST /inspections/rematch`: Bulk rematch all unmatched inspections for a carrier

## Key Decisions
- Matching prioritizes VIN > plate > CDL > name (descending confidence)
- Risk event severity: OOS=high (weight 10), 3+ violations=medium (5), any violations=low (2), clean=low (1)
- Dedup events by (driver_id, event_source, source_id) unique constraint
- Manual match overrides auto-match with confidence 1.0
