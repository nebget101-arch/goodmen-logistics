# FN-1517: [Work Order] Basics tab — Scheduled/Start/Completion dates and Vehicle Make/Model/Year not populating on open

## Requirement

When opening an existing Work Order, the Basics tab Work Order Info dates (Scheduled / Start / Completion) and Vehicle Info Make / Model / Year fields are blank. Vehicle fields are present in the API payload but not bound; the three date fields have no backing DB columns at all.

## Acceptance Criteria

- [ ] Migration adds `scheduled_date / start_date / completion_date DATE` columns to `work_orders`
- [ ] Backend GET / PUT / POST read & write the three new dates
- [ ] Frontend binds Make/Model/Year + the three dates on load
- [ ] Round-trip: enter values, save, refresh → values persist
- [ ] No regression in totals, FN-1367 wiring

## Implementation Summary

### FN-1523 (subtask)
Added Knex migration `20260507220000_add_workorder_planned_dates.js` that adds three nullable `DATE` columns to `work_orders`: `scheduled_date`, `start_date`, `completion_date`. The migration is idempotent — each column is added only if missing, and the down step drops each only if present. The existing `completed_at TIMESTAMPTZ` audit column is untouched.

Verified locally:
- Up on a fresh state adds all three columns as `date`, nullable.
- Down drops all three cleanly.
- Up/down cycle is repeatable.
- All 1000 existing `work_orders` rows have `NULL` in the three new columns (no backfill needed).
