## Implementation Summary

### FN-1251 (frontend subtask)
Built admin UI for vendor management at `/admin/vendors`:

**Files created:**
- `frontend/src/app/services/vendors.service.ts` — Extended with admin CRUD methods (`listVendors`, `getVendor`, `createVendor`, `updateVendor`, `setVendorStatus`) using `/logistics/vendors` endpoint. Legacy MasterEntity search/create methods preserved unchanged.
- `frontend/src/app/components/admin/vendors/vendors-list/vendors-list.component.ts` — List view with status filter (all/active/suspended), edit and suspend/reactivate actions, confirmation modal.
- `frontend/src/app/components/admin/vendors/vendors-list/vendors-list.component.html`
- `frontend/src/app/components/admin/vendors/vendors-list/vendors-list.component.css`
- `frontend/src/app/components/admin/vendors/vendor-form/vendor-form.component.ts` — Create/edit modal with name, capacity, skill picker (10 predefined roadside skills), and lat/lng coordinate inputs for base location.
- `frontend/src/app/components/admin/vendors/vendor-form/vendor-form.component.html`
- `frontend/src/app/components/admin/vendors/vendor-form/vendor-form.component.css`
- `frontend/src/app/components/admin/vendors/vendors-admin.component.ts` — Container routing between list and form views.
- `frontend/src/app/components/admin/vendors/vendors-admin.module.ts` — Lazy-loaded module with `AuthGuard + PermissionGuard` (ROLES_MANAGE or ACCESS_ADMIN).
- `frontend/src/app/components/admin/vendors/vendors-list/vendors-list.component.spec.ts` — Unit tests covering load, filter, edit emit, toggle status, and display helpers.
- `frontend/src/app/app-routing.module.ts` — Added `admin/vendors` lazy route.

**Key decisions:**
- Used `/logistics/vendors` as the admin API prefix to avoid collision with the legacy MasterEntity vendor search at `/vendors/search`.
- Skill picker uses a predefined list of 10 roadside skills (Towing, Heavy Duty Towing, Tire Change, Fuel Delivery, Lockout Service, Battery Jump, Accident Recovery, Light Mechanical, Winching, Flatbed Transport) rendered as toggle chips.
- Base location input uses lat/lng text fields (validated against -90/90 and -180/180 ranges) rather than an interactive map; the backend stores these as PostGIS points. A map-picker can replace this once the mapping UX pattern is standardized.
- Dark AI theme (same CSS tokens as fmcsa-imports admin page).
