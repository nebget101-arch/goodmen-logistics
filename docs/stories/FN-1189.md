## Implementation Summary

Frontend triage panel widget built for the Roadside v2 dispatcher board (Story 1.3). Delivers the acceptance criteria from FN-1189:

- `TriagePanelComponent` (`app-triage-panel`) renders severity, category, urgency, vendor skills, rationale text, and a `prompt_version` badge (with model_name tooltip)
- Override button opens `OverrideModalComponent` with an optional dispatcher hint text field that POSTs to `POST /incidents/:id/triage/override`
- Loading and error states handled; retry button on error
- Tenant theming via CSS custom properties (`--fn-surface`, `--fn-accent`, etc.)
- Telemetry hooks: `triage_panel_view` and `triage_override_click` via `window.__fn_telemetry?.track`
- Component spec covers render, override flow, error state, badge class helpers

## Key Decisions

- Used `override-modal` as a child component (not a shared modal service) — keeps state local to the triage panel
- Inline template/styles on `OverrideModalComponent` to keep it self-contained as a leaf component
- Telemetry via `window.__fn_telemetry` guard (fire-and-forget, no crash if absent)
- Severity/urgency badges use existing design-token CSS variables (no new hex values)
