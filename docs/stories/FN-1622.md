# FN-1622: Settlement wizard "Create period" inline form is not user-friendly

## Requirement
On the New Settlement wizard (Step 1: Pay period), the "Or create a new period" inline form renders the **Create period** action as plain unstyled text instead of a proper button. Date pickers and the action are crammed into one row with no visual grouping. Restyle the sub-form to match the wizard's AI dark theme and add proper validation gating.

- [x] Button shows a leading icon (`event_available`) for affordance.
- [x] Section is grouped visually as a subtle card panel with rounded background and border.
- [x] Start / End date inputs and the action align cleanly on desktop (â‰Ą 960px) and stack on narrow widths (â‰¤ 768px).
- [x] Button is `disabled` when Start or End is empty, when End < Start, OR while `creatingPeriod` is true. Disabled state has reduced opacity and `not-allowed` cursor.
- [x] Inline validation message ("End date must be on or after the start date.") appears when End < Start.
- [x] On success, the newly created period is auto-selected in the dropdown and the inline form clears (`newPeriodStart`/`newPeriodEnd` reset).
- [x] No regression to the existing Period dropdown selection flow.

## Implementation Summary
Restyled the inline "Or create a new period" sub-form on the New Settlement wizard's Period step. The bare unstyled `.btn-secondary` text is now a properly themed icon button. The block is visually grouped as its own panel. ## Key Decisions
- **Dedicated `.btn-create-period` class instead of nesting `.btn-secondary` selectors.** Mirrors the wizard's existing pattern (each region â€” `.header-actions`, `.wizard-actions` â€” defines its own scoped secondary-button ruleset). Keeps visual changes local and avoids leaking styles to other `.btn-secondary` elsewhere.
- **Card surface (background + border) over a stronger top divider.** A subtle rounded panel reads as a distinct sub-form, which is what the AC asks for. Reuses the same color tokens already used by `.summary-block`.
- **38px fixed height on the button.** The `mat-form-field outline` date picker rounds to ~38px once vertical padding is included; matching that yields a clean baseline alignment with `align-items: flex-end`.
- **Clear `newPeriodStart`/`newPeriodEnd` on success.** Previous behavior left them populated, which combined with the now-enabled button would let the user accidentally re-create the same period. Clearing them mirrors the AC.
